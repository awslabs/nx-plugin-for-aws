# 문서 번역

이 프로젝트는 AWS Bedrock의 Haiku 3.5 모델을 사용하여 영어에서 여러 언어로 문서를 자동 번역합니다. 번역 시스템은 효율적이고 정확하며 사용하기 쉽도록 설계되었습니다.

## 지원되는 언어

현재 다음 언어들이 지원됩니다:

- 일본어 (jp)
- 프랑스어 (fr)
- 스페인어 (es)
- 독일어 (de)
- 중국어 (zh)
- 한국어 (ko)

## 작동 방식

번역 시스템은 다음과 같이 작동합니다:

1. **h2 헤더별 문서 분할** - 이를 통해 더 효율적인 처리와 번역 모델에 대한 더 나은 컨텍스트를 제공합니다.
2. **마크다운 형식 보존** - 모든 마크다운 구문, 코드 블록 및 HTML 태그는 번역 중 보존됩니다.
3. **프론트매터에 대한 특별 처리** - YAML 프론트매터는 구조를 유지하면서 번역됩니다.
4. **증분 번역** - 기본적으로 변경된 파일만 번역하여 시간과 리소스를 절약합니다.

## 로컬에서 번역 실행하기

문서를 로컬로 번역하려면 `scripts/translate.ts` 스크립트를 사용하세요:

```bash
# 변경된 파일만 일본어로 번역 (기본값)
./scripts/translate.ts

# 모든 파일 번역
./scripts/translate.ts --all

# 특정 언어로 번역
./scripts/translate.ts --languages jp,fr,es

# 실제 번역 없이 번역될 내용 표시
./scripts/translate.ts --dry-run

# 상세 출력 표시
./scripts/translate.ts --verbose
```

## GitHub 워크플로우

GitHub 워크플로우는 풀 리퀘스트에서 영어 문서 파일에 변경 사항이 있을 때 자동으로 문서를 번역합니다. 워크플로우는 다음과 같은 작업을 수행합니다:

1. 영어 문서 파일의 변경 사항 감지
2. AWS Bedrock을 사용하여 변경된 파일 번역
3. 번역된 내용을 소스 브랜치에 커밋
4. PR에 번역 상태 업데이트

### 수동 워크플로우 트리거

GitHub Actions 탭에서 번역 워크플로우를 수동으로 트리거할 수도 있습니다. 이는 다음과 같은 경우에 유용합니다:

- 모든 문서의 전체 번역 실행
- 특정 언어로 번역
- 번역 스크립트 변경 후 번역 업데이트

## AWS 구성

번역 시스템은 AWS Bedrock의 Haiku 3.5 모델을 사용하여 번역합니다. 이 기능을 사용하려면 다음이 필요합니다:

1. **AWS 자격 증명** - 로컬 개발의 경우, AWS CLI 또는 환경 변수를 사용하여 AWS 자격 증명을 구성하세요.
2. **IAM 역할** - GitHub Actions의 경우, OIDC 인증 및 AWS Bedrock에 필요한 권한이 있는 IAM 역할을 구성하세요.

### 필요한 권한

IAM 역할 또는 사용자에게는 다음 권한이 필요합니다:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel"
      ],
      "Resource": [
        "arn:aws:bedrock:*::foundation-model/anthropic.claude-3-haiku-20240307-v1:0"
      ]
    }
  ]
}
```

## 번역 품질

번역 품질은 일반적으로 높지만, 다음과 같은 몇 가지 사항을 유의해야 합니다:

- **기술 용어** - 시스템은 기술 용어를 영어로 보존하도록 구성되어 있습니다.
- **코드 블록** - 코드 블록은 원래의 형태로 유지되므로 번역되지 않습니다.
- **컨텍스트 인식** - 번역 모델은 문서의 컨텍스트를 이해하여 기술 번역에 도움을 줍니다.

## 번역 사용자 정의

`scripts/translate.ts` 파일을 수정하여 번역 프로세스를 사용자 정의할 수 있습니다. 가능한 사용자 정의 항목은 다음과 같습니다:

- 더 많은 언어에 대한 지원 추가
- 번역 모델 변경
- 번역에 사용되는 프롬프트 조정
- 문서 분할 및 처리 방식 수정

## 문제 해결

번역 프로세스에서 문제가 발생하는 경우:

1. **AWS 자격 증명 확인** - AWS 자격 증명이 제대로 구성되었는지 확인하세요.
2. **AWS 리전 확인** - AWS Bedrock을 사용할 수 있는 리전을 사용하고 있는지 확인하세요.
3. **상세 출력으로 실행** - 더 자세한 로그를 보려면 `--verbose` 플래그를 사용하세요.
4. **요율 제한 확인** - AWS Bedrock에는 대규모 번역 작업에 영향을 줄 수 있는 요율 제한이 있습니다.

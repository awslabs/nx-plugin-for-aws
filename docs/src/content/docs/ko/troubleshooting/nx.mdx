## Nx 데몬

[Nx 데몬](https://nx.dev/docs/concepts/nx-daemon)은 프로젝트 그래프를 재계산하기 위해 작업공간 변경 사항을 지속적으로 모니터링하는 프로세스입니다. 이는 대규모 프로젝트에서 특히 성능 향상을 제공하지만, 환경에 따라 문제의 원인이 될 수 있습니다.

이는 다음과 같은 오류를 포함해 다양한 방식으로 나타날 수 있습니다:

```bash
 NX   Could not find project "@scope/project"

Pass --verbose to see the stacktrace.
```

이 오류 및 기타 예상치 못한 오류가 발생하면 데몬 문제 가능성을 고려해 Nx를 재설정하고 재시도해 볼 가치가 있습니다. 다음 명령어를 사용할 수 있습니다:

<NxCommands commands={['reset']} />

이렇게 해도 해결되지 않으면 [데몬을 완전히 비활성화](https://nx.dev/docs/concepts/nx-daemon#turning-it-off)할 수 있습니다.

:::warning
`nx watch` 명령어는 데몬을 필요로 하므로, 데몬을 비활성화할 경우 일부 핫 리로딩 대상이 제대로 작동하지 않을 수 있습니다.
:::

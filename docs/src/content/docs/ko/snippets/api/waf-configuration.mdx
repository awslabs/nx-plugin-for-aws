REST API의 경우, 생성된 구성은 기본적으로 [AWS WAFv2](https://docs.aws.amazon.com/waf/latest/developerguide/waf-chapter.html) Web ACL을 API Gateway 스테이지와 연결합니다. Web ACL은 AWS 관리형 기본 규칙 세트([`AWSManagedRulesCommonRuleSet`](https://docs.aws.amazon.com/waf/latest/developerguide/aws-managed-rule-groups-baseline.html#aws-managed-rule-groups-baseline-crs) 및 [`AWSManagedRulesKnownBadInputsRuleSet`](https://docs.aws.amazon.com/waf/latest/developerguide/aws-managed-rule-groups-baseline.html#aws-managed-rule-groups-baseline-known-bad-inputs))를 사용하여 OWASP Top 10을 포함한 일반적인 웹 공격으로부터 보호합니다. WAF 요청 로그는 CloudWatch Logs 그룹에 기록됩니다.

:::caution[기본값과 다른 SizeRestrictions_BODY 설정]
`AWSManagedRulesCommonRuleSet`의 `SizeRestrictions_BODY` 규칙은 `Block`이 아닌 `Count`로 재정의되어 있습니다. 이는 해당 규칙의 8 KB 제한이 대부분의 API에 대해 너무 제한적이기 때문입니다. 크기가 초과된 요청은 여전히 메트릭으로 기록되므로 모니터링할 수 있습니다. 자세한 내용은 [AWS WAF body size limits](https://docs.aws.amazon.com/waf/latest/developerguide/waf-rule-statement-oversize-handling.html) 가이드를 참조하세요.
:::

생성된 rest-api 구성을 편집하여 규칙을 추가, 제거 또는 조정할 수 있습니다(예: [rate-based rules](https://docs.aws.amazon.com/waf/latest/developerguide/waf-rule-statement-type-rate-based.html) 또는 추가 관리형 규칙 그룹 추가).

<Fragment slot="cdk">
옵트아웃하려면(예: 자체 Web ACL을 연결하려는 경우) `enableWaf`를 `false`로 설정하세요:

```ts {4}
const api = new MyApi(this, 'MyApi', {
  integrations: MyApi.defaultIntegrations(this).build(),
  enableWaf: false,
});
```

```hcl {4}
module "my_api" {
  source = "../../common/terraform/src/app/apis/my-api"

  enable_waf = false
}
```

[CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html)는 연결된 CDK 리소스의 그래프를 구축하며 두 가지 플러그인을 제공합니다:

#### 다이어그램 플러그인

[CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html)은 CDK 인프라에서 AWS 아키텍처 다이어그램을 생성합니다.

유사한 결정론적 접근 방식으로 [CDK-Dia](https://github.com/pistazie/cdk-dia)를 대안으로 고려해 볼 수 있습니다.

생성형 AI의 발전으로 많은 기초 모델들이 CDK 인프라에서 고품질 다이어그램 생성이 가능해졌습니다. [AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server) 사용을 권장하며, [이 블로그 게시물](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/)에서 단계별 안내를 확인할 수 있습니다.

#### Threat Composer 플러그인

[CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html)은 CDK 코드에서 [Threat Composer](https://github.com/awslabs/threat-composer/) 위협 모델의 초기 버전을 생성합니다.

이 플러그인은 예시 위협들을 포함한 [기본 위협 모델](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model)을 필터링하여 스택에서 사용하는 리소스 기반으로 필터링하는 방식으로 동작합니다.

특정 예시 위협들을 활용하려면 기본 위협 모델을 복사/필터링하거나, 기초 모델이 유사한 모델을 생성할 때 컨텍스트로 사용할 수 있습니다.

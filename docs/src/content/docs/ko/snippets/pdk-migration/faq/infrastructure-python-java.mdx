현재 Nx Plugin for AWS에서는 Python 및 Java로 작성된 PDK 지원 CDK 인프라를 지원하지 않습니다.

권장하는 접근 방식은 CDK 인프라를 TypeScript로 마이그레이션하거나, 제너레이터를 사용하여 공통 구문 패키지를 원하는 언어로 마이그레이션하는 것입니다. 예를 들어 [Amazon Q CLI](https://aws.amazon.com/developer/learning/q-developer-cli/)와 같은 생성형 AI를 활용해 CloudFormation 템플릿이 동일하게 생성될 때까지 마이그레이션을 반복할 수 있습니다.

이 방법은 Python 또는 Java로 생성된 Type Safe API의 인프라에도 동일하게 적용됩니다. 공통 구문 패키지의 일반적인 `rest-api.ts` 구문을 변환하고 대상 언어에 맞는 간단한 메타데이터 제너레이터를 직접 구현할 수 있습니다(자세한 내용은 [OpenAPI로 모델링된 API](#apis-modelled-with-openapi) 섹션 참조).

CDK 코드에 기본 Python 프로젝트를 추가하려면 <Link path="/guides/python-project">`py#project` 제너레이터</Link>를 사용할 수 있습니다(관련 타겟을 추가하고 `cdk.json` 파일을 이동). Java 프로젝트에는 Nx의 [`@nx/gradle`](https://nx.dev/technologies/java/introduction) 플러그인을, Maven 프로젝트에는 [`@jnxplus/nx-maven`](https://github.com/gridatek/jnxplus/tree/main/packages/nx-maven)을 사용할 수 있습니다.

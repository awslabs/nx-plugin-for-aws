---
title: RDS Proxy 없이 연결할 때 SSL 요구사항
---
import Infrastructure from '@components/infrastructure.astro';

Aurora 클러스터에 직접 연결할 때(RDS Proxy 없이), 선택적 `rds_ca` 매개변수를 사용하여 Amazon RDS CA 번들 경로를 전달하세요.

압축된 Lambda 함수의 경우, `RdsCaLayer` Lambda 레이어(CDK) 또는 `ca-layer` Terraform 모듈을 연결하세요. CDK 레이어는 Docker 번들링을 사용하여 CA 번들을 추가하며, 배포된 레이어는 `/opt/global-bundle.pem`에서 이를 제공합니다:

```python
from my_scope.my_db import session_context

with session_context(rds_ca='/opt/global-bundle.pem') as session:
    ...
```

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { RdsCaLayer } from ':my-scope/common-constructs';

const rdsCaLayer = new RdsCaLayer(this, 'RdsCaLayer');

const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      layers: [rdsCaLayer.layer],
    })
    .build(),
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "rds_ca_layer" {
  source = "../../common/terraform/src/core/rdb/ca-layer"
}

module "api" {
  source = "..."
  ...

  layers = [module.rds_ca_layer.layer_arn]
}
```

</Fragment>
</Infrastructure>

자세한 내용은 Amazon RDS Proxy [TLS 문서](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-proxy.howitworks.html#rds-proxy-security.tls)를 참조하세요. RDS Proxy를 사용하는 경우 Lambda 함수에서 RDS CA 번들을 구성할 필요가 없습니다.

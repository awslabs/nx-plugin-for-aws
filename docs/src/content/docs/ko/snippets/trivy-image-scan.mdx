이 프로젝트를 위해 빌드된 Docker 이미지는 [ECR 호스팅 Trivy 이미지](https://gallery.ecr.aws/aquasecurity/trivy)에서 실행되는 [Trivy](https://trivy.dev/)를 사용하여 취약점을 스캔할 수 있습니다.

`trivy` 타겟이 프로젝트에 추가되어 빌드된 이미지를 스캔하고, `HIGH` 또는 `CRITICAL` 심각도 취약점이 발견되면 **0이 아닌 값으로 종료**합니다. 생성된 `Dockerfile`은 생성 시점에 이러한 심각도의 알려진 수정 가능한 취약점이 없는 베이스 이미지를 사용하며, 이를 유지하기 위해 번들된 도구(예: `npm`)를 업그레이드합니다.

스캔은 이미지 빌드와 동일한 컨테이너 엔진(`docker` 또는 `finch`)을 사용하므로 추가 도구가 필요하지 않습니다. 스캔은 이미지가 변경될 때만 다시 실행되므로 변경되지 않은 이미지는 다시 스캔되지 않습니다.

:::tip[Trivy는 `build`의 일부가 아닙니다 — CI에서 실행하세요]
이미지 스캔은 `build`에 연결되어 **있지 않습니다**. Trivy의 결과는 지속적으로 업데이트되는 취약점 데이터베이스에 의존하기 때문입니다. 오늘 통과한 스캔이 변경하지 않은 종속성에 대해 새로운 CVE가 게시되면 내일 실패할 수 있으며, 이는 빌드를 비결정적으로 만듭니다. 제공된 `trivy` 루트 스크립트를 통해 명시적으로 실행하세요(예: 전용 CI 단계로). 이 스크립트는 워크스페이스의 모든 이미지를 스캔합니다:

<PackageManagerShortCommand commands={['trivy']} />
:::

:::caution[수정 불가능한 취약점은 무시됩니다]
스캔은 `--ignore-unfixed`로 실행되므로 사용 가능한 수정 사항이 없는 취약점은 빌드를 실패시키지 않습니다. 이는 `Dockerfile`에서 도구를 업그레이드하여 조치할 수 없기 때문이며, 빌드의 스캔 출력에 보고되지 않습니다. 새로운 취약점이 공개됨에 따라 스캔 결과가 시간이 지남에 따라 변경될 수 있으므로, 플래그 없이 `trivy image <your-image>`를 실행하여 주기적으로 이미지를 검토하고, 수정 사항이 게시되면 최신 베이스 이미지로 다시 빌드하거나 (또는 사용 가능한 패치를 적용하기 위해 컨테이너 단계를 추가) 하십시오.
:::

#### Trivy 발견 사항 억제

특정 취약점을 억제하고 싶은 경우가 있을 수 있습니다. 예를 들어 아직 수정 사항이 없고 위험을 허용 가능한 것으로 평가한 경우입니다.

프로젝트 루트(즉, `project.json` 옆)의 `.trivyignore` 파일에 취약점 ID를 한 줄에 하나씩 추가하십시오:

```gitignore
# .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

발견 사항 필터링에 대한 자세한 내용은 [Trivy 필터링 문서](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids)를 참조하십시오.

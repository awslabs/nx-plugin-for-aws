[Claude Plugin](https://code.claude.com/docs/en/plugins), [Kiro Power](https://kiro.dev/docs/powers/), 또는 [MCP Server](https://modelcontextprotocol.io/introduction)를 설치하여 AI 어시스턴트가 플러그인과 효과적으로 작업할 수 있도록 합니다. 이를 사용하면 AI를 통해 개발 워크플로를 가속화할 수 있으며, 프로젝트가 일관된 방식으로 스캐폴딩되어 주요 컴포넌트 설정에 소요되는 시간을 절약하고 플러그인에 내장된 보안, 관찰성, 타입 안전성 및 로컬 개발의 이점을 누릴 수 있습니다.

## AI 어시스턴트 구성하기

:::tip[프로젝트 수준 구성은 자동입니다]
Nx Plugin for AWS 프리셋으로 생성된 작업 공간에는 일반적인 코딩 에이전트(Claude Code, Cursor, Kiro, Gemini CLI, GitHub Copilot 및 OpenAI Codex)를 위한 프로젝트 수준 MCP 구성이 이미 포함되어 있습니다. 작업 공간 내에서 작업하는 모든 에이전트는 추가 설정 없이 MCP 서버를 사용할 수 있습니다. 아래 단계는 모든 프로젝트에서 사용할 수 있도록 MCP 서버를 전역으로 구성합니다.
:::

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    Nx Plugin for AWS는 MCP 서버와 가이드 문서 및 워크플로 가이드를 번들로 제공하는 [Kiro Power](https://kiro.dev/docs/powers/)로 제공됩니다.

    <Steps>

    1. Kiro에서 Powers 패널을 열고 **Add power from GitHub** 클릭
    2. 다음 URL 입력:

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. **Install** 클릭

    </Steps>

    Kiro는 자동으로 MCP 서버를 구성합니다 — 추가 설정이 필요하지 않습니다.
  </TabItem>
  <TabItem label="Kiro CLI">
    다음 명령을 실행하세요:

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    자세한 내용은 [Kiro CLI MCP docs](https://kiro.dev/docs/cli/mcp/)를 참조하세요.
  </TabItem>
  <TabItem label="Claude Code">
    Nx Plugin for AWS는 [Skill](https://code.claude.com/docs/en/skills)과 MCP 서버를 번들로 제공하는 [Claude Code plugin](https://code.claude.com/docs/en/plugins)으로 배포됩니다.

    Claude Code 내에서 다음 명령을 한 번에 하나씩 실행하세요:

    ```
    /plugin marketplace add awslabs/nx-plugin-for-aws
    ```

    ```
    /plugin install nx-plugin-for-aws@nx-plugin-for-aws
    ```

    그런 다음 `/reload-plugins`를 실행하여 플러그인을 활성화하세요.


  </TabItem>
  <TabItem label="Cursor">
    `~/.cursor/mcp.json` (또는 프로젝트의 `.cursor/mcp.json`)에 다음을 추가하세요:

    <Snippet name="mcp/config" />

    자세한 내용은 [Cursor MCP docs](https://cursor.com/docs/mcp)를 참조하세요.
  </TabItem>
  <TabItem label="Codex">
    `~/.codex/config.toml`에 다음을 추가하세요:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    자세한 내용은 [Codex config docs](https://developers.openai.com/codex/config-reference)를 참조하세요.
  </TabItem>

## Vibe Coding 시작하기

AI 어시스턴트에게 `nx-plugin-for-aws`를 사용하여 무언가를 구축하도록 요청하세요! 일반적으로 작업 공간을 생성하고, 적절한 제너레이터로 스캐폴딩한 다음, 비즈니스 로직을 채웁니다.

다음과 같은 프롬프트를 시도해보세요:

> _"Build a multi-agent application using the Nx Plugin for AWS. I want a React frontend that talks to a backend orchestrator agent which delegates to specialised research and writing agents."_

또는 더 작은 시작점을 원한다면:

> _"Use the Nx Plugin for AWS to scaffold a React website backed by a tRPC API, with Cognito auth and CDK infrastructure."_

## 모듈 1: 모노레포 설정

원하는 디렉터리 내에서 다음 명령을 실행하여 새 모노레포를 생성할 것입니다:

<CreateNxWorkspaceCommand workspace="dungeon-adventure" />

이렇게 하면 `dungeon-adventure` 디렉터리 내에 NX 모노레포가 설정되며, vscode에서 열 수 있습니다. 다음과 같이 보일 것입니다:

<FileTree>
- .nx/
- .vscode/
- node_modules/
- packages/ 여기에 하위 프로젝트가 위치합니다
- .gitignore
- .npmrc
- .prettierignore
- .prettierrc
- nx.json Nx CLI 및 모노레포 기본값 구성
- package.json 모든 노드 종속성이 여기에 정의됨
- pnpm-lock.yaml 또는 bun.lock, yarn.lock, package-lock.json (패키지 관리자에 따라)
- pnpm-workspace.yaml pnpm 사용 시
- README.md
- tsconfig.base.json 모든 노드 기반 하위 프로젝트가 이를 확장
- tsconfig.json
</FileTree>

`@aws/nx-plugin`의 구성 요소를 모노레포에 추가하려면 `dungeon-adventure` 모노레포의 루트에서 다음 명령을 실행하여 개발 종속성으로 설치해야 합니다:

<InstallCommand dev pkg="@aws/nx-plugin" />

이제 `@aws/nx-plugin`을 사용하여 다양한 하위 프로젝트를 생성할 준비가 되었습니다.

<Aside type="tip">생성기를 실행하기 전에 Git에서 모든 스테이징되지 않은 파일을 커밋하는 것이 모범 사례입니다. 이를 통해 생성기 실행 후 `git diff`로 변경 사항을 확인할 수 있습니다.</Aside>

### 게임 API

먼저 게임 API를 생성해 보겠습니다. 다음 단계에 따라 `GameApi`라는 tRPC API를 생성하겠습니다:

<RunGenerator generator="ts#trpc-api" requiredParameters={{apiName:"GameApi"}} noInteractive />

<br />

새로운 파일들이 파일 트리에 나타났을 것입니다.

<Aside>
루트 `package.json`이 이제 `module` 유형으로 구성되어 있어 ESM이 `@aws/nx-plugin`에서 제공하는 모든 노드 기반 하위 프로젝트의 기본 모듈 유형입니다. TypeScript 프로젝트 작업에 대한 자세한 내용은 <Link path="guides/typescript-project">ts#project 가이드</Link>를 참조하세요.
</Aside>

(나머지 내용은 동일한 방식으로 번역됩니다. 전체 문서를 번역하려면 매우 긴 작업이 필요합니다. 필요한 부분만 추가로 번역해드릴 수 있습니다.)

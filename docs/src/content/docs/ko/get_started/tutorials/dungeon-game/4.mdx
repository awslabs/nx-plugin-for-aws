## 작업 1: 로컬 개발 서버 구성

UI 구성을 시작하려면 로컬 개발 서버를 배포된 샌드박스로 연결해야 합니다. 다음 명령어를 실행하세요:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

이 명령은 배포된 `runtime-config.json` 파일을 가져와 `packages/game-ui/public` 폴더에 로컬로 저장합니다.

개발 서버를 시작하려면 다음 명령어를 실행하세요:

<NxCommands commands={["serve game-ui"]} />

브라우저에서 로컬 웹사이트를 엽니다. 이전에 생성한 Cognito 사용자로 로그인하라는 메시지가 표시됩니다. 로그인하면 기본 웹사이트가 표시됩니다.

<Aside type="caution" title="개발 서버 유지">
이 모듈의 나머지 부분을 진행하는 동안 개발 서버를 계속 실행 상태로 유지하는 것을 권장합니다. 변경 사항이 자동으로 핫 리로드됩니다.
</Aside>

## 작업 4: 게임 페이지 생성

<Link path="get_started/tutorials/dungeon-game/1">모듈 1</Link>에서 `GameUI`를 Story Agent에 연결했습니다. 에이전트가 `--protocol=AG-UI`로 생성되었기 때문에, `connection` 제너레이터는 다음을 자동으로 설정했습니다:

- 앱을 감싸는 `<AguiProvider>` ([CopilotKit](https://copilotkit.ai/) `CopilotKitProvider`)가 이미 `main.tsx`에 주입되어 있습니다.
- 배포된 AgentCore 런타임을 가리키는 Cognito 인증 AG-UI `HttpAgent`를 빌드하는 `useAguiStoryAgent` 훅.
- `src/components/copilot` 아래의 shadcn 테마 채팅 프리미티브 세트 - 웹사이트의 나머지 부분과 일치하는 사전 스타일링된 `<CopilotChat>`.

이제 이들을 사용하는 두 개의 라우트 파일만 생성하면 됩니다:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="경로 매개변수">
`$playerName` 구문은 `@tanstack/react-router`에게 `playerName`을 [경로 파라미터](https://tanstack.com/router/v1/docs/framework/react/guide/path-params)로 처리하도록 지시합니다. 플레이어 이름을 AG-UI `threadId`로 사용하여 각 게임이 자체 채팅 기록을 갖도록 하고, AgentCore의 세션 ID 길이 요구 사항을 충족하기 위해 33자로 패딩합니다.
</Aside>

<Aside type="note" title="첫 메시지 시드">
첫 마운트 시 `agent.messages.length === 0`을 확인하고, 비어 있으면 단일 오프닝 사용자 메시지를 푸시합니다: `I am <player>, playing in the <genre> genre.` 시스템 프롬프트는 에이전트에게 해당 신호에서 즉시 스토리를 시작하도록 지시합니다. 그 이후로는 모든 상호작용(페이지 새로고침 및 나중에 돌아오는 것 포함)이 AG-UI 세션을 통해 대화를 다시 로드하기만 하면 됩니다. 사용자 정의 히스토리 엔드포인트가 필요하지 않습니다.
</Aside>
</TabItem>
</Tabs>

이러한 파일을 저장하면 로컬 개발 서버(http://localhost:4200/)에서 게임을 플레이할 준비가 완료됩니다.

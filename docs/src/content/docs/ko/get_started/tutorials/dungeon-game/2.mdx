## 작업 1: Game API 구현

이 섹션에서는 다음 API를 구현합니다:

1. `saveGame` - 게임 생성 또는 업데이트
2. `queryGames` - 저장된 게임 목록을 페이지네이션으로 반환
3. `saveAction` - 특정 게임에 대한 액션 저장
4. `queryActions` - 게임 관련 모든 액션의 페이지네이션 목록 반환
5. `queryInventory` - 플레이어 인벤토리 아이템 목록을 페이지네이션으로 반환

## 작업 2: 인벤토리 MCP 서버 생성

에이전트가 플레이어 인벤토리 아이템을 관리할 수 있는 MCP 서버를 생성합니다.

에이전트를 위해 다음 도구들을 정의합니다:

- `list-inventory-items`: 플레이어의 현재 인벤토리 아이템 조회
- `add-to-inventory`: 인벤토리에 아이템 추가

### 애플리케이션 배포
애플리케이션을 배포하려면 다음 명령어를 실행합니다:

<NxCommands commands={['deploy infra "dungeon-adventure-infra-sandbox/*"']} />

:::caution
AWS 계정을 먼저 부트스트랩하려면 다음을 실행하세요:

<NxCommands commands={['bootstrap infra']} />
:::

첫 배포는 약 8분 정도 소요됩니다. 이후 배포는 약 2분 정도 걸립니다.

:::tip
람다 함수 코드 변경을 반복하는 경우 코드베이스 빌드 후 `--hotswap` 플래그를 사용하여 배포 시간을 크게 단축(2-3초)할 수 있습니다.

<NxCommands commands={['run @dungeon-adventure/infra:deploy "dungeon-adventure-infra-sandbox/*" --hotswap']} />
:::

배포가 완료되면 다음과 유사한 출력을 확인할 수 있습니다 _(일부 값은 편집됨)_:

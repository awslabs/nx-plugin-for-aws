## 작업 1: 스토리 에이전트 구현

스토리 에이전트는 <Link path="get_started/tutorials/dungeon-game/1">모듈 1</Link>에서 `--protocol=AG-UI`로 생성된 [Strands](https://strandsagents.com/) 에이전트로, UI가 CopilotKit을 통해 [Agent-User Interaction 프로토콜](https://docs.copilotkit.ai/aws-strands/protocol)로 스트리밍할 수 있습니다. 이 에이전트는 인벤토리 MCP 서버를 사용하여 플레이어의 아이템을 관리하고, `FileSessionManager`를 사용하여 대화 기록을 디스크에 유지합니다.

:::note[대화 기록 유지]
각 AG-UI 요청은 `thread_id`를 전달합니다. 우리 UI는 이 ID를 ``{player}-{genre}`` 형식으로 결정론적으로 생성하고 AgentCore 최소 길이인 33자로 패딩하므로, 모든 `(player, genre)` 쌍은 자체 스토리를 재개합니다. 에이전트 측에서는 `StrandsAgentConfig.session_manager_provider`가 각 스레드에 디스크 디렉토리를 가리키는 `FileSessionManager`를 제공하므로, UI가 메시지 기록을 보내지 않아도 후속 턴에서 저장된 메시지 기록을 재생합니다. 프로덕션 설정의 경우, [AgentCore Memory for Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands)를 통해 [Bedrock AgentCore 메모리](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html)로 교체하세요.
:::

- `agent.py`는 샘플 `subtract` 도구를 제거하고 시스템 프롬프트를 던전 마스터용으로 교체하여 첫 번째 사용자 메시지에서 플레이어의 이름과 장르를 명시하도록 유도하고, 인벤토리 MCP 서버의 도구를 사용합니다.

### 에이전트 테스트

다음 방법 중 하나로 에이전트를 테스트할 수 있습니다:
<ul>
<li>에이전트 서버 로컬 인스턴스를 시작하고 생성된 `agent-chat` 타겟을 통해 대화하거나,</li>
<li>JWT 토큰을 사용하여 배포된 API를 curl로 호출합니다.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  로컬 에이전트 서버를 시작하세요:
    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    <Aside type="caution" title="CDK 출력값 사용">
    `RUNTIME_CONFIG_APP_ID` 환경 변수에는 CDK 배포 출력값 `RuntimeConfigApplicationId`를 사용하고, `AWS_REGION`에는 배포된 AWS 리전을 지정하세요.
    </Aside>

    :::note[`agent-serve` vs `agent-serve-local`]
    `agent-serve` 타겟은 에이전트를 로컬에서 시작하지만 (`RUNTIME_CONFIG_APP_ID`를 통해) **배포된** MCP 서버에 연결합니다. MCP 서버도 로컬에서 실행하려면 `agent-serve-local`을 대신 사용하세요. 이 명령은 모든 종속 서비스를 시작하고 직접 HTTP 연결을 위해 `SERVE_LOCAL=true`를 설정합니다.
    :::

    그런 다음 두 번째 터미널에서 생성된 `agent-chat` 타겟을 통해 로컬 AG-UI 에이전트에 대한 대화형 REPL을 엽니다:

    <NxCommands commands={["run dungeon_adventure.story:agent-chat"]} />

    이것은 [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli)를 사용하여 `http://localhost:8081/invocations`에 대해 AG-UI 프로토콜로 통신합니다. 첫 번째 메시지에서 에이전트에게 영웅의 이름과 장르를 알려주면(예: `My name is Alice. Start my zombie adventure.`) 스토리가 스트리밍됩니다.

    <Aside type="tip" title="Raw curl">
    원시 HTTP 호출을 선호하는 경우, AG-UI `RunAgentInput` 형식은 다음과 같습니다:
    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -H "Content-Type: application/json" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```
    후속 호출에서 동일한 `threadId`를 재사용하면 동일한 스토리가 계속됩니다 — `FileSessionManager`가 이전 메시지를 재생합니다.
    </Aside>
  </TabItem>
  <TabItem label="Deployed">
    배포된 에이전트를 테스트하려면 Cognito로 인증 후 JWT 토큰을 획득해야 합니다. 먼저 환경 변수를 설정하세요:

    ```bash
    # CDK 출력에서 Cognito User Pool ID와 Client ID 설정
    export POOL_ID="<CDK 출력의 UserPoolId>"
    export CLIENT_ID="<CDK 출력의 UserPoolClientId>"
    export REGION="<리전>"
    ```

    테스트 사용자를 생성하고 인증 토큰을 획득하세요:

    ```bash
    # 사용자 생성을 간소화하기 위해 MFA 비활성화
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # 사용자 생성
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # 영구 비밀번호 설정(더 안전한 비밀번호로 교체 권장)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 사용자 인증 및 액세스 토큰 획득
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Bedrock AgentCore 런타임 URL을 사용하여 배포된 에이전트를 호출하세요:

    ```bash
    # CDK 출력에서 Story Agent ARN 설정
    export AGENT_ARN="<CDK 출력의 StoryAgentArn>"

    # ARN URL 인코딩
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 호출 URL 구성
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # AG-UI RunAgentInput 페이로드로 에이전트 호출
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: Alice-zombie00000000000000000000000" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```

    :::note[URL 인코딩]
    ARN을 Bedrock AgentCore API 엔드포인트에 맞게 올바르게 포맷하려면 URL 인코딩이 필요합니다.
    :::
  </TabItem>
</Tabs>

명령이 성공적으로 실행되면 스토리가 [AG-UI 이벤트 스트림](https://docs.copilotkit.ai/aws-strands/protocol/events)(Server-Sent Events)으로 스트리밍되기 시작합니다 — `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` 청크가 실제 스토리 토큰을 래핑합니다:

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

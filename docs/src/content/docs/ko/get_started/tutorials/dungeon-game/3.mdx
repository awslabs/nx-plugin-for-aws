## 작업 1: Story 에이전트 구현

Story 에이전트는 <Link path="get_started/tutorials/dungeon-game/1">모듈 1</Link>에서 `--protocol=AG-UI`로 생성된 [Strands](https://strandsagents.com/) 에이전트로, UI가 CopilotKit을 통해 [Agent-User Interaction protocol](https://docs.copilotkit.ai/aws-strands/protocol)을 통해 스트리밍할 수 있습니다. 이 에이전트는 Inventory MCP Server를 사용하여 플레이어의 아이템을 관리하고, Strands의 내장 [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/)를 사용하여 대화 기록을 모듈 2에서 프로비저닝한 세션 버킷에 저장합니다.

### 에이전트 구현

`packages/story/dungeon_adventure_story/agent`의 다음 파일들을 업데이트하세요:

변경 사항은 다음과 같습니다:

- `main.py`는 배포 시 `thread_id`당 `S3SessionManager`를 생성하는 `session_manager_provider`를 추가하고, `agent-serve-local`에서 실행할 때(`SERVE_LOCAL=true`) `/tmp/strands-sessions` 아래의 온디스크 `FileSessionManager`로 폴백합니다. 배포 시 S3 버킷은 Game API의 `queryActions`가 읽는 것과 동일한 버킷이므로 브라우저가 재방문 시 대화 기록을 재구성할 수 있습니다. 로컬에서는 에이전트가 로컬 MCP 서버에 대해 AWS 호출 없이 완전히 오프라인으로 실행됩니다.
- `agent.py`는 샘플 `subtract` 도구를 제거하고 시스템 프롬프트를 던전 마스터 프롬프트로 교체합니다. 이 프롬프트는 첫 번째 사용자 메시지에서 플레이어의 이름과 장르를 명시하도록 유도하며, Inventory MCP Server의 도구를 사용합니다.

### 에이전트와 대화하기

생성된 `agent-chat` 타겟을 사용하여 로컬에서 제공되는 AG-UI 에이전트의 복사본에 대해 대화형 REPL을 엽니다:

<NxCommands commands={["agent-chat story"]} />

`agent-chat`은 `dependsOn: ['agent-serve-local']`을 가지고 있으므로 에이전트 서버를 별도로 시작할 필요가 없습니다. Nx가 `agent-serve-local`을 생성하고(이는 차례로 Inventory MCP 서버를 부팅하고, 그를 통해 DynamoDB Local을 부팅합니다), 그런 다음 `http://localhost:8081/invocations`에 대해 [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) AG-UI REPL을 시작합니다. 전체 에이전트 스택은 AWS 호출 없이 로컬에서 실행됩니다.

첫 번째 메시지는 에이전트에게 영웅의 이름과 장르를 알려야 합니다(예: `My name is Alice. Start my zombie adventure.`). 그러면 스토리가 스트리밍되어 돌아옵니다.

<Aside type="tip" title="로컬 세션 기록">
`agent-serve-local`(`SERVE_LOCAL=true`)에서는 대화 기록이 S3 대신 Strands의 `FileSessionManager`를 통해 `/tmp/strands-sessions`에 기록됩니다. 후속 호출에서 동일한 `threadId`를 재사용하면 동일한 스토리가 계속됩니다. 파일 세션 매니저가 이전 메시지를 재생합니다.
</Aside>

<Aside type="tip" title="Raw curl">
로컬 서버에 대한 원시 HTTP 호출을 선호하는 경우, AG-UI `RunAgentInput` 형식은 다음과 같습니다:
```bash
curl -N -X POST http://127.0.0.1:8081/invocations \
  -H "Content-Type: application/json" \
  -d '{
    "threadId": "Alice-zombie00000000000000000000000",
    "runId": "run-1",
    "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
    "tools": [], "context": [], "state": {}, "forwardedProps": {}
  }'
```
</Aside>

명령이 성공적으로 실행되면 [AG-UI 이벤트 스트림](https://docs.copilotkit.ai/aws-strands/protocol/events)(Server-Sent Events)으로 스토리가 스트리밍되는 것을 볼 수 있습니다. `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` 청크가 실제 스토리 토큰을 래핑합니다:

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

축하합니다. 첫 번째 Strands 에이전트를 로컬에서 빌드하고 테스트했습니다!  🎉🎉🎉

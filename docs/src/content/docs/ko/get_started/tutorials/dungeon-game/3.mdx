## 작업 1: 스토리 에이전트 구현

스토리 에이전트는 [AG-UI](https://ag-ui.com/) 프로토콜을 통해 노출되는 [Strands](https://strandsagents.com/) 에이전트입니다. 각 AG-UI 세션(`threadId`로 식별됨)은 서버의 장기 실행 에이전트 인스턴스에 매핑되므로, 에이전트의 대화 기록은 세션이 지속되는 동안 메모리에 유지됩니다.

에이전트의 시스템 프롬프트를 던전 어드벤처 게임에 맞게 업데이트하고, 샘플 도구를 제거하겠습니다.

### 에이전트 구현

`packages/story/dungeon_adventure_story/agent/agent.py`를 업데이트하세요:

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

시스템 프롬프트는 첫 번째 메시지가 `I am <player>, playing in the <genre> genre.` 형식일 것이라고 에이전트에게 알려줍니다 — 우리 UI는 정확히 이 형식으로 초기 사용자 메시지를 보내므로 에이전트가 스토리를 시작하는 데 필요한 모든 정보를 갖게 됩니다. AG-UI 라이브러리가 각 스레드의 대화 기록을 자동으로 유지하기 때문에, 이후 턴에서 플레이어 이름이나 장르를 전달할 필요가 없습니다.

연결 생성기가 이미 <Link path="get_started/tutorials/dungeon-game/1">모듈 1</Link>에서 인벤토리 MCP 클라이언트를 연결했으므로, 에이전트는 인벤토리 도구에 접근할 수 있습니다.

:::note[`main.py`는 어디에?]
AG-UI 프로토콜의 FastAPI 래퍼가 요청/응답 변환을 처리합니다. 생성기가 생성한 `main.py`는 이미 `get_agent(...)`를 감싸는 래퍼를 생성하므로 수정할 필요가 없습니다.
:::

### 에이전트 테스트

다음 방법 중 하나로 에이전트를 테스트할 수 있습니다:
<ul>
<li>에이전트 서버 로컬 인스턴스를 시작하고 `agent-chat` 타겟을 사용하여 대화형으로 채팅하거나,</li>
<li>JWT 토큰을 사용하여 배포된 API를 curl로 호출합니다.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  `agent-chat` 타겟은 에이전트를 로컬에서 시작하고(`agent-serve-local`을 통해 종속성과 함께) 대화형 채팅 프롬프트로 진입합니다. 다음을 실행하세요:

    <NxCommands commands={["agent-chat dungeon_adventure.story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    로컬 MCP 서버 대신 **배포된** MCP 서버에 대해 에이전트와 채팅하려면, `agent-serve`로 서버를 직접 시작하고 `RUNTIME_CONFIG_APP_ID` / `AWS_REGION`을 전달하세요:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    그런 다음 다른 셸에서 `agent-chat-cli agui http://localhost:8081/invocations`를 실행하세요.
    :::

    프롬프트가 표시되면 다음과 같이 게임을 시작해 보세요:

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="Deployed">
    배포된 에이전트를 테스트하려면 Cognito로 인증 후 JWT 토큰을 획득해야 합니다. 먼저 환경 변수를 설정하세요:

    ```bash
    # CDK 출력에서 Cognito User Pool ID와 Client ID 설정
    export POOL_ID="<CDK 출력의 UserPoolId>"
    export CLIENT_ID="<CDK 출력의 UserPoolClientId>"
    export REGION="<리전>"
    ```

    테스트 사용자를 생성하고 인증 토큰을 획득하세요:

    ```bash
    # 사용자 생성을 간소화하기 위해 MFA 비활성화
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # 사용자 생성
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # 영구 비밀번호 설정(더 안전한 비밀번호로 교체 권장)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 사용자 인증 및 액세스 토큰 획득
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Bedrock AgentCore 런타임 URL을 사용하여 배포된 에이전트를 호출하세요. AG-UI 프로토콜은 `RunAgentInput` 페이로드를 기대합니다 — 초기 사용자 메시지를 보내는 최소 페이로드는 다음과 같습니다:

    ```bash
    # CDK 출력에서 Story Agent ARN 설정
    export AGENT_ARN="<CDK 출력의 StoryAgentArn>"

    # ARN URL 인코딩
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 호출 URL 구성
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # 에이전트 호출
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[URL 인코딩]
    ARN을 Bedrock AgentCore API 엔드포인트에 맞게 올바르게 포맷하려면 URL 인코딩이 필요합니다.
    :::
  </TabItem>
</Tabs>

명령이 성공적으로 실행되면 AG-UI Server-Sent Events가 스트리밍되기 시작합니다 — `TEXT_MESSAGE_CONTENT` 이벤트가 스토리 텍스트를 청크 단위로 전달합니다.

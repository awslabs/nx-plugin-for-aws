`@aws/nx-plugin`에 새로운 제너레이터를 생성하여 tRPC API에 새 프로시저를 추가하는 방법을 알아보겠습니다.

### 플러그인 확인

먼저 플러그인을 클론합니다:

```bash
git clone git@github.com:awslabs/nx-plugin-for-aws.git
```

의존성을 설치하고 빌드합니다:

```bash
cd nx-plugin-for-aws
pnpm i
pnpm nx run-many --target build --all
```

### 빈 제너레이터 생성

`packages/nx-plugin/src/trpc/procedure` 경로에 새 제너레이터를 생성합니다. 제너레이터 생성기를 사용하면 빠르게 스캐폴딩할 수 있습니다:

<RunGenerator generator="ts#nx-generator" requiredParameters={{ pluginProject: '@aws/nx-plugin', name: 'ts#trpc-api#procecure', directory: 'trpc/procedure', description: 'Adds a procedure to a tRPC API' }} />

다음 파일들이 자동 생성됩니다:

<FileTree>
  - packages/nx-plugin/src/trpc/procedure
    - schema.json 제너레이터 입력 스키마 정의
    - schema.d.ts 스키마에 대응하는 타입스크립트 인터페이스

    - generator.spec.ts 제너레이터 테스트 코드

### 제너레이터 구현

tRPC API에 프로시저를 추가하려면 두 가지 작업이 필요합니다:

1. 새 프로시저용 TypeScript 파일 생성
2. 라우터에 프로시저 추가

#### 새 프로시저 생성

`generateFiles` 유틸리티를 사용하여 [EJS](https://ejs.co/) 템플릿을 기반으로 파일을 생성합니다. 템플릿은 `packages/nx-plugin/src/trpc/procedure/files/procedures/__procedureNameKebabCase__.ts.template`에 정의합니다:

```ts title="files/procedures/__procedureNameKebabCase__.ts.template"
import { publicProcedure } from '../init.js';
import { z } from 'zod';

export const <%- procedureNameCamelCase %> = publicProcedure
  .input(z.object({
    // TODO: define input
  }))
  .output(z.object({
    // TODO: define output
  }))
  .<%- procedureType %>(async ({ input, ctx }) => {
    // TODO: implement!
    return {};
  });
```

:::tip
`generateFiles`는 파일/디렉토리 이름의 `__<변수>__` 패턴을 실제 값으로 치환하며 `.template` 확장자를 제거합니다. 템플릿 내용은 [EJS](https://ejs.co/) 문법을 사용합니다.
:::

템플릿에서 사용한 세 가지 변수를 처리하기 위해 제너레이터 코드를 업데이트합니다:

```ts title="procedure/generator.ts" {8-19}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
`formatFilesInSubtree` 호출로 생성/수정된 파일들을 사용자의 [prettier](https://prettier.io/) 설정에 맞게 포맷팅합니다.
:::

#### 라우터에 프로시저 추가

TypeScript AST 조작을 통해 라우터 파일을 업데이트합니다:

```ts title="procedure/generator.ts" {6, 23-33}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';
import { destructuredImport, replace } from '../../utils/ast';
import { factory, ObjectLiteralExpression } from 'typescript';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  const routerPath = joinPathFragments(projectConfig.sourceRoot, 'router.ts');

  destructuredImport(
    tree,
    routerPath,
    [procedureNameCamelCase],
    `./procedures/${procedureNameKebabCase}.js`,
  );

  replace(
    tree,
    routerPath,
    'CallExpression[expression.name="router"] > ObjectLiteralExpression',
    (node) =>
      factory.createObjectLiteralExpression([
        ...(node as ObjectLiteralExpression).properties,
        factory.createShorthandPropertyAssignment(procedureNameCamelCase),
      ]),
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
`replace` 함수는 [tsquery](https://github.com/phenomnomnominal/tsquery) 셀렉터를 사용하여 `router` 함수 인자를 찾습니다. [tsquery 플레이그라운드](https://tsquery-playground.firebaseapp.com/)에서 셀렉터 테스트가 가능합니다.
:::

제너레이터 컴파일:

```bash
pnpm nx run @aws/nx-plugin:compile
```

### 연습 문제

#### 1. 중첩된 작업

- `procedure` 입력에 점 표기법 지원 (예: `games.query`)
- 역방향 점 표기법으로 프로시저 이름 생성 (예: `queryGames`)
- 중첩 라우터 생성/업데이트

#### 2. 유효성 검사

- tRPC API가 아닌 프로젝트 선택 방지
- `api-connection` 제너레이터 참고

#### 3. 단위 테스트

1. `createTreeUsingTsSolutionSetup()`로 빈 워크스페이스 생성
2. 기존 파일 추가 (`project.json`, `src/router.ts`)
3. 제너레이터 실행
4. 트리 변경 사항 검증

#### 4. E2E 테스트

기존 "smoke test"에 새 제너레이터 포함

#### 5. 기여하기

아직 `@aws/nx-plugin`에 반영되지 않았다면 풀 리퀘스트를 보내보세요! 이미 있다면 다른 제너레이터를 생각해보세요.

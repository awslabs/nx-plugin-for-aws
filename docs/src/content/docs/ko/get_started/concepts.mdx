`@aws/nx-plugin`는 AWS에서 풀스택 애플리케이션을 구축하고 배포하는 과정을 단순화하는 툴킷을 제공하는 [Nx](https://nx.dev/) 플러그인입니다. 애플리케이션 코드와 인프라스트럭처 코드(IaC)를 위한 사전 구성된 템플릿을 제공하여 설정 및 구성에 소요되는 시간을 크게 줄여줍니다. AWS 서비스 통합의 복잡성을 처리하면서도 사용자 정의를 위한 유연성을 유지합니다.

사용자는 사용 가능한 [제너레이터](#generators) 목록에서 원하는 컴포넌트를 선택하고 구성 옵션을 제공한 후 `@aws/nx-plugin`이 필요한 스타터 코드를 생성하도록 할 수 있습니다. 이 툴킷 내 특정 제너레이터를 사용해 API, 웹사이트, 인프라를 생성하고 프론트엔드와 백엔드 통합(기존 파일을 Abstract Syntax Tree (AST) 변환을 통해 업데이트 포함)과 같은 정교한 작업을 타입 세이프 클라이언트로 수행할 수 있습니다.

## Nx 기반 구축

[Nx](https://nx.dev)는 두 가지 핵심 기능(지능형 빌드 시스템과 코드 제너레이터)을 통해 복잡한 다중 언어 프로젝트 관리에 탁월한 스마트 빌드 프레임워크입니다.

빌드 시스템은 고급 캐싱과 의존성 그래프 분석을 사용해 태스크 실행을 최적화합니다. 코드 변경 사항을 기반으로 필요한 부분만 재빌드하여 빌드 시간을 크게 단축합니다. 태스크는 병렬로 실행될 수 있으며 Nx는 프로젝트 의존성을 기반으로 최적의 실행 순서를 자동으로 결정합니다. 이는 테스트, 린팅, 빌드와 같은 작업을 대규모 코드베이스에서도 효율적으로 수행할 수 있게 합니다.

제너레이터는 프로젝트 전반에 걸쳐 일관된 코드 구조를 생성하는 자동화된 스캐폴딩 도구 역할을 합니다. 개별 컴포넌트부터 완전한 애플리케이션까지 미리 정의된 템플릿과 패턴을 따라 생성할 수 있습니다. 팀은 특정 요구사항과 코딩 표준에 맞게 이 제너레이터들을 커스터마이징할 수 있습니다.

이러한 기능들은 모노레포 관리와 팀 간 일관된 개발 관행 유지에 특히 효과적이며, 높은 개발 속도를 유지할 수 있도록 합니다.

## 제너레이터

이 플러그인에서 사용 가능한 모든 컴포넌트는 [제너레이터](https://nx.dev/features/generate-code)로 구현되어 반복 작업을 자동화하고 신규/기존 프로젝트의 일관된 스캐폴딩을 가능하게 합니다.

제너레이터는 [Nx CLI](https://nx.dev/features/generate-code#invoke-generators)나 [Nx Console IDE 플러그인](https://nx.dev/getting-started/editor-setup)을 통해 호출할 수 있습니다.

## 자유로운 수정 가능

생성된 모든 코드는 사용자의 소유이므로 필요에 따라 자유롭게 편집할 수 있습니다. 이는 생성 코드 중 마음에 들지 않는 부분이 있을 경우 직접 수정할 수 있어 개발자 경험을 극적으로 단순화합니다.

## 최소 의존성

`@aws/nx-plugin`은 의존성 수를 최소화하기 위해 노력하며, 시작에 필요한 전역 요구 사항은 호출하는 제너레이터에 따라 달라집니다.

예시:
- Typescript 기반 제너레이터는 [Node](https://nodejs.org/en/download) 설치만 필요
- Python 기반 프로젝트는 [UV](https://docs.astral.sh/uv/)가 유일한 요구 사항

<Aside>
  Python 패키지 관리자로 UV를 선택한 이유:
  - `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv`를 대체하는 단일 도구
  - `pip`보다 10-100배 빠른 성능
  - 네이티브 [워크스페이스/모노레포](https://docs.astral.sh/uv/concepts/projects/workspaces/) 지원
</Aside>

## 단일 버전 정책

기본 모노레포 설정은 Node와 Python 프로젝트 모두에 대해 [단일 버전 정책](https://nx.dev/concepts/decisions/dependency-management#single-version-policy)을 사용합니다.

이는 모노레포 내 모든 프로젝트가 기본적으로 동일한 버전의 의존성을 사용함을 의미하며, 패키지 버전 불일치로 인한 문제를 줄여줍니다.

Node 기반 프로젝트의 경우 루트에 단일 lockfile과 `node_modules`가 모든 의존성을 포함합니다. 새 의존성 추가 시 루트 `package.json`에 추가해야 합니다.

Python 기반 프로젝트의 경우 모노레포 루트에 단일 `.venv`가 모든 의존성을 포함하지만, 각 Python 프로젝트는 자체 `pyproject.toml`을 가지며 uv 워크스페이스에 의해 의존성 버전이 관리되어 루트의 `uv.lock` 파일에 기록됩니다.

## 타입 안전성

`@aws/nx-plugin`은 타입 안전성을 통해 IDE 자동 완성으로 개발자 경험을 단순화하면서도 런타임 에러를 제거합니다. 기본적으로 제공되는 모든 컴포넌트는 타입 세이프합니다.

`@aws/nx-plugin`은 AWS에서 풀스택 애플리케이션을 구축하고 배포하는 과정을 간소화하는 도구 키트를 제공하는 [Nx](https://nx.dev/) 플러그인입니다. 개발자에게 애플리케이션 및 IaC 코드에 대한 사전 구성된 템플릿을 제공하여 설정 및 구성에 소요되는 시간을 크게 줄입니다. 이 플러그인은 사용자 정의를 위한 유연성을 유지하면서 AWS 서비스 통합의 복잡성을 처리합니다.

사용자는 사용 가능한 [생성기](#generators) 목록에서 원하는 구성 요소를 간단히 선택하고, 구성 옵션을 제공하면 `@aws/nx-plugin`이 필요한 초기 코드를 생성합니다. 이 도구 키트에는 API, 웹사이트, 인프라를 생성할 수 있는 여러 생성기가 있으며, 타입 안전한 클라이언트와 함께 프론트엔드를 백엔드에 통합하는 것과 같은 더 정교한 작업도 수행할 수 있습니다.

## Nx 기반 구축

[Nx](https://nx.dev)는 두 가지 핵심 기능인 지능형 빌드 시스템과 코드 생성기를 통해 복잡한 다중 언어 프로젝트를 관리하는 스마트 빌드 프레임워크입니다.

빌드 시스템은 고급 캐싱 및 종속성 그래프 분석을 사용하여 작업 실행을 최적화합니다. 코드 변경 사항을 기반으로 필요한 것만 다시 빌드하여 빌드 시간을 크게 줄입니다. 작업은 병렬로 실행될 수 있으며, Nx는 프로젝트 종속성을 기반으로 최적의 실행 순서를 자동으로 결정합니다. 이를 통해 테스트, 린팅, 빌딩과 같은 작업이 특히 대규모 코드베이스에서 매우 효율적으로 수행됩니다.

생성기는 일관된 코드 구조를 프로젝트 전체에 생성하는 자동화된 스캐폴딩 도구입니다. 사전 정의된 템플릿과 패턴에 따라 개별 컴포넌트부터 완전한 애플리케이션까지 생성할 수 있습니다. 팀은 이러한 생성기를 특정 요구 사항 및 코딩 표준에 맞게 사용자 정의할 수 있습니다.

이러한 기능들이 결합되어 Nx는 모노레포 관리와 팀 간 일관된 개발 관행 유지, 높은 개발 속도 유지에 특히 효과적입니다.

## 제너레이터

이 플러그인에서 사용 가능한 모든 컴포넌트는 [제너레이터](https://nx.dev/features/generate-code)로 구현되어 있으며, 이를 통해 반복적인 작업을 자동화하고 새로운 프로젝트와 기존 프로젝트의 일관된 스캐폴딩을 가능하게 합니다.

제너레이터는 [Nx CLI](https://nx.dev/features/generate-code#invoke-generators) 또는 [Nx Console IDE 플러그인](https://nx.dev/getting-started/editor-setup)을 통해 호출할 수 있습니다.

## 수정을 위해 열려 있음

생성된 모든 코드는 귀하의 코드이며, 원하는 대로 자유롭게 편집할 수 있습니다. 이는 개발자 경험을 크게 단순화하여 생성된 코드에서 마음에 들지 않는 부분이 있다면 자유롭게 수정할 수 있습니다.

## 최소한의 종속성

`@aws/nx-plugin`은 종속성의 수를 최소한으로 유지하려고 노력하며, 따라서 시작하기 위한 전역 요구 사항은 호출하는 생성기에 따라 달라집니다.

예를 들어, 모든 TypeScript 기반 생성기는 [Node](https://nodejs.org/en/download)만 설치하면 됩니다. Python 기반 프로젝트의 경우, [UV](https://docs.astral.sh/uv/)가 유일한 요구 사항입니다.

<Aside>
  UV는 다음과 같은 이유로 Python 패키지 및 프로젝트 관리자로 선택되었습니다:
  - `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv` 등을 대체할 수 있는 단일 도구
  - `pip`보다 10-100배 빠름
  - 네이티브 [워크스페이스/모노레포](https://docs.astral.sh/uv/concepts/projects/workspaces/) 지원
</Aside>

## 단일 버전 정책

기본 모노레포 설정은 Node와 Python 기반 프로젝트 모두에 대해 [단일 버전 정책](https://nx.dev/concepts/decisions/dependency-management#single-version-policy)을 사용합니다.

이는 모노레포 내의 모든 프로젝트가 기본적으로 동일한 버전의 종속성을 사용한다는 의미이며, 동일한 모노레포 내에서 패키지 간 버전 불일치 문제를 줄입니다.

Node 관점에서 이는 루트에 단일 잠금 파일과 모든 종속성을 포함하는 단일 `node_modules`가 있음을 의미합니다. 새로운 종속성을 추가해야 하는 경우 루트의 `package.json`에서 수행합니다.

Python 관점에서 이는 모노레포 루트에 모든 종속성이 설치된 단일 `.venv`가 있음을 의미합니다. 그러나 각 Python 프로젝트는 자체 `pyproject.toml`을 가지며, 해당 종속성의 버전은 uv 워크스페이스에 의해 관리되고 루트의 `uv.lock` 파일에 기록됩니다.

## 타입 안전성

`@aws/nx-plugin`은 IDE 자동 완성을 통해 개발자 경험을 간소화하고, 타입 안전하지 않은 구현에서만 발생할 수 있는 런타임 오류를 제거하기 위해 타입 안전성을 활용합니다. 따라서 제공되는 모든 컴포넌트는 기본적으로 타입 안전합니다.

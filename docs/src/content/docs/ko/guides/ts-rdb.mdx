이 생성기는 [Amazon Aurora](https://aws.amazon.com/rds/aurora/) PostgreSQL을 기반으로 하는 새로운 관계형 데이터베이스 프로젝트를 생성합니다. AWS CDK 또는 Terraform을 사용하여 데이터베이스를 프로비저닝하고 관리하는 데 필요한 애플리케이션 코드와 인프라를 생성합니다.

생성된 인프라에는 다음이 포함됩니다:

- 재사용 가능한 Aurora 구성 요소 또는 모듈
- 애플리케이션별 데이터베이스 구성 요소 또는 모듈
- IAM 인증 애플리케이션 데이터베이스 사용자 핸들러
- AWS AppConfig에 런타임 구성 등록
- 자동 Prisma 마이그레이션 배포를 위한 마이그레이션 핸들러

## 데이터베이스 구현

### 데이터 모델링

생성된 프로젝트는 데이터베이스 스키마를 정의하고 타입 안전 클라이언트를 생성하기 위해 [Prisma ORM](https://www.prisma.io/docs/orm)을 사용합니다. `prisma/schema.prisma`에서 테이블, 관계 및 필드 타입을 모델링합니다.

예제 `User` 모델:

```ts title="packages/postgres/prisma/schema.prisma"
model User {
  id          Int      @id @default(autoincrement()) 
  firstName   String
  lastName    String
}
```

자세한 내용은 공식 [Prisma data modelling guide](https://www.prisma.io/docs/orm/core-concepts/data-modeling#data-modeling-with-prisma-orm)를 참조하세요.

### 데이터베이스 클라이언트 생성

생성기는 프로젝트를 빌드할 때마다 타입 안전 TypeScript Prisma 클라이언트를 생성하도록 `generate` 타겟을 자동으로 구성합니다. 클라이언트는 `generated/prisma`에 작성됩니다(`.gitignore`에 추가됨).

언제든지 수동으로 클라이언트를 생성할 수도 있습니다:


  </TabItem>
  <TabItem label="pnpm" icon="seti:pnpm">

```bash

```

  </TabItem>
  <TabItem label="yarn" icon="seti:yarn">

  </TabItem>
  <TabItem label="bun" icon="seti:bun">

`src/prisma.ts`의 런타임 래퍼는 다음을 내보냅니다:

- `DB_PACKAGE_NAME` - AWS AppConfig의 `database` 런타임 구성 네임스페이스에서 사용되는 키
- `getPrisma()` - AWS AppConfig에서 데이터베이스 연결 설정을 로드하고 IAM 인증을 사용하여 Prisma 클라이언트를 생성합니다

클라이언트는 자동으로:
- `RUNTIME_CONFIG_APP_ID` 환경 변수를 사용하여 AWS AppConfig에서 데이터베이스 구성을 검색합니다
- IAM 인증을 위해 AWS RDS Signer를 통해 임시 인증 토큰을 생성합니다
- 인증서 검증과 함께 SSL/TLS 연결을 관리합니다
- 영구 데이터베이스 연결 풀을 통해 연결 풀링을 처리합니다

tRPC API에서의 사용 예시:

```ts title="packages/api/src/procedures/list-user.ts"
import { getPrisma } from ':aws-nx-poc/postgres';
import { publicProcedure } from '../init.js';
import { ListUserOutputSchema } from '../schema/index.js';

export const listUser = publicProcedure
  .output(ListUserOutputSchema)
  .query(async () => {
    const prisma = await getPrisma();
    const users = await prisma.user.findMany({
      orderBy: { id: 'asc' },
    });

    return { users };
  });
```

### 마이그레이션 생성

:::note[패키지 디렉토리에서 Prisma 실행]

:::

Prisma Migrate는 스키마를 현재 데이터베이스 상태와 비교하고 SQL 변경 사항을 생성하기 위해 [shadow database](https://www.prisma.io/docs/orm/prisma-migrate/understanding-prisma-migrate/shadow-database)가 필요합니다.

```bash
export DATABASE_URL="postgresql://username:password@hostname:5432/dbname"
```

## AWS에 배포

관계형 데이터베이스 생성기는 선택한 `iacProvider`에 따라 CDK 또는 Terraform 인프라를 생성합니다.

<Infrastructure>
<Fragment slot="cdk">
CDK 구성 요소는 `common/constructs`에 생성됩니다. 사용 예시:

```ts title="packages/infra/src/stacks/application-stack.ts"
import { Postgres } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const vpc = new Vpc(this, 'Vpc', {
      subnetConfiguration: [
        {
          name: 'public',
          subnetType: SubnetType.PUBLIC,
        },
        {
          name: 'private_with_egress',
          subnetType: SubnetType.PRIVATE_WITH_EGRESS,
        },
        {
          name: 'private_isolated',
          subnetType: SubnetType.PRIVATE_ISOLATED,
        },
      ],
    });

    const db = new Postgres(this, 'Db', {
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_ISOLATED,
      }
    });
  }
}
```

이것은 RDS Proxy, 관리자 자격 증명, 애플리케이션 데이터베이스 사용자, 런타임 구성 등록 및 마이그레이션 핸들러와 함께 Aurora 클러스터를 프로비저닝합니다. 직접적인 인터넷 액세스를 방지하기 위해 프라이빗 격리 서브넷에 배포하세요.

생성된 인프라는 두 개의 데이터베이스 사용자를 생성합니다:
- **관리자 사용자** - 클러스터 프로비저닝 중에 생성되며 자격 증명은 AWS Secrets Manager에 저장됩니다
- **애플리케이션 사용자** - Lambda 사용자 지정 리소스를 통해 생성되며 IAM 인증이 활성화되고 애플리케이션 데이터베이스에 대한 전체 권한이 있습니다
</Fragment>
<Fragment slot="terraform">
Terraform 모듈은 `common/terraform`에 생성됩니다. 사용 예시:

```hcl title="packages/infra/src/main.tf"
module "postgres" {

```

이것은 RDS Proxy, 관리자 자격 증명, create-db-user Lambda, 런타임 구성 등록, 마이그레이션 Lambda 및 컨테이너 레지스트리 리소스와 함께 Aurora 클러스터를 프로비저닝합니다.

생성된 인프라는 두 개의 데이터베이스 사용자를 생성합니다:
- **관리자 사용자** - 클러스터 프로비저닝 중에 생성되며 자격 증명은 AWS Secrets Manager에 저장됩니다
- **애플리케이션 사용자** - Lambda 함수를 통해 생성되며 IAM 인증이 활성화되고 애플리케이션 데이터베이스에 대한 전체 권한이 있습니다

### Lambda 함수 연결

Lambda 함수에서 Prisma 클라이언트를 사용하려면 다음 요구 사항이 충족되는지 확인하세요:

1. 데이터베이스와 동일한 VPC에 함수 배포
2. 데이터베이스 포트에 대한 네트워크 액세스 허용
3. 생성된 런타임 사용자에 대한 IAM `rds-db:connect` 권한 부여
4. 필수 환경 변수 설정:
   - `NODE_EXTRA_CA_CERTS=/var/runtime/ca-cert.pem` (TLS 검증에 필요)
   - `RUNTIME_CONFIG_APP_ID` (생성된 인프라에 의해 자동으로 설정됨)
   - `AWS_REGION` (Lambda 실행 환경에서 자동으로 설정됨)

:::note[Prisma CLI vs 런타임 클라이언트]
**Prisma CLI** 명령(`prisma migrate dev`, `prisma migrate deploy`)은 환경 변수에서 `DATABASE_URL`을 읽습니다.

**런타임 클라이언트** (`src/prisma.ts`)는 AWS AppConfig에서 연결 설정을 로드하고 IAM 인증을 사용합니다.
:::

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { RuntimeConfig } from ':my-scope/common-constructs';

const runtimeConfig = RuntimeConfig.ensure(this);

const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_WITH_EGRESS,
      },
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});

Object.entries(api.integrations).forEach(([operation, integration]) => {
  db.allowDefaultPortFrom(
    integration.handler,
    `Allow ${operation} to connect to the database`,
  );
  db.grantConnect(integration.handler);
});
```

이것은 각 API Lambda 함수가 동일한 VPC에서 실행되고, AppConfig에서 런타임 구성을 읽고, IAM 인증을 사용하여 데이터베이스에 연결하도록 구성합니다.
</Fragment>
<Fragment slot="terraform">
Lambda 함수를 데이터베이스에 연결할 때 다음 모듈 출력을 사용하세요:

- `appconfig_application_id` - 컴퓨팅 환경에 노출
- `cluster_port` - 네트워크 액세스 허용
- `database_runtime_user` - `rds-db:connect` 권한 부여

컴퓨팅이 동일한 VPC 및 프라이빗 서브넷에서 실행되는지 확인하고, 컴퓨팅 역할에 AppConfig 읽기 권한을 부여하세요.
</Fragment>
</Infrastructure>

### RDS Proxy

생성된 인프라에는 기본적으로 RDS Proxy가 포함되어 있으며, 이는 애플리케이션과 Aurora 클러스터 사이에 위치합니다. RDS Proxy는 여러 이점을 제공합니다:

- **연결 풀링** - 애플리케이션 인스턴스 간에 공유할 수 있는 데이터베이스 연결 풀을 유지하여 새 연결 설정의 오버헤드를 줄입니다
- **연결 복원력** - Aurora 인스턴스 교체 또는 유지 관리 중 장애 조치 및 재연결을 자동으로 처리합니다
- **IAM 인증** - IAM 기반 데이터베이스 인증을 지원하여 애플리케이션 코드에서 데이터베이스 자격 증명을 관리할 필요가 없습니다
- **보안 향상** - 모든 연결에 대해 TLS 암호화를 강제합니다

<Infrastructure>
<Fragment slot="cdk">

기본적으로 RDS Proxy가 활성화되어 있습니다. 생성된 런타임 클라이언트(`getPrisma()`)는 프록시 엔드포인트를 통해 자동으로 연결됩니다. 필요한 경우 비활성화할 수 있습니다:

```ts title="packages/infra/src/stacks/application-stack.ts"
const db = new Postgres(this, 'Db', {
  ...
  enableRdsProxy: false,
});
```

RDS Proxy가 비활성화되면 애플리케이션이 Aurora 클러스터 엔드포인트에 직접 연결됩니다.
</Fragment>
<Fragment slot="terraform">

기본적으로 RDS Proxy가 활성화되어 있습니다. 생성된 런타임 클라이언트(`getPrisma()`)는 프록시 엔드포인트를 통해 자동으로 연결됩니다. 필요한 경우 비활성화할 수 있습니다:

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"
  ...
  enable_rds_proxy = false
}
```

RDS Proxy가 비활성화되면 애플리케이션이 Aurora 클러스터 엔드포인트에 직접 연결됩니다. 프록시는 TLS 암호화를 요구하고 IAM 인증을 사용하도록 구성되어 있습니다.
</Fragment>
</Infrastructure>

## 데이터베이스 아키텍처 사용자 정의

생성된 인프라는 워크로드 요구 사항에 맞게 사용자 정의할 수 있습니다. 다음 예시는 사용 가능한 몇 가지 일반적인 사용자 정의 옵션을 보여줍니다.

### 클러스터 인스턴스

Aurora 클러스터의 writer 및 reader 인스턴스를 구성합니다.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
const db = new Postgres(this, 'Db', {
  ...
  writer: ClusterInstance.serverlessV2('writer'),
  readers: [ClusterInstance.serverlessV2('reader')],
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"
  ...
  instance_count = 2  # 1 writer + 1 reader
}
```
</Fragment>
</Infrastructure>

</Fragment>
</Infrastructure>

### 삭제 보호

실수로 인한 삭제를 방지하기 위해 수명 주기 설정을 구성합니다.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { RemovalPolicy } from 'aws-cdk-lib';

const db = new Postgres(this, 'Db', {
  ...
  deletionProtection: true,
  removalPolicy: RemovalPolicy.SNAPSHOT,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"
  ...
  deletion_protection = true
  skip_final_snapshot = false
}
```
</Fragment>
</Infrastructure>

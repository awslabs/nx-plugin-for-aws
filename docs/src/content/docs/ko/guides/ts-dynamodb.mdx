이 생성기는 [Amazon DynamoDB](https://aws.amazon.com/dynamodb/)를 기반으로 하는 새로운 TypeScript 프로젝트를 생성하며, 타입 안전한 엔티티 모델링을 위해 [ElectroDB](https://electrodb.dev/)를 사용합니다. AWS CDK 또는 Terraform을 사용하여 DynamoDB 테이블을 프로비저닝하고 관리하는 데 필요한 애플리케이션 코드와 인프라를 생성하며, 단일 테이블 설계 지원과 DynamoDB Local을 통한 로컬 개발 환경을 제공합니다.

## 로컬 개발

### 로컬 DynamoDB 시작

생성기는 Docker를 통해 [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html) 인스턴스를 시작하고 테이블을 생성하는 `serve-local` 타겟을 구성합니다:

<NxCommands commands={['run <project-name>:serve-local']} />

이 명령은 자동으로:
1. DynamoDB Local Docker 이미지를 가져옵니다 (`docker-pull` 타겟)
2. Docker 컨테이너를 시작합니다
3. 사전 정의된 인덱스로 로컬 테이블을 생성합니다

### 데이터 모델링

생성된 프로젝트는 단일 DynamoDB 테이블에서 타입 안전한 엔티티 모델링을 위해 [ElectroDB](https://electrodb.dev/)를 사용합니다. 생성된 예제 엔티티를 시작점으로 사용하여 `src/entities/` 아래에 엔티티 파일을 추가하거나 업데이트하세요.

엔티티 정의 예제:

```ts title="packages/my-table/src/entities/user.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createUserEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'user',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: { type: 'string', required: true },
        email: { type: 'string', required: true },
        createdAt: {
          type: 'string',
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: { field: 'pk', composite: ['id'] },
          sk: { field: 'sk', composite: [] },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

자세한 내용은 [ElectroDB 엔티티 문서](https://electrodb.dev/en/modeling/entities/)를 참조하세요.

### DynamoDB 클라이언트 사용

생성된 `src/client.ts`는 두 가지 주요 유틸리티를 내보냅니다:

- `getDynamoDBClient()` — 캐시된 싱글톤 `DynamoDBClient`를 반환합니다. `SERVE_LOCAL=true`일 때는 로컬 DynamoDB Local 인스턴스에 연결하고, 그렇지 않으면 기본 자격 증명 체인을 사용하여 AWS 클라이언트를 생성합니다.
- `resolveTableName()` — DynamoDB 테이블 이름을 반환합니다. `SERVE_LOCAL=true`일 때는 로컬 테이블 이름 상수를 반환하고, 그렇지 않으면 `RUNTIME_CONFIG_APP_ID` 환경 변수를 사용하여 AWS AppConfig에서 이름을 가져와 후속 호출을 위해 캐시합니다.

## 글로벌 보조 인덱스 추가/제거

먼저 `src/gsi.ts`의 `GLOBAL_SECONDARY_INDEXES`를 업데이트하세요 — 이것이 GSI 정의의 단일 진실 공급원이며, 다음 `serve-local` 실행 시 스크립트가 자동으로 로컬 테이블의 인덱스를 추가하거나 제거하여 새 목록을 반영합니다. CDK를 사용하는 경우 이러한 변경 사항은 다음 배포 시 AWS 테이블에도 자동으로 반영됩니다.

<OptionFilter when={{ iacProvider: 'Terraform' }}>
  CDK와 달리 `src/gsi.ts`의 변경 사항은 Terraform 모듈에 자동으로 반영되지 않습니다. 배포하기 전에 해당 GSI 정의를 추가하거나 제거하도록 Terraform 모듈도 수동으로 업데이트해야 합니다.
</OptionFilter>

## 테이블 연결

모든 TypeScript 프로젝트에서 DynamoDB 패키지의 엔티티 팩토리를 가져와 직접 사용할 수 있습니다:

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

## 테이블 배포

DynamoDB 생성기는 선택한 `iacProvider`에 따라 CDK 또는 Terraform 인프라를 생성합니다.

<Infrastructure>
<Fragment slot="cdk">
CDK 구성은 `common/constructs`에 생성됩니다. 사용 예제:

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const table = new MyTable(this, 'Table');
  }
}
```

다음과 같은 DynamoDB 테이블을 프로비저닝합니다:
- `pk` (파티션 키) 및 `sk` (정렬 키), 둘 다 `String` 타입
- 온디맨드 (`PAY_PER_REQUEST`) 요금 청구
- 자동 키 교체가 활성화된 고객 관리형 KMS 암호화
- 특정 시점 복구 활성화
- 삭제 보호 활성화
- AWS AppConfig의 `dynamodb` 네임스페이스 아래 Runtime Config에 테이블 이름 등록
</Fragment>

### 삭제 보호

삭제 보호는 기본적으로 활성화되어 실수로 테이블이 삭제되는 것을 방지합니다.

#### 삭제 보호 비활성화

단기 개발 또는 프리뷰 스택과 같이 테이블 삭제가 예상되는 환경에서는 비활성화하세요.

<Fragment slot="cdk">

### 요금 청구 모드

테이블은 기본적으로 온디맨드 (`PAY_PER_REQUEST`) 요금 청구를 사용합니다. 예측 가능한 고처리량 워크로드의 경우 프로비저닝된 용량으로 전환하세요.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { BillingMode } from 'aws-cdk-lib/aws-dynamodb';
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  billingMode: BillingMode.PROVISIONED,
  readCapacity: 5,
  writeCapacity: 5,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source       = "../../common/terraform/src/app/dynamodb/my-table"
  billing_mode = "PROVISIONED"
}
```
</Fragment>
</Infrastructure>

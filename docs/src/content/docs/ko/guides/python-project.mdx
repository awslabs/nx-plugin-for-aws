Python 프로젝트 생성기는 [Python](https://www.python.org/)의 최신 라이브러리 또는 애플리케이션을 생성하는 데 사용할 수 있으며, [UV](https://docs.astral.sh/uv/)로 관리되고, 단일 락파일과 가상 환경의 [UV 워크스페이스](https://docs.astral.sh/uv/concepts/projects/workspaces/)에서 구성되며, 테스트 실행을 위한 [pytest](https://docs.pytest.org/en/stable/), 정적 분석을 위한 [Ruff](https://docs.astral.sh/ruff/)를 포함합니다.

## 생성기 출력

생성기는 `<directory>/<name>` 디렉토리에 다음과 같은 프로젝트 구조를 생성합니다:

<FileTree>

  - \<module-name>
    - \_\_init\_\_.py 모듈 초기화
    - hello.py 예시 Python 소스 파일
  - tests
    - \_\_init\_\_.py 모듈 초기화
    - conftest.py 테스트 구성
    - test_hello.py 예시 테스트
  - project.json 프로젝트 구성 및 빌드 대상
  - pyproject.toml UV에서 사용하는 패키징 구성 파일
  - .python-version 프로젝트의 Python 버전 포함

</FileTree>

작업 공간의 루트에 다음 파일들이 생성/업데이트될 수 있습니다:

<FileTree>

  - pyproject.toml UV용 작업 공간 수준 패키징 구성
  - .python-version 작업 공간 Python 버전 포함
  - uv.lock Python 종속성 잠금 파일

</FileTree>

## Python 소스 코드 작성하기

`<모듈-이름>` 디렉토리에 Python 소스 코드를 추가하세요.

### 다른 프로젝트에서 라이브러리 코드 가져오기

[UV 워크스페이스](https://docs.astral.sh/uv/concepts/projects/workspaces/)가 이미 설정되어 있으므로, 워크스페이스의 다른 Python 프로젝트에서 Python 프로젝트를 참조할 수 있습니다:

```python title="packages/my_other_project/my_other_project/main.py"
from "my_library.hello" import say_hello
```

위에서 `my_library`는 모듈 이름, `hello`는 Python 소스 파일 `hello.py`에 해당하며, `say_hello`는 `hello.py`에 정의된 메서드입니다.

### 의존성

프로젝트에 의존성을 추가하려면 Python 프로젝트에서 `add` 대상을 실행할 수 있습니다. 예를 들어:

<NxCommands commands={['run my_scope.my_library:add some-pip-package']} />

이렇게 하면 프로젝트의 `pyproject.toml` 파일에 의존성이 추가되고 루트 `uv.lock`이 업데이트됩니다.

#### 런타임 코드

Python 프로젝트를 런타임 코드(예: AWS 람다 함수의 핸들러)로 사용할 때는 소스 코드와 모든 의존성의 번들을 생성해야 합니다. 이를 위해 `project.json` 파일에 다음과 같은 대상을 추가할 수 있습니다:

```json title="project.json"
{
  ...
  "targets": {
    ...
    "bundle": {
      "cache": true,
      "executor": "nx:run-commands",
      "outputs": ["{workspaceRoot}/dist/packages/my_library/bundle"],
      "options": {
        "commands": [
          "uv export --frozen --no-dev --no-editable --project my_library -o dist/packages/my_library/bundle/requirements.txt",
          "uv pip install -n --no-installer-metadata --no-compile-bytecode --python-platform x86_64-manylinux2014 --python `uv python pin` --target dist/packages/my_library/bundle -r dist/packages/my_library/bundle/requirements.txt"
        ],
        "parallel": false
      },
      "dependsOn": ["compile"]
    },
  },
}
```

### 빌드

Python 프로젝트는 `project.json`에 정의된 `build` 대상으로 구성되어 있으며, 다음과 같이 실행할 수 있습니다:

<NxCommands commands={['run <project-name>:build']} />

여기서 `<project-name>`은 프로젝트의 정규화된 이름입니다.

`build` 대상은 프로젝트를 컴파일, 린트 및 테스트합니다.

빌드 출력은 워크스페이스의 루트 `dist` 폴더에서 찾을 수 있으며, 패키지 및 대상의 디렉토리 내부에 있습니다. 예: `dist/packages/<my-library>/build`

## 테스트

[pytest](https://docs.pytest.org/en/stable/)가 프로젝트 테스트를 위해 구성되어 있습니다.

### 테스트 작성

테스트는 프로젝트의 `test` 디렉토리 내에 `test_` 접두사가 붙은 Python 파일에 작성해야 합니다. 예를 들어:

<FileTree>
  - my_library
    - hello.py
  - test
    - test_hello.py hello.py에 대한 테스트
</FileTree>

테스트는 `test_`로 시작하는 메서드이며, 기대 사항을 확인하기 위해 어설션을 만듭니다. 예를 들어:

```python title="test/test_hello.py"
from my_library.hello import say_hello

def test_say_hello():
  assert say_hello("Darth Vader") == "Hello, Darth Vader!"
```

테스트 작성 방법에 대한 자세한 내용은 [pytest 문서](https://docs.pytest.org/en/stable/how-to/assert.html#)를 참조하세요.

### 테스트 실행

테스트는 프로젝트의 `build` 대상의 일부로 실행되지만, `test` 대상을 실행하여 별도로 실행할 수도 있습니다:

<NxCommands commands={['run <project-name>:test']} />

`-k` 플래그를 사용하여 테스트 파일 또는 메서드 이름을 지정하여 개별 테스트 또는 테스트 모음을 실행할 수 있습니다:

<NxCommands commands={["run <project-name>:test -k 'test_say_hello'"]} />

## 린팅

Python 프로젝트는 린팅을 위해 [Ruff](https://docs.astral.sh/ruff/)를 사용합니다.

### 린터 실행하기

프로젝트를 검사하기 위해 린터를 호출하려면 `lint` 대상을 실행할 수 있습니다.

<NxCommands commands={["run <project-name>:lint"]} />

### 린트 문제 수정하기

대부분의 린팅 또는 포맷팅 문제는 자동으로 수정할 수 있습니다. `--configuration=fix` 인수와 함께 실행하여 Ruff에 린트 문제를 수정하도록 지시할 수 있습니다.

<NxCommands commands={["run <project-name>:lint --configuration=fix"]} />

마찬가지로 작업 공간의 모든 패키지에서 모든 린트 문제를 수정하려면 다음을 실행할 수 있습니다:

<NxCommands commands={["run-many --target lint --all --configuration=fix"]} />

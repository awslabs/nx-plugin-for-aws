완전한 TypeScript [Nx Plugin](https://nx.dev/extending-nx/intro/getting-started)을 통합 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 서버와 함께 생성합니다. 이 생성기는 AI 어시스턴트가 자동 코드 생성 및 프로젝트 스캐폴딩에 사용할 수 있는 커스텀 제너레이터 구축을 위한 기반을 제공합니다.

## 사용 방법

### Nx 플러그인 생성

TypeScript Nx 플러그인을 두 가지 방법으로 생성할 수 있습니다:

<RunGenerator generator="ts#nx-plugin" />

### 옵션

<GeneratorParameters generator="ts#nx-plugin" />

## 생성기 출력

생성기는 다음 프로젝트 구조를 생성합니다:

<FileTree>
- your-plugin/
  - src/
    - index.ts 플러그인의 빈 진입점
    - mcp-server/
      - index.ts 서버 내보내기
      - server.ts 주요 MCP 서버 구성
      - stdio.ts STDIO 전송을 사용한 MCP 서버 진입점
      - http.ts Streamable HTTP 전송을 사용한 MCP 서버 진입점
      - tools/
        - create-workspace-command.ts 워크스페이스 생성 가이드 도구
        - general-guidance.ts Nx 및 플러그인 일반 가이드 도구
        - list-generators.ts 사용 가능한 제너레이터 목록 도구
        - generator-guide.ts 상세 제너레이터 정보 도구
      - utils.ts MCP 서버 유틸리티 함수
  - generators.json Nx 제너레이터 구성 (초기 빈 상태)
  - package.json MCP 서버 바이너리가 포함된 플러그인 패키지 구성
  - tsconfig.json TypeScript 구성 (Nx 호환성을 위한 CommonJS)
  - project.json 빌드 및 패키지 타겟이 포함된 Nx 프로젝트 구성
</FileTree>

## Nx 플러그인 작업

### 제너레이터 추가

플러그인 프로젝트 생성 후 <Link path="/guides/ts-nx-generator">`ts#nx-generator`</Link> 생성기를 사용하여 제너레이터를 추가할 수 있습니다:

<RunGenerator generator="ts#nx-generator" requiredParameters={{ pluginProject: 'your-plugin' }} />

이렇게 하면 플러그인에 새 제너레이터가 추가됩니다.

:::tip
제너레이터 구현 방법에 대한 상세 내용은 <Link path="/guides/ts-nx-generator">`ts#nx-generator` 가이드</Link>를 참조하세요.
:::

MCP 서버의 `generator-guide` 도구에서 사용되므로 제너레이터에 대한 상세한 `README.md` 작성을 권장합니다.

### 프로젝트 타겟

생성된 플러그인에는 다음과 같은 주요 타겟이 포함됩니다:

#### 빌드 타겟
TypeScript 코드를 컴파일하고 플러그인 사용 준비:

<NxCommands commands={['build your-plugin']} />

#### 패키지 타겟
NPM 배포 준비가 된 배포용 패키지 생성:

<NxCommands commands={['build your-plugin']} />

패키지 타겟에는 다음 자산이 포함됩니다:
- 컴파일된 JavaScript 파일
- TypeScript 선언 파일
- 제너레이터 및 실행기 구성 파일
- 문서 및 라이선스 파일

## MCP 서버 통합

플러그인은 <Link path="/guides/ts-mcp-server">`ts#mcp-server`</Link> 생성기를 사용하여 MCP 서버를 구성합니다.

### 도구

플러그인에 포함된 MCP 서버는 AI 어시스턴트에 다음 도구를 제공합니다:

- `general-guidance`: Nx 및 플러그인 작업 모범 사례
- `create-workspace-command`: 플러그인 사용 가능한 새 워크스페이스 생성 방법
- `list-generators`: 플러그인 내 사용 가능한 모든 제너레이터 목록
- `generator-guide`: 스키마 및 사용법을 포함한 특정 제너레이터 상세 정보

## 플러그인 배포

### 배포 준비

1. **패키지 정보 업데이트**: 플러그인의 [`package.json`](https://docs.npmjs.com/cli/v10/configuring-npm/package-json)에 적절한 이름, 버전, 설명, 저장소 정보를 입력하세요.

2. **패키지 빌드**:
   <NxCommands commands={['package your-plugin']} />

3. **로컬 테스트**: 다른 워크스페이스에 설치하여 플러그인 테스트 가능:
   ```bash
   npm install /path/to/your/workspace/dist/your-plugin/package
   ```

### NPM 배포

플러그인 준비 완료 시:

```bash
cd dist/your-plugin/package
npm publish
```

### 배포된 플러그인 사용

배포 후 다른 사용자가 플러그인 설치 및 사용 가능:

<InstallCommand pkg="your-plugin-name" />

제너레이터 사용:

<RunGenerator namespace="your-plugin-name" generator="your-generator-name" />

#### 배포된 플러그인으로 AI 어시스턴트 구성

배포된 플러그인의 MCP 서버는 다음과 같이 구성할 수 있습니다:

```json {3-6}
{
  "mcpServers": {
    "aws-nx-mcp": {
      "command": "npx",
      "args": ["-y", "-p", "your-plugin-name", "your-plugin-name-mcp-server"]
    }
  }
}
```

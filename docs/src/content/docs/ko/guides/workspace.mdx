`@aws/nx-plugin`으로 새 워크스페이스를 생성하면, 프리셋 생성기가 AWS에서 빌드하기 위한 합리적인 기본값으로 [Nx monorepo](https://nx.dev)를 설정합니다 — TypeScript, Prettier, 워크스페이스 레이아웃 및 보안 도구를 포함합니다.

## 워크스페이스 생성

<CreateNxWorkspaceCommand workspace="my-project" />

### 옵션

| 매개변수 | 타입 | 기본값 | 설명 |
|-----------|------|---------|-------------|
| `iacProvider` | `CDK` \| `Terraform` | — | Infrastructure as Code 공급자 선호도 |
| `gitSecrets` | `boolean` | `true` | AWS 자격 증명 커밋을 방지하기 위한 git-secrets pre-commit 훅 구성 |

<Infrastructure>
<Fragment slot="cdk">
<CreateNxWorkspaceCommand workspace="my-project" iacProvider="CDK" />
</Fragment>
<Fragment slot="terraform">
<CreateNxWorkspaceCommand workspace="my-project" iacProvider="Terraform" />
</Fragment>
</Infrastructure>

git-secrets 스캔을 비활성화하려면:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

## 워크스페이스 구조

<FileTree>
  - packages/ 프로젝트가 여기에 위치합니다
  - .git-secrets/
    - git-secrets 자격 증명 스캔을 위한 벤더링된 git-secrets bash 스크립트
    - setup.sh 로컬 git 설정에 AWS 자격 증명 패턴 등록
  - .husky/
    - pre-commit 각 커밋 전에 git-secrets 스캔 실행
  - nx.json Nx 구성 (타겟, 동기화 생성기, 분석)
  - package.json 스크립트, devDependencies 및 prepare 훅
  - tsconfig.json 루트 TypeScript 구성
  - aws-nx-plugin.config.mts 플러그인 구성 (IaC 공급자, 라이선스 등)
</FileTree>

## Git Secrets

새 워크스페이스에는 각 커밋 전에 스테이징된 파일에서 AWS 자격 증명 패턴을 스캔하는 [git-secrets](https://github.com/awslabs/git-secrets) pre-commit 훅이 포함되어 있습니다. 이는 액세스 키, 시크릿 키 및 기타 민감한 값을 실수로 커밋하는 것을 방지합니다.

### 작동 방식

<Steps>
1. `prepare` 스크립트는 패키지 설치 후 실행되며, husky git 훅을 초기화하고 로컬 git 설정에 AWS 자격 증명 패턴을 등록합니다.

2. 각 `git commit`에서 husky pre-commit 훅이 git-secrets를 호출하여 스테이징된 파일을 스캔합니다.

3. AWS 자격 증명과 일치하는 패턴이 감지되면 커밋이 차단됩니다.
</Steps>

### 감지되는 패턴

- **AWS Access Key IDs** — `AKIA...`, `ASIA...` 및 기타 IAM 키 접두사
- **AWS Secret Access Keys** — `SECRET_ACCESS_KEY` 레이블이 있는 40자 base64 문자열
- **AWS Account IDs** — `ACCOUNT_ID` 레이블이 있는 12자리 숫자
- **Bedrock API Keys** — 장기(`ABSK...`) 및 단기 키
- **Credentials file values** — `~/.aws/credentials`의 리터럴 키 값

잘 알려진 예제 키(예: `AKIAIOSFODNN7EXAMPLE`)는 자동으로 허용 목록에 추가됩니다.

### 오탐 허용

```sh
# 특정 패턴 허용
git config --add secrets.allowed 'my-pattern-here'
```

또는 저장소 루트에 `.gitallowed` 파일에 패턴을 추가합니다(한 줄에 하나의 정규식).

일회성 재정의의 경우:

```sh
git commit --no-verify
```

### 요구사항

- **Git**이 워크스페이스에서 초기화되어야 합니다(`git init`)
- **Bash**가 PATH에서 사용 가능해야 합니다(Windows에서는 Git for Windows와 함께 제공됨)
- 패턴은 `prepare`가 실행될 때 등록됩니다 — 복제 후 패키지 매니저의 설치 명령을 실행하여 다시 등록하세요

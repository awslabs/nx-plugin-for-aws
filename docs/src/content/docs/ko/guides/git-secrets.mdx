`@aws/nx-plugin`으로 생성된 새 워크스페이스에는 기본적으로 [git-secrets](https://github.com/awslabs/git-secrets) pre-commit 훅이 구성되어 있습니다. 이는 각 커밋 전에 스테이징된 파일에서 AWS 자격 증명과 일치하는 패턴을 스캔하여 실수로 자격 증명이 유출되는 것을 방지합니다.

## 포함된 내용

새 워크스페이스를 생성하면 다음이 자동으로 설정됩니다:

<FileTree>
  - .git-secrets/
    - git-secrets The git-secrets bash script (vendored from awslabs/git-secrets)
    - setup.sh Registers AWS credential patterns in local git config
  - .husky/
    - pre-commit Runs git-secrets scan before each commit
  - package.json Includes husky and prepare script for automatic setup
</FileTree>

## 작동 방식

1. `prepare` 스크립트는 `npm install` / `pnpm install` 후에 실행되며, husky git 훅을 초기화하고 로컬 git 구성에 AWS 자격 증명 패턴을 등록합니다.
2. 각 `git commit`마다 husky pre-commit 훅이 `git-secrets`를 호출하여 스테이징된 파일을 스캔합니다.
3. AWS 자격 증명과 일치하는 패턴이 감지되면 오류 메시지와 함께 커밋이 차단됩니다.

## 감지되는 패턴

다음 패턴들이 스캔됩니다:

- **AWS Access Key IDs** — `AKIA...`, `ASIA...` 및 기타 IAM 키 접두사 뒤에 16자의 대문자 영숫자 문자가 오는 형식
- **AWS Secret Access Keys** — `SECRET_ACCESS_KEY` 레이블 앞에 오는 40자 base64 문자열
- **AWS Account IDs** — `ACCOUNT_ID` 레이블 앞에 오는 12자리 숫자
- **Bedrock API Keys** — 장기(`ABSK...`) 및 단기 Bedrock 키
- **Credentials file values** — `~/.aws/credentials`에서 찾은 리터럴 키 값

잘 알려진 예제 키(예: `AKIAIOSFODNN7EXAMPLE`)는 자동으로 허용 목록에 추가됩니다.

## 오탐 허용하기

git-secrets가 오탐을 포함한 커밋을 차단하는 경우:

```sh
# Allow a specific pattern
git config --add secrets.allowed 'my-pattern-here'
```

또는 저장소 루트에 `.gitallowed` 파일을 추가하여 패턴을 추가할 수 있습니다(한 줄에 하나의 정규식).

일회성 우회의 경우:

```sh
git commit --no-verify
```

## 비활성화하기

git-secrets 없이 워크스페이스를 생성하려면:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

또는 프리셋 생성기에 `--gitSecrets=false`를 전달하세요.

## 요구 사항

- 워크스페이스에서 **Git**이 초기화되어 있어야 합니다(`git init`)
- **Bash**가 PATH에서 사용 가능해야 합니다(Windows에서는 Git for Windows와 함께 제공됨)
- 패턴은 `prepare` 실행 시 등록됩니다 — 새로 복제한 경우 패키지 매니저의 install 명령을 실행하여 다시 등록하세요

`connection` 생성기는 <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link>를 원격 [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) 에이전트(<Link path="guides/ts-strands-agent">TypeScript</Link> 또는 <Link path="guides/py-strands-agent">Python</Link>)에 연결하여, 에이전트가 다른 에이전트를 도구로 위임할 수 있도록 합니다.

생성기는 에이전트가 AWS에 배포되었을 때(Bedrock AgentCore를 통해)와 로컬에서 실행될 때 모두 원격 A2A 에이전트를 검색하고 호출할 수 있도록 필요한 모든 연결을 설정합니다.

## 사전 요구사항

이 생성기를 사용하기 전에 다음을 확인하세요:

1. <Link path="guides/ts-strands-agent">Strands Agent</Link> 컴포넌트(모든 프로토콜)가 있는 TypeScript 프로젝트
2. `--protocol=A2A` 및 `--auth=IAM`으로 생성된 Strands Agent 컴포넌트가 있는 프로젝트(<Link path="guides/ts-strands-agent">`ts#strands-agent`</Link> 또는 <Link path="guides/py-strands-agent">`py#strands-agent`</Link>)
3. `computeType: BedrockAgentCoreRuntime`으로 생성된 두 컴포넌트

## 생성기 출력

생성기는 공유 `agent-connection` 패키지를 생성하고 에이전트 코드를 수정합니다:

<FileTree>

- packages/common/agent-connection
  - src
    - app
      - \<target-agent-name>-client.ts 연결된 A2A 에이전트용 고수준 클라이언트
    - core
      - agentcore-a2a-client.ts SigV4 인증을 사용하는 저수준 AgentCore A2A 클라이언트
    - index.ts 모든 클라이언트 내보내기
  - project.json
  - tsconfig.json

</FileTree>

또한 다음을 수행합니다:
- 에이전트의 `agent.ts`를 변환하여 원격 A2A 에이전트를 Strands `tool`로 등록
- 에이전트의 `serve-local` 타겟을 업데이트하여 타겟 에이전트의 `serve-local` 타겟에 의존하도록 설정
- 필요한 종속성 설치

## 연결된 A2A 에이전트 사용

생성기는 에이전트의 `agent.ts`를 변환하여 원격 A2A 에이전트를 도구로 래핑합니다:

```ts title="packages/example/src/my-agent/agent.ts" {2,5-11,14}
import { Agent, tool } from '@strands-agents/sdk';
import { RemoteAgentClient } from ':my-scope/agent-connection';
import { z } from 'zod';

export const getAgent = async (sessionId: string) => {
  const remoteAgent = await RemoteAgentClient.create(sessionId);
  const remoteAgentTool = tool({
    name: 'askRemoteAgent',
    description: 'Delegate a question to the remote RemoteAgent A2A agent and return its reply.',
    inputSchema: z.object({ prompt: z.string() }),
    callback: async ({ prompt }) => (await remoteAgent.invoke(prompt)).toString(),
  });
  return new Agent({
    systemPrompt: '...',
    tools: [remoteAgentTool],
  });
};
```

`sessionId` 매개변수는 호출자로부터 전달되어 [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html)의 일관성을 보장합니다.

내부적으로 `RemoteAgentClient.create(sessionId)`는 AWS에 배포되었을 때 SigV4 서명 `clientFactory`로 구성된 Strands `A2AAgent`를 반환하고, `SERVE_LOCAL=true`일 때는 일반 `http://localhost:<port>/` 엔드포인트를 반환합니다.

## 로컬 개발

생성기는 호스트 에이전트의 `serve-local` 타겟을 다음과 같이 구성합니다:
1. 연결된 A2A 에이전트를 자동으로 시작
2. `SERVE_LOCAL=true`를 설정하여 생성된 클라이언트가 AgentCore 대신 `http://localhost:<port>/`에 직접 연결하도록 설정

다음 명령으로 에이전트를 로컬에서 실행하세요:

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

이렇게 하면 호스트 에이전트와 연결된 모든 A2A 에이전트가 시작되며, 호스트 에이전트는 할당된 로컬 포트에서 일반 HTTP를 통해 원격 에이전트를 호출합니다.

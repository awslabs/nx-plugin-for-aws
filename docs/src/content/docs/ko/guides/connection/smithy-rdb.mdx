`connection` 생성기는 <Link path="guides/ts-smithy-api">Smithy API</Link>를 <Link path="guides/ts-rdb">관계형 데이터베이스</Link> 프로젝트에 연결하여, 모든 작업 구현이 데이터베이스에 액세스할 수 있도록 서비스 컨텍스트에 Prisma 클라이언트를 주입합니다.

## 사전 요구 사항

이 생성기를 사용하기 전에 다음이 필요합니다:

1. <Link path="guides/ts-smithy-api">Smithy TypeScript API</Link> 프로젝트 (`ts#api`를 `--framework=smithy`와 함께 사용하여 생성)
2. <Link path="guides/ts-rdb">`ts#rdb`</Link> 프로젝트

## 생성기 출력

생성기는 Smithy API 백엔드의 기존 파일 세 개를 수정합니다:

<FileTree>

- packages/api/src
  - context.ts `ServiceContext`에 `db` 속성 추가
  - handler.ts `lambdaHandler` 내부에서 Prisma 클라이언트 생성, `serviceHandler.handle`로 전달
  - local-server.ts 요청 핸들러 내부에서 Prisma 클라이언트 생성, `serviceHandler.handle`로 전달

</FileTree>

또한 API의 `dev` 타겟을 업데이트하여 데이터베이스를 자동으로 시작합니다.

## 작동 방식

### ServiceContext

생성기는 `context.ts`의 `ServiceContext`에 타입이 지정된 `db` 속성을 추가합니다:

```ts title="packages/api/src/context.ts" {1,7}
import { getPrisma as getMyDb } from ':my-scope/my-db';

export interface ServiceContext {
  tracer: Tracer;
  logger: Logger;
  metrics: Metrics;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
}
```

### Lambda Handler

Prisma 클라이언트는 `lambdaHandler` 내부에서 인스턴스화되고 서비스 컨텍스트를 통해 전달됩니다:

```ts title="packages/api/src/handler.ts" {1,10}
import { getPrisma as getMyDb } from '@my-scope/my-db';

export const lambdaHandler = async (event: APIGatewayProxyEvent) => {
  const httpRequest = convertEvent(event);
  const myDb = await getMyDb();
  const httpResponse = await serviceHandler.handle(httpRequest, {
    tracer,
    logger,
    metrics,
    myDb,
  });
  return convertVersion1Response(httpResponse);
};
```

### 작업에서 데이터베이스 사용

작업 구현에서 컨텍스트로부터 `db`에 액세스합니다:

```ts title="packages/api/src/operations/list-users.ts" {8}
import { ListUsersOperationInput, ListUsersOperationOutput } from '../generated/ssdk/index.js';
import { ServiceContext } from '../context.js';

export const listUsers = async (
  input: ListUsersOperationInput,
  ctx: ServiceContext,
): Promise<ListUsersOperationOutput> => {
  const users = await ctx.myDb.user.findMany();
  return { users };
};
```

## 여러 데이터베이스

다른 대상으로 생성기를 다시 실행하면 첫 번째 데이터베이스와 함께 두 번째 데이터베이스가 추가됩니다. 두 클라이언트 모두 `ServiceContext`에 추가되고 `handler.ts`에서 인스턴스화됩니다:

```ts title="packages/api/src/context.ts" {5,6}
export interface ServiceContext {
  tracer: Tracer;
  logger: Logger;
  metrics: Metrics;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
  otherDb: Awaited<ReturnType<typeof getOtherDb>>;
}
```

```ts title="packages/api/src/handler.ts" {1,2,7,8}
const myDb = await getMyDb();
const otherDb = await getOtherDb();
const httpResponse = await serviceHandler.handle(httpRequest, {
  tracer,
  logger,
  metrics,
  myDb,
  otherDb,
});
```

## 로컬 개발

생성기는 `local-server.ts`의 요청 핸들러 내부에 동일한 Prisma 클라이언트 주입을 적용합니다:

```ts title="packages/api/src/local-server.ts" {1,5,10}
import { getPrisma as getMyDb } from '@my-scope/my-db';

const server = createServer(async function (req, res) {
  const httpRequest = convertRequest(req);
  const myDb = await getMyDb();
  const httpResponse = await serviceHandler.handle(httpRequest, {
    tracer,
    logger,
    metrics,
    myDb,
  });
  return writeResponse(httpResponse, res);
});
```

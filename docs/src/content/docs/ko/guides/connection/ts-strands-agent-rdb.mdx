`connection` 생성기는 <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link>를 <Link path="guides/ts-rdb">관계형 데이터베이스</Link> 프로젝트에 연결하여 에이전트의 요청 수명 주기 전반에 걸쳐 Prisma 클라이언트를 사용할 수 있도록 합니다.

주입 전략은 에이전트의 프로토콜에 따라 달라집니다 — **HTTP** (tRPC 기반) 또는 **A2A** (Agent-to-Agent).

## 전제 조건

이 생성기를 사용하기 전에 다음이 필요합니다:

1. <Link path="guides/ts-strands-agent">`ts#strands-agent`</Link> 프로젝트
2. <Link path="guides/ts-rdb">`ts#rdb`</Link> 프로젝트

## 생성기 출력

수정되는 파일은 에이전트의 프로토콜에 따라 다릅니다.

**HTTP 프로토콜** — 네 개의 파일을 수정합니다:

<FileTree>

- packages/my-service/src/my-agent
  - init.ts `Context` 인터페이스에 `db` 속성 추가
  - index.ts `createContext`가 비동기로 변경됨; Prisma 클라이언트가 생성되어 컨텍스트에 반환됨
  - router.ts `opts.ctx`에서 `db`를 구조 분해하여 `getAgent`에 전달
  - agent.ts `getAgent`가 두 번째 매개변수로 `{ db }`를 받도록 업데이트됨

</FileTree>

**A2A 프로토콜** — 두 개의 파일을 수정합니다:

<FileTree>

- packages/my-service/src/my-agent
  - index.ts `A2AExpressServer` 전에 Prisma 클라이언트 생성; `getAgent`에 전달됨
  - agent.ts `getAgent`가 두 번째 매개변수로 `{ db }`를 받도록 업데이트됨

</FileTree>

또한 에이전트의 `<agent-name>-serve-local` 타겟이 데이터베이스의 `serve-local` 타겟에 의존하도록 업데이트됩니다.

## 작동 방식

### HTTP 프로토콜

HTTP 에이전트의 경우 데이터베이스는 tRPC 요청 컨텍스트를 통해 전달됩니다 — `createContext`는 요청당 Prisma 클라이언트를 생성하고 `opts.ctx`를 통해 모든 프로시저에서 사용할 수 있게 됩니다. 그런 다음 라우터가 이를 명시적으로 `getAgent`에 전달합니다.

#### Context 인터페이스

Prisma 클라이언트 타입이 `init.ts`의 tRPC `Context`에 추가됩니다:

```ts title="packages/my-service/src/my-agent/init.ts" {1,5}
import { getPrisma as getMyDb } from ':my-scope/my-db';

export interface Context {
  sessionId: string;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
}
```

#### 요청 컨텍스트

`createContext`가 비동기로 변경되고 Prisma 클라이언트가 인스턴스화되어 반환됩니다:

```ts title="packages/my-service/src/my-agent/index.ts" {1,6,10}
import { getPrisma as getMyDb } from ':my-scope/my-db';

const createContext = async (
  opts: CreateHTTPContextOptions | CreateWSSContextFnOptions,
): Promise<Context> => {
  const myDb = await getMyDb();
  const sessionId = /* ... */;
  return {
    sessionId,
    myDb,
  };
};
```

#### Router와 Agent

데이터베이스는 구독 핸들러를 통해 컨텍스트에서 `getAgent`로 전달됩니다:

```ts title="packages/my-service/src/my-agent/router.ts" {2,3}
.subscription(async function* (opts) {
  const { myDb } = opts.ctx;
  const agent = await getAgent(opts.ctx.sessionId, { myDb });
  // ...
})
```

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

### A2A 프로토콜

A2A 에이전트의 경우 tRPC 컨텍스트가 없습니다. 데이터베이스는 시작 시 한 번 생성되고 `A2AExpressServer`가 시작되기 전에 `getAgent`에 직접 전달됩니다.

#### 서버 시작

Prisma 클라이언트가 `A2AExpressServer` 전에 생성되고 `getAgent`에 전달됩니다:

```ts title="packages/my-service/src/my-agent/index.ts" {1,4,6}
import { getPrisma as getMyDb } from ':my-scope/my-db';

void (async () => {
  const myDb = await getMyDb();
  const server = new A2AExpressServer({
    agent: await getAgent('default', { myDb }),
    // ...
  });
})();
```

#### Agent 정의

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

## 여러 데이터베이스

다른 타겟으로 생성기를 다시 실행하면 첫 번째 데이터베이스와 함께 두 번째 데이터베이스가 추가됩니다. HTTP 에이전트의 경우 둘 다 `Context` 인터페이스에 나타나고 라우터에서 함께 구조 분해됩니다:

```ts title="packages/my-service/src/my-agent/router.ts"
const { postgresDb, mysqlDb } = opts.ctx;
const agent = await getAgent(opts.ctx.sessionId, { postgresDb, mysqlDb });
```

## 인프라

생성된 에이전트 구성은 `IGrantable` 및 `IConnectable`을 구현하므로 구성에서 직접 데이터베이스에 대한 네트워크 및 IAM 액세스를 부여할 수 있습니다.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myAgent = new MyAgent(this, 'MyAgent', { vpc, ... });

db.allowDefaultPortFrom(myAgent);
db.grantConnect(myAgent);
```

`allowDefaultPortFrom`은 에이전트 런타임이 데이터베이스 포트에 도달할 수 있도록 보안 그룹 규칙을 엽니다. `grantConnect`는 에이전트의 실행 역할에 IAM `rds-db:connect` 권한을 부여합니다.

</Fragment>

module "my_database" {

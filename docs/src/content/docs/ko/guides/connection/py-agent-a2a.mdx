`connection` 생성기는 <Link path="guides/py-agent">Python Agent</Link>를 원격 [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) 에이전트(<Link path="guides/ts-agent">TypeScript</Link> 또는 <Link path="guides/py-agent">Python</Link>)에 연결하여, 에이전트가 다른 에이전트를 도구로 위임할 수 있도록 합니다.

생성기는 에이전트가 원격 A2A 에이전트를 발견하고 호출할 수 있도록 필요한 모든 배선을 설정하며, AWS에 배포된 경우(Bedrock AgentCore를 통해)와 로컬에서 실행되는 경우 모두 작동합니다.

## 전제 조건

이 생성기를 사용하기 전에 다음을 준비해야 합니다:

1. <Link path="guides/py-agent">Python Agent</Link> 컴포넌트가 있는 Python 프로젝트 (Strands 또는 LangChain)
2. `--protocol=a2a` 및 `--auth=iam`으로 생성된 Agent 컴포넌트가 있는 프로젝트 (<Link path="guides/ts-agent">`ts#agent`</Link> 또는 <Link path="guides/py-agent">`py#agent`</Link>)
3. `infra: agentcore`으로 생성된 두 컴포넌트

## 생성기 출력

생성기는 `packages/common/agent_connection/`에 공유 `agent_connection` Python 프로젝트를 생성합니다(아직 존재하지 않는 경우). 연결별 클라이언트 모듈이 이 공유 프로젝트에 생성됩니다:

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py 연결별 클라이언트 재내보내기
    - core
      - agentcore\_endpoints.py 프레임워크 독립적인 ARN/URL 해석
      - agentcore\_a2a\_client\_config.py 프레임워크 독립적인 A2A 클라이언트 구성 (서명된 `ClientConfig`)
      - agentcore\_a2a\_client\_\<framework>.py 에이전트의 프레임워크를 위한 구성을 래핑하는 A2A 클라이언트
      - auth/ 프레임워크 독립적인 SigV4 / 세션 전달 `httpx.Auth`
    - app
      - \<target\_agent\_name>\_client\_\<framework>.py 각 A2A 에이전트에 대한 연결별 A2A 클라이언트

</FileTree>

클라이언트 접미사는 에이전트의 프레임워크(`_strands` 또는 `_langchain`)와 일치합니다. 두 클라이언트 모두 동일한 프레임워크 독립적인 서명된 `ClientConfig`를 래핑합니다: Strands 클라이언트는 Strands `A2AAgent`를 래핑하고, LangChain 클라이언트는 [a2a SDK](https://pypi.org/project/a2a-sdk/)를 직접 구동하므로 LangChain 에이전트는 Strands 의존성을 포함하지 않습니다.

또한 생성기는:
- 에이전트의 `agent.py`를 변환하여 원격 A2A 에이전트를 `@tool`을 사용하여 도구로 등록합니다
- `agent_connection` 프로젝트를 에이전트 프로젝트의 워크스페이스 의존성으로 추가합니다
- 에이전트의 `serve-local` 대상을 대상 에이전트의 `serve-local` 대상에 의존하도록 업데이트합니다

## 연결된 A2A 에이전트 사용

생성기는 에이전트의 `agent.py`를 변환하여 원격 A2A 에이전트를 도구로 래핑합니다. 원격 에이전트는 `@tool`로 장식된 델리게이트로 등록되며 — 데코레이터의 import와 에이전트 생성자는 프레임워크에 따라 다릅니다:

<Tabs syncKey="agent-framework">
<TabItem label="Strands" _filter={{ framework: 'strands' }}>

```python title="packages/my-project/my_module/agent/agent.py" {4,8-13,15}
from contextlib import contextmanager
from strands import Agent, tool

from my_scope_agent_connection import RemoteAgentClientStrands

@contextmanager
def get_agent():
    remote_agent = RemoteAgentClientStrands.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    yield Agent(
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
<TabItem label="LangChain" _filter={{ framework: 'langchain' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,7-12,14}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse
from langchain_core.tools import tool

from my_scope_agent_connection import RemoteAgentClientLangChain

def get_agent():
    remote_agent = RemoteAgentClientLangChain.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
</Tabs>

두 클라이언트 모두 직접 호출 가능하며 원격 에이전트의 응답을 반환하고, AWS에 배포된 경우 SigV4로 요청에 서명하고 `LOCAL_DEV=true`인 경우 일반 `http://localhost:<port>/` 엔드포인트를 사용하는 `httpx.AsyncClient`를 래핑합니다. Strands 클라이언트는 Strands `A2AAgent`를 래핑하고, LangChain 클라이언트는 a2a SDK를 직접 구동합니다.

AgentCore 세션 ID는 프레임워크에 관계없이 `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id` 헤더를 통해 원격 에이전트로 자동으로 전파됩니다: 에이전트 서버는 인바운드 요청의 세션을 비동기 컨텍스트에 바인딩하고, 연결 클라이언트의 서명된 `httpx.Auth`는 모든 아웃바운드 호출에 이를 스탬프하여 [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html)의 일관성을 보장합니다.

## 로컬 개발

생성기는 호스트 에이전트의 `serve-local` 대상을 다음과 같이 구성합니다:
1. 연결된 A2A 에이전트를 자동으로 시작
2. `SERVE_LOCAL=true`를 설정하여 생성된 클라이언트가 AgentCore 대신 `http://localhost:<port>/`에 직접 연결

다음 명령으로 에이전트를 로컬에서 실행합니다:

<NxCommands commands={["<agent-name>-dev <project-name>"]} />

이렇게 하면 호스트 에이전트와 연결된 모든 A2A 에이전트가 시작되며, 호스트 에이전트는 할당된 로컬 포트에서 일반 HTTP를 통해 원격 에이전트를 호출합니다.

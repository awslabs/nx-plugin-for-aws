I progetti connessi espongono due target per eseguirli sulla tua macchina: `serve` e `serve-local`. La differenza riguarda l'**ambito** ŌĆö quanto della tua applicazione viene eseguito localmente rispetto a quanto punta all'infrastruttura AWS distribuita.

Considera un workspace con le seguenti connessioni: un sito web che chiama un'`api` tRPC, e chiama anche un `agent` che a sua volta chiama un server `mcp`.

## `serve`

Il target `serve` esegue **solo il progetto di destinazione** localmente. Ogni altro progetto a cui si connette ├© previsto che sia **distribuito**, e viene raggiunto al suo URL AWS reale.

<NxCommands commands={['serve website']} />

Poich├® il progetto di destinazione comunica con risorse distribuite, deve sapere dove si trovano. Per un sito web questo ├© fornito dal suo `runtime-config.json` (vedi <Link path="guides/react-website#local-runtime-config">Local Runtime Config</Link>), che carichi da un'applicazione distribuita. Per i progetti lato server (API e agent) che leggono la <Link path="guides/runtime-config">Runtime Configuration</Link>, imposta la variabile d'ambiente `RUNTIME_CONFIG_APP_ID` per puntare alla tua applicazione AppConfig distribuita.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
}
deployed: Deployed {
  style.stroke-dash: 3
  api: api
  agent: agent
  mcp: mcp
}

local.website -> deployed.api
local.website -> deployed.agent
deployed.agent -> deployed.mcp
```

Usa `serve` quando vuoi iterare su un singolo progetto rispetto alle versioni "reali" distribuite di tutto ci├▓ da cui dipende.

## `serve-local`

Il target `serve-local` esegue il **progetto di destinazione** e **ogni progetto connesso ad esso transitivamente**, tutto sulla tua macchina. Il generatore di connessioni configura questo automaticamente ŌĆö eseguire `serve-local` sul sito web avvia anche i server locali per l'`api`, l'`agent` e il server `mcp` che raggiunge attraverso l'agent.

<NxCommands commands={['serve-local website']} />

Quando eseguito in questo modo, il `runtime-config.json` del sito web viene automaticamente sovrascritto (tramite [`MODE` di Vite](https://vite.dev/guide/env-and-mode), impostato su `serve-local`) in modo che punti ai tuoi server in esecuzione locale invece che agli URL distribuiti.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
  api: api
  agent: agent
  mcp: mcp

  website -> api
  website -> agent
  agent -> mcp
}
```

Ogni progetto viene eseguito localmente, quindi non ci sono dipendenze distribuite.

Usa `serve-local` quando stai lavorando su diversi progetti connessi contemporaneamente e vuoi iterare rapidamente senza distribuire la tua infrastruttura.

:::tip[Scorciatoia `dev`]
Uno script `dev` radice viene aggiunto come scorciatoia per il primo sito web che generi nel tuo workspace, cos├¼ puoi avviare il sito web e tutti i componenti connessi con:

<PackageManagerShortCommand commands={["dev"]} />
:::

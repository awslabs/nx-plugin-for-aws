I progetti connessi espongono due target per eseguirli sulla tua macchina: `serve` e `serve-local`. La differenza riguarda l'**ambito** — quanto della tua applicazione viene eseguito localmente rispetto a quanto punta all'infrastruttura AWS distribuita.

Considera un workspace con le seguenti connessioni: un sito web che chiama un'`api` tRPC, e chiama anche un `agent` che a sua volta chiama un server `mcp`.

## `serve`

Il target `serve` esegue **solo il progetto di destinazione** localmente. Ogni altro progetto a cui si connette è previsto che sia **distribuito**, e viene raggiunto al suo URL AWS reale.

<NxCommands commands={['serve website']} />

Poiché il progetto di destinazione comunica con risorse distribuite, deve sapere dove si trovano. Per un sito web questo è fornito dal suo `runtime-config.json` (vedi <Link path="guides/react-website#local-runtime-config">Local Runtime Config</Link>), che carichi da un'applicazione distribuita. Per i progetti lato server (API e agent) che leggono la <Link path="guides/runtime-config">Runtime Configuration</Link>, imposta la variabile d'ambiente `RUNTIME_CONFIG_APP_ID` per puntare alla tua applicazione AppConfig distribuita.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
}
deployed: Deployed {
  style.stroke-dash: 3
  api: api
  agent: agent
  mcp: mcp
}

local.website -> deployed.api
local.website -> deployed.agent
deployed.agent -> deployed.mcp
```

Usa `serve` quando vuoi iterare su un singolo progetto rispetto alle versioni "reali" distribuite di tutto ciò da cui dipende.

## `serve-local`

Il target `serve-local` esegue il **progetto di destinazione** e **ogni progetto connesso ad esso transitivamente**, tutto sulla tua macchina. Il generatore di connessioni configura questo automaticamente — eseguire `serve-local` sul sito web avvia anche i server locali per l'`api`, l'`agent` e il server `mcp` che raggiunge attraverso l'agent.

<NxCommands commands={['serve-local website']} />

Quando eseguito in questo modo, il `runtime-config.json` del sito web viene automaticamente sovrascritto (tramite [`MODE` di Vite](https://vite.dev/guide/env-and-mode), impostato su `serve-local`) in modo che punti ai tuoi server in esecuzione locale invece che agli URL distribuiti.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
  api: api
  agent: agent
  mcp: mcp

  website -> api
  website -> agent
  agent -> mcp
}
```

Ogni progetto viene eseguito localmente, quindi non ci sono dipendenze distribuite.

Usa `serve-local` quando stai lavorando su diversi progetti connessi contemporaneamente e vuoi iterare rapidamente senza distribuire la tua infrastruttura.

## `dev`

`dev` è il comando canonico per lo sviluppo locale. Ogni progetto con un target `serve-local` espone un target `dev` come alias per esso, quindi puoi avviare un progetto (e tutto ciò che è connesso ad esso) con:

<NxCommands commands={['dev website']} />

Preferisci `dev` rispetto a `serve-local` — è più breve, coerente in ogni tipo di progetto, e per i progetti con più componenti li avvia tutti (vedi sotto). `serve-local` rimane disponibile se hai bisogno di targetizzare un singolo componente.

### Progetti con più componenti

Alcuni tipi di progetto possono contenere più componenti in un singolo progetto (ad esempio un progetto TypeScript o Python contenente diversi agent e server MCP). Per questi:

- Ogni componente espone un alias `<component-name>-dev` per il suo target `<component-name>-serve-local`, così puoi eseguire un singolo componente da solo:

<NxCommands commands={['run my-project:my-component-dev']} />
- Il target `dev` a livello di progetto avvia **tutti** i componenti del progetto insieme. Ogni componente viene aggiunto a `dev` man mano che viene generato:

<NxCommands commands={['dev my-project']} />

Poiché Nx identifica i task tramite `project:target`, qualsiasi target `serve-local` condiviso tra componenti (come un server MCP comune connesso a due agent) viene eseguito esattamente una volta quando avvii `dev`.

:::tip[Script `dev` del workspace]
Il `package.json` radice del tuo workspace include uno script `dev` che esegue `nx run-many --target dev`, avviando il target `dev` per ogni progetto che ne ha uno. Avvia tutti i server di sviluppo locale del tuo workspace con:

<PackageManagerShortCommand commands={["dev"]} />
:::

Per maggiori dettagli sul server di sviluppo locale di ciascun tipo di progetto, consulta la guida pertinente — ad esempio la guida <Link path="guides/react-website#local-development-server">React Website</Link>.

Questo generatore viene utilizzato per collegare progetti tra loro, come ad esempio siti web che chiamano API. Basta selezionare il progetto sorgente (ad esempio il progetto che effettuerà le chiamate alla tua API) e il progetto di destinazione (ad esempio il tuo progetto API), e questo generatore si occuperà di integrarli.

:::

## Sviluppo Locale

I progetti connessi espongono due target per eseguirli sulla tua macchina: `serve` e `serve-local`. La differenza riguarda l'**ambito** — quanto della tua applicazione viene eseguito localmente rispetto a quanto punta all'infrastruttura AWS distribuita.

Considera un workspace con le seguenti connessioni: un sito web che chiama un'API tRPC `api`, e chiama anche un `agent` che a sua volta chiama un server `mcp`.

### `serve`

Il target `serve` esegue **solo il progetto di destinazione** localmente. Ogni altro progetto a cui si connette deve essere **distribuito**, e viene raggiunto al suo URL AWS reale.

<NxCommands commands={['serve website']} />

Poiché il progetto di destinazione comunica con risorse distribuite, deve sapere dove si trovano. Per un sito web questo viene fornito dal suo `runtime-config.json` (vedi <Link path="guides/react-website#local-runtime-config">Local Runtime Config</Link>), che carichi da un'applicazione distribuita. Per i progetti lato server (API e agenti) che leggono la <Link path="guides/runtime-config">Configurazione Runtime</Link>, imposta la variabile d'ambiente `RUNTIME_CONFIG_APP_ID` per puntare alla tua applicazione AppConfig distribuita.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
agent: agent {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
mcp: mcp {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Usa `serve` quando vuoi iterare su un singolo progetto rispetto alle versioni "reali" e distribuite di tutto ciò da cui dipende.

### `serve-local`

Il target `serve-local` esegue il progetto di destinazione **e ogni progetto ad esso connesso transitivamente**, tutto sulla tua macchina. Il generatore di connessioni configura questo automaticamente — eseguire `serve-local` sul sito web avvia anche server locali per l'`api`, l'`agent` e il server `mcp` che raggiunge attraverso l'agente.

<NxCommands commands={['serve-local website']} />

Quando eseguito in questo modo, il `runtime-config.json` del sito web viene automaticamente sovrascritto (tramite il [`MODE` di Vite](https://vite.dev/guide/env-and-mode), impostato su `serve-local`) in modo che punti ai tuoi server in esecuzione localmente invece che agli URL distribuiti.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#c8e6c9"
}
agent: agent {
  style.fill: "#c8e6c9"
}
mcp: mcp {
  style.fill: "#c8e6c9"
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Usa `serve-local` quando stai lavorando su più progetti connessi contemporaneamente e vuoi iterare rapidamente senza distribuire la tua infrastruttura.

:::note[Risorse AWS in `serve-local`]
I tuoi server locali vengono eseguiti con le credenziali AWS che hai configurato localmente, quindi possono comunque raggiungere le risorse AWS distribuite (come tabelle DynamoDB o modelli Bedrock) dove necessario. Puoi fornire qualsiasi variabile d'ambiente di cui i tuoi server locali hanno bisogno, ad esempio:

<NxCommands env={{RUNTIME_CONFIG_APP_ID: 'xxxxx'}} commands={['serve-local website']} />
:::

Per maggiori dettagli sul server di sviluppo locale di ciascun tipo di progetto, consulta la guida pertinente — ad esempio la guida <Link path="guides/react-website#local-development-server">React Website</Link>.

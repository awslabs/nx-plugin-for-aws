[AWS CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) è un framework per definire infrastrutture cloud tramite codice e provisionarle attraverso AWS CloudFormation.

Il generatore di infrastrutture TypeScript crea un'applicazione AWS CDK per infrastrutture scritta in TypeScript. L'applicazione generata include best practice di sicurezza tramite controlli [CFN Guard](https://docs.aws.amazon.com/cfn-guard/latest/ug/what-is-guard.html).

## Implementare la tua infrastruttura CDK

Puoi iniziare a scrivere la tua infrastruttura CDK all'interno di `src/stacks/application-stack.ts`, ad esempio:

```ts title="src/stacks/application-stack.ts" {9-10}
import * as cdk from 'aws-cdk-lib';
import { Bucket } from 'aws-cdk-lib/aws-s3'
import { Construct } from 'constructs';

export class ApplicationStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Dichiarare la propria infrastruttura qui
    new Bucket(this, 'MyBucket');
  }
}
```

### Infrastruttura per API

Se hai utilizzato i generatori <Link path="guides/trpc">API tRPC</Link> o <Link path="guides/fastapi">FastAPI</Link>, noterai che sono già disponibili alcuni costrutti in `packages/common/constructs` per distribuirli.

Ad esempio, se hai creato un'API tRPC chiamata `my-api`, puoi semplicemente importare e istanziare il costrutto per aggiungere tutta l'infrastruttura necessaria:

```ts title="src/stacks/application-stack.ts" {3, 9-10}
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import { MyApi } from ':my-scope/common-constructs';

export class ApplicationStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Aggiungi l'infrastruttura per la tua API
    new MyApi(this, 'MyApi');
  }
}
```

### Infrastruttura per siti web

Se hai utilizzato il generatore <Link path="guides/cloudscape-website">sito web CloudScape</Link>, noterai che è già disponibile un costrutto in `packages/common/constructs` per distribuirlo. Esempio:

```ts title="src/stacks/application-stack.ts" {3, 9-10}
import * as cdk from 'aws-cdk-lib';
import { Construct } from 'constructs';
import { MyWebsite } from ':my-scope/common-constructs';

export class ApplicationStack extends cdk.Stack {
  constructor(scope: Construct, id: string, props?: cdk.StackProps) {
    super(scope, id, props);

    // Aggiungi l'infrastruttura per il sito web
    new MyWebsite(this, 'MyWebsite');
  }
}
```

:::warning
È importante dichiarare il sito web _dopo_ eventuali costrutti API per includere tutte le configurazioni API nella <Link path="guides/cloudscape-website#runtime-configuration">Configurazione Runtime</Link> del sito web.
:::

## Sintetizzare la tua infrastruttura

Oltre agli <Link path="guides/typescript-project#building">obiettivi predefiniti di compilazione, lint e test</Link>, il target `build` della tua infrastruttura _sintetizza_ anche il progetto in CloudFormation. Puoi eseguire questa operazione separatamente con il target `synth`:

<NxCommands commands={['run <my-infra>:synth']} />

Troverai l'assembly cloud sintetizzato nella cartella `dist` principale, sotto `dist/packages/<my-infra-project>/cdk.out`.

## Bootstrapping degli account AWS

Se stai distribuendo un'applicazione CDK su un account AWS per la prima volta, è necessario eseguire il bootstrap.

Prima, assicurati di aver [configurato le credenziali per il tuo account AWS](https://docs.aws.amazon.com/sdkref/latest/guide/access.html).

Poi, usa il comando `cdk bootstrap`:

```bash
npx cdk bootstrap aws://<account-id>/<region>
```

Per maggiori dettagli, consulta la [documentazione CDK](https://docs.aws.amazon.com/cdk/v2/guide/bootstrapping-env.html).

## Distribuire su AWS

Dopo una build, puoi distribuire l'infrastruttura su AWS usando il target `deploy`.

Prima, assicurati di aver [configurato le credenziali per il tuo account AWS](https://docs.aws.amazon.com/sdkref/latest/guide/access.html).

Poi, esegui il target deploy:

<NxCommands commands={['run <my-infra>:deploy --all']} />

:::tip
Il comando sopra distribuisce _tutti_ gli stack definiti in `main.ts`. Potresti voler destinare uno stack specifico, specialmente se hai configurato più ambienti:

<NxCommands commands={['run <my-infra>:deploy my-sandbox-stack']} />
:::

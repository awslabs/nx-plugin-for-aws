Il generatore `connection` collega un <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> a un progetto <Link path="guides/ts-rdb">Database Relazionale</Link>, rendendo disponibile un client Prisma durante l'intero ciclo di vita delle richieste del tuo agent.

La strategia di iniezione dipende dal protocollo dell'agent — **HTTP** (basato su tRPC) o **A2A** (Agent-to-Agent).

## Prerequisiti

Prima di utilizzare questo generatore, assicurati di avere:

1. Un progetto <Link path="guides/ts-strands-agent">`ts#strands-agent`</Link>
2. Un progetto <Link path="guides/ts-rdb">`ts#rdb`</Link>

## Output del Generatore

I file modificati dipendono dal protocollo dell'agent.

**Protocollo HTTP** — modifica quattro file:

<FileTree>

  - index.ts `createContext` reso asincrono; client Prisma creato e restituito nel contesto

**Protocollo A2A** — modifica due file:

<FileTree>

- packages/my-service/src/my-agent
  - index.ts client Prisma creato prima di `A2AExpressServer`; passato a `getAgent`
  - agent.ts `getAgent` aggiornato per accettare `{ db }` come secondo parametro

</FileTree>

Inoltre, il target `<agent-name>-serve-local` dell'agent viene aggiornato per dipendere dal target `serve-local` del database.

## Come Funziona

### Protocollo HTTP

Per gli agent HTTP, il database fluisce attraverso il contesto della richiesta tRPC — `createContext` crea un client Prisma per ogni richiesta e diventa disponibile per ogni procedura tramite `opts.ctx`. Il router lo passa quindi esplicitamente a `getAgent`.

#### Interfaccia Context

Il tipo del client Prisma viene aggiunto al `Context` di tRPC in `init.ts`:

```ts title="packages/my-service/src/my-agent/init.ts" {1,5}
import { getPrisma as getMyDb } from ':my-scope/my-db';

export interface Context {
  sessionId: string;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
}
```

#### Contesto della Richiesta

`createContext` viene reso asincrono e il client Prisma viene istanziato e restituito:

```ts title="packages/my-service/src/my-agent/index.ts" {1,6,10}
import { getPrisma as getMyDb } from ':my-scope/my-db';

const createContext = async (
  opts: CreateHTTPContextOptions | CreateWSSContextFnOptions,
): Promise<Context> => {
  const myDb = await getMyDb();
  const sessionId = /* ... */;
  return {
    sessionId,
    myDb,
  };
};
```

#### Router e Agent

Il database viene passato dal contesto a `getAgent` tramite il gestore di sottoscrizione:

```ts title="packages/my-service/src/my-agent/router.ts" {2,3}
.subscription(async function* (opts) {
  const { myDb } = opts.ctx;
  const agent = await getAgent(opts.ctx.sessionId, { myDb });
  // ...
})
```

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

### Protocollo A2A

Per gli agent A2A, non esiste un contesto tRPC. Il database viene creato una volta all'avvio e passato direttamente a `getAgent` prima che `A2AExpressServer` si avvii.

#### Avvio del Server

Il client Prisma viene creato prima di `A2AExpressServer` e passato a `getAgent`:

```ts title="packages/my-service/src/my-agent/index.ts" {1,4,6}
import { getPrisma as getMyDb } from ':my-scope/my-db';

void (async () => {
  const myDb = await getMyDb();
  const server = new A2AExpressServer({
    agent: await getAgent('default', { myDb }),
    // ...
  });
})();
```

#### Definizione dell'Agent

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

## Database Multipli

Eseguire nuovamente il generatore con una destinazione diversa aggiunge il secondo database accanto al primo. Per gli agent HTTP, entrambi appaiono nell'interfaccia `Context` e vengono destrutturati insieme nel router:

```ts title="packages/my-service/src/my-agent/router.ts"
const { postgresDb, mysqlDb } = opts.ctx;
const agent = await getAgent(opts.ctx.sessionId, { postgresDb, mysqlDb });
```

## Infrastruttura

Il costrutto agent generato implementa `IGrantable` e `IConnectable`, quindi puoi concedere l'accesso di rete e IAM al database direttamente sul costrutto.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myAgent = new MyAgent(this, 'MyAgent', { vpc, ... });

db.allowDefaultPortFrom(myAgent);
db.grantConnect(myAgent);
```

`allowDefaultPortFrom` apre la regola del security group in modo che il runtime dell'agent possa raggiungere la porta del database. `grantConnect` concede il permesso IAM `rds-db:connect` al ruolo di esecuzione dell'agent.

</Fragment>
<Fragment slot="terraform">

Passa gli output del modulo database nel tuo modulo agent in modo che possa raggiungere il database e leggere la sua configurazione di runtime:

```hcl title="packages/infra/src/main.tf"
module "my_database" {
  source              = "../../common/terraform/src/app/dbs/my-database"
  vpc_id              = module.vpc.vpc_id
  database_subnet_ids = module.vpc.private_isolated_subnet_ids
}

module "my_agent" {
  source = "../../common/terraform/src/app/agents/my-agent"

  appconfig_application_id     = module.my_database.appconfig_application_id
  database_cluster_resource_id = module.my_database.cluster_resource_id
  database_runtime_user        = module.my_database.database_runtime_user
  database_security_group_id   = module.my_database.security_group_id
  database_port                = module.my_database.cluster_port
}
```

Assicurati che il ruolo di esecuzione dell'agent abbia il permesso `rds-db:connect` e che il suo security group possa raggiungere il security group del database sulla porta del database.

</Fragment>
</Infrastructure>

## Sviluppo Locale

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Questo avvia l'agent e tutti i database connessi. La variabile d'ambiente `SERVE_LOCAL=true` fa sì che ogni client Prisma si connetta al proprio database Docker locale invece che ad Aurora.

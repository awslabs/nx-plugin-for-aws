Il generatore `connection` può connettere il tuo <Link path="guides/py-agent">Python Agent</Link> a un agente [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) remoto — sia <Link path="guides/ts-agent">TypeScript</Link> che <Link path="guides/py-agent">Python</Link> — in modo che il tuo agente possa delegare a un altro agente come strumento.

Il generatore configura tutto il cablaggio necessario affinché il tuo agente possa scoprire e invocare l'agente A2A remoto, sia quando viene distribuito su AWS (tramite Bedrock AgentCore) che quando viene eseguito localmente.

## Prerequisiti

Prima di utilizzare questo generatore, assicurati di avere:

1. Un progetto Python con un componente <Link path="guides/py-agent">Python Agent</Link> (Strands o LangChain)
2. Un progetto con un componente Agent generato con `--protocol=a2a` e `--auth=iam` (sia <Link path="guides/ts-agent">`ts#agent`</Link> che <Link path="guides/py-agent">`py#agent`</Link>)
3. Entrambi i componenti creati con `infra: agentcore`

## Output del Generatore

Il generatore crea un progetto Python condiviso `agent_connection` in `packages/common/agent_connection/` (se non esiste già). I moduli client per-connessione vengono generati in questo progetto condiviso:

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py Ri-esporta i client per-connessione
    - core
      - agentcore\_endpoints.py Risoluzione ARN/URL indipendente dal framework
      - agentcore\_a2a\_client\_config.py Configurazione client A2A indipendente dal framework (`ClientConfig` firmato)
      - agentcore\_a2a\_client\_\<framework>.py Client A2A che avvolge la configurazione per il framework del tuo agente
      - auth/ `httpx.Auth` SigV4 / session-forwarding indipendente dal framework
    - app
      - \<target\_agent\_name>\_client\_\<framework>.py Client A2A per-connessione per ogni agente A2A

</FileTree>

Il suffisso del client corrisponde al framework del tuo agente (`_strands` o `_langchain`). Entrambi avvolgono lo stesso `ClientConfig` firmato indipendente dal framework: il client Strands avvolge uno Strands `A2AAgent`, mentre il client LangChain utilizza direttamente l'[a2a SDK](https://pypi.org/project/a2a-sdk/) in modo che un agente LangChain non includa alcuna dipendenza da Strands.

Inoltre, il generatore:
- Trasforma il file `agent.py` del tuo agente per registrare l'agente A2A remoto come strumento utilizzando `@tool`
- Aggiunge il progetto `agent_connection` come dipendenza workspace del progetto del tuo agente
- Aggiorna il target `serve-local` dell'agente per dipendere dal target `serve-local` dell'agente di destinazione

## Utilizzo dell'Agente A2A Connesso

Il generatore trasforma il file `agent.py` del tuo agente per avvolgere l'agente A2A remoto come strumento. L'agente remoto viene registrato con un delegato decorato con `@tool` — l'importazione del decoratore e il costruttore dell'agente differiscono per framework:

<Tabs syncKey="agent-framework">
<TabItem label="Strands" _filter={{ framework: 'strands' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,8-13,15}
from contextlib import contextmanager
from strands import Agent, tool

from my_scope_agent_connection import RemoteAgentClientStrands

@contextmanager
def get_agent():
    remote_agent = RemoteAgentClientStrands.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    yield Agent(
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
<TabItem label="LangChain" _filter={{ framework: 'langchain' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,7-12,14}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse
from langchain_core.tools import tool

from my_scope_agent_connection import RemoteAgentClientLangChain

def get_agent():
    remote_agent = RemoteAgentClientLangChain.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
</Tabs>

Entrambi i client sono direttamente invocabili, restituendo la risposta dell'agente remoto, e avvolgono un `httpx.AsyncClient` che firma le richieste con SigV4 quando distribuito su AWS e utilizza un endpoint semplice `http://localhost:<port>/` quando `LOCAL_DEV=true`. Il client Strands avvolge uno Strands `A2AAgent`; il client LangChain utilizza direttamente l'a2a SDK.

L'ID di sessione AgentCore viene propagato automaticamente all'agente remoto tramite l'header `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id`, indipendentemente dal framework: il server dell'agente associa la sessione della richiesta in entrata a un contesto asincrono, e l'`httpx.Auth` firmato del client di connessione lo applica su ogni chiamata in uscita — garantendo coerenza per [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html).

## Sviluppo Locale

Il generatore configura il target `serve-local` dell'agente host per:
1. Avviare automaticamente gli agenti A2A connessi
2. Impostare `SERVE_LOCAL=true` in modo che il client generato si connetta direttamente a `http://localhost:<port>/` invece che ad AgentCore

Esegui l'agente localmente con:

<NxCommands commands={["<agent-name>-dev <project-name>"]} />

Questo avvierà sia l'agente host che tutti gli agenti A2A connessi, con l'agente host che chiama gli agenti remoti tramite HTTP semplice sulle loro porte locali assegnate.

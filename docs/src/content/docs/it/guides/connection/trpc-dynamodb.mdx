Il generatore `connection` collega un'<Link path="guides/trpc">API tRPC</Link> a un progetto <Link path="guides/ts-dynamodb">DynamoDB</Link>, configurando lo sviluppo locale in modo che entrambi si avviino automaticamente insieme.

## Prerequisiti

Prima di utilizzare questo generatore, assicurati di avere:

1. Un progetto <Link path="guides/trpc">`ts#trpc-api`</Link>
2. Un progetto <Link path="guides/ts-dynamodb">`ts#dynamodb`</Link>

## Output del Generatore

Il generatore aggiorna il file `project.json` della tua API tRPC per aggiungere una dipendenza dal suo target `serve-local` al target `serve-local` del progetto DynamoDB. Nessun file sorgente viene modificato.

## Utilizzare DynamoDB nelle Procedure

Importa le factory di entità dal pacchetto DynamoDB e utilizzale all'interno delle tue procedure tRPC:

```ts title="packages/api/src/procedures/example.ts"
import { createExampleEntity } from ':my-scope/my-table';
import { publicProcedure } from '../init.js';

export const listExamples = publicProcedure
  .query(async () => {
    const entity = await createExampleEntity();
    const result = await entity.scan.go();
    return result.data;
  });
```

## Sviluppo Locale

Il generatore configura il target `serve-local` della tua API tRPC in modo che dipenda dal target `serve-local` del progetto DynamoDB. Eseguendo:

<NxCommands commands={["serve-local <api-project-name>"]} />

viene avviata automaticamente l'istanza DynamoDB locale insieme alla tua API. La variabile d'ambiente `SERVE_LOCAL=true` viene impostata automaticamente, quindi `getDynamoDBClient()` e `resolveTableName()` si connettono all'istanza locale di DynamoDB Local invece che ad AWS.

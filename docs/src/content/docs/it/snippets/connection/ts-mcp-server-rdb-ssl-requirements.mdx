---
title: Requisiti SSL per la Connessione senza RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

Il generatore di connessione aggiorna il Dockerfile per installare il bundle CA di Amazon RDS in `/usr/local/share/ca-certificates/rds-bundle.crt`. Imposta `NODE_EXTRA_CA_CERTS` su quel percorso in modo che Node.js si fidi del certificato quando ci si connette senza RDS Proxy:

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
new MyMcpServer(this, 'MyMcpServer', {
  ...
  environmentVariables: {
    NODE_EXTRA_CA_CERTS: '/usr/local/share/ca-certificates/rds-bundle.crt',
  },
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_mcp_server" {
  ...
  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/usr/local/share/ca-certificates/rds-bundle.crt"
  }
}
```

</Fragment>
</Infrastructure>

Per maggiori dettagli, consulta la [documentazione SSL/TLS di Amazon RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/UsingWithRDS.SSL.html). Quando si utilizza RDS Proxy, non è necessario configurare `NODE_EXTRA_CA_CERTS`.

---
title: Avviso Sviluppo Locale Windows
---

L'arresto di `dev` (ad es. con `Ctrl+C`) rimuove automaticamente il container DynamoDB Local, ma preserva il volume nominato in modo che i tuoi dati persistano tra i riavvii.

:::caution[Windows]
A causa di limitazioni nella gestione dei segnali su Windows, il container non viene rimosso automaticamente quando `dev` viene arrestato. Sarà necessario rimuoverlo manualmente:

```bash
<engine> rm -f <scope>-dynamodb
```

Sostituisci `<engine>` con il tuo motore di container (`docker` o `finch`) e `<scope>` con lo scope del tuo workspace Nx (ad es. `proj`).
:::

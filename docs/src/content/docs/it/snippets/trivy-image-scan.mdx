---
title: Scansione delle Immagini Container
---

L'immagine Docker creata per questo progetto viene scansionata alla ricerca di vulnerabilità come parte del processo di build utilizzando [Trivy](https://trivy.dev/), eseguito dall'[immagine Trivy ospitata su ECR](https://gallery.ecr.aws/aquasecurity/trivy).

La scansione viene eseguita dopo la creazione dell'immagine e **fa fallire la build** se viene trovata qualsiasi vulnerabilità di gravità `HIGH` o `CRITICAL` risolvibile. Le vulnerabilità senza una correzione disponibile vengono ignorate in modo che gli avvisi delle immagini base non risolvibili non blocchino la build.

La scansione utilizza lo stesso motore di container della build dell'immagine (`docker` o `finch`), quindi non sono necessari strumenti aggiuntivi.

:::caution[Immagini Base Pulite]
Il `Dockerfile` generato utilizza un'immagine base senza vulnerabilità `HIGH` o `CRITICAL` note al momento della generazione e aggiorna gli strumenti inclusi (come `npm`) per mantenerla tale. Nuove vulnerabilità vengono divulgate nel tempo, quindi una build che passa oggi potrebbe fallire in seguito quando il database delle vulnerabilità viene aggiornato. Quando ciò accade, aggiorna a un'immagine base più recente in cui la vulnerabilità è corretta, oppure sopprimi il risultato come descritto di seguito.
:::

#### Soppressione dei Risultati di Trivy

Potrebbero esserci casi in cui si desidera sopprimere una vulnerabilità specifica, ad esempio quando non è ancora disponibile una correzione e si è valutato il rischio come accettabile.

Aggiungi l'ID della vulnerabilità (uno per riga) al file `.trivyignore` nella radice del progetto:

```text
// .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

Per maggiori dettagli sul filtraggio dei risultati, consulta la [documentazione sul filtraggio di Trivy](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids).

[CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html) costruisce grafici delle tue risorse CDK connesse e fornisce due plugin:

#### Plugin Diagramma

Il [CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html) genera diagrammi dell'architettura AWS dalla tua infrastruttura CDK.

Un'alternativa valida con approccio deterministico simile è [CDK-Dia](https://github.com/pistazie/cdk-dia).

Con i progressi nell'AI generativa, molti modelli di base sono in grado di creare diagrammi di alta qualità dalla tua infrastruttura CDK. Consigliamo di provare [AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server). Consulta [questo blog post](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/) per una guida passo passo.

#### Plugin Threat Composer

Il [CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html) genera un modello di minaccia iniziale per [Threat Composer](https://github.com/awslabs/threat-composer/) dal tuo codice CDK.

Questo plugin funziona semplicemente filtrando un [modello di minaccia base](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model) contenente minacce di esempio, selezionando quelle rilevanti in base alle risorse utilizzate dal tuo stack.

Se sei interessato a queste minacce di esempio specifiche, puoi copiare e filtrare il modello base, o usarlo come contesto per far generare un modello simile da un modello di AI generativa.

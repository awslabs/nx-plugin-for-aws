Il PDK forniva un [`PDKPipelineProject`](https://aws.github.io/aws-pdk/developer_guides/pipeline/index.html) che configurava un progetto di infrastruttura CDK e utilizzava un costrutto CDK che incapsulava alcune risorse delle [CDK Pipelines](https://docs.aws.amazon.com/cdk/api/v2/docs/aws-cdk-lib.pipelines-readme.html).

Per migrare da questo approccio, puoi utilizzare direttamente i costrutti delle CDK Pipelines. Tuttavia, nella pratica è spesso più semplice utilizzare soluzioni come GitHub Actions o GitLab CI/CD, dove si definiscono [CDK Stages](https://docs.aws.amazon.com/cdk/v2/guide/stages.html) e si esegue direttamente il comando di deploy per lo stage appropriato.

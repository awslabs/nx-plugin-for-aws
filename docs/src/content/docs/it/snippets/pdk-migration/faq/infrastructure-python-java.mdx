PDK supporta infrastrutture CDK scritte in Python e Java. Al momento non offriamo supporto per questo nel Plugin Nx per AWS.

La soluzione consigliata è migrare la tua infrastruttura CDK a TypeScript, oppure utilizzare i nostri generatori e migrare il pacchetto common constructs alla lingua desiderata. Puoi utilizzare l'Intelligenza Artificiale Generativa per accelerare queste migrazioni, ad esempio con [Amazon Q CLI](https://aws.amazon.com/developer/learning/q-developer-cli/). Un agente IA può iterare sulla migrazione finché i template CloudFormation sintetizzati non risultano identici.

Lo stesso vale per l'infrastruttura generata da Type Safe API in Python o Java - puoi tradurre il costrutto generico `rest-api.ts` dal pacchetto common constructs e implementare il tuo generatore di metadati semplificato per il linguaggio target (fai riferimento alla sezione [APIs Modelled with OpenAPI](#apis-modelled-with-openapi)).

Puoi utilizzare il <Link path="/guides/python-project">generatore `py#project`</Link> come base per progetti Python nel tuo codice CDK (spostando il file `cdk.json` e aggiungendo target rilevanti). Per progetti Java puoi usare il plugin [`@nx/gradle`](https://nx.dev/technologies/java/introduction) di Nx, oppure [`@jnxplus/nx-maven`](https://github.com/gridatek/jnxplus/tree/main/packages/nx-maven) per Maven.

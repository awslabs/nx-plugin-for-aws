# Traduzione della Documentazione

Questo progetto utilizza il modello Haiku 3.5 di AWS Bedrock per tradurre automaticamente la documentazione dall'inglese a più lingue. Il sistema di traduzione è progettato per essere efficiente, accurato e semplice da utilizzare.

## Lingue Supportate

Attualmente sono supportate le seguenti lingue:

- Giapponese (jp)
- Francese (fr)
- Spagnolo (es)
- Tedesco (de)
- Cinese (zh)
- Coreano (ko)

## Funzionamento

Il sistema di traduzione funziona:

1. **Dividendo i documenti per intestazioni h2** - Permette un processing più efficiente e un migliore contesto per il modello di traduzione.
2. **Preservando la formattazione markdown** - Tutta la sintassi markdown, i blocchi di codice e i tag HTML vengono preservati durante la traduzione.
3. **Gestione speciale del frontmatter** - Il frontmatter YAML viene tradotto mantenendo la sua struttura.
4. **Traduzione incrementale** - Per default vengono tradotti solo i file modificati, risparmiando tempo e risorse.

## Eseguire Traduzioni in Locale

Per tradurre la documentazione in locale, utilizza lo script `scripts/translate.ts`:

```bash
# Traduce solo i file modificati in giapponese (default)
./scripts/translate.ts

# Traduce tutti i file
./scripts/translate.ts --all

# Traduce in lingue specifiche
./scripts/translate.ts --languages jp,fr,es

# Mostra le operazioni senza eseguirle
./scripts/translate.ts --dry-run

# Mostra output dettagliato
./scripts/translate.ts --verbose
```

## Workflow GitHub

Un workflow GitHub traduce automaticamente la documentazione quando vengono modificati i file della documentazione inglese nelle pull request. Il workflow:

1. Rileva le modifiche ai file della documentazione inglese
2. Traduce i file modificati utilizzando AWS Bedrock
3. Inserisce le traduzioni nel branch sorgente
4. Aggiorna la PR con lo stato della traduzione

### Attivazione Manuale del Workflow

È possibile attivare manualmente il workflow di traduzione dalla sezione GitHub Actions. Utile per:

- Eseguire una traduzione completa di tutta la documentazione
- Tradurre in lingue specifiche
- Aggiornare le traduzioni dopo modifiche allo script di traduzione

## Configurazione AWS

Il sistema utilizza il modello Haiku 3.5 di AWS Bedrock. Per utilizzare questa funzionalità, è necessario:

1. **Credenziali AWS** - Per lo sviluppo locale, configurare le credenziali AWS usando la CLI AWS o variabili d'ambiente.
2. **Ruolo IAM** - Per GitHub Actions, configurare un ruolo IAM con autenticazione OIDC e i permessi necessari per AWS Bedrock.

### Permessi Richiesti

Il ruolo o utente IAM necessita dei seguenti permessi:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel"
      ],
      "Resource": [
        "arn:aws:bedrock:*::foundation-model/anthropic.claude-3-haiku-20240307-v1:0"
      ]
    }
  ]
}
```

## Qualità della Traduzione

La qualità della traduzione è generalmente alta, ma è importante considerare:

- **Termini tecnici** - Il sistema è configurato per preservare i termini tecnici in inglese.
- **Blocchi di codice** - I blocchi di codice non vengono tradotti.
- **Consapevolezza del contesto** - Il modello comprende il contesto tecnico della documentazione.

## Personalizzazione della Traduzione

È possibile personalizzare il processo modificando il file `scripts/translate.ts`. Alcune personalizzazioni possibili:

- Aggiunta di supporto per nuove lingue
- Modifica del modello di traduzione
- Regolazione dei prompt di traduzione
- Modifica della logica di divisione dei documenti

## Risoluzione dei Problemi

In caso di problemi con la traduzione:

1. **Verificare le credenziali AWS** - Assicurarsi che siano configurate correttamente.
2. **Verificare la regione AWS** - Utilizzare una regione dove AWS Bedrock è disponibile.
3. **Eseguire con output dettagliato** - Usare il flag `--verbose` per log più completi.
4. **Controllare i limiti di frequenza** - AWS Bedrock applica limiti che potrebbero influenzare lavori massicci.

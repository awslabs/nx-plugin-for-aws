Il plugin `@aws/nx-plugin` è un componente per [Nx](https://nx.dev/) che fornisce un toolkit per semplificare lo sviluppo e il deployment di applicazioni full-stack su AWS. Offre template preconfigurati sia per il codice applicativo che per l'Infrastructure-as-Code (IaC), riducendo drasticamente il tempo dedicato alla configurazione iniziale. Gestisce la complessità dell'integrazione con i servizi AWS mantenendo al contempo flessibilità per personalizzazioni.

Gli utenti possono selezionare i componenti desiderati dall'elenco dei [generatori](#generators) disponibili, fornire opzioni di configurazione e far generare a `@aws/nx-plugin` il codice iniziale richiesto. È possibile utilizzare generatori specifici per creare API, siti web, infrastrutture e componenti avanzati come l'integrazione type-safe tra frontend e backend (inclusa la modifica di file esistenti tramite trasformazioni AST).

## Basato su Nx

[Nx](https://nx.dev) è un framework di build avanzato specializzato nella gestione di progetti poliglotti complessi attraverso due funzionalità chiave: un sistema di build intelligente e generatori di codice.

Il sistema di build utilizza caching avanzato e analisi del grafo delle dipendenze per ottimizzare l'esecuzione dei task. Ricostruisce solo ciò che è necessario in base alle modifiche, riducendo significativamente i tempi di compilazione. I task possono essere eseguiti in parallelo, con Nx che determina automaticamente l'ordine ottimale in base alle dipendenze. Ciò rende operazioni come test, linting e build estremamente efficienti, specialmente in codebase di grandi dimensioni.

I generatori fungono da strumenti di scaffolding automatizzati che creano strutture di codice consistenti. Possono generare qualsiasi cosa, da singoli componenti a intere applicazioni, seguendo template e pattern predefiniti. I team possono personalizzare questi generatori per adattarli ai propri standard e requisiti specifici.

Insieme, queste caratteristiche rendono Nx particolarmente efficace per gestire monorepo e garantire pratiche di sviluppo coerenti tra team, mantenendo un'elevata velocità di sviluppo.

## Generatori

Tutti i componenti disponibili in questo plugin sono implementati come [Generatori](https://nx.dev/features/generate-code) che automatizzano task ripetitivi e consentono lo scaffolding coerente di progetti nuovi ed esistenti.

È possibile invocare i generatori tramite la [CLI di Nx](https://nx.dev/features/generate-code#invoke-generators) o il [plugin IDE Nx Console](https://nx.dev/getting-started/editor-setup).

## Modificabile liberamente

Tutto il codice generato è di tua proprietà e può essere modificato liberamente. Questo semplifica drasticamente l'esperienza dello sviluppatore: se qualcosa nel codice generato non ti soddisfa, puoi modificarlo direttamente.

## Dipendenze minime

`@aws/nx-plugin` mantiene al minimo il numero di dipendenze globali, che variano in base ai generatori utilizzati.

Esempi:

- I generatori basati su Typescript richiedono solo [Node](https://nodejs.org/en/download)
- I progetti Python richiedono esclusivamente [UV](https://docs.astral.sh/uv/)

<Aside>
  Abbiamo scelto UV come gestore di pacchetti Python per questi motivi:
  - Sostituisce unificato per `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv`
  - Velocità 10-100x superiore a `pip`
  - Supporto nativo per [workspace/monorepo](https://docs.astral.sh/uv/concepts/projects/workspaces/)
</Aside>

## Politica di versione unica

La configurazione predefinita del monorepo adotta una [single version policy](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) sia per progetti Node che Python.

Tutti i progetti nel monorepo utilizzano per default la stessa versione delle dipendenze, riducendo problemi di incompatibilità tra pacchetti.

Per progetti Node:
- Singolo lockfile alla radice
- Unico `node_modules` contenente tutte le dipendenze
- Nuove dipendenze vanno aggiunte al `package.json` radice

Per progetti Python:
- Singolo `.venv` nella radice del monorepo
- Ogni progetto Python ha il proprio `pyproject.toml`
- Le versioni delle dipendenze sono gestite dall'uv workspace e sincronizzate nel file `uv.lock` radice

## Type safety

`@aws/nx-plugin` utilizza la type-safety per semplificare lo sviluppo tramite completamenti IDE ed eliminare errori runtime tipici delle implementazioni non type-safe. Tutti i componenti forniti sono type-safe per impostazione predefinita.

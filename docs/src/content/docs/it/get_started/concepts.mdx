Il plugin `@aws/nx-plugin` è un'estensione per [Nx](https://nx.dev/) che fornisce un toolkit per semplificare lo sviluppo e il deployment di applicazioni full-stack su AWS. Offre template preconfigurati sia per il codice applicativo che per l'infrastruttura (IaC), riducendo drasticamente il tempo dedicato alla configurazione iniziale. Gestisce la complessità dell'integrazione con i servizi AWS mantenendo al contempo flessibilità per personalizzazioni.

Gli utenti possono semplicemente selezionare i componenti desiderati dall'elenco dei [Generator](#generators) disponibili, fornire le opzioni di configurazione e lasciare che `@aws/nx-plugin` generi il codice iniziale necessario. Il toolkit include generatori per creare API, siti web, infrastrutture e persino per operazioni avanzate come l'integrazione type-safe di frontend e backend (aggiornando i file esistenti tramite trasformazioni AST!).

## Basato su Nx

[Nx](https://nx.dev) è un framework di build intelligente specializzato nella gestione di progetti complessi e poliglotti attraverso due funzionalità chiave: un sistema di build avanzato e generatori di codice.

Il sistema di build utilizza caching intelligente e analisi del grafo delle dipendenze per ottimizzare l'esecuzione dei task. Ricostruisce solo ciò che è necessario in base alle modifiche, riducendo significativamente i tempi di compilazione. I task vengono eseguiti in parallelo seguendo l'ordine ottimizzato determinato automaticamente da Nx, rendendo operazioni come test, linting e build estremamente efficienti specialmente in codebase di grandi dimensioni.

I generatori fungono da strumenti di scaffolding automatizzati che creano strutture di codice consistenti. Possono generare qualsiasi cosa, da singoli componenti a intere applicazioni, seguendo template e pattern predefiniti. I team possono personalizzare questi generatori per aderire ai propri standard e requisiti specifici.

Insieme, queste caratteristiche rendono Nx particolarmente efficace per la gestione di monorepo e il mantenimento di pratiche di sviluppo coerenti across team, garantendo alta velocità operativa.

## Generator

Tutti i componenti disponibili in questo plugin sono implementati come [Generator](https://nx.dev/features/generate-code) che automatizzano task ripetitivi e consentono lo scaffolding coerente sia per nuovi progetti che per quelli esistenti.

I generatori possono essere invocati tramite [Nx CLI](https://nx.dev/features/generate-code#invoke-generators) o il plugin IDE [Nx Console](https://nx.dev/getting-started/editor-setup).

## Modificabile liberamente

Tutto il codice generato è di tua proprietà e può essere modificato liberamente. Questo semplifica drasticamente l'esperienza degli sviluppatori: se qualcosa nel codice generato non ti piace, puoi modificarlo direttamente.

## Dipendenze minime

`@aws/nx-plugin` mantiene al minimo il numero di dipendenze globali, limitando i requisiti iniziali in base ai generatori utilizzati.

Ad esempio, i generatori basati su Typescript richiedono solo l'installazione di [Node](https://nodejs.org/en/download). Per progetti Python è sufficiente [UV](https://docs.astral.sh/uv/).

<Aside>
  UV è stato scelto come gestore di pacchetti Python per diverse ragioni:
  - Sostituisce unificato per `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv` e altri
  - 10-100x più veloce di `pip` 
  - Supporto nativo per [workspace/monorepo](https://docs.astral.sh/uv/concepts/projects/workspaces/)
</Aside>

## Politica a versione unica

La configurazione predefinita del monorepo utilizza una [politica a versione unica](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) sia per progetti Node che Python.

Tutti i progetti nel monorepo utilizzano per default la stessa versione delle dipendenze, riducendo i problemi di incompatibilità tra pacchetti.

Per Node ciò significa un singolo lockfile alla root e una cartella `node_modules` condivisa. Le nuove dipendenze vengono aggiunte al `package.json` root.

Per Python, viene utilizzato un singolo `.venv` alla root del monorepo. Ogni progetto Python ha il proprio `pyproject.toml`, ma le versioni delle dipendenze sono gestite dal workspace uv e sincronizzate nel file `uv.lock` root.

## Type safety

`@aws/nx-plugin` implementa la type-safety per semplificare l'esperienza degli sviluppatori attraverso completamenti nell'IDE, eliminando al contempo errori runtime tipici delle implementazioni non type-safe. Tutti i componenti generati sono type-safe per default.

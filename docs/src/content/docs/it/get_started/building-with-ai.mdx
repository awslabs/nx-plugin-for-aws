Il Plugin Nx per AWS include una [Skill](https://code.claude.com/docs/en/skills)/[Power](https://kiro.dev/docs/powers/) e un [MCP Server](https://modelcontextprotocol.io/introduction) che insieme consentono agli assistenti AI di lavorare efficacemente con il plugin. Utilizzandoli, puoi accelerare il tuo flusso di lavoro di sviluppo con l'AI beneficiando della generazione deterministica dei progetti — risparmiando tempo e risorse del contesto nella configurazione dei componenti principali.

## Configura il tuo Assistente AI

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    Il Plugin Nx per AWS è distribuito come [Kiro Power](https://kiro.dev/docs/powers/) che raggruppa il MCP server con documentazione di guida e guide per i flussi di lavoro.

    <Steps>

    1. Apri il pannello Powers in Kiro e clicca su **Add power from GitHub**
    2. Inserisci il seguente URL:

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. Clicca su **Install**

    </Steps>

    Kiro configura automaticamente il MCP server — non è richiesta alcuna configurazione aggiuntiva.
  </TabItem>
  <TabItem label="Kiro CLI">
    Esegui il seguente comando:

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    Consulta la [documentazione MCP di Kiro CLI](https://kiro.dev/docs/cli/mcp/) per maggiori dettagli.
  </TabItem>
  <TabItem label="Claude Code">
    Il Plugin Nx per AWS è distribuito come [plugin di Claude Code](https://code.claude.com/docs/en/plugins) che raggruppa una [Skill](https://code.claude.com/docs/en/skills) con il MCP server.

    Esegui i seguenti comandi all'interno di Claude Code:

    ```
    /plugin marketplace add awslabs/nx-plugin-for-aws
    /plugin install nx-plugin-for-aws@nx-plugin-for-aws
    ```

    Quindi esegui `/reload-plugins` per attivare il plugin.
  </TabItem>
  <TabItem label="Cursor">
    Aggiungi quanto segue a `~/.cursor/mcp.json` (o `.cursor/mcp.json` nel tuo progetto):

    <Snippet name="mcp/config" />

    Consulta la [documentazione MCP di Cursor](https://cursor.com/docs/mcp) per maggiori dettagli.
  </TabItem>
  <TabItem label="Codex">
    Aggiungi quanto segue a `~/.codex/config.toml`:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    Consulta la [documentazione di configurazione di Codex](https://developers.openai.com/codex/config-reference) per maggiori dettagli.
  </TabItem>

## Inizia il Vibe Coding

Chiedi al tuo assistente AI di costruire qualcosa utilizzando `nx-plugin-for-aws`! Tipicamente creerà un workspace, genererà componenti con i generatori applicabili e poi implementerà la logica di business.

Prova un prompt come:

> *"Build a multi-agent application using the Nx Plugin for AWS. I want a React frontend that talks to a backend orchestrator agent which delegates to specialised research and writing agents."*

Oppure per un punto di partenza più semplice:

> *"Use the Nx Plugin for AWS to scaffold a React website backed by a tRPC API, with Cognito auth and CDK infrastructure."*

Utilizzando questo tutorial, costruirai un gioco di avventura in stile dungeon potenziato da Agentic AI con `@aws/nx-plugin`. Questo tutorial non richiede alcuna conoscenza pregressa di `@aws/nx-plugin` o tecnologie correlate.
Le tecniche che apprenderai in questo tutorial ti aiuteranno a:
- Costruire qualsiasi applicazione basata su `@aws/nx-plugin`,
- Ottenere una comprensione approfondita di `@aws/nx-plugin`, e
- Acquisire una comprensione di base su come utilizzare il framework [NX](https://nx.dev/).

<Aside title="Apprendimento Pratico">Questo tutorial è pensato per chi preferisce imparare facendo e vuole realizzare rapidamente qualcosa di tangibile. Se preferisci apprendere ogni concetto passo dopo passo, consulta le guide dei singoli componenti utilizzando la navigazione a sinistra.</Aside>

Al termine del tutorial, acquisirai le conoscenze per:

- utilizzare `@aws/nx-plugin` per creare nuove applicazioni,
- usare NX per gestire/costruire la codebase,
- costruire API con tRPC,
- sviluppare Agent con Strands,
- implementare server MCP in TypeScript,
- utilizzare Tanstack router per creare nuove pagine,
- usare Tanstack query per chiamare API backend,
- modellare dati in DynamoDB e sviluppare localmente con DynamoDB Local, e
- creare e distribuire infrastrutture CDK.

### Architettura dell'applicazione

Il gioco di avventura in stile dungeon potenziato da Agentic AI è costruito utilizzando la seguente architettura:

```d2 inline=true
direction: down

browser: Web Browser {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/client.svg
}

cognito: Cognito / IAM {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/cognito.svg
}

cloudfront: CloudFront {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/cloudfront.svg
}

s3: Static Assets\n(S3) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/s3.svg
}

apigw: API Gateway\n(Game API) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/api-gateway.svg
}

lambda: tRPC API\n(Lambda) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/lambda.svg
}

story: Story Agent\n(AgentCore) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/bedrock-agentcore.svg
}

mcp: Inventory MCP\n(AgentCore) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/bedrock-agentcore.svg
}

ddb: Game State\n(DynamoDB) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/dynamodb.svg
}

sessions: Story Sessions\n(S3) {
  shape: image
  icon: /nx-plugin-for-aws/icons/aws/s3.svg
}

browser -> cognito: Sign in
browser -> cloudfront
cloudfront -> s3
browser -> apigw
apigw -> lambda
lambda -> ddb
lambda -> sessions: Read transcripts
browser -> story: AG-UI stream
story -> sessions: Persist turns
story -> mcp: Tool calls
mcp -> ddb
ddb -> sessions: {
  style.opacity: 0
}
```

- Frontend React/Vite che utilizza:
  - Amazon Cognito/Identity Pools per chiamate API sicure.
  - [Tanstack router](https://tanstack.com/router/latest) con supporto al routing basato su file type-safe.
  - SDK generati per chiamare la Game API.
- API [tRPC](https://trpc.io/) che usa [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) per gestire lo stato del gioco.
- Agent [Strands](https://strandsagents.com/) ospitato su [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) per eseguire il gioco.
- Server MCP TypeScript ospitato su [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) per consentire all'agent di gestire l'inventario del giocatore.
- Infrastruttura [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) per il deployment dell'applicazione.

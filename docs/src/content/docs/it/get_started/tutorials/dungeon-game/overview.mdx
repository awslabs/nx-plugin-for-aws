## Panoramica

Durante questo tutorial costruirai un gioco di avventura in stile dungeon potenziato da Agentic AI. Questo tutorial non richiede alcuna conoscenza pregressa di `@aws/nx-plugin` o tecnologie correlate. Le tecniche che apprenderai sono fondamentali per costruire qualsiasi applicazione basata su `@aws/nx-plugin`, e una piena comprensione ti darà una conoscenza approfondita del plugin e una comprensione di base dell'utilizzo di [NX](https://nx.dev/).

<Aside>Questo tutorial è pensato per chi preferisce imparare facendo e vuole realizzare rapidamente qualcosa di tangibile. Se preferisci apprendere ogni concetto passo dopo passo, consulta le guide dei singoli componenti</Aside>

Gli utenti che completeranno questo tutorial acquisiranno le conoscenze per:

- utilizzare `@aws/nx-plugin` per creare nuove applicazioni
- usare NX per gestire/costruire la codebase
- costruire API con tRPC
- sviluppare Agent con Strands
- implementare server MCP in TypeScript
- utilizzare Tanstack router per creare nuove pagine
- usare Tanstack query per chiamare API backend
- creare e distribuire infrastrutture CDK

### Componenti dell'applicazione

Il gioco sarà costruito utilizzando la seguente architettura a componenti:

<Image style="margin: auto" src={dungeonAdventureArchitecturePng} alt="dungeon-game-architecture.png" height="600" />

- Frontend React/Vite che utilizza:
  - Amazon Cognito/Identity Pools per chiamate API sicure
  - [Tanstack router](https://tanstack.com/router/latest) con supporto al routing basato su file type-safe
  - SDK generati per interagire con la Game API
- API [tRPC](https://trpc.io/) che usa [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) per gestire lo stato del gioco
- Agent [Strands](https://strandsagents.com/latest/) ospitato su [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) per eseguire il gioco
- Server MCP TypeScript ospitato su [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) per gestire l'inventario del giocatore
- Infrastruttura [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) per il deployment dell'applicazione

## Prerequisiti

Sono necessarie le seguenti dipendenze globali prima di procedere:

<Snippet name="required-prerequisites" />
- [Docker](https://www.docker.com/) è richiesto per la costruzione dei componenti AgentCore
- Assicurati che il tuo account AWS abbia abilitato l'accesso al [modello predefinito di Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) in Bedrock per la tua regione target seguendo i passaggi della [guida ufficiale](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access-modify.html). Al momento della stesura si tratta di Anthropic Claude Sonnet 4.

Se utilizzi un assistente AI come Kiro, Amazon Q Developer, Cursor, Claude Code o Cline, potresti voler <Link path="/get_started/building-with-ai">installare il plugin Nx per AWS MCP server.</Link>
:::

## Panoramica

Durante questo tutorial costruirai un gioco di avventura dungeon potenziato da IA. Questo tutorial non presuppone alcuna conoscenza pregressa del `@aws/nx-plugin` o tecnologie correlate. Le tecniche che imparerai sono fondamentali per costruire qualsiasi applicazione basata su `@aws/nx-plugin`, e una piena comprensione ti darà una conoscenza approfondita del plugin e una comprensione di base dell'utilizzo di [NX](https://nx.dev/).

<Aside>Questo tutorial è progettato per chi preferisce imparare facendo e vurealizzare rapidamente qualcosa di concreto. Se preferisci apprendere ogni concetto passo dopo passo, consulta le singole guide dei componenti dalla navigazione sinistra.</Aside>

Gli utenti che completeranno questo tutorial impareranno come:

- utilizzare `@aws/nx-plugin` per creare nuove applicazioni,
- usare NX per gestire/compilare il codice,
- costruire API usando sia tRPC che FastAPI,
- utilizzare TanStack router per creare nuove pagine,
- usare TanStack query per chiamare API backend,
- modificare codice generato per adattarlo allo scopo,
- creare e distribuire infrastrutture CDK.

### Componenti dell'applicazione

Costruiremo il gioco utilizzando la seguente architettura a componenti.

<Image class="white-bg" src={dungeonAdventureArchitecturePng} alt="dungeon-game-architecture.png" width="800" height="600" />

1. L'API [tRPC](https://trpc.io/) utilizza [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) per gestire lo stato del gioco
2. La funzione Lambda Python/[FastAPI](https://fastapi.tiangolo.com/) con risposte in streaming per la generazione della storia (alimentata da [Amazon Bedrock](https://aws.amazon.com/bedrock/))
3. Un frontend React/Vite che utilizza:
  - Amazon Cognito/Identity Pools per chiamate API sicure
  - [TanStack router](https://tanstack.com/router/latest) con routing basato su file e type-safe
  - SDK generati per chiamare le API di Gioco e Story (con [TanStack query](https://tanstack.com/query/latest))

Un'infrastruttura [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) per distribuire i componenti dell'applicazione.

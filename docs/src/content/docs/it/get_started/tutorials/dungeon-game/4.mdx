## Task 1: Configurare il server di sviluppo locale

Per iniziare a costruire l'interfaccia utente, dobbiamo configurare il server di sviluppo locale per puntare alla sandbox distribuita. Esegui il seguente comando:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Questo comando scaricherà il file `runtime-config.json` distribuito e lo memorizzerà localmente nella cartella `packages/game-ui/public`.

Per avviare il server di sviluppo, esegui il seguente comando:

<NxCommands commands={["serve game-ui"]} />

Apri il sito locale nel browser. Ti verrà chiesto di effettuare l'accesso con l'utente Cognito creato in precedenza. Una volta effettuato l'accesso, dovresti vedere il sito base.

<Aside type="caution" title="Mantieni il server di sviluppo attivo">
Consigliamo di mantenere il server di sviluppo in esecuzione per tutto il resto di questo modulo, poiché ricaricherà automaticamente tutte le modifiche apportate.
</Aside>

## Task 4: Creare le pagine del gioco

Nel <Link path="get_started/tutorials/dungeon-game/1">Modulo 1</Link>, abbiamo connesso la nostra `GameUI` allo Story Agent. Poiché l'agente è stato generato con `--protocol=AG-UI`, il generatore di connessione ha configurato un `AguiProvider` di CopilotKit e un hook per-agente (`useAguiStoryAgent`) invece del solito client OpenAPI. Possiamo renderizzare la chat con un singolo componente `<CopilotChat />` e guidarla tramite `threadId` (che usiamo come id di sessione di AgentCore).

Crea due nuovi file in `packages/game-ui/src/routes/game/`:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="Parametri del percorso">
La sintassi `$playerName` indica a `@tanstack/react-router` di trattare `playerName` come un [parametro di percorso](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). Usiamo il nome del giocatore come `threadId` di AG-UI in modo che ogni partita abbia la propria cronologia della chat, e lo riempiamo a 33 caratteri per soddisfare il requisito di lunghezza dell'id di sessione di AgentCore.
</Aside>

<Aside type="note" title="Inizializzazione del primo messaggio">
Al primo montaggio controlliamo `agent.messages.length === 0` e, se vuoto, inviamo un singolo messaggio utente di apertura: `I am <player>, playing in the <genre> genre.` Il prompt di sistema dice all'agente di iniziare immediatamente la storia da quel segnale. Da quel momento in poi, ogni interazione (inclusi i refresh della pagina e i ritorni successivi) ricarica semplicemente la conversazione tramite la sessione AG-UI — non è richiesto alcun endpoint di cronologia personalizzato.
</Aside>
</TabItem>
</Tabs>

Una volta salvati questi file, il server di sviluppo locale (http://localhost:4200/) dovrebbe ora avere il gioco pronto per essere giocato.

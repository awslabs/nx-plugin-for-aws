## Task 1: Configurare il server di sviluppo locale

Scarica localmente il file `runtime-config.json` distribuito in modo che il server di sviluppo possa comunicare con il backend reale:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Quindi avvia il server di sviluppo:

<NxCommands commands={["serve game-ui"]} />

Apri il sito locale nel browser, effettua l'accesso e segui le istruzioni per creare un nuovo utente. Dovresti vedere il sito base Shadcn:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution" title="Mantieni il server di sviluppo attivo">
Lascia il server di sviluppo in esecuzione per il resto di questo modulo — ricarica automaticamente ogni modifica qui sotto.
</Aside>

## Task 2: Dove CopilotKit è già configurato

Quando hai eseguito il generatore `connection` per `game-ui → story` nel <Link path="get_started/tutorials/dungeon-game/1">Modulo 1</Link>, l'integrazione AG-UI del sito Shadcn è stata generata per te. Vale la pena dare un'occhiata veloce:

<FileTree>
- packages/game-ui/src/
  - components/
    - AguiProvider.tsx Singolo `CopilotKitProvider` registrato con ogni agente AG-UI.
    - copilot/
      - index.tsx Ri-esporta `CopilotChat` / `CopilotSidebar` / `CopilotPopup` con tema.
      - ShadcnAssistantMessage.tsx, ShadcnUserMessage.tsx, ShadcnChatInput.tsx, ShadcnCursor.tsx, copilot.css
  - hooks/
    - useAguiStoryAgent.tsx Istanzia un `HttpAgent` di `@ag-ui/client` che punta allo Story Agent distribuito e riempie `threadId` fino al minimo di 33 caratteri richiesto da AgentCore per l'id di sessione.
  - main.tsx Avvolge `<App />` in `<AguiProvider>`
</FileTree>

Tutto ciò che dobbiamo fare è inserire un `<CopilotChat agentId="agent" threadId={...} />` in una route. Per maggiori dettagli su come è strutturata l'integrazione, consulta la <Link path="guides/connection/react-agui">guida alla connessione React → AG-UI</Link>.

## Task 4: Creare le route del gioco

Abbiamo bisogno di due route — una per scegliere un eroe, una per giocare. Entrambe usano componenti shadcn e la chat CopilotKit; non c'è un'interfaccia di chat personalizzata.

<Tabs>
<TabItem label="routes/index.tsx">
<E2EDiff before="dungeon-adventure/4/routes/index.tsx.old.template" after="dungeon-adventure/4/routes/index.tsx.template" lang="tsx" />

Questo è il selettore del gioco: form per nuovo gioco (shadcn `Input` + `Button` + `Card`) più un elenco "Continua" alimentato da `useGameApi().games.query` con `useInfiniteQuery` — un `<div>` sentinella in fondo osservato da un `IntersectionObserver` chiama automaticamente `fetchNextPage()` quando viene visualizzato nello scroll, e gli spinner accanto all'intestazione e sotto l'elenco mostrano lo stato di caricamento. Avviare un gioco esegue `saveGame` della coppia `(playerName, genre)` (in modo che appaia la prossima volta) e naviga alla route di gioco.
</TabItem>
<TabItem label="routes/game/$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

Questa è la route di gioco. Costruisce il `threadId` deterministico (``{player}-{genre}`` riempito a 33 caratteri — l'hook AG-UI lo invierà testualmente come id di sessione AgentCore), renderizza `<CopilotChat agentId="agent" threadId={threadId} />` e sovrappone l'inventario da `useGameApi().inventory.query` sopra. Al montaggio, `useGameApi().actions.query({ sessionId: threadId })` legge la cronologia della conversazione che l'agente ha memorizzato in S3 e — se ce n'è una — chiama `agent.setMessages(...)` per reidratare la chat; altrimenti invia un messaggio utente di inizializzazione per avviare la storia. `agent.messages` è sottoscritto tramite `useAgent({ updates: [OnMessagesChanged] })` quindi ogni nuovo turno ricarica anche la query dell'inventario (le chiamate agli strumenti MCP mutano DynamoDB direttamente).

<Aside type="tip" title="Aspetta `isFetching`, non `isLoading`">
La cache predefinita di React Query mantiene il primo risultato per 5 minuti, quindi una visita di ritorno inizialmente vede l'elenco vuoto obsoleto dal primissimo caricamento. Impostiamo `staleTime: 0` + `refetchOnMount: 'always'` e proteggiamo il `useEffect` di idratazione su `!isFetching && isSuccess` in modo che il percorso di re-inizializzazione si attivi solo su un thread genuinamente vuoto.
</Aside>

<Aside type="tip" title="I threadId deterministici rendono /game/$playerName?genre=... condivisibile">
Rivisitare lo stesso URL — sia dopo un ricaricamento, una ridistribuzione o su un altro dispositivo — si risolve nello stesso `threadId`, quindi l'agente continua a raccontare la stessa storia.
</Aside>

<Aside type="tip" title="Parametri del percorso">
La sintassi `$playerName` indica a `@tanstack/react-router` di trattare `playerName` come un [parametro di percorso](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). `validateSearch` rende `genre` fortemente tipizzato rispetto al nostro enum.
</Aside>
</TabItem>
</Tabs>

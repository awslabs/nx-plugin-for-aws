## Task 1: Implementare lo Story Agent

Lo Story Agent è un agente [Strands](https://strandsagents.com/) che, dato un `Game` e una lista di `Action` come contesto, farà progredire una storia. Configureremo l'agente per interagire con il nostro server MCP dell'inventario per gestire gli oggetti disponibili di un giocatore.

:::note[Implementazione Python]
Per questo tutorial, implementeremo l'Agente in modo "senza stato", dove ogni invocazione è una sessione indipendente che include l'intera cronologia dei messaggi. Per ambienti di produzione, considera l'integrazione con [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html). Per ulteriori informazioni, consulta la pagina [using AgentCore Memory with Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

### Testare il tuo Agent

Puoi testare il tuo Agent in due modi:
<ul>
<li>Avviando un'istanza locale del server Agent e invocandola con `curl`, oppure</li>
<li>Chiamando l'API deployata usando curl con un token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  Il generatore di agenti fornisce una CLI di chat interattiva da terminale (`story-chat`) che utilizza il client TypeScript type-safe generato dalla specifica OpenAPI dell'agente. Poiché abbiamo modificato la forma di input dell'agente per accettare `playerName`, `genre` e `actions`, dobbiamo prima aggiornare lo script di chat generato per fornire questi campi.

  Sostituisci il contenuto di `packages/story/scripts/story/chat.ts` con:

    ```ts
    // packages/story/scripts/story/chat.ts
    import { chatLoop, type ChatAdapter } from 'agent-chat-cli';
    import { Story } from './generated/client.gen.js';

    const port = process.env.PORT ?? '8081';

    class StoryAdapter implements ChatAdapter {
      private client!: Story;
      private readonly actions: { role: string; content: string }[] = [];

      async connect(url: string) {
        this.client = new Story({ url });
        return { agentName: 'Story' };
      }

      async *sendMessage(text: string): AsyncIterable<string> {
        this.actions.push({ role: 'user', content: text });
        let response = '';
        for await (const chunk of this.client.invoke({
          playerName: 'UnnamedHero',
          genre: 'superhero',
          actions: this.actions,
        })) {
          response += chunk.content;
          yield chunk.content;
        }
        this.actions.push({ role: 'assistant', content: response });
      }
    }

    await chatLoop(new StoryAdapter(), `http://localhost:${port}`);
    ```

    Quindi avvia una chat interattiva con il tuo agente Story in esecuzione locale:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["run @dungeon-adventure/story:story-chat"]} />

    <Aside type="caution" title="Usa i valori di output CDK">
    Utilizza il valore di output del deploy CDK `RuntimeConfigApplicationId` per la variabile d'ambiente `RUNTIME_CONFIG_APP_ID`, e la tua regione AWS deployata per `AWS_REGION`.
    </Aside>

    Il target `story-chat` avvia automaticamente `story-serve-local` (che avvia l'agente e tutte le dipendenze del server MCP con `SERVE_LOCAL=true`) ed esegue la generazione della specifica OpenAPI + client prima di portarti nella chat.

    :::note[`story-serve` vs `story-serve-local`]
    `story-chat` dipende da `story-serve-local`, che avvia l'agente con `SERVE_LOCAL=true` e avvia anche tutte le dipendenze del server MCP localmente. Se preferisci che l'agente si connetta al tuo server MCP **deployato** invece (tramite `RUNTIME_CONFIG_APP_ID`), esegui il target `story-serve` in un terminale separato e poi invoca l'agente manualmente con `agent-chat-cli` o `curl`.
    :::
  </TabItem>
  <TabItem label="Deployed">
    Per testare l'agente deployato, dovrai autenticarti con Cognito e ottenere un token JWT. Prima, imposta le tue variabili d'ambiente:

    ```bash
    # Imposta User Pool ID e Client ID dagli output CDK
    export POOL_ID="<UserPoolId dagli output CDK>"
    export CLIENT_ID="<UserPoolClientId dagli output CDK>"
    export REGION="<tua-regione>"
    ```

    Crea un utente test e ottieni un token di autenticazione:

    ```bash
    # Disabilita MFA per semplificare la creazione dell'utente
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Crea Utente
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Imposta Password Permanente (sostituisci con qualcosa di più sicuro!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Autentica l'Utente e cattura l'Access Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoca l'agente deployato usando l'URL Bedrock AgentCore runtime:

    ```bash
    # Imposta Story Agent ARN dagli output CDK
    export AGENT_ARN="<StoryAgentArn dagli output CDK>"

    # Codifica URL dell'ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Costruisci l'URL di invocazione
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoca l'agente
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}'
    ```

    :::note[Codifica URL]
    La codifica URL è necessaria per formattare correttamente l'ARN per l'endpoint API di Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

## Task 1: Implementare lo Story Agent

Lo Story Agent è un agente [Strands](https://strandsagents.com/) generato con `--protocol=AG-UI` nel <Link path="get_started/tutorials/dungeon-game/1">Modulo 1</Link>, in modo che l'interfaccia utente possa ricevere lo streaming da esso tramite il [protocollo Agent-User Interaction](https://docs.copilotkit.ai/aws-strands/protocol) via CopilotKit. Utilizza il server MCP dell'inventario per gestire gli oggetti del giocatore e l'[`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) integrato di Strands per persistere la cronologia delle conversazioni nel bucket delle sessioni che abbiamo fornito nel Modulo 2.

Le modifiche sono:

- `main.py` aggiunge un `session_manager_provider` che crea un `S3SessionManager` per ogni `thread_id` quando deployato, e ricade su un `FileSessionManager` su disco sotto `/tmp/strands-sessions` quando eseguito con `agent-serve-local` (`SERVE_LOCAL=true`). In deployment, il bucket S3 è lo stesso da cui legge il `queryActions` della Game API, così il browser può ricostruire le trascrizioni alla rivisitazione; localmente l'agente funziona completamente offline contro il server MCP locale senza chiamate AWS.
- `agent.py` rimuove lo strumento di esempio `subtract` e sostituisce il prompt di sistema con uno da dungeon-master che invita il primo messaggio dell'utente a dichiarare il nome del giocatore e il genere, e utilizza gli strumenti del server MCP dell'inventario.

### Chattare con il tuo Agent

Apri un REPL interattivo contro una copia dell'agente AG-UI servita localmente utilizzando il target generato `agent-chat`:

<NxCommands commands={["agent-chat story"]} />

`agent-chat` ha `dependsOn: ['agent-serve-local']`, quindi non è necessario avviare il server dell'agente separatamente — Nx avvierà `agent-serve-local` (che a sua volta avvia il server MCP dell'inventario, e attraverso di esso DynamoDB Local), quindi lancerà il REPL AG-UI [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) contro `http://localhost:8081/invocations`. L'intero stack dell'agente viene eseguito localmente senza chiamate AWS.

Il tuo primo messaggio dovrebbe dire all'agente il nome del tuo eroe e il genere (per esempio: `My name is Alice. Start my zombie adventure.`) e la storia verrà trasmessa in streaming.

<Aside type="tip" title="Cronologia delle sessioni locali">
Con `agent-serve-local` (`SERVE_LOCAL=true`), la cronologia delle conversazioni viene scritta in `/tmp/strands-sessions` tramite il `FileSessionManager` di Strands invece che su S3. Riutilizzare lo stesso `threadId` nelle chiamate successive continua la stessa storia — il file session manager riproduce i messaggi precedenti.
</Aside>

<Aside type="tip" title="curl grezzo">
Se preferisci una chiamata HTTP grezza contro il server locale, la forma `RunAgentInput` di AG-UI è:
```bash
curl -N -X POST http://127.0.0.1:8081/invocations \
  -H "Content-Type: application/json" \
  -d '{
    "threadId": "Alice-zombie00000000000000000000000",
    "runId": "run-1",
    "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
    "tools": [], "context": [], "state": {}, "forwardedProps": {}
  }'
```
</Aside>

Se il comando viene eseguito con successo, dovresti iniziare a vedere la storia trasmessa in streaming come un [flusso di eventi AG-UI](https://docs.copilotkit.ai/aws-strands/protocol/events) (Server-Sent Events) — i chunk `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` avvolgono i token effettivi della storia:

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

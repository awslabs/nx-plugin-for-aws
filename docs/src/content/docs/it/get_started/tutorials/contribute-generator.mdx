Creiamo un nuovo generatore per contribuire a `@aws/nx-plugin`. Il nostro obiettivo sarà generare una nuova procedure per la nostra API tRPC.

### Clona il Plugin

Innanzitutto cloniamo il plugin:

```bash
git clone git@github.com:awslabs/nx-plugin-for-aws.git
```

Quindi installiamo e compiliamo:

```bash
cd nx-plugin-for-aws
pnpm i
pnpm nx run-many --target build --all
```

### Implementa il Generatore

Per aggiungere una procedure a un'API tRPC dobbiamo:

1. Creare un file TypeScript per la nuova procedure
2. Aggiungere la procedure al router

#### Crea la Nuova Procedure

Utilizziamo l'utility `generateFiles` con un template [EJS](https://ejs.co/) per generare il file:

```ts title="files/procedures/__procedureNameKebabCase__.ts.template"
import { publicProcedure } from '../init.js';
import { z } from 'zod';

export const <%- procedureNameCamelCase %> = publicProcedure
  .input(z.object({
    // TODO: define input
  }))
  .output(z.object({
    // TODO: define output
  }))
  .<%- procedureType %>(async ({ input, ctx }) => {
    // TODO: implement!
    return {};
  });
```

:::tip
`generateFiles` sostituisce `__<variabile>__` nei nomi file/cartelle e rimuove `.template`. Il contenuto usa sintassi EJS `<% ... %>`.
:::

Nel template usiamo tre variabili:

- `procedureNameCamelCase`
- `procedureNameKebabCase`
- `procedureType`

Aggiorniamo il generatore per passarli:

```ts title="procedure/generator.ts" {8-19}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
`formatFilesInSubtree` formatta i file modificati secondo le impostazioni [prettier](https://prettier.io/) dell'utente.
:::

#### Collega la Procedure al Router

Modifichiamo il file sorgente del router usando AST manipulation:

```ts title="procedure/generator.ts" {6, 23-33}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';
import { destructuredImport, replace } from '../../utils/ast';
import { factory, ObjectLiteralExpression } from 'typescript';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  const routerPath = joinPathFragments(projectConfig.sourceRoot, 'router.ts');

  destructuredImport(
    tree,
    routerPath,
    [procedureNameCamelCase],
    `./procedures/${procedureNameKebabCase}.js`,
  );

  replace(
    tree,
    routerPath,
    'CallExpression[expression.name="router"] > ObjectLiteralExpression',
    (node) =>
      factory.createObjectLiteralExpression([
        ...(node as ObjectLiteralExpression).properties,
        factory.createShorthandPropertyAssignment(procedureNameCamelCase),
      ]),
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
Il selettore `replace` usa [tsquery](https://github.com/phenomnomnominal/tsquery). Puoi testare i selettori nel [tsquery playground](https://tsquery-playground.firebaseapp.com/).
:::

Compiliamo il generatore:

```bash
pnpm nx run @aws/nx-plugin:compile
```

### Esercizi

Proposte per estendere il generatore:

#### 1. Operazioni Annidate

Supportare notazione puntata per `procedure` (es. `games.query`):
- Generare nomi procedurali invertiti (es. `queryGames`)
- Aggiornare i router annidati

#### 2. Validazione

Impedire la selezione di progetti non tRPC. Esempio nel generatore `api-connection`.

#### 3. Test Unitari

Scrivere test che:
1. Creano un workspace vuoto con `createTreeUsingTsSolutionSetup()`
2. Aggiungono file preesistenti
3. Eseguono il generatore
4. Verificano le modifiche attese

#### 4. Test End-to-End

Aggiornare lo "smoke test" esistente per includere il nuovo generatore.

#### 5. Contribuisci!

Invia una pull request se il generatore non è ancora stato contribuito. Altrimenti, pensa a un nuovo generatore da proporre.

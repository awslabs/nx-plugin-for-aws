## Nx 守护进程

[Nx 守护进程](https://nx.dev/docs/concepts/nx-daemon)是一个持续监视工作区变更以重新计算项目图的进程。这能显著提升性能（特别是大型项目），但根据环境不同有时可能引发问题。

该问题可能以不同形式表现，其中一种如下所示：

```bash
 NX   找不到项目 "@scope/project"

添加 --verbose 参数查看堆栈跟踪。
```

对于此错误及其他意外错误，建议先重置 Nx 并重试以排除守护进程问题。可使用以下命令：

<NxCommands commands={['reset']} />

若问题依旧，可尝试[完全禁用守护进程](https://nx.dev/docs/concepts/nx-daemon#turning-it-off)。

:::warning
`nx watch` 命令需要依赖守护进程，因此禁用后部分热重载目标可能无法正常工作。
:::

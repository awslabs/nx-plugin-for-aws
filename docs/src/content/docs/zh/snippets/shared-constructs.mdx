由于该生成器会根据您选择的 `iacProvider` 以基础设施即代码的形式输出，它将在 `packages/common` 目录下创建一个包含相关 CDK 构造体或 Terraform 模块的项目。

通用的基础设施即代码项目结构如下：

<Infrastructure>
<Fragment slot="cdk">
<FileTree>
  - packages/common/constructs
    - src
      - app/ 针对特定项目/生成器的基础设施构造体
      - core/ 被 `app` 目录构造体重用的通用构造体
      - index.ts 导出 `app` 目录构造体的入口文件
    - project.json 项目构建目标与配置
</FileTree>

:::note
此项目使用 <Link path="/guides/typescript-project">`ts#project`</Link> 生成器创建，因此配置了相同的构建目标。
:::
</Fragment>
<Fragment slot="terraform">
<FileTree>
  - packages/common/terraform
    - src
      - app/ 针对特定项目/生成器的 Terraform 模块
      - core/ 被 `app` 目录模块重用的通用模块
    - project.json 项目构建目标与配置
</FileTree>

:::note
此项目使用 <Link path="/guides/terraform-project">`terraform#project`</Link> 生成器创建，因此配置了相同的构建目标。
:::
</Fragment>
</Infrastructure>

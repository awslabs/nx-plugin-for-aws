---
title: Serve Local Windows 注意事项
---

停止 `serve-local`（例如使用 `Ctrl+C`）会自动删除 DynamoDB Local 容器，但会保留命名卷，以便您的数据在重启后保持不变。

:::caution[Windows]
由于 Windows 上信号处理的限制，当 `serve-local` 停止时，容器不会自动删除。您需要手动删除它：

```bash
<engine> rm -f <scope>-dynamodb
```

将 `<engine>` 替换为您的容器引擎（`docker` 或 `finch`），将 `<scope>` 替换为您的 Nx 工作空间作用域（例如 `proj`）。
:::

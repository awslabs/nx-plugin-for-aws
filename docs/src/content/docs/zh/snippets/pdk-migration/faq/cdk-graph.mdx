[CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html) 可构建连接CDK资源的图谱，并提供两个插件：

#### 图表插件

[CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html) 能够根据CDK基础设施生成AWS架构图。

若需类似的确定性方法，[CDK-Dia](https://github.com/pistazie/cdk-dia) 是可行的替代方案。

随着生成式AI的进步，许多基础模型已能根据CDK基础设施生成高质量图表。我们推荐尝试 [AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server)。通过[这篇博客](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/)可获取详细操作指南。

#### 威胁建模插件

[CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html) 能够基于CDK代码生成初始的[Threat Composer](https://github.com/awslabs/threat-composer/)威胁模型。

该插件通过筛选包含示例威胁的[基础威胁模型](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model)，并根据应用栈使用的资源进行过滤来运作。

若对这些示例威胁感兴趣，可复制并过滤基础威胁模型，或将其作为上下文辅助基础模型生成类似模型。

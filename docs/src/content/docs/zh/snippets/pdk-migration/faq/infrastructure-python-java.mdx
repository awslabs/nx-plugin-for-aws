当前 PDK 支持的 CDK 基础设施使用 Python 和 Java 编写。截至本文撰写时，Nx Plugin for AWS 暂不支持此功能。

建议的解决方案是：将您的 CDK 基础设施迁移至 TypeScript，或使用我们的生成器并将通用构造包迁移至目标语言。您可以使用生成式 AI 加速此类迁移，例如 [Amazon Q CLI](https://aws.amazon.com/developer/learning/q-developer-cli/)。通过 AI 代理持续迭代迁移过程，直至生成的 CloudFormation 模板完全一致。

此原则同样适用于 Type Safe API 生成的 Python 或 Java 基础设施——您可以转换通用构造包中的 `rest-api.ts` 构造，并为目标语言实现简单的元数据生成器（参考 [使用 OpenAPI 建模的 API](#apis-modelled-with-openapi) 章节）。

您可以使用 <Link path="/guides/python-project">`py#project` 生成器</Link> 为 CDK 代码创建基础 Python 项目（迁移 `cdk.json` 文件并添加相关构建目标）。对于 Java 项目，可使用 Nx 的 [`@nx/gradle`](https://nx.dev/technologies/java/introduction) 插件；对于 Maven 项目，可使用 [`@jnxplus/nx-maven`](https://github.com/gridatek/jnxplus/tree/main/packages/nx-maven)。

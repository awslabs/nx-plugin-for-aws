---
title: 不使用 RDS Proxy 连接时的 SSL 要求
---
import Infrastructure from '@components/infrastructure.astro';

连接生成器会更新 Dockerfile，将 Amazon RDS CA 证书包安装到 `/usr/local/share/ca-certificates/rds-bundle.crt`。将 `NODE_EXTRA_CA_CERTS` 设置为该路径，以便 Node.js 在不使用 RDS Proxy 连接时信任该证书：

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
new MyMcpServer(this, 'MyMcpServer', {
  ...
  environmentVariables: {
    NODE_EXTRA_CA_CERTS: '/usr/local/share/ca-certificates/rds-bundle.crt',
  },
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_mcp_server" {
  ...
  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/usr/local/share/ca-certificates/rds-bundle.crt"
  }
}
```

</Fragment>
</Infrastructure>

有关更多详细信息，请参阅 [Amazon RDS SSL/TLS 文档](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/UsingWithRDS.SSL.html)。使用 RDS Proxy 时，无需配置 `NODE_EXTRA_CA_CERTS`。

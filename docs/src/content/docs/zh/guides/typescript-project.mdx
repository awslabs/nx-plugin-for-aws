## 编写 TypeScript 源代码

在 `src` 目录中添加您的 TypeScript 代码。

### ESM 导入语法

由于 TypeScript 项目是 ES 模块，请确保使用正确的 ESM 语法编写导入语句，明确指定文件扩展名：

```ts title="index.ts" ".js"
import { sayHello } from './hello.js';
```

:::note
尽管使用 TypeScript 且 `sayHello` 定义于 `hello.ts`，但导入时仍需使用 `.js` 扩展名。详见[此处](https://www.typescriptlang.org/docs/handbook/modules/reference.html)。
:::

### 导出供其他 TypeScript 项目使用

项目的入口文件是 `src/index.ts`，可在此处添加希望其他项目导入的导出项：

```ts title="src/index.ts"
export { sayHello } from './hello.js';
export * from './algorithms/index.js';
```

### 在其他项目中导入库代码

工作区 `tsconfig.base.json` 中配置了项目的 [TypeScript 别名](https://www.typescriptlang.org/docs/handbook/modules/reference.html#paths)，允许其他 TypeScript 项目引用您的项目：

```ts title="packages/my-other-project/src/index.ts"
import { sayHello } from ':my-scope/my-library';
```

:::note
项目别名以 `:` 开头而非传统的 `@`，避免本地工作区包与 [NPM](https://www.npmjs.com/) 远程包名称冲突。
:::

首次在项目中使用新导入时，IDE 可能出现类似错误：

<details>
<summary>导入错误示例</summary>

```bash wrap

  File is ECMAScript module because '/path/to/my/workspace/package.json' has field "type" with value "module" ts(6307)
```

这是因为尚未设置[项目引用](https://www.typescriptlang.org/docs/handbook/project-references.html)。

TypeScript 项目默认配置了 Nx TypeSync 生成器，无需手动配置项目引用。运行以下命令让 Nx 添加必要配置：

<NxCommands commands={['sync']} />

执行后 IDE 错误应消失，即可正常使用库。

:::tip
构建项目时也会收到提示：

```bash wrap

This will result in an error in CI.

? Would you like to sync the identified changes to get your workspace up to date?

No, run the tasks without syncing the changes
```

选择 `Yes` 允许 Nx 更新项目引用。
:::

### 依赖管理

项目没有 `package.json` 文件，这可能与传统 TypeScript monorepo 不同。

要为 monorepo 中的 TypeScript 包添加依赖，只需将依赖添加到工作区根目录的 `package.json`。可通过包管理器命令行操作：

<InstallCommand pkg="some-npm-package" />

此后，依赖即可被工作区中所有 TypeScript 项目使用。

#### 运行时代码

当项目作为运行时代码（如 AWS Lambda 函数处理程序）时，建议使用 [`esbuild`](https://esbuild.github.io/) 等工具打包，以便通过 [tree-shaking](https://esbuild.github.io/api/#tree-shaking) 确保仅包含实际使用的依赖。

可在 `project.json` 中添加类似构建目标：

```json
{
  ...
  "targets": {
    ...
    "bundle": {
      "cache": true,
      "executor": "nx:run-commands",
      "outputs": ["{workspaceRoot}/dist/packages/my-library/bundle"],
      "options": {
        "command": "esbuild packages/my-library/src/index.ts --bundle --outfile=dist/packages/my-library/bundle/index.js --platform=node --format=cjs"
      }
    },
  },
}
```

:::note
示例中选择 `src/index.ts` 作为打包入口，该文件导出的代码及其依赖将被包含在包中。
:::

#### 发布到 NPM

若需将项目发布到 NPM，必须创建 `package.json` 文件。

需声明项目引用的依赖。由于构建时依赖解析基于工作区根目录的 `package.json`，建议配置 [Nx 依赖检查 ESLint 插件](https://nx.dev/nx-api/eslint-plugin/documents/dependency-checks)，确保发布的 `package.json` 包含项目所有依赖。

### 构建

项目配置了 `build` 目标（定义于 `project.json`），可通过以下命令运行：

<NxCommands commands={['run <project-name>:build']} />

其中 `<project-name>` 为项目的完整限定名称。

`build` 目标将编译、检查并测试项目。

构建输出位于工作区根目录的 `dist` 文件夹中，路径如 `dist/packages/<my-library>/tsc`。

## 测试

使用 [Vitest](https://vitest.dev/) 配置项目测试。

### 编写测试

测试文件应为 `.spec.ts` 或 `.test.ts`，与源码共置于 `src` 目录。

例如：

<FileTree>
  - src
    - hello.ts 库源码
    - hello.spec.ts 测试文件
</FileTree>

Vitest 提供类似 Jest 的测试语法，支持 `describe`、`it`、`test` 和 `expect` 等工具。

```ts title="hello.spec.ts"
import { sayHello } from './hello.js';

describe('sayHello', () => {

  it('should greet the caller', () => {
    expect(sayHello('Darth Vader')).toBe('Hello, Darth Vader!');
  });

});
```

更多测试编写方法及 Mock 依赖等特性，请参考 [Vitest 文档](https://vitest.dev/guide/#writing-tests)。

### 运行测试

测试会随 `build` 目标自动运行，也可单独执行 `test` 目标：

<NxCommands commands={['run <project-name>:test']} />

使用 `-t` 参数运行特定测试：

<NxCommands commands={["run <project-name>:test -t 'sayHello'"]} />

:::tip
VSCode 用户建议安装 [真正可用的 Vitest Runner 扩展](https://marketplace.visualstudio.com/items?itemName=rluvaton.vscode-vitest)，支持在 IDE 中运行、观察或调试测试。
:::

## 代码检查

项目使用 [ESLint](https://eslint.org/) 进行代码检查，[Prettier](https://prettier.io/) 进行格式化。

建议在工作区根目录的 `eslint.config.mjs` 中配置 ESLint，变更将应用于所有项目以确保一致性。Prettier 配置同理可置于根目录 `.prettierrc`。

### 运行检查器

执行 `lint` 目标检查代码：

<NxCommands commands={["run <project-name>:lint"]} />

### 修复问题

大部分检查或格式问题可自动修复。添加 `--configuration=fix` 参数运行：

<NxCommands commands={["run <project-name>:lint --configuration=fix"]} />

修复所有项目的问题可运行：

<NxCommands commands={["run-many --target lint --all --configuration=fix"]} />

使用 `@aws/nx-plugin` 创建的新工作空间默认包含配置好的 [git-secrets](https://github.com/awslabs/git-secrets) 预提交钩子。它会在每次提交前扫描暂存文件中匹配 AWS 凭证的模式，防止意外泄露凭证。

## 包含内容

当您创建新工作空间时，会自动设置以下内容:

<FileTree>
  - .git-secrets/
    - git-secrets git-secrets bash 脚本(从 awslabs/git-secrets 供应)
    - setup.sh 在本地 git 配置中注册 AWS 凭证模式
  - .husky/
    - pre-commit 在每次提交前运行 git-secrets 扫描
  - package.json 包含 husky 和用于自动设置的 prepare 脚本
</FileTree>

## 工作原理

1. `prepare` 脚本在 `npm install` / `pnpm install` 后运行,它会初始化 husky git 钩子并在本地 git 配置中注册 AWS 凭证模式。
2. 在每次 `git commit` 时,husky 预提交钩子会调用 `git-secrets` 扫描暂存文件。
3. 如果检测到匹配 AWS 凭证的模式,提交将被阻止并显示错误消息。

## 检测模式

会扫描以下模式:

- **AWS 访问密钥 ID** — `AKIA...`、`ASIA...` 和其他 IAM 密钥前缀,后跟 16 个大写字母数字字符
- **AWS 秘密访问密钥** — 前面带有 `SECRET_ACCESS_KEY` 标签的 40 字符 base64 字符串
- **AWS 账户 ID** — 前面带有 `ACCOUNT_ID` 标签的 12 位数字
- **Bedrock API 密钥** — 长期(`ABSK...`)和短期 Bedrock 密钥
- **凭证文件值** — 在 `~/.aws/credentials` 中找到的字面密钥值

已知的示例密钥(例如 `AKIAIOSFODNN7EXAMPLE`)会自动加入白名单。

## 允许误报

如果 git-secrets 阻止了包含误报的提交:

```sh
# 允许特定模式
git config --add secrets.allowed 'my-pattern-here'
```

或者在仓库根目录添加一个 `.gitallowed` 文件(每行一个正则表达式)。

一次性覆盖:

```sh
git commit --no-verify
```

## 选择退出

创建不包含 git-secrets 的工作空间:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

或者向预设生成器传递 `--gitSecrets=false`。

## 要求

- 工作空间中必须初始化 **Git**(`git init`)
- PATH 中必须有 **Bash**(Windows 上随 Git for Windows 一起提供)
- 模式在 `prepare` 运行时注册 — 如果您克隆了一个新副本,请运行包管理器的安装命令以重新注册

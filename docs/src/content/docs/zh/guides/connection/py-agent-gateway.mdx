`connection` 生成器可以将您的 <Link path="guides/py-agent">Python Agent</Link> 连接到 <Link path="guides/agentcore-gateway">AgentCore Gateway</Link>。

该生成器会配置 agent，使其在部署时使用 IAM SigV4（通过 `httpx` 请求签名）向 Gateway 进行身份验证，并在本地运行时连接到 Gateway 项目启动的本地网关。

## 前置条件

在使用此生成器之前，请确保您具备：

1. 一个包含 <Link path="guides/py-agent">Agent</Link> 组件的 Python 项目（`infra: agentcore`）
2. 一个 <Link path="guides/agentcore-gateway">`agentcore-gateway`</Link> 项目

## 生成器输出

生成器会将共享的 core-gateway 模块发送到您的 `agent_connection` Python 项目中，加上一个针对每个 Gateway 的包装器，并修改您的 agent：

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - core/
      - agentcore\_gateway\_mcp\_client.py 使用 `httpx` 的 SigV4 MCP 客户端
      - agentcore\_gateway\_mcp\_local\_client.py 用于附加 MCP 服务器的本地多路复用客户端
    - app/
      - \<gateway\_snake>\_client.py 带有 `ATTACHED_MCP_SERVERS` 列表的针对每个 Gateway 的包装器
    - \_\_init\_\_.py 重新导出 Gateway 客户端

</FileTree>

此外，生成器还会：

- 修改您的 agent 的 `agent.py` 以导入 Gateway 客户端，通过 `with` 块进入它，并在 `tools` 中注册其工具
- 将 `agent_connection` 添加为 agent 的工作区依赖项
- 将 agent 的 `<agent>-serve-local` 目标配置为依赖于 Gateway 的 `<gateway>-serve-local` 聚合器

## 使用连接的 Gateway

生成器会转换您的 agent 的 `agent.py` 以使用 Gateway 客户端：

```python title="packages/example/example/my_agent/agent.py" {4,8,9-13}
from contextlib import contextmanager
from strands import Agent

from my_scope_agent_connection import MyGatewayClient

@contextmanager
def get_agent():
    my_gateway = MyGatewayClient.create()
    with (
        my_gateway,
    ):
        yield Agent(
            system_prompt="...",
            tools=[*my_gateway.list_tools_sync()],
        )
```

`MyGatewayClient.create()` 返回一个可以作为上下文管理器的单一客户端，其 `list_tools_sync()` 会产生通过 Gateway 可用的每个工具：

- **部署模式**（未设置 `SERVE_LOCAL`）：指向 Gateway 的 MCP 端点的单个 `MCPClient`，使用 SigV4 签名。
- **本地模式**（`SERVE_LOCAL=true`）：一个 `AgentCoreGatewayLocalMultiplexClient`，它会扇出到每个附加的 MCP 服务器并呈现它们工具的并集。

会话 ID 会通过 `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id` 标头自动传播到下游 MCP 服务器。

## 本地开发

生成器会配置 agent 的 `serve-local` 目标以：

1. 启动连接的 Gateway 的本地网关和每个附加的 MCP 服务器
2. 设置 `SERVE_LOCAL=true`，以便生成的客户端指向本地网关，而不是已部署的 Gateway

使用以下命令在本地运行 agent：

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

要在本地运行 agent **针对已部署的 Gateway**（例如，为了执行 Cedar 策略），请使用 agent 的 `serve` 目标。在未设置 `SERVE_LOCAL` 的情况下，客户端会从运行时配置中解析已部署的 Gateway URL，并使用您的本地 AWS 凭证对请求进行 SigV4 签名：

<NxCommands commands={["<agent-name>-serve <project-name>"]} />

### 本地保真度

本地网关代替已部署的 Gateway，因此：

- **没有 Cedar 策略评估。** 无论策略如何，agent 都可以看到每个工具。使用 `serve` target 针对已部署的 Gateway 执行策略。
- **工具名称前缀被保留。** 每个本地 MCP 服务器的工具都以 `<target-name>___<tool-name>` 的形式公开，与已部署的 Gateway 发出的内容相匹配。这使得 agent 的系统提示和您引用的 Cedar 操作名称在本地和部署运行中保持一致。

`connection` 生成器将 <Link path="guides/py-mcp-server">Python MCP Server</Link> 连接到 <Link path="guides/py-rdb">Python 关系数据库</Link>项目，使数据库会话可供服务器中注册的所有工具使用。

## 前提条件

在使用此生成器之前，请确保您拥有：

1. 一个 <Link path="guides/py-mcp-server">`py#mcp-server`</Link> 项目
2. 一个 <Link path="guides/py-rdb">`py#rdb`</Link> 项目

## 生成器输出

生成器将数据库包添加为工作区依赖项，更新 MCP 服务器的 `<mcp-server-name>-dev` 目标以依赖于数据库的 `dev` 目标，以便在本地开发期间两者一起启动，并使用用于直接 Aurora 连接的 RDS CA 证书包更新生成的 MCP 服务器 Dockerfile。

## 工作原理

数据库包导出 `session_context`，这是一个上下文管理器，它使用生成的数据库客户端打开一个 SQLModel `Session`，并在 `with` 块退出时关闭它。

对于本地开发，`session_context()` 连接到本地数据库容器。在部署的运行时中，它从 AWS AppConfig 读取数据库配置，生成 IAM 身份验证令牌，并验证数据库 TLS 证书。

## 在 MCP 工具中使用数据库

从您的数据库包中导入 `session_context` 并在您的 MCP 服务器工具中使用它：

```python title="packages/my_service/my_service/my_mcp_server/server.py"
from sqlmodel import select
from my_scope.my_db import session_context
from my_scope.my_db.models.example import ExampleModel

@mcp.tool()
def list_examples() -> str:
    """List all example records."""
    with session_context() as session:
        items = session.exec(select(ExampleModel)).all()
        return str([item.model_dump() for item in items])
```

## 多个数据库

使用不同的目标再次运行生成器会将第二个数据库添加到第一个数据库旁边。所有工具都可以使用两个会话上下文：

```python title="packages/my_service/my_service/my_mcp_server/server.py"
from my_scope.my_db import session_context as my_db_session_context
from my_scope.other_db import session_context as other_db_session_context
```

## 基础设施

生成的 MCP 服务器构造实现了 `IGrantable` 和 `IConnectable`，因此您可以直接在构造上授予对数据库的网络和 IAM 访问权限。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myMcpServer = new MyMcpServer(this, 'MyMcpServer', { vpc, ... });

db.allowDefaultPortFrom(myMcpServer);
db.grantConnect(myMcpServer);
```

`allowDefaultPortFrom` 打开安全组规则，以便 MCP 服务器运行时可以访问数据库端口。`grantConnect` 向服务器的执行角色授予 IAM `rds-db:connect` 权限。

</Fragment>

将数据库模块输出传递到您的 MCP 服务器模块中，以便它可以访问数据库并读取其运行时配置：

```hcl title="packages/infra/src/main.tf"
module "my_database" {

```

确保 MCP 服务器的执行角色具有 `rds-db:connect` 权限，并且其安全组可以在数据库端口上访问数据库安全组。

### 在不使用 RDS Proxy 连接时的 SSL 要求

当 MCP 服务器直接连接到 Aurora 集群（不使用 RDS Proxy）时，连接生成器会更新生成的 MCP 服务器 Dockerfile，以在 `/opt/global-bundle.pem` 处添加 Amazon RDS CA 证书包：

```dockerfile
ADD --chmod=0644 https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem /opt/global-bundle.pem
```

使用可选的 `rds_ca` 参数将该路径传递给 `session_context`：

```python title="packages/my_service/my_service/my_mcp_server/server.py"
from sqlmodel import select
from my_scope.my_db import session_context
from my_scope.my_db.models.example import ExampleModel

@mcp.tool()
def list_examples() -> str:
    """List all example records."""
    with session_context(rds_ca='/opt/global-bundle.pem') as session:
        items = session.exec(select(ExampleModel)).all()
        return str([item.model_dump() for item in items])
```

使用 RDS Proxy 时，您无需在 MCP 服务器运行时中配置 RDS CA 证书包。

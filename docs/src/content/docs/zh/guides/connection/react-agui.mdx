Nx Plugin for AWS 提供了一个生成器，用于将 React 网站连接到公开 [AG-UI 协议](https://docs.ag-ui.com/)的 Strands Agent。它在您的网站上将 [CopilotKit](https://docs.copilotkit.ai/aws-strands) 与 [`@ag-ui/client`](https://docs.ag-ui.com/) `HttpAgent` 连接起来，并支持 AWS IAM 和 Cognito 身份验证。

:::caution[TypeScript Strands Agents]
AG-UI 尚不支持 TypeScript Strands Agents。请参阅 [strands-agents/sdk-typescript#347](https://github.com/strands-agents/sdk-typescript/issues/347) 了解最新状态。目前，请使用带有 `protocol=AG-UI` 的 <Link path="guides/py-strands-agent">`py#strands-agent` 生成器</Link>。
:::

## 前置条件

在使用此生成器之前，请确保您拥有：

1. 一个 React 网站（使用 <Link path="guides/react-website">`ts#react-website` 生成器</Link>生成）
2. 一个带有 `protocol=AG-UI` 的 Python Strands Agent（使用 <Link path="guides/py-strands-agent">`py#strands-agent` 生成器</Link>生成）
3. 对于已部署的代理，通过 <Link path="/guides/react-website-auth">`ts#react-website-auth` 生成器</Link>添加 Cognito Auth

## 生成器输出

生成器创建一个**单一共享的** `AguiProvider` 组件和每个连接的代理一个钩子：

<FileTree>

- src
  - components
    - AguiProvider.tsx 为每个 AG-UI 代理提供单一的 `CopilotKitProvider`。在第一次运行 `connection` 时创建，并在后续运行时更新以注册每个新代理。
  - hooks
    - useAgui\<AgentName>.tsx 注册一个 AG-UI 代理。每次运行 `connection` 生成一个文件。
    - useSigV4.tsx SigV4 签名（仅 IAM）

</FileTree>

第二次为不同的代理运行 `connection` 会**添加一个新的 `useAgui<AgentName>.tsx` 钩子**并更新 `AguiProvider.tsx` 以注册两个钩子——您对提供者所做的任何自定义编辑都会被保留。`main.tsx` 保持其单一的 `<AguiProvider>` 包装器——您永远不会出现嵌套的提供者。

以下依赖项被添加到根 `package.json`：

- `@copilotkit/react-core` — 提供 `CopilotKitProvider` 和聊天组件（`CopilotChat`、`CopilotSidebar`、`CopilotPopup`）
- `@ag-ui/client` — 生成的钩子使用的 `HttpAgent`
- `aws4fetch`、`oidc-client-ts`、`react-oidc-context`、`@aws-sdk/credential-providers` — 仅 IAM 身份验证
- `react-oidc-context` — Cognito 身份验证

## 工作原理

### AG-UI 连接

每个 `useAgui<AgentName>` 钩子从<Link path="guides/runtime-config">运行时配置</Link>读取其代理的运行时值，并实例化一个 `@ag-ui/client` `HttpAgent`：

- **已部署**：运行时值是 Bedrock AgentCore Runtime ARN，它被转换为 AgentCore HTTPS 端点：`https://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/invocations?qualifier=DEFAULT`
- **本地开发**：`serve-local` 将该值覆盖为代理的本地 URL（例如 `http://localhost:8081`）

共享的 `AguiProvider` 调用每个生成的钩子，并将每个钩子展开到单个 `CopilotKitProvider` 的 `selfManagedAgents` 中，从而将它们全部暴露给 CopilotKit 组件。

### CopilotKit 集成

[CopilotKit](https://docs.copilotkit.ai/aws-strands) 是 AG-UI 协议的第一方参考 React 客户端，并提供现成的聊天组件：

- `<CopilotChat />` — 完整的聊天界面
- `<CopilotSidebar />` — 固定侧面板聊天
- `<CopilotPopup />` — 浮动聊天弹窗

将这些组件中的任何一个放置在 `<AguiProvider>` 包装器内的任何位置（已为您连接到 `main.tsx`）。

### 身份验证

生成的代码根据您的代理配置处理身份验证：

- **IAM**（默认）：使用 AWS SigV4 签名的 HTTP 请求。凭证从配置了您网站身份验证的 Cognito Identity Pool 获取。
- **Cognito**：将 JWT 访问令牌作为 Bearer 令牌嵌入 `Authorization` 标头中。

## 使用生成的代码

### 添加聊天界面

使用 `agentId` 实例化 CopilotKit 组件以选择要使用的代理。该 id 是代理的名称——与您运行 <Link path="guides/py-strands-agent">`py#strands-agent` 生成器</Link>时选择的名称相同——您也可以在生成的钩子文件中找到它（例如从 `packages/web/src/hooks/useAgui<AgentName>.tsx` 返回的键）：

```tsx
import { CopilotChat } from '@copilotkit/react-core/v2';

function ChatPage() {
  return (
    <CopilotChat
      agentId="agent"
      labels={{
        welcomeMessageText: 'How can I help you today?',
        chatInputPlaceholder: 'Ask me anything...',
      }}
    />
  );
}
```

### 连接多个 AG-UI 代理

为每个代理运行一次 `connection` 生成器。通过共享的 `AguiProvider` 注册的每个代理都可以从应用程序的任何地方看到——使用不同的 `agentId` 实例化 CopilotKit 组件，将每个聊天路由到您想要的代理：

```tsx
<CopilotChat agentId="story" />    {/* talks to StoryAgent */}
<CopilotChat agentId="research" /> {/* talks to ResearchAgent */}
```

## 自定义外观和感觉

`<CopilotChat />`（以及 `<CopilotSidebar />`、`<CopilotPopup />`）使用递归的**插槽系统**——您可以使用 Tailwind 类字符串、prop 对象或自定义 React 组件覆盖任何子组件。请参阅 [CopilotKit 插槽指南](https://docs.copilotkit.ai/aws-strands/custom-look-and-feel/slots)了解完整的插槽树。

### 通过插槽进行 Tailwind 样式设置

```tsx
<CopilotChat
  agentId="agent"
  // style the input and its children
  input={{
    textArea: 'text-blue-600',
    sendButton: 'bg-blue-600 hover:bg-blue-700',
  }}
  // style nested message slots
  messageView={{
    assistantMessage: 'bg-blue-50 rounded-xl p-2',
    userMessage: 'bg-blue-100 rounded-xl',
  }}
/>
```

### 用自定义组件替换插槽

任何插槽都可以接受 React 组件而不是 className，因此您可以完全替换默认值：

```tsx
import { CopilotChat } from '@copilotkit/react-core/v2';

const MySendButton: React.FC<{ onClick: () => void }> = ({ onClick }) => (
  <button onClick={onClick} className="my-send-btn">
    ✨ Send
  </button>
);

<CopilotChat
  agentId="agent"
  input={{ sendButton: MySendButton }}
/>;
```

更深层的覆盖遵循相同的形式——例如，仅替换助手消息上的复制按钮：

```tsx
<CopilotChat
  agentId="agent"
  messageView={{
    assistantMessage: {
      copyButton: ({ onClick }) => <button onClick={onClick}>Copy</button>,
    },
  }}
/>
```

## 本地开发

连接生成器自动配置 `serve-local` 集成：

1. 运行 `nx serve-local <website>` 也会启动代理的本地服务器
2. 运行时配置被覆盖以指向本地 AG-UI URL（例如 `http://localhost:8081`）
3. 网站和代理一起热重载

<NxCommands commands={['serve-local <WebsiteProject>']} />

:::tip[热重载]
网站和连接的代理一起热重载，使您能够快速迭代双方而无需部署到 AWS。
:::

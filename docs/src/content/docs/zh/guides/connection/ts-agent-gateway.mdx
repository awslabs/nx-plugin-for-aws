`connection` 生成器可以将您的 <Link path="guides/ts-agent">TypeScript Agent</Link> 连接到 <Link path="guides/agentcore-gateway">AgentCore Gateway</Link>。

该生成器会配置 agent，使其在部署时使用 IAM SigV4 向 Gateway 进行身份验证，并在本地运行时直接多路复用到附加的 MCP 服务器。

## 前提条件

在使用此生成器之前，请确保您具有：

1. 一个包含 <Link path="guides/ts-agent">Agent</Link> 组件的 TypeScript 项目（`infra: agentcore`）
2. 一个 <Link path="guides/agentcore-gateway">`agentcore-gateway`</Link> 项目

## 生成器输出

生成器会将共享的核心客户端文件输出到您的 `agent-connection` 包中，加上每个 Gateway 的包装器，并修改您的 agent：

<FileTree>

- packages/common/agent-connection
  - src
    - core/
      - agentcore-gateway-mcp-client.ts 用于已部署 Gateway 的 SigV4 MCP 客户端
    - app/
      - \<gateway-kebab>-client.ts 每个 Gateway 的客户端包装器
    - index.ts 重新导出 Gateway 客户端

</FileTree>

此外，生成器还会：

- 修改您的 agent 的 `agent.ts` 以导入 Gateway 客户端类，调用 `<Gateway>Client.create()`，并在 `tools` 数组中注册返回的客户端
- 将 agent 的 `<agent>-serve-local` 目标配置为依赖于 Gateway 的 `<gateway>-serve-local` 聚合器
- 安装所需的 SigV4 / MCP 依赖项

## 使用已连接的 Gateway

生成器会转换您的 agent 的 `agent.ts` 以使用 Gateway 客户端：

```ts title="packages/example/src/my-agent/agent.ts" {2,5,8}
import { Agent } from '@strands-agents/sdk';
import { MyGatewayClient } from ':my-scope/agent-connection';

export const getAgent = async () => {
  const myGateway = await MyGatewayClient.create();
  return new Agent({
    systemPrompt: '...',
    tools: [myGateway],
  });
};
```

`create()` 始终返回单个 `McpClient`。当部署时（未设置 `SERVE_LOCAL`），它指向 Gateway 的 MCP 端点并使用 SigV4 进行身份验证。当 `SERVE_LOCAL=true` 时，它指向由 Gateway 项目的 `serve-local` 目标启动的本地 gateway，因此相同的 `agent.ts` 在两种模式下都能统一工作。

会话 ID 会自动传播到下游 MCP 服务器：底层传输通过 [Strands TS 会话上下文](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html) 读取请求范围的会话，并将其作为 `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id` 标头转发到 Gateway。

## 基础设施

运行生成器后，您必须授予 agent 调用 Gateway 的权限。

<Infrastructure>
<Fragment slot="cdk">

const gateway = new MyGateway(this, 'MyGateway');

gateway.grantInvokeAccess(myAgent);

## 本地开发

生成器会将 agent 的 `serve-local` 目标配置为：

1. 启动已连接 Gateway 的本地 gateway 和每个附加的 MCP 服务器
2. 设置 `SERVE_LOCAL=true`，以便生成的客户端指向本地 gateway，而不是已部署的 Gateway

使用以下命令在本地运行 agent：

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

要在本地运行 agent **针对已部署的 Gateway**（例如，为了测试 Cedar 策略），请使用 agent 的 `serve` target。未设置 `SERVE_LOCAL` 时，客户端会从运行时配置解析已部署的 Gateway URL，并使用您的本地 AWS 凭证对请求进行 SigV4 签名：

<NxCommands commands={["<agent-name>-serve <project-name>"]} />

### 本地保真度

本地 gateway 代替已部署的 Gateway，因此：

- **没有 Cedar 策略评估。** 无论策略如何，agent 都可以看到每个工具。使用 `serve` 目标针对已部署的 Gateway 测试策略。
- **保留工具名称前缀。** 每个本地 MCP 服务器的工具都被包装以公开 `<target-name>___<tool-name>` 形式的名称，与已部署的 Gateway 发出的名称相匹配。这使得 agent 的系统提示和您引用的 Cedar 操作名称在本地和已部署运行中保持一致。

Nx Plugin for AWS 提供了一个生成器，可以快速将您的 <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> 与 React 网站集成。它设置了通过 tRPC over WebSocket 连接到您的代理所需的所有配置，包括 AWS IAM 和 Cognito 身份验证支持。该集成在您的前端和代理的 tRPC 路由器之间提供完整的端到端类型安全。

## 前置条件

在使用此生成器之前，请确保您具有：

1. 一个 React 网站（使用 <Link path="guides/react-website">`ts#react-website` 生成器</Link>生成）
2. 一个 TypeScript Strands Agent（使用 <Link path="guides/ts-strands-agent">`ts#strands-agent` 生成器</Link>生成）
3. 如果连接使用 Cognito 或 IAM 身份验证的代理，需要通过 <Link path="/guides/react-website-auth">`ts#react-website-auth` 生成器</Link>添加 Cognito Auth

## 生成器输出

生成器在您的 React 应用程序中创建以下结构：

<FileTree>

- src
  - components
    - \<AgentName>AgentClientProvider.tsx 设置 tRPC WebSocket 客户端和与您的代理的 tRPC 路由器的绑定
    - QueryClientProvider.tsx TanStack React Query 客户端提供程序
  - hooks
    - useSigV4.tsx 用于使用 SigV4 签名请求的钩子（仅限 IAM）
    - use\<AgentName>Agent.tsx 返回用于 TanStack Query 集成的 tRPC 选项代理的钩子
    - use\<AgentName>AgentClient.tsx 返回用于直接代理调用的原生 tRPC 客户端的钩子

</FileTree>

此外，它还会安装所需的依赖项：

  - `@trpc/client`
  - `@trpc/tanstack-react-query`
  - `@tanstack/react-query`
  - `aws4fetch`（如果使用 IAM 身份验证）

## 工作原理

### WebSocket 连接

生成的客户端通过 tRPC over WebSocket 连接到您的 Strands Agent。代理通过 WebSocket 端点公开一个 tRPC 路由器（包括用于流式传输代理响应的 `invoke` 订阅）。

- **已部署**：代理运行时 ARN 从<Link path="guides/runtime-config">运行时配置</Link>加载。ARN 遵循 Bedrock AgentCore Runtime WebSocket 协议转换为 WebSocket URL：`wss://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/ws`
- **本地开发**：使用 `serve-local` 运行时，运行时配置覆盖将值设置为本地 `ws://` URL（例如，`ws://localhost:8081/ws`），客户端直接连接

### 身份验证

生成的代码根据您的代理配置处理身份验证：

- **IAM**（默认）：使用 AWS SigV4 预签名 URL 对 WebSocket 连接进行身份验证。凭证从与您网站的身份验证配置的 Cognito Identity Pool 获取
- **Cognito**：将 JWT 访问令牌作为 base64url 编码的 bearer 令牌嵌入到 `Sec-WebSocket-Protocol` 标头中，遵循 [AgentCore WebSocket 身份验证协议](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-get-started-websocket.html)
- **无身份验证**：直接连接而不进行身份验证（适用于本地开发）

## 使用生成的代码

### 使用 useSubscription 流式传输代理响应

最常见的用例是使用 `invoke` 订阅流式传输代理的响应：

```tsx {1-2,7-22}
import { useSubscription } from '@trpc/tanstack-react-query';
import { useMyAgent } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const trpc = useMyAgent();

  const subscription = useSubscription(
    trpc.invoke.subscriptionOptions(
      { message: 'What can you help me with?' },
      {
        enabled: true,
        onStarted: () => {
          console.log('Agent started responding');
        },
        onData: (token) => {
          console.log('Received token:', token);
        },
        onError: (error) => {
          console.error('Agent error:', error);
        },
      },
    ),
  );

  return (
    <div>
      <p>Status: {subscription.status}</p>
      {subscription.data && <p>Latest token: {subscription.data}</p>}
      {subscription.error && <p>Error: {subscription.error.message}</p>}
    </div>
  );
}
```

### 使用原生 tRPC 客户端

`use<AgentName>AgentClient` 钩子提供对原生 tRPC 客户端的访问，以便更好地控制订阅生命周期：

```tsx {2,5,9-22,25}
import { useState, useEffect } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const client = useMyAgentClient();
  const [messages, setMessages] = useState<string[]>([]);

  const sendMessage = (message: string) => {
    const subscription = client.invoke.subscribe(
      { message },
      {
        onData: (token) => {
          setMessages((prev) => [...prev, token]);
        },
        onComplete: () => {
          console.log('Agent finished');
        },
        onError: (error) => {
          console.error('Error:', error);
        },
      },
    );

    // Clean up when done
    return () => subscription.unsubscribe();
  };

  return (
    <div>
      <button onClick={() => sendMessage('Hello!')}>Send</button>
      <div>
        {messages.map((msg, i) => (
          <span key={i}>{msg}</span>
        ))}
      </div>
    </div>
  );
}
```

### 构建聊天界面

以下是使用生成的客户端构建简单聊天界面的示例：

```tsx
import { useState, useCallback } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function Chat() {
  const client = useMyAgentClient();
  const [input, setInput] = useState('');
  const [response, setResponse] = useState('');
  const [isStreaming, setIsStreaming] = useState(false);

  const handleSend = useCallback(() => {
    if (!input.trim() || isStreaming) return;

    setIsStreaming(true);
    setResponse('');

    client.invoke.subscribe(
      { message: input },
      {
        onData: (token) => {
          setResponse((prev) => prev + token);
        },
        onComplete: () => {
          setIsStreaming(false);
        },
        onError: (error) => {
          console.error(error);
          setIsStreaming(false);
        },
      },
    );

    setInput('');
  }, [client, input, isStreaming]);

  return (
    <div>
      <div style={{ whiteSpace: 'pre-wrap' }}>{response}</div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        onKeyDown={(e) => e.key === 'Enter' && handleSend()}
        disabled={isStreaming}
        placeholder="Ask the agent..."
      />
      <button onClick={handleSend} disabled={isStreaming}>
        {isStreaming ? 'Streaming...' : 'Send'}
      </button>
    </div>
  );
}
```

## 本地开发

连接生成器自动配置 `serve-local` 集成：

1. 运行 `nx run <website>:serve-local` 也会启动代理的本地服务器
2. 运行时配置被覆盖以指向本地 WebSocket URL（例如，`ws://localhost:8081/ws`）
3. 客户端检测到本地 URL 并直接连接，无需身份验证开销

这意味着您可以使用单个命令在本地开发和测试您的代理集成。

## 类型安全

该集成提供完整的端到端类型安全。您的 IDE 将为所有代理过程调用提供完整的自动完成和类型检查。类型会自动从您的代理的 tRPC 路由器定义中推断，确保对代理 API 的任何更改都会立即反映在您的前端代码中。

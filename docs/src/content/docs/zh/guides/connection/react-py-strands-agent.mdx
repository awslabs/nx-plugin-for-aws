Nx Plugin for AWS 提供了一个生成器，可以快速将您的 <Link path="guides/py-strands-agent">Python Strands Agent</Link> 与 React 网站集成。它会设置所有必要的配置，通过类型安全的 OpenAPI 生成客户端连接到您的代理，包括 AWS IAM 和 Cognito 身份验证支持。

:::tip[AG-UI 代理]
如果您的 Python Strands Agent 是使用 `protocol=AG-UI` 生成的，连接生成器将在 React 网站上配置 [CopilotKit](https://docs.copilotkit.ai/aws-strands) 和 [`@ag-ui/client`](https://docs.ag-ui.com/) `HttpAgent`。有关详细信息，请参阅 <Link path="guides/connection/react-agui">React 到 AG-UI Agent</Link> 指南。
:::

## 前置条件

在使用此生成器之前，请确保您拥有：

1. 一个 React 网站（使用 <Link path="guides/react-website">`ts#react-website` 生成器</Link>生成）
2. 一个使用 `protocol=HTTP` 的 Python Strands Agent（使用 <Link path="guides/py-strands-agent">`py#strands-agent` 生成器</Link>生成）
3. 通过 <Link path="/guides/react-website-auth">`ts#react-website-auth` 生成器</Link>添加的 Cognito Auth

生成器在您的 Python Strands Agent 项目中创建以下内容：

<FileTree>

- scripts
  - \<agent_name>_openapi.py 从代理的 FastAPI 应用生成 OpenAPI 规范的脚本
- project.json 添加了一个新的 `<agent-name>-openapi` 目标

</FileTree>

生成器在您的 React 应用程序中创建以下结构：

<FileTree>

- src
  - components
    - \<AgentName>Provider.tsx OpenAPI 客户端的提供者
    - QueryClientProvider.tsx TanStack React Query 客户端提供者
  - hooks
    - useSigV4.tsx 用于使用 SigV4 签名请求的 Hook（仅限 IAM）
    - use\<AgentName>.tsx 返回代理 API 的 TanStack Query 选项代理的 Hook
    - use\<AgentName>Client.tsx 返回原始 API 客户端的 Hook
  - generated
    - \<agent-name>
      - types.gen.ts 从代理的 Pydantic 模型生成的类型
      - client.gen.ts 用于调用代理 API 的类型安全客户端
      - options-proxy.gen.ts 用于与代理交互的 TanStack Query hooks 选项
- project.json 添加了用于客户端生成和监视更改的目标
- .gitignore 默认情况下忽略生成的客户端文件

</FileTree>

## 工作原理

### OpenAPI 客户端生成

在构建时，会内省 Python Strands Agent 的 FastAPI 应用以生成 OpenAPI 规范。然后使用此规范生成带有 TanStack Query hooks 的类型安全 TypeScript 客户端，遵循与 <Link path="guides/connection/react-fastapi">React 到 FastAPI 连接</Link>相同的模式。

每个代理都有自己作用域的 OpenAPI 脚本（例如 `scripts/agent_openapi.py`），以便具有多个代理的项目可以生成各自的规范。

### 运行时配置

运行此连接生成器还会修补代理生成的 CDK/Terraform 构造，将其 AgentCore 运行时 ARN 发布到网站的 `runtime-config.json`（在 `connection` 命名空间下），因此只有您明确连接的代理才会暴露给前端。有关详细信息，请参阅<Link path="guides/runtime-config">运行时配置</Link>。

### 身份验证

生成的代码根据您的代理配置处理身份验证：

- **IAM**（默认）：使用 AWS SigV4 签名 HTTP 请求。凭证从配置了网站身份验证的 Cognito Identity Pool 获取
- **Cognito**：在 Authorization 标头中嵌入 JWT 访问令牌
- **None**：无身份验证

## 使用生成的代码

### 使用 API Hook

`use<AgentName>` hook 提供用于调用代理 API 端点的 TanStack Query 选项：

```tsx
import { useState } from 'react';
import { useMutation } from '@tanstack/react-query';
import { useMyAgent } from '../hooks/useMyAgent';
import type { StreamChunk } from '../generated/my-agent/types.gen';

function ChatComponent() {
  const api = useMyAgent();
  const [chunks, setChunks] = useState<StreamChunk[]>([]);

  const invoke = useMutation(api.invoke.mutationOptions({
    onSuccess: async (stream) => {
      setChunks([]);
      for await (const chunk of stream) {
        setChunks((prev) => [...prev, chunk]);
      }
    },
  }));

  const handleSend = (message: string) => {
    invoke.mutate({
      prompt: message,
      sessionId: 'my-session',
    });
  };

  return (
    <div>
      <button onClick={() => handleSend('Hello!')}>Send</button>
      {invoke.isPending && <p>Agent is thinking...</p>}
      {chunks.map((chunk, i) => (
        <span key={i}>{chunk.content}</span>
      ))}
    </div>
  );
}
```

### 使用原始客户端

`use<AgentName>Client` hook 提供对 API 客户端的直接访问：

```tsx
import { useState } from 'react';
import { useMyAgentClient } from '../hooks/useMyAgentClient';
import type { StreamChunk } from '../generated/my-agent/types.gen';

function ChatComponent() {
  const client = useMyAgentClient();
  const [chunks, setChunks] = useState<StreamChunk[]>([]);

  const handleSend = async (message: string) => {
    setChunks([]);
    for await (const chunk of client.invoke({
      prompt: message,
      sessionId: 'my-session',
    })) {
      setChunks((prev) => [...prev, chunk]);
    }
  };

  return (
    <div>
      <button onClick={() => handleSend('Hello!')}>Send</button>
      {chunks.map((chunk, i) => (
        <span key={i}>{chunk.content}</span>
      ))}
    </div>
  );
}
```

## 本地开发

连接生成器会自动配置 `serve-local` 集成：

1. 运行 `nx serve-local <website>` 也会启动代理的本地 FastAPI 服务器
2. 运行时配置被覆盖以指向本地 HTTP URL（例如 `http://localhost:8081/`）
3. 当代理的 API 发生变化时，TypeScript 客户端会自动重新生成

<NxCommands commands={['serve-local <WebsiteProject>']} />

:::tip[热重载]
网站和连接的代理将热重载，使您能够快速迭代两者，而无需部署到 AWS。
:::

## 本地开发

### 启动本地 DynamoDB

生成器配置了一个 `serve-local` 目标，启动 [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html) 实例并创建表：

<NxCommands commands={['run <project-name>:serve-local']} />

这会自动：
1. 拉取 DynamoDB Local 镜像（`pull-image` 目标）
2. 启动容器
3. 创建包含 `dynamodb.config.json` 中定义的索引的本地表

### 数据建模

生成的项目使用 [ElectroDB](https://electrodb.dev/) 在单个 DynamoDB 表上进行类型安全的实体建模。在 `src/entities/` 下添加或更新实体文件，使用生成的示例实体作为起点。

示例实体定义：

```ts title="packages/my-table/src/entities/example.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createExampleEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'example',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: {
          type: 'string',
          required: true,
        },
        createdAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: {
            field: 'pk',
            composite: ['id'],
          },
          sk: {
            field: 'sk',
            composite: [],
          },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

有关更多详细信息，请参阅 [ElectroDB 实体文档](https://electrodb.dev/en/modeling/entities/)。

### 使用 DynamoDB 客户端

生成的 `src/client.ts` 导出两个关键实用程序：

- `getDynamoDBClient()` — 返回缓存的单例 `DynamoDBClient`。当 `SERVE_LOCAL=true` 时，连接到本地 DynamoDB Local 实例；否则使用默认凭证链创建 AWS 客户端。
- `resolveTableName()` — 返回 DynamoDB 表名。当 `SERVE_LOCAL=true` 时，返回本地表名常量；否则使用 `RUNTIME_CONFIG_APP_ID` 环境变量从 AWS AppConfig 获取名称，并为后续调用缓存它。

### 停止本地 DynamoDB

停止 `serve-local`（例如使用 `Ctrl+C`）会自动删除 DynamoDB Local 容器，但会保留命名卷，以便您的数据在重启后保持不变。

:::caution[Windows]
由于 Windows 上信号处理的限制，停止 `serve-local` 时不会自动删除容器。您需要手动删除它：

```bash
<engine> rm -f <scope>-dynamodb
```

将 `<engine>` 替换为您的容器引擎（`docker` 或 `finch`），将 `<scope>` 替换为您的 Nx 工作区作用域（例如 `proj`）。
:::

## 添加/删除全局二级索引

GSI 在项目根目录的 `dynamodb.config.json` 中定义。为每个 GSI 添加一个条目到 `globalSecondaryIndexes`，遵循 [单表设计](https://electrodb.dev/en/core-concepts/single-table-relationships/) 的 GSI 键命名约定：

```json title="packages/my-table/dynamodb.config.json"
{
  "globalSecondaryIndexes": [
    {
      "indexName": "gsi1pk-gsi1sk-index",
      "partitionKey": "gsi1pk",
      "sortKey": "gsi1sk"
    }
  ]
}
```

`sortKey` 字段对于仅哈希键的 GSI 是可选的。

此配置文件由所有使用者直接从项目根目录读取：
- **本地开发** — `serve-local` 读取 `dynamodb.config.json` 并创建或更新本地表以匹配 GSI 列表
- **CDK** — 构造在合成时读取 `dynamodb.config.json`，因此 GSI 更改会在下次 `cdk deploy` 时反映
- **Terraform** — 模块在计划/应用时读取 `dynamodb.config.json`

### 每次部署一个 GSI

:::caution
DynamoDB [不允许在单次表更新中创建或删除多个 GSI](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/GSI.OnlineOps.html)。

每次部署最多可以添加或删除**一个 GSI**。如果您需要添加或删除多个 GSI，请一次执行一个 — 更新 `dynamodb.config.json`，部署堆栈，然后重复下一个更改。
:::

## 连接到表

在任何 TypeScript 项目中，从您的 DynamoDB 包导入实体工厂并直接使用它们：

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

:::note[运行时配置]
在 AWS 中运行时，`resolveTableName()` 使用 `RUNTIME_CONFIG_APP_ID` 环境变量从 AWS AppConfig 获取表名。使用此插件构建的项目（tRPC API、Smithy API、代理、MCP 服务器）已自动配置此变量。对于其他 TypeScript 项目，请确保在运行时环境中设置 `RUNTIME_CONFIG_APP_ID`，其值为您的基础设施配置的 AppConfig 应用程序 ID。有关更多信息，请参阅<Link path="guides/runtime-config">运行时配置指南</Link>。
:::

### 删除保护

默认情况下启用删除保护以防止意外删除表。

#### 禁用删除保护

对于预期删除表的环境（例如短期开发或预览堆栈），请禁用它。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  deletionProtection: false,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source                      = "../../common/terraform/src/app/dynamodb/my-table"
  deletion_protection_enabled = false
}
```
</Fragment>
</Infrastructure>

### 计费模式

表默认为按需（`PAY_PER_REQUEST`）计费。对于可预测的高吞吐量工作负载，切换到预配置容量。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { BillingMode } from 'aws-cdk-lib/aws-dynamodb';
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  billingMode: BillingMode.PROVISIONED,
  readCapacity: 5,
  writeCapacity: 5,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source       = "../../common/terraform/src/app/dynamodb/my-table"
  billing_mode = "PROVISIONED"
}
```
</Fragment>
</Infrastructure>

### 时间点恢复

[时间点恢复](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Point-in-time-recovery.html)默认情况下启用，允许您将表恢复到过去 35 天内的任何时间点。

#### 禁用时间点恢复

<Infrastructure>

import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  pointInTimeRecoverySpecification: { pointInTimeRecoveryEnabled: false },

</Fragment>

为 TypeScript 项目添加一个 [Nx Generator](https://nx.dev/extending-nx/recipes/local-generators)，帮助自动化重复任务，例如组件脚手架或强制特定项目结构。

## 使用方式

### 生成生成器

有两种方式可以生成生成器：

<RunGenerator generator="ts#nx-generator" />

### 选项配置

<GeneratorParameters schema={schema} />

## 生成器输出

生成器将在指定的 `pluginProject` 中创建以下项目文件：

<FileTree>
  - src/\<name>/
    - schema.json 生成器输入参数模式
    - schema.d.ts 模式的 TypeScript 类型定义
    - generator.ts 生成器实现存根
    - generator.spec.ts 生成器测试
  - generators.json 定义生成器的 Nx 配置文件
</FileTree>

## 本地生成器

:::tip
推荐先使用 `ts#project` 生成器为所有生成器创建专用 TypeScript 项目。例如：

<RunGenerator generator="ts#project" requiredParameters={{ name: 'nx-plugin', directory: 'tools' }} />
:::

运行 `ts#nx-generator` 生成器时选择本地 `nx-plugin` 项目，指定名称及可选的目录和描述。

### 定义模式

`schema.json` 文件定义生成器接受的选项，遵循 [JSON Schema](https://json-schema.org/) 格式并带有 Nx 扩展。

#### 基础结构

schema.json 文件基本结构如下：

```json
{
  "$schema": "https://json-schema.org/schema",
  "$id": "YourGeneratorName",
  "title": "生成器标题",
  "description": "生成器功能描述",
  "type": "object",
  "properties": {
    // 生成器选项在此定义
  },
  "required": ["必填参数1", "必填参数2"]
}
```

#### 简单示例

基础选项示例：

```json
{
  "$schema": "https://json-schema.org/schema",
  "$id": "ComponentGenerator",
  "title": "创建组件",
  "description": "创建新 React 组件",
  "type": "object",
  "properties": {
    "name": {
      "type": "string",
      "description": "组件名称",
      "x-priority": "important"
    },
    "directory": {
      "type": "string",
      "description": "组件创建目录",
      "default": "src/components"
    },
    "withTests": {
      "type": "boolean",
      "description": "是否生成测试文件",
      "default": true
    }
  },
  "required": ["name"]
}
```

#### CLI 交互提示

通过 `x-prompt` 属性自定义 CLI 运行时显示的提示：

```json
"name": {
  "type": "string",
  "description": "组件名称",
  "x-prompt": "请输入组件名称："
}
```

布尔类型选项使用是/否提示：

```json
"withTests": {
  "type": "boolean",
  "description": "是否生成测试文件",
  "x-prompt": "需要生成测试文件吗？"
}
```

#### 下拉选择

固定选项使用 `enum` 定义：

```json
"style": {
  "type": "string",
  "description": "使用的样式方案",
  "enum": ["css", "scss", "styled-components", "none"],
  "default": "css"
}
```

#### 项目选择下拉框

从工作区现有项目中选择：

```json
"project": {
  "type": "string",
  "description": "添加组件的目标项目",
  "x-prompt": "请选择要添加组件的项目：",
  "x-dropdown": "projects"
}
```

`x-dropdown: "projects"` 属性指示 Nx 使用工作区所有项目填充下拉列表。

#### 位置参数

配置 CLI 位置参数：

```json
"name": {
  "type": "string",
  "description": "组件名称",
  "x-priority": "important",
  "$default": {
    "$source": "argv",
    "index": 0
  }
}
```

用户可执行 `nx g your-generator my-component` 替代 `nx g your-generator --name=my-component`。

#### 优先级设置

使用 `x-priority` 标记重要选项：

```json
"name": {
  "type": "string",
  "description": "组件名称",
  "x-priority": "important"
}
```

优先级支持 `"important"` 或 `"internal"`，用于 VSCode 插件选项排序。

#### 默认值

为选项提供默认值：

```json
"directory": {
  "type": "string",
  "description": "组件创建目录",
  "default": "src/components"
}
```

#### 更多信息

详见 [Nx 生成器选项文档](https://nx.dev/extending-nx/recipes/generator-options)。

#### TypeScript 类型定义

生成器同时创建 `schema.d.ts` 文件，为选项提供类型安全：

```typescript
export interface YourGeneratorSchema {
  name: string;
  directory?: string;
  withTests?: boolean;
}
```

该接口用于生成器实现中的类型校验：

```typescript
import { YourGeneratorSchema } from './schema';

export default async function (tree: Tree, options: YourGeneratorSchema) {
  // TypeScript 知晓所有选项类型
  const { name, directory = 'src/components', withTests = true } = options;
  // ...
}
```

:::caution
修改 `schema.json` 后必须同步更新 `schema.d.ts`，包括：

- 添加/删除属性
- 修改属性类型
- 调整必填/可选状态（可选属性使用 `?` 后缀）

TypeScript 接口需准确反映 JSON 模式结构。
:::

### 实现生成器

创建生成器后，可在 `generator.ts` 编写实现逻辑。生成器函数通过操作虚拟文件系统 (`Tree`) 实现变更。

常见操作示例：

#### 文件读写

```typescript
// 读取文件
const content = tree.read('path/to/file.ts', 'utf-8');

// 写入文件
tree.write('path/to/new-file.ts', 'export const hello = "world";');

// 检查文件存在性
if (tree.exists('path/to/file.ts')) {
  // 执行操作
}
```

#### 模板文件生成

```typescript
import { generateFiles, joinPathFragments } from '@nx/devkit';

// 从模板生成文件
generateFiles(
  tree,
  joinPathFragments(__dirname, 'files'), // 模板目录
  'path/to/output', // 输出目录
  {
    // 模板替换变量
    name: options.name,
    nameCamelCase: camelCase(options.name),
    nameKebabCase: kebabCase(options.name),
    // 添加更多变量
  },
);
```

#### TypeScript AST 操作

推荐使用 [TSQuery](https://github.com/phenomnomnominal/tsquery) 进行 AST 操作：

```typescript
import { tsquery } from '@phenomnomnominal/tsquery';
import * as ts from 'typescript';

// 示例：更新版本号

// 解析文件生成 AST
const sourceFile = tsquery.ast(tree.read('path/to/version.ts', 'utf-8'));

// 查找目标节点
const nodes = tsquery.query(
  sourceFile,
  'VariableDeclaration:has(Identifier[name="VERSION"]) NumericLiteral',
);

if (nodes.length > 0) {
  // 获取数值节点
  const numericNode = nodes[0] as ts.NumericLiteral;

  // 版本号递增
  const currentVersion = Number(numericNode.text);
  const newVersion = currentVersion + 1;

  // 替换 AST 节点
  const result = tsquery.replace(
    sourceFile,
    'VariableDeclaration:has(Identifier[name="VERSION"]) NumericLiteral',
    () => ts.factory.createNumericLiteral(newVersion),
  );

  // 写回修改
  tree.write(
    'path/to/version.ts',
    ts
      .createPrinter({
        newLine: ts.NewLineKind.LineFeed,
      })
      .printNode(ts.EmitHint.Unspecified, result, sourceFile),
  );
}
```

:::tip
可在 [TSQuery Playground](https://tsquery-playground.firebaseapp.com/) 在线测试选择器。
:::

#### 添加依赖

```typescript
import { addDependenciesToPackageJson } from '@nx/devkit';

addDependenciesToPackageJson(

    'new-dev-dependency': '^2.0.0',

```

#### 文件格式化

```typescript
import { formatFiles } from '@nx/devkit';

// 格式化所有修改文件
await formatFiles(tree);
```

#### JSON 文件操作

```typescript
import { readJson, updateJson } from '@nx/devkit';

// 读取 JSON
const packageJson = readJson(tree, 'package.json');

// 更新 JSON
updateJson(tree, 'tsconfig.json', (json) => {
  json.compilerOptions = {
    ...json.compilerOptions,
    strict: true,
  };
  return json;
});
```

### 测试生成器

单元测试实现示例：

```typescript
import { createTreeWithEmptyWorkspace } from '@nx/devkit/testing';
import { yourGenerator } from './generator';

describe('生成器测试', () => {
  let tree;

  beforeEach(() => {
    // 创建空工作区树
    tree = createTreeWithEmptyWorkspace();

    // 添加预置文件
    tree.write(
      'project.json',
      JSON.stringify({
        name: 'test-project',
        sourceRoot: 'src',
      }),
    );

    tree.write('src/existing-file.ts', 'export const existing = true;');
  });

  it('应生成预期文件', async () => {
    // 执行生成器
    await yourGenerator(tree, {
      name: 'test',
      // 其他必要参数
    });

    // 验证文件生成
    expect(tree.exists('src/test/file.ts')).toBeTruthy();

    // 校验文件内容
    const content = tree.read('src/test/file.ts', 'utf-8');
    expect(content).toContain('export const test');

    // 使用快照测试
    expect(tree.read('src/test/file.ts', 'utf-8')).toMatchSnapshot();
  });

  it('应更新现有文件', async () => {
    await yourGenerator(tree, {
      name: 'test',
      // 其他必要参数
    });

    const content = tree.read('src/existing-file.ts', 'utf-8');
    expect(content).toContain('import { test } from');
  });

  it('应处理错误', async () => {
    await expect(
      yourGenerator(tree, {
        name: 'invalid',
        // 触发错误的条件
      }),
    ).rejects.toThrow('预期错误信息');
  });
});
```

测试关键点：

- 使用 `createTreeWithEmptyWorkspace()` 创建虚拟文件系统
- 预先设置必要文件
- 测试新文件生成与现有文件更新
- 复杂内容使用快照测试
- 验证错误处理机制

## 向 @aws/nx-plugin 贡献生成器

使用 `ts#nx-generator` 可在 `@aws/nx-plugin` 内创建生成器模板。运行时将生成：

<FileTree>
  - packages/nx-plugin/src/\<name>/
    - schema.json 生成器参数模式
    - schema.d.ts TypeScript 类型定义
    - generator.ts 生成器实现
    - generator.spec.ts 测试文件
  - docs/src/content/docs/guides/
    - \<name>.mdx 生成器文档页
  - packages/nx-plugin/generators.json 更新包含新生成器
</FileTree>

随后即可开始实现生成器逻辑。

此生成器创建一个由 [Amazon Aurora](https://aws.amazon.com/rds/aurora/) PostgreSQL 支持的新关系数据库项目。它生成使用 AWS CDK 或 Terraform 配置和管理数据库所需的应用程序代码和基础设施。

生成的基础设施包括:

- 可重用的 Aurora 构造或模块
- 特定于应用程序的数据库构造或模块
- IAM 认证的应用程序数据库用户处理程序
- 在 AWS AppConfig 中注册运行时配置
- 用于自动部署 Prisma 迁移的迁移处理程序

## 实现您的数据库

### 数据建模

生成的项目使用 [Prisma ORM](https://www.prisma.io/docs/orm) 来定义数据库架构并生成类型安全的客户端。在 `prisma/schema.prisma` 中对表、关系和字段类型进行建模。

`User` 模型示例:

```ts title="packages/postgres/prisma/schema.prisma"
model User {
  id          Int      @id @default(autoincrement()) 
  firstName   String
  lastName    String
}
```

有关更多详细信息,请参阅官方 [Prisma 数据建模指南](https://www.prisma.io/docs/orm/core-concepts/data-modeling#data-modeling-with-prisma-orm)。

### 生成数据库客户端

生成器会自动配置 `generate` 目标,以便在构建项目时创建类型安全的 TypeScript Prisma 客户端。客户端被写入 `generated/prisma`(已添加到 `.gitignore`)。

您也可以随时手动生成客户端:

<Tabs>
  <TabItem label="npm" icon="seti:npm">

```bash
npm run nx generate <your-db-package-name>
```

  </TabItem>
  <TabItem label="pnpm" icon="seti:pnpm">

```bash
pnpm nx generate <your-db-package-name>
```

  </TabItem>
  <TabItem label="yarn" icon="seti:yarn">

```bash
yarn nx generate <your-db-package-name>
```

  </TabItem>
  <TabItem label="bun" icon="seti:bun">

```bash
bun nx generate <your-db-package-name>
```

  </TabItem>
</Tabs>

`src/prisma.ts` 中的运行时包装器导出:

- `DB_PACKAGE_NAME` - 在 AWS AppConfig 的 `database` 运行时配置命名空间下使用的键
- `getPrisma()` - 从 AWS AppConfig 加载数据库连接设置并使用 IAM 认证创建 Prisma 客户端

客户端自动:
- 使用 `RUNTIME_CONFIG_APP_ID` 环境变量从 AWS AppConfig 检索数据库配置
- 通过 AWS RDS Signer 生成临时认证令牌以进行 IAM 认证
- 通过证书验证管理 SSL/TLS 连接
- 通过持久数据库连接池处理连接池

在 tRPC API 中的使用示例:

```ts title="packages/api/src/procedures/list-user.ts"
import { getPrisma } from ':aws-nx-poc/postgres';
import { publicProcedure } from '../init.js';
import { ListUserOutputSchema } from '../schema/index.js';

export const listUser = publicProcedure
  .output(ListUserOutputSchema)
  .query(async () => {
    const prisma = await getPrisma();
    const users = await prisma.user.findMany({
      orderBy: { id: 'asc' },
    });

    return { users };
  });
```

### 创建迁移

:::note[从包目录运行 Prisma]
使用 Prisma CLI 时,从生成的数据库包目录运行命令(例如 `packages/postgres`),以便 Prisma 可以找到 `prisma/schema.prisma` 和 `prisma.config.ts`。
:::

Prisma Migrate 需要一个[影子数据库](https://www.prisma.io/docs/orm/prisma-migrate/understanding-prisma-migrate/shadow-database)来将您的架构与当前数据库状态进行比较并生成 SQL 更改。

要在本地生成迁移,通过导出 `DATABASE_URL` 将 Prisma 指向开发数据库:

```bash
export DATABASE_URL="postgresql://username:password@hostname:5432/dbname"
```

然后创建迁移文件而不应用它们:

```bash

```

  </TabItem>
  <TabItem label="pnpm" icon="seti:pnpm">

pnpm prisma migrate dev --create-only
```

  </TabItem>
  <TabItem label="yarn" icon="seti:yarn">

  </TabItem>

```

  </TabItem>
</Tabs>

## 部署到 AWS

关系数据库生成器根据您选择的 `iacProvider` 创建 CDK 或 Terraform 基础设施。

<Infrastructure>
<Fragment slot="cdk">
CDK 构造在 `common/constructs` 中创建。使用示例:

```ts title="packages/infra/src/stacks/application-stack.ts"
import { Postgres } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const vpc = new Vpc(this, 'Vpc', {
      subnetConfiguration: [
        {
          name: 'public',
          subnetType: SubnetType.PUBLIC,
        },
        {
          name: 'private_with_egress',
          subnetType: SubnetType.PRIVATE_WITH_EGRESS,
        },
        {
          name: 'private_isolated',
          subnetType: SubnetType.PRIVATE_ISOLATED,
        },
      ],
    });

    const db = new Postgres(this, 'Db', {
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_ISOLATED,
      }
    });
  }
}
```

这会配置一个带有 RDS Proxy、管理员凭证、应用程序数据库用户、运行时配置注册和迁移处理程序的 Aurora 集群。部署到私有隔离子网以防止直接互联网访问。

生成的基础设施创建两个数据库用户:
- **管理员用户** - 在集群配置期间创建,凭证存储在 AWS Secrets Manager 中
- **应用程序用户** - 通过 Lambda 自定义资源创建,启用 IAM 认证并对应用程序数据库具有完全权限
</Fragment>
<Fragment slot="terraform">
Terraform 模块在 `common/terraform` 中创建。使用示例:

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"

  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_isolated_subnet_ids

  tags = local.common_tags
}
```

这会配置一个带有 RDS Proxy、管理员凭证、create-db-user Lambda、运行时配置注册、迁移 Lambda 和容器注册表资源的 Aurora 集群。

生成的基础设施创建两个数据库用户:
- **管理员用户** - 在集群配置期间创建,凭证存储在 AWS Secrets Manager 中
- **应用程序用户** - 通过 Lambda 函数创建,启用 IAM 认证并对应用程序数据库具有完全权限
</Fragment>

### 连接 Lambda 函数

要从 Lambda 函数使用 Prisma 客户端,请确保满足以下要求:

1. 在与数据库相同的 VPC 中部署函数
2. 允许对数据库端口的网络访问
3. 为生成的运行时用户授予 IAM `rds-db:connect` 权限
4. 设置所需的环境变量:
   - `NODE_EXTRA_CA_CERTS=/var/runtime/ca-cert.pem`(TLS 验证所需)
   - `RUNTIME_CONFIG_APP_ID`(由生成的基础设施自动设置)
   - `AWS_REGION`(在 Lambda 执行环境中自动设置)

:::note[Prisma CLI 与运行时客户端]
**Prisma CLI** 命令(`prisma migrate dev`、`prisma migrate deploy`)从环境变量读取 `DATABASE_URL`。

**运行时客户端**(`src/prisma.ts`)从 AWS AppConfig 加载连接设置并使用 IAM 认证。
:::

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { RuntimeConfig } from ':my-scope/common-constructs';

const runtimeConfig = RuntimeConfig.ensure(this);

const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_WITH_EGRESS,
      },
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});

Object.entries(api.integrations).forEach(([operation, integration]) => {
  db.allowDefaultPortFrom(
    integration.handler,
    `Allow ${operation} to connect to the database`,
  );
  db.grantConnect(integration.handler);
});
```

这会配置每个 API Lambda 函数在同一 VPC 中运行,从 AppConfig 读取运行时配置,并使用 IAM 认证连接到数据库。
</Fragment>
<Fragment slot="terraform">
将 Lambda 函数连接到数据库时,使用这些模块输出:

- `appconfig_application_id` - 暴露给计算环境
- `cluster_port` - 允许网络访问
- `database_runtime_user` - 授予 `rds-db:connect` 权限

确保计算在同一 VPC 和私有子网中运行,并向计算角色授予 AppConfig 读取权限。
</Fragment>
</Infrastructure>

### RDS Proxy

生成的基础设施默认包含一个 RDS Proxy,它位于您的应用程序和 Aurora 集群之间。RDS Proxy 提供多项好处:

- **连接池** - 维护一个可在应用程序实例间共享的数据库连接池,减少建立新连接的开销
- **连接弹性** - 在 Aurora 实例替换或维护期间自动处理故障转移和重新连接
- **IAM 认证** - 支持基于 IAM 的数据库认证,无需在应用程序代码中管理数据库凭证
- **改进的安全性** - 对所有连接强制执行 TLS 加密

<Infrastructure>
<Fragment slot="cdk">

默认情况下,RDS Proxy 已启用。生成的运行时客户端(`getPrisma()`)会自动通过代理端点连接。如果需要,您可以禁用它:

```ts title="packages/infra/src/stacks/application-stack.ts"
const db = new Postgres(this, 'Db', {
  ...
  enableRdsProxy: false,
});
```

禁用 RDS Proxy 后,您的应用程序将直接连接到 Aurora 集群端点。
</Fragment>
<Fragment slot="terraform">

默认情况下,RDS Proxy 已启用。生成的运行时客户端(`getPrisma()`)会自动通过代理端点连接。如果需要,您可以禁用它:

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"
  ...
  enable_rds_proxy = false
}
```

禁用 RDS Proxy 后,您的应用程序将直接连接到 Aurora 集群端点。代理配置为需要 TLS 加密并使用 IAM 认证。
</Fragment>
</Infrastructure>

## 本地开发

已连接的项目公开了两个目标，用于在您的机器上运行它们：`serve` 和 `serve-local`。区别在于**范围** —— 您的应用程序有多少部分在本地运行，而不是指向已部署的 AWS 基础设施。

考虑一个具有以下连接的工作空间：一个调用 tRPC `api` 的网站，还调用一个 `agent`，而该 agent 又调用一个 `mcp` 服务器。

### `serve`

`serve` 目标**仅在本地运行目标项目**。它连接到的每个其他项目都应该是**已部署的**，并通过其真实的 AWS URL 访问。

<NxCommands commands={['serve website']} />

由于目标项目与已部署的资源通信，它需要知道它们在哪里。对于网站，这由其 `runtime-config.json` 提供（参见<Link path="guides/react-website#local-runtime-config">本地运行时配置</Link>），您可以从已部署的应用程序加载它。对于读取<Link path="guides/runtime-config">运行时配置</Link>的服务器端项目（API 和 agent），请设置 `RUNTIME_CONFIG_APP_ID` 环境变量以指向您已部署的 AppConfig 应用程序。

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
agent: agent {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
mcp: mcp {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

当您想要针对它所依赖的所有内容的"真实"已部署版本迭代单个项目时，请使用 `serve`。

### `serve-local`

`serve-local` 目标运行目标项目**以及与其传递连接的每个项目**，全部在您的机器上。连接生成器会自动设置这一点 —— 在网站上运行 `serve-local` 还会为 `api`、`agent` 以及它通过 agent 访问的 `mcp` 服务器启动本地服务器。

<NxCommands commands={['serve-local website']} />

以这种方式运行时，网站的 `runtime-config.json` 会自动被覆盖（通过 [Vite 的 `MODE`](https://vite.dev/guide/env-and-mode)，设置为 `serve-local`），以便它指向您本地运行的服务器而不是已部署的 URL。

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#c8e6c9"
}
agent: agent {
  style.fill: "#c8e6c9"
}
mcp: mcp {
  style.fill: "#c8e6c9"
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

当您同时处理多个已连接的项目并希望快速迭代而无需部署基础设施时，请使用 `serve-local`。

:::note[`serve-local` 中的 AWS 资源]
您的本地服务器使用您在本地配置的 AWS 凭证运行，因此它们仍然可以在需要时访问已部署的 AWS 资源（例如 DynamoDB 表或 Bedrock 模型）。您可以提供本地服务器所需的任何环境变量，例如：

<NxCommands env={{RUNTIME_CONFIG_APP_ID: 'xxxxx'}} commands={['serve-local website']} />
:::

有关每种项目类型的本地开发服务器的更多详细信息，请参阅相关指南 —— 例如 <Link path="guides/react-website#local-development-server">React Website</Link> 指南。

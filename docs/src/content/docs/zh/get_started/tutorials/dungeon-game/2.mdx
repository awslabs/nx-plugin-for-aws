## 模块 2：游戏 API 实现

我们将从实现游戏 API 开始。为此需要总共创建 4 个 API：

1. `createGame` - 用于创建新游戏实例
2. `queryGames` - 返回分页的已保存游戏列表
3. `saveAction` - 保存指定游戏的操作记录
4. `queryActions` - 返回指定游戏相关的分页操作记录

### 部署与测试

首先构建代码库：

<NxCommands commands={['run-many --target build --all']} />

<Aside type="tip">
如遇 lint 错误，可运行以下命令自动修复：

<NxCommands commands={['run-many --target lint --configuration=fix --all']} />
</Aside>

运行以下命令部署应用：

<NxCommands commands={['run @dungeon-adventure/infra:deploy dungeon-adventure-infra-sandbox']} />

首次部署约需 8 分钟，后续部署约 2 分钟。

:::tip
如需迭代修改 lambda 函数代码，可在构建后使用 `--hotswap` 标志进行热部署（2-3 秒）：

<NxCommands commands={['run @dungeon-adventure/infra:deploy dungeon-adventure-infra-sandbox --hotswap']} />
:::

<Drawer title="部署命令" trigger="点击查看批量部署详情">

可通过以下命令一次性部署所有堆栈：

<NxCommands commands={['run @dungeon-adventure/infra:deploy --all']} />

**不推荐此方式**，因为实际场景中可能需要分阶段部署不同环境（如生产环境 infra-prod）。使用 `--all` 会尝试部署所有堆栈，可能导致意外部署！

</Drawer>

<ul>
<li>启动 tRPC 后端本地实例并使用 `curl` 调用 API</li>
<li>
<Drawer title="支持 Sigv4 签名的 curl" trigger="调用已部署 API">
可将以下脚本添加至 `.bashrc` 文件（需执行 `source`）或直接粘贴到终端：
```bash
// ~/.bashrc
acurl () {
    REGION=$1
    SERVICE=$2
    shift; shift;
    curl --aws-sigv4 "aws:amz:$REGION:$SERVICE" --user "$(aws configure get aws_access_key_id):$(aws configure get aws_secret_access_key)" -H "X-Amz-Security-Token: $(aws configure get aws_session_token)" "$@"
}
```

调用示例：

###### API 网关
```bash
acurl ap-southeast-2 execute-api -X GET https://xxx
```

###### Lambda 函数 URL
```bash
acurl ap-southeast-2 lambda -N -X POST https://xxx
```
</Drawer>
</li>
</ul>

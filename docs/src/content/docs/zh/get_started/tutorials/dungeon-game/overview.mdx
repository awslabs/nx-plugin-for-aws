## 概述

在本教程中，您将构建一个由AI驱动的地牢冒险游戏。本教程不要求事先具备`@aws/nx-plugin`或相关技术的知识。您将学习到的技术是构建任何基于`@aws/nx-plugin`应用程序的基础，充分理解这些内容将使您深入掌握`@aws/nx-plugin`，并对如何使用[NX](https://nx.dev/)有基本了解。

<Aside>本教程适合那些喜欢在实践中学习并希望快速尝试构建具体项目的人群。如果您更倾向于逐步学习每个概念，请参考各个组件的单独指南</Aside>

完成本教程的用户将掌握以下技能：
- 使用`@aws/nx-plugin`创建新应用
- 使用NX管理和构建代码库
- 使用tRPC和FastAPI构建API
- 使用Tanstack Router创建新页面
- 使用Tanstack Query调用后端API
- 修改生成代码以满足特定需求
- 创建并部署CDK基础设施

### 应用组件

AI驱动的地牢冒险游戏将采用以下组件架构：

<Image class="white-bg" src={dungeonAdventureArchitecturePng} alt="dungeon-game-architecture.png" width="800" height="600" />

- [tRPC](https://trpc.io/) API：使用[ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/)管理游戏状态
- Python/[FastAPI](https://fastapi.tiangolo.com/) Lambda函数：通过流式响应生成故事情节（基于[Amazon Bedrock](https://aws.amazon.com/bedrock/)）
- React/Vite前端网站包含：
  - 使用Amazon Cognito/Identity Pools实现安全API调用
  - 支持类型安全文件路由的[Tanstack Router](https://tanstack.com/router/latest)
  - 调用游戏和故事API的生成SDK（利用[Tanstack Query](https://tanstack.com/query/latest)）
- 部署应用的[CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html)基础设施

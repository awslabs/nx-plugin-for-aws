## 总结

恭喜！您已成功创建了一个 Agentic AI 地牢冒险游戏,该游戏使用了 `@aws/nx-plugin` 中提供的多种生成器。🎉🎉🎉

## 后续步骤

我们建议您尝试自行扩展代码库,添加以下功能:

1. 为项目添加<Link path="guides/license">许可证生成器</Link>,自动管理工作区中的 LICENSE 文件与源代码头部信息。
2. <Link path="get_started/tutorials/contribute-generator">贡献自定义生成器</Link>来简化新 tRPC API 的添加流程。
3. 将 [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html) 与 Story Agent 集成,使长时间运行的冒险在服务器重启后仍能保留。`ag_ui_strands` 适配器提供了 `session_manager_provider` 钩子来实现此功能 - 请参阅 [在 Strands 中使用 AgentCore Memory](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands) 文档。
4. 使用左侧导航栏探索各种组件指南。

## 销毁资源

1. 要销毁已创建的 AWS 资源,请运行以下命令:

<NxCommands commands={['destroy infra "dungeon-adventure-infra-sandbox/*"']} />

    该命令将提示选择要删除的堆栈列表,通常包含 `dungeon-adventure-infra-sandbox/Application/GameUI/waf` 和 `dungeon-adventure-infra-sandbox/Application`。

2. 输入 `Y` 继续。CloudFormation 将销毁您的堆栈。

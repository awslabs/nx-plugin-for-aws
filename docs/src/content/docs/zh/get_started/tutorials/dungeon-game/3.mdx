## 任务 1：实现 Story 代理

Story 代理是一个在<Link path="get_started/tutorials/dungeon-game/1">模块 1</Link>中使用 `--protocol=AG-UI` 生成的 [Strands](https://strandsagents.com/) 代理，因此 UI 可以通过 CopilotKit 使用 [Agent-User Interaction 协议](https://docs.copilotkit.ai/aws-strands/protocol)从中进行流式传输。它使用 Inventory MCP Server 来管理玩家的物品，并使用 Strands 内置的 [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) 将对话历史持久化到我们在模块 2 中配置的会话存储桶中。

更改内容如下：

- `main.py` 添加了一个 `session_manager_provider`，在部署时为每个 `thread_id` 创建一个 `S3SessionManager`，并在 `agent-serve-local` 下运行时（`SERVE_LOCAL=true`）回退到 `/tmp/strands-sessions` 下的磁盘 `FileSessionManager`。在部署时，S3 存储桶与 Game API 的 `queryActions` 读取的存储桶相同，因此浏览器可以在重新访问时重建记录；在本地，代理将会话持久化到磁盘并与本地 MCP 服务器通信，无需部署。
- `agent.py` 删除了示例 `subtract` 工具，并将系统提示替换为地牢主持人提示，邀请第一条用户消息说明玩家的名字和类型，并使用 Inventory MCP Server 的工具。

### 与您的代理聊天

生成的 `agent-chat` 目标会针对您的代理打开一个交互式 REPL。它独立运行并连接到您本地运行的代理，因此首先在一个终端中启动代理的本地服务器：

<NxCommands commands={["agent-serve-local story"]} />

然后，在第二个终端中启动聊天：

<NxCommands commands={["agent-chat story"]} />

您的第一条消息应该告诉代理您的英雄名字和类型（例如：`My name is Alice. Start my zombie adventure.`），故事将流式返回。

<Aside type="caution" title="需要 AWS 凭证">
尽管代理在本地运行，但它会调用 [Amazon Bedrock](https://aws.amazon.com/bedrock/) 来使用 LLM。您需要在环境中配置 AWS 凭证，并具有访问 [Strands 默认模型](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/)的权限。有关详细信息，请参阅 [Strands 模型提供程序文档](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/)。
</Aside>

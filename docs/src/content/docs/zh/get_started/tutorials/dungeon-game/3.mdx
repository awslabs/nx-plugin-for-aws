## 任务 1：实现故事代理

故事代理是一个通过 [AG-UI](https://ag-ui.com/) 协议公开的 [Strands](https://strandsagents.com/) 代理。每个 AG-UI 会话（由 `threadId` 标识）映射到服务器上的一个长期存在的代理实例，因此代理的对话历史在会话的生命周期内保存在内存中。

我们将更新代理的系统提示以进行地牢冒险游戏，并删除示例工具。

### 代理实现

更新 `packages/story/dungeon_adventure_story/agent/agent.py`：

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

系统提示告诉代理第一条消息的格式为 `I am <player>, playing in the <genre> genre.` — 我们的 UI 正好发送这个作为开场用户消息，因此代理拥有开始故事所需的一切。由于 AG-UI 库会自动保留每个线程的对话历史，我们不需要在后续回合中传递玩家名称或类型。

连接生成器已在 <Link path="get_started/tutorials/dungeon-game/1">模块 1</Link> 中配置好库存管理控制协议（MCP）客户端，因此代理可以访问库存工具。

:::note[`main.py` 在哪里？]
AG-UI 协议的 FastAPI 包装器为我们处理请求/响应转换。生成器生成的 `main.py` 已经在 `get_agent(...)` 周围创建了包装器 — 我们不需要修改它。
:::

### 测试代理

可通过以下方式测试代理：

<li>启动代理服务器本地实例并使用 `agent-chat` 目标进行交互式聊天，或</li>

    然后在另一个 shell 中运行 `agent-chat-cli agui http://localhost:8081/invocations`。
    :::

    当提示时，尝试用类似以下内容开始游戏：

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="已部署">
    测试已部署代理需通过 Cognito 认证获取 JWT 令牌。首先设置环境变量：

    ```bash
    # 从 CDK 输出中设置 Cognito 用户池 ID 和客户端 ID
    export POOL_ID="<CDK 输出的 UserPoolId>"
    export CLIENT_ID="<CDK 输出的 UserPoolClientId>"
    export REGION="<区域>"
    ```

    创建测试用户并获取认证令牌：

    ```bash
    # 禁用 MFA 以简化用户创建
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # 创建用户
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # 设置永久密码（请替换为更安全的密码！）
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 用户认证并获取访问令牌
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    通过 Bedrock AgentCore 运行时 URL 调用已部署代理。AG-UI 协议期望一个 `RunAgentInput` 负载 — 一个发送开场用户消息的最小负载如下所示：

    ```bash
    # 设置 CDK 输出的 Story Agent ARN
    export AGENT_ARN="<CDK 输出的 StoryAgentArn>"

    # URL 编码 ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 构造调用 URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # 调用代理
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[URL 编码]
    URL 编码是正确格式化 Bedrock AgentCore API 端点 ARN 的必要步骤。
    :::
  </TabItem>
</Tabs>

若命令运行成功，将开始看到 AG-UI 服务器发送事件流式返回 — `TEXT_MESSAGE_CONTENT` 事件逐块携带故事文本。

## 任务 1：实现 Story 代理

Story 代理是一个在<Link path="get_started/tutorials/dungeon-game/1">模块 1</Link>中使用 `--protocol=AG-UI` 生成的 [Strands](https://strandsagents.com/) 代理，因此 UI 可以通过 CopilotKit 使用 [Agent-User Interaction 协议](https://docs.copilotkit.ai/aws-strands/protocol)从中进行流式传输。它使用 Inventory MCP Server 来管理玩家的物品，并使用 Strands 内置的 [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) 将对话历史持久化到我们在模块 2 中配置的会话存储桶中。

更改内容如下：

- `main.py` 添加了一个 `session_manager_provider`，在部署时为每个 `thread_id` 创建一个 `S3SessionManager`，并在 `agent-serve-local` 下运行时（`SERVE_LOCAL=true`）回退到 `/tmp/strands-sessions` 下的磁盘 `FileSessionManager`。在部署时，S3 存储桶与 Game API 的 `queryActions` 读取的存储桶相同，因此浏览器可以在重新访问时重建记录；在本地，代理完全离线运行，针对本地 MCP 服务器，无需 AWS 调用。
- `agent.py` 删除了示例 `subtract` 工具，并将系统提示替换为地牢主持人提示，邀请第一条用户消息说明玩家的名字和类型，并使用 Inventory MCP Server 的工具。

### 与您的代理聊天

使用生成的 `agent-chat` 目标针对本地服务的 AG-UI 代理副本打开交互式 REPL：

<NxCommands commands={["agent-chat story"]} />

`agent-chat` 具有 `dependsOn: ['agent-serve-local']`，因此无需单独启动代理服务器 — Nx 将生成 `agent-serve-local`（它反过来启动 Inventory MCP 服务器，并通过它启动 DynamoDB Local），然后针对 `http://localhost:8081/invocations` 启动 [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) AG-UI REPL。整个代理堆栈在本地运行，无需 AWS 调用。

您的第一条消息应该告诉代理您的英雄名字和类型（例如：`My name is Alice. Start my zombie adventure.`），故事将流式返回。

<Aside type="tip" title="本地会话历史">
在 `agent-serve-local`（`SERVE_LOCAL=true`）下，对话历史通过 Strands 的 `FileSessionManager` 写入 `/tmp/strands-sessions` 而不是 S3。在后续调用中重用相同的 `threadId` 会继续相同的故事 — 文件会话管理器会重放早期的消息。
</Aside>

<Aside type="tip" title="原始 curl">
如果您更喜欢针对本地服务器的原始 HTTP 调用，AG-UI `RunAgentInput` 的形式为：
```bash
curl -N -X POST http://127.0.0.1:8081/invocations \
  -H "Content-Type: application/json" \
  -d '{
    "threadId": "Alice-zombie00000000000000000000000",
    "runId": "run-1",
    "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
    "tools": [], "context": [], "state": {}, "forwardedProps": {}
  }'
```
</Aside>

如果命令成功运行，您应该开始看到故事以 [AG-UI 事件流](https://docs.copilotkit.ai/aws-strands/protocol/events)（Server-Sent Events）的形式流式返回 — `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` 块包装实际的故事令牌：

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

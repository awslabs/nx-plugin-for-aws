## 任务 1：实现故事代理

故事代理是一个在 <Link path="get_started/tutorials/dungeon-game/1">模块 1</Link> 中使用 `--protocol=AG-UI` 生成的 [Strands](https://strandsagents.com/) 代理，因此 UI 可以通过 CopilotKit 以 [Agent-User Interaction 协议](https://docs.copilotkit.ai/aws-strands/protocol)从中流式传输数据。它使用库存管理控制协议（MCP）服务器来管理玩家的物品，并使用 Strands 内置的 [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) 将对话历史持久化到我们在模块 2 中配置的会话存储桶中。

:::note[持久化对话历史]
每个 AG-UI 请求都携带一个 `thread_id`。我们的 UI 以确定性方式构建该 id，格式为 ``{player}-{genre}``，并填充到 AgentCore 最小长度 33 个字符，因此每个 `(player, genre)` 对都会恢复其自己的故事。在代理端，`StrandsAgentConfig.session_manager_provider` 为每个线程提供一个指向 `StorySessions` 存储桶的 `S3SessionManager` —— 因此后续回合会重放存储的消息历史，而无需 UI 发送它。由于 Game API 直接从同一存储桶读取消息，浏览器也可以在页面重新加载时显示完整的对话记录。对于生产环境设置，可通过 [AgentCore Memory for Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands) 将其替换为 [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html)。
:::

<Tabs>
  <TabItem label="本地">
  启动本地代理服务器：
    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    <Aside type="caution" title="使用CDK输出值">
    请使用 CDK 部署输出的 `RuntimeConfigApplicationId` 值作为 `RUNTIME_CONFIG_APP_ID` 环境变量，并使用已部署的 AWS 区域作为 `AWS_REGION`。
    </Aside>

    :::note[`agent-serve` 与 `agent-serve-local`]
    `agent-serve` 目标在本地启动代理，但连接到**已部署**的 MCP 服务器（通过 `RUNTIME_CONFIG_APP_ID`）。如果您希望同时在本地运行 MCP 服务器，请改用 `agent-serve-local`，它会启动所有依赖服务并设置 `SERVE_LOCAL=true` 以进行直接 HTTP 连接。
    :::

    然后，在第二个终端中，通过生成的 `agent-chat` 目标打开针对本地 AG-UI 代理的交互式 REPL：

    <NxCommands commands={["run dungeon_adventure.story:agent-chat"]} />

    这使用 [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) 通过 AG-UI 协议与 `http://localhost:8081/invocations` 通信。您的第一条消息应该告诉代理您的英雄姓名和类型（例如：`My name is Alice. Start my zombie adventure.`），故事将流式返回。

    <Aside type="tip" title="原始 curl">
    如果您更喜欢原始 HTTP 调用，AG-UI `RunAgentInput` 格式为：
    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -H "Content-Type: application/json" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```
    在后续调用中重用相同的 `threadId` 将继续相同的故事 —— `S3SessionManager` 会重放之前的消息。
    </Aside>
  </TabItem>
  <TabItem label="已部署">
    测试已部署代理需通过 Cognito 认证获取 JWT 令牌。首先设置环境变量：

    ```bash
    # 从 CDK 输出中设置 Cognito 用户池 ID 和客户端 ID
    export POOL_ID="<CDK 输出的 UserPoolId>"
    export CLIENT_ID="<CDK 输出的 UserPoolClientId>"
    export REGION="<区域>"
    ```

    创建测试用户并获取认证令牌：

    ```bash
    # 禁用 MFA 以简化用户创建
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # 创建用户
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # 设置永久密码（请替换为更安全的密码！）
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 用户认证并获取访问令牌
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    通过 Bedrock AgentCore 运行时 URL 调用已部署代理：

    ```bash
    # 设置 CDK 输出的 Story Agent ARN
    export AGENT_ARN="<CDK 输出的 StoryAgentArn>"

    # URL 编码 ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 构造调用 URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # 使用 AG-UI RunAgentInput 负载调用代理
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: Alice-zombie00000000000000000000000" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```

    :::note[URL 编码]
    URL 编码是正确格式化 Bedrock AgentCore API 端点 ARN 的必要步骤。
    :::

</Tabs>

若命令运行成功，将开始看到故事以 [AG-UI 事件流](https://docs.copilotkit.ai/aws-strands/protocol/events)（Server-Sent Events）的形式流式返回 —— `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` 块包装实际的故事令牌：


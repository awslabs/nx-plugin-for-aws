## 任务一：配置本地开发服务器

要开始构建用户界面，我们需要将本地开发服务器配置指向已部署的沙盒环境。运行以下命令：

<NxCommands commands={["run game-ui:load:runtime-config"]} />

该命令将拉取已部署的 `runtime-config.json` 文件，并存储在本地 `packages/game-ui/public` 目录中。

要启动开发服务器，运行以下命令：

<NxCommands commands={["serve game-ui"]} />

在浏览器中打开本地网站。系统将提示您使用之前创建的 Cognito 用户登录。登录后，您将看到基础网站界面。

<Aside type="caution" title="保持开发服务器运行">
我们建议在本模块剩余部分保持开发服务器运行，因为它会自动热重载我们所做的任何修改。
</Aside>

## 任务四：创建游戏页面

在 <Link path="get_started/tutorials/dungeon-game/1">模块一</Link> 中，我们将 `GameUI` 连接到故事代理。由于代理是使用 `--protocol=AG-UI` 生成的，`connection` 生成器为我们配置了以下内容：

- 一个包裹应用的 `<AguiProvider>`（一个 [CopilotKit](https://copilotkit.ai/) `CopilotKitProvider`），已注入到 `main.tsx` 中。
- 一个 `useAguiStoryAgent` 钩子，用于构建指向已部署 AgentCore 运行时的 Cognito 认证 AG-UI `HttpAgent`。
- `src/components/copilot` 下的一组 shadcn 主题聊天原语——一个与网站其余部分匹配的预样式化 `<CopilotChat>`。

我们只需要创建两个使用它们的路由文件：

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="路径参数">
`$playerName` 语法指示 `@tanstack/react-router` 将 `playerName` 作为[路径参数](https://tanstack.com/router/v1/docs/framework/react/guide/path-params)处理。我们使用玩家名称作为 AG-UI `threadId`，这样每个游戏都有自己的聊天历史记录，并将其填充到 33 个字符以满足 AgentCore 的会话 id 长度要求。
</Aside>

<Aside type="note" title="初始化第一条消息">
在首次挂载时，我们检查 `agent.messages.length === 0`，如果为空，则推送一条开场用户消息：`I am <player>, playing in the <genre> genre.` 系统提示告诉代理立即从该提示开始故事。从那时起，每次交互（包括页面刷新和稍后返回）只需通过 AG-UI 会话重新加载对话——无需自定义历史端点。
</Aside>
</TabItem>
</Tabs>

保存这些文件后，您的本地开发服务器（http://localhost:4200/）现在应该已准备好游戏供您游玩。

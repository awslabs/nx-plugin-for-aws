## 模块四：用户界面实现

要开始构建用户界面，我们需要将本地开发服务器配置指向已部署的沙箱环境。运行以下命令：

<NxCommands commands={["run @dungeon-adventure/game-ui:load:runtime-config"]} />

该命令将拉取已部署的 `runtime-config.json` 文件，并存储在本地 `packages/game-ui/public` 目录中。

现在可以通过以下命令启动开发服务器：

<NxCommands commands={["run @dungeon-adventure/game-ui:serve"]} />

随后在浏览器中打开本地网站，系统将提示登录并按照指引创建新用户。完成后您将看到基础网站界面：

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution">
_在本模块剩余部分，请保持开发服务器运行，它将自动热重载所有修改。_
</Aside>

:::tip
您也可以运行 <NxCommands commands={["run @dungeon-adventure/game-ui:serve-local"]} />，该命令会同时启动所有关联的API本地服务器，方便您同步迭代tRPC API和网站开发。
:::

### 创建新路由 '/game'

我们将通过创建类型安全路由来展示 `@tanstack/react-router` 的功能。在指定位置创建空文件：`packages/game-ui/src/routes/game/index.tsx`，文件将立即自动更新。

`@tanstack/react-router` 已自动配置新路由，您会注意到新建文件已包含路由路径：

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

访问 `http://localhost:4200/game` 即可看到新页面已渲染！

<Image src={baselineGamePng} alt="baseline-game.png" width="800" height="600" />

### 故事代理集成

接下来创建用于初始化故事代理客户端的钩子：

<Tabs>
<TabItem label="hooks/useStoryAgent.tsx">
<E2ECode path="dungeon-adventure/4/hooks/useStoryAgent.tsx.template" lang="tsx" />
</TabItem>
</Tabs>

该钩子实现以下功能：
- 从 `runtime-config.json` 获取代理ARN
- 根据ARN构建AgentCore运行时调用URL
- 使用已登录用户的JWT令牌和随机会话ID调用代理
- 返回用于消费流式消息的异步迭代器

:::tip
您会注意到与游戏API的集成已自动配置——使用 `api-connection` 生成器后无需手动操作。如需类似代理体验，请为[此GitHub议题](https://github.com/awslabs/nx-plugin-for-aws/issues/326)点赞。
:::

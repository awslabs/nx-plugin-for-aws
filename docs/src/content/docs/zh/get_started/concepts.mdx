`@aws/nx-plugin` 是一个 [Nx](https://nx.dev/) 插件，为在 AWS 上构建和部署全栈应用程序提供工具包。它为您提供一系列[生成器](#generators)，可以脚手架生成生产就绪的应用程序代码_以及_部署所需的基础设施——类型安全、可本地运行且随时可发布。

您（或您的 AI 代理）无需从空白页面开始，只需选择所需的组件——API、网站、身份验证、AI 代理、基础设施——提供一些配置选项，插件就会生成最佳实践的初始代码。它甚至会为您连接项目（包括通过 AST 转换更新现有文件），在前端和后端之间生成类型安全的客户端。

## 基于 Nx 及其生成器构建

[Nx](https://nx.dev) 是一个用于管理单体仓库的智能构建框架。它与语言无关，让 TypeScript、Python、基础设施等可以在单个工作区中共存和构建。其构建系统使用缓存和依赖图分析，仅重建发生变化的部分并并行运行任务，随着工作区的增长保持测试、代码检查和构建的快速。

`@aws/nx-plugin` 中的每一项功能都作为 Nx [生成器](https://nx.dev/features/generate-code)提供——自动化脚手架，从预定义模板生成一致的代码结构。生成器可以通过 [Nx CLI](https://nx.dev/features/generate-code#invoke-generators)、[Nx Console IDE 插件](https://nx.dev/getting-started/editor-setup)或您的 AI 助手通过 <Link path="/get_started/building-with-ai">Nx Plugin for AWS MCP Server</Link> 调用。

因为每个组件都是一个生成器，您只需在需要时添加所需内容。从 API 开始，稍后添加网站，准备好时连接它们——每一步都只需一个命令。

## 使用经过验证的框架构建

`@aws/nx-plugin` 刻意避免在您已经熟悉的工具之上构建自定义抽象。它不是发明定制框架、运行时或包装器，而是使用已建立的、广泛采用的开源项目来脚手架生成惯用代码。这意味着您现有的技能可以直接迁移，您可以依赖每个框架的官方文档，并且永远不会被锁定在只有此插件理解的层中。

## 开放修改

所有生成的代码都是_您的_代码。`@aws/nx-plugin` 是一个构建时工具，而不是运行时依赖——一旦生成器运行完成，插件就不再参与其中。您可以阅读它生成的每个文件，提交它，并随意修改它。

这意味着：

- **无需逃生舱口。** 当您需要做生成器未预料到的事情时，没有专有的配置界面需要学习或应对。您可以直接编辑生成的代码，就像编辑自己编写的任何代码一样。
- **没有新的抽象层。** 生成器为底层框架生成简洁、惯用的代码。在您和 React、tRPC、FastAPI、CDK 或任何其他工具之间没有包装 API 或魔法间接层——您看到的就是运行的。
- **插件不是依赖项。** 它生成代码后就退出了。您的应用程序在运行时不会导入或依赖 `@aws/nx-plugin`，因此您永远不会被它阻塞，也永远不会被锁定。

如果您不喜欢生成代码中的某些内容，可以自由修改它。生成器为您提供起步优势，而不是牢笼。

## 最小依赖

`@aws/nx-plugin` 致力于将全局依赖的数量保持在最低限度。入门所需的内容取决于您调用的生成器。

例如，任何基于 TypeScript 的生成器只需安装 [Node](https://nodejs.org/en/download)。对于基于 Python 的项目，[UV](https://docs.astral.sh/uv/) 是唯一的要求。

## 类型安全

`@aws/nx-plugin` 采用类型安全来简化开发体验，通过 IDE 自动补全，同时消除在非类型安全实现中才会出现的运行时错误。因此，所有提供的组件默认都是类型安全的。

类型安全跨越项目边界：当您将网站连接到 API 时，生成的客户端与后端共享类型，因此对 API 契约的更改会在前端显示为编译时错误，而不是生产环境中的错误。放心重构。

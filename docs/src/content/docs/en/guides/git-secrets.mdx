New workspaces created with `@aws/nx-plugin` include [git-secrets](https://github.com/awslabs/git-secrets) pre-commit hooks configured by default. This scans staged files for patterns matching AWS credentials before each commit, preventing accidental credential leaks.

## What's Included

When you create a new workspace, the following is set up automatically:

<FileTree>
  - .git-secrets/
    - git-secrets The git-secrets bash script (vendored from awslabs/git-secrets)
    - setup.sh Registers AWS credential patterns in local git config
  - .husky/
    - pre-commit Runs git-secrets scan before each commit
  - package.json Includes husky and prepare script for automatic setup
</FileTree>

## How It Works

1. The `prepare` script runs after `npm install` / `pnpm install`, which initializes husky git hooks and registers AWS credential patterns in your local git config.
2. On each `git commit`, the husky pre-commit hook invokes `git-secrets` to scan staged files.
3. If a pattern matching an AWS credential is detected, the commit is blocked with an error message.

## Detected Patterns

The following patterns are scanned:

- **AWS Access Key IDs** — `AKIA...`, `ASIA...`, and other IAM key prefixes followed by 16 uppercase alphanumeric characters
- **AWS Secret Access Keys** — 40-character base64 strings preceded by a `SECRET_ACCESS_KEY` label
- **AWS Account IDs** — 12-digit numbers preceded by an `ACCOUNT_ID` label
- **Bedrock API Keys** — Long-lived (`ABSK...`) and short-lived Bedrock keys
- **Credentials file values** — Literal key values found in `~/.aws/credentials`

Well-known example keys (e.g., `AKIAIOSFODNN7EXAMPLE`) are automatically allowlisted.

## Allowing False Positives

If git-secrets blocks a commit that contains a false positive:

```sh
# Allow a specific pattern
git config --add secrets.allowed 'my-pattern-here'
```

Or add patterns to a `.gitallowed` file at the repository root (one regex per line).

For a one-time override:

```sh
git commit --no-verify
```

## Opting Out

To create a workspace without git-secrets:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

Or pass `--gitSecrets=false` to the preset generator.

## Requirements

- **Git** must be initialized in the workspace (`git init`)
- **Bash** must be available on PATH (ships with Git for Windows on Windows)
- Patterns are registered when `prepare` runs — if you clone a fresh copy, run your package manager's install command to re-register

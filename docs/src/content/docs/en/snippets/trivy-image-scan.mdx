---
title: Container Image Scanning
---

The Docker image built for this project is scanned for vulnerabilities as part of the build using [Trivy](https://trivy.dev/), running from the [ECR-hosted Trivy image](https://gallery.ecr.aws/aquasecurity/trivy).

A `trivy` target is added to your project which scans the built image and **fails the build** if any `HIGH` or `CRITICAL` severity vulnerability is found.

The scan uses the same container engine as your image build (`docker` or `finch`), so no additional tooling is required. Since the scan is only re-run when the image changes, an unchanged image is not re-scanned.

:::caution[Unfixable vulnerabilities are ignored]
The scan runs with `--ignore-unfixed`, so vulnerabilities without an available fix do not fail the build, since they cannot be actioned by upgrading tooling in your `Dockerfile`. Review these periodically — for example with `trivy image <your-image>` without the flag — and rebuild against a newer base image once a fix is published.
:::

:::caution[Green Base Images]
The generated `Dockerfile` uses a base image with no known fixable `HIGH` or `CRITICAL` vulnerabilities at time of generation, and upgrades bundled tooling (such as `npm`) to keep it that way. New vulnerabilities are disclosed over time, so a build that passes today may fail later as the vulnerability database is updated. When this happens, update to a newer base image where the vulnerability is fixed, or add container steps to upgrade specific tooling and apply available patches.
:::

#### Suppressing Trivy Findings

There may be instances where you want to suppress a specific vulnerability, for example when no fix is yet available and you have assessed the risk as acceptable.

Add the vulnerability ID (one per line) to the `.trivyignore` file in the root of your project (i.e. next to your `project.json`):

```gitignore
# .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

For more details on filtering findings, refer to the [Trivy filtering documentation](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids).

## Task 1: Configure local dev server

To start building the UI, we will need to configure our local dev server to point to our deployed sandbox. To do this, run the following command:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

This command will pull down the `runtime-config.json` that is deployed and store it locally within the `packages/game-ui/public` folder.

To start the dev server, run the following command:

<NxCommands commands={["serve game-ui"]} />

Open your local website in a browser. You will be prompted to log in with the Cognito user you created earlier. Once logged in, you should see the baseline website.

<Aside type="caution" title="Keep Dev Server Running">
We recommend keeping the dev server running throughout the remainder of this module as it will automatically hot-reload any changes we make.
</Aside>

## Task 4: Create the game pages

In <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, we connected our `GameUI` to the Story Agent. Because the agent was generated with `--protocol=AG-UI`, the connection generator wired up a CopilotKit `AguiProvider` and a per-agent hook (`useAguiStoryAgent`) instead of the usual OpenAPI client. We can render the chat with a single `<CopilotChat />` component and drive it by `threadId` (which we use as the AgentCore session id).

Create two new files under `packages/game-ui/src/routes/game/`:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="Path Parameters">
The `$playerName` syntax instructs `@tanstack/react-router` to treat `playerName` as a [path param](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). We use the player name as the AG-UI `threadId` so each game has its own chat history, and pad it to 33 characters to satisfy AgentCore's session id length requirement.
</Aside>

<Aside type="note" title="Seeding the first message">
On first mount we check `agent.messages.length === 0` and, if empty, push a single opening user message: `I am <player>, playing in the <genre> genre.` The system prompt tells the agent to begin the story immediately from that cue. From then on, every interaction (including page refreshes and returning later) just reloads the conversation via the AG-UI session — no custom history endpoint required.
</Aside>
</TabItem>
</Tabs>

Once you save these files, your local dev server (http://localhost:4200/) should now have your game ready to play.

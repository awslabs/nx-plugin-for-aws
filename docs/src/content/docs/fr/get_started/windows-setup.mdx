Plusieurs générateurs du plugin Nx pour AWS produisent des cibles de build basées sur Docker (par exemple, l'infrastructure CDK avec des images de conteneur Lambda, la compilation de modèles d'API Smithy et les déploiements de serveurs MCP / agents Strands). Ces cibles construisent des images Docker **Linux**, ce qui nécessite un daemon Docker Linux.

Sur macOS et Linux, cela fonctionne directement. Sur Windows, Docker utilise par défaut le **mode conteneur Windows**, donc une étape de configuration supplémentaire est nécessaire pour fournir un environnement de build Linux. Il existe deux options :

## Option A : Docker Desktop (Recommandé)

[Docker Desktop](https://www.docker.com/products/docker-desktop/) inclut une VM Linux gérée et prend en charge de manière transparente les builds de conteneurs Linux. C'est la solution la plus simple pour le développement local.

<Steps>
1. **Installez Docker Desktop** depuis [docker.com](https://www.docker.com/products/docker-desktop/).

2. **Activez le backend WSL 2** (par défaut sur les installations modernes) :
   - Ouvrez Docker Desktop **Settings > General**
   - Assurez-vous que **Use the WSL 2 based engine** est coché

3. **Vérifiez** en ouvrant un terminal et en exécutant :
   ```bash
   docker version
   docker buildx ls
   ```
   Vous devriez voir `OS/Arch: linux/amd64` dans la section serveur.
</Steps>

C'est tout — Docker Desktop gère la VM Linux, Buildx et le support multi-plateforme pour vous.

## Option B : Docker Engine dans WSL 2 (Sans Docker Desktop)

Si vous préférez ne pas installer Docker Desktop (par exemple, pour des raisons de licence), vous pouvez installer Docker Engine directement dans une distribution Linux WSL 2 et utiliser un petit script wrapper pour que les outils côté Windows (comme Nx) puissent invoquer `docker` de manière transparente.

### Prérequis

- **Windows 10 version 2004+** ou **Windows 11**
- **WSL 2** activé (`wsl --install` si ce n'est pas déjà fait)

### Installer Ubuntu + Docker Engine

<Steps>
1. **Installez une distribution Ubuntu dans WSL 2 :**

   ```powershell
   wsl --install Ubuntu-24.04 --no-launch
   wsl --set-default Ubuntu-24.04
   ```

2. **Installez Docker Engine dans WSL 2 :**

   Ouvrez le shell Ubuntu (`wsl -d Ubuntu-24.04`) et exécutez :

   ```bash
   # Add Docker's official GPG key and repository
   sudo apt-get update -qq
   sudo apt-get install -y ca-certificates curl
   sudo install -m 0755 -d /etc/apt/keyrings
   sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
     -o /etc/apt/keyrings/docker.asc
   sudo chmod a+r /etc/apt/keyrings/docker.asc
   echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc] \
     https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
     | sudo tee /etc/apt/sources.list.d/docker.list

   # Install Docker Engine + Buildx
   sudo apt-get update -qq
   sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
     docker-buildx-plugin
   ```

3. **Démarrez Docker et créez un builder Buildx :**

   ```bash
   sudo dockerd &>/var/log/dockerd.log &

   # Wait for the daemon to be ready
   until docker info &>/dev/null; do sleep 1; done

   # Set up a Buildx builder for multi-platform builds
   docker buildx create --name linux-builder --driver docker-container --use
   docker buildx inspect --bootstrap --builder linux-builder
   ```

4. **Créez un wrapper `docker.cmd`** pour que les outils côté Windows utilisent le daemon WSL 2 :

   Enregistrez ce qui suit sous `C:\docker-wsl\docker.cmd` (ou tout répertoire que vous ajoutez à votre `PATH`) :

   ```bat
   @echo off
   for /f "usebackq tokens=*" %%i in (`wsl -d Ubuntu-24.04 wslpath "%CD%"`) do set WSL_CWD=%%i
   if "%1"=="build" (
     echo %* | findstr /i /c:"--output" >nul
     if errorlevel 1 (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %* --load
     ) else (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %*
     )
   ) else (
     wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- docker %*
   )
   ```

   Ensuite, ajoutez `C:\docker-wsl` au **début** de votre `PATH` système pour qu'il ait la priorité sur tout `docker.exe` existant.

5. **Vérifiez :**

   ```powershell
   docker version
   docker buildx ls
   ```
</Steps>

:::note
Avec l'option B, vous devez démarrer le daemon Docker dans WSL 2 à chaque redémarrage. Vous pouvez automatiser cela en ajoutant `sudo dockerd &>/dev/null &` à votre profil de shell WSL, ou en créant une entrée dans le Planificateur de tâches Windows.
:::

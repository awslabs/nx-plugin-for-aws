Installez le [plugin Claude](https://code.claude.com/docs/en/plugins), le [Kiro Power](https://kiro.dev/docs/powers/), ou le [serveur MCP](https://modelcontextprotocol.io/introduction) pour permettre aux assistants IA de travailler efficacement avec le plugin. En les utilisant, vous pouvez accélérer votre flux de développement avec l'IA tout en bénéficiant d'une génération cohérente de vos projets, en passant moins de temps à configurer les composants principaux et en profitant de la sécurité, de l'observabilité, de la sûreté des types et du développement local intégrés au plugin.

## Configurez votre assistant IA

:::tip[La configuration au niveau du projet est automatique]
Les espaces de travail créés avec le preset Nx Plugin pour AWS incluent déjà une configuration MCP au niveau du projet pour les agents de codage courants (Claude Code, Cursor, Kiro, Gemini CLI, GitHub Copilot et OpenAI Codex). Tout agent travaillant à l'intérieur de l'espace de travail peut utiliser le serveur MCP sans configuration supplémentaire. Les étapes ci-dessous configurent le serveur MCP globalement pour une utilisation dans tous vos projets.
:::

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    Le Plugin Nx pour AWS est livré sous forme de [Kiro Power](https://kiro.dev/docs/powers/) qui regroupe le serveur MCP avec la documentation de pilotage et les guides de flux de travail.

    <Steps>

    1. Ouvrez le panneau Powers dans Kiro et cliquez sur **Add power from GitHub**
    2. Entrez l'URL suivante :

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. Cliquez sur **Install**

    </Steps>

    Kiro configure automatiquement le serveur MCP — aucune configuration supplémentaire n'est requise.
  </TabItem>
  <TabItem label="Kiro CLI">
    Exécutez la commande suivante :

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    Consultez la [documentation MCP de Kiro CLI](https://kiro.dev/docs/cli/mcp/) pour plus de détails.
  </TabItem>
  <TabItem label="Claude Code">
    Le Plugin Nx pour AWS est distribué sous forme de [plugin Claude Code](https://code.claude.com/docs/en/plugins) qui regroupe une [Skill](https://code.claude.com/docs/en/skills) avec le serveur MCP.

    Exécutez les commandes suivantes dans Claude Code, une à la fois :

    ```
    /plugin marketplace add awslabs/nx-plugin-for-aws
    ```

    ```
    /plugin install nx-plugin-for-aws@nx-plugin-for-aws
    ```

    Ensuite, exécutez `/reload-plugins` pour activer le plugin.

    :::tip
    Pour essayer la version candidate 1.0, installez plutôt `nx-plugin-for-aws-next@nx-plugin-for-aws`.
    :::
  </TabItem>

## Commencez le Vibe Coding

Demandez à votre assistant IA de construire quelque chose en utilisant le `nx-plugin-for-aws` ! Il créera généralement un espace de travail, générera les squelettes avec les générateurs appropriés, puis remplira la logique métier.

Essayez un prompt comme :

> *"Construis une application multi-agents en utilisant le Plugin Nx pour AWS. Je veux un frontend React qui communique avec un agent orchestrateur backend qui délègue à des agents spécialisés de recherche et d'écriture."*

Ou pour un point de départ plus simple :

> *"Utilise le Plugin Nx pour AWS pour générer un site web React soutenu par une API tRPC, avec l'authentification Cognito et l'infrastructure CDK."*

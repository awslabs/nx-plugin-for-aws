## Tâche 1 : Configurer le serveur de développement local

Pour commencer à développer l'interface utilisateur, nous devons configurer notre serveur de développement local pour pointer vers notre sandbox déployée. Exécutez la commande suivante :

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Cette commande récupérera le fichier `runtime-config.json` déployé et le stockera localement dans le dossier `packages/game-ui/public`.

Pour démarrer le serveur de développement, exécutez la commande suivante :

<NxCommands commands={["serve game-ui"]} />

Ouvrez le site local dans un navigateur. Vous serez invité à vous connecter avec l'utilisateur Cognito que vous avez créé précédemment. Une fois connecté, vous devriez voir le site de base.

<Aside type="caution" title="Garder le serveur de dev actif">
Nous recommandons de garder le serveur de développement en fonctionnement pendant le reste de ce module, car il rechargera automatiquement toutes les modifications apportées.
</Aside>

## Tâche 4 : Créer les pages du jeu

Dans le <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, nous avons connecté notre `GameUI` à l'Agent Narratif. Comme l'agent a été généré avec `--protocol=AG-UI`, le générateur `connection` a configuré les éléments suivants pour nous :

- Un `<AguiProvider>` qui enveloppe l'application (un `CopilotKitProvider` [CopilotKit](https://copilotkit.ai/)), déjà injecté dans `main.tsx`.
- Un hook `useAguiStoryAgent` qui construit un `HttpAgent` AG-UI authentifié par Cognito pointant vers le runtime AgentCore déployé.
- Un ensemble de primitives de chat thématisées shadcn sous `src/components/copilot` - un `<CopilotChat>` pré-stylisé qui correspond au reste du site web.

Nous devons simplement créer deux fichiers de route qui les utilisent :

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="Paramètres de chemin">
La syntaxe `$playerName` indique à `@tanstack/react-router` de traiter `playerName` comme un [paramètre de chemin](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). Nous utilisons le nom du joueur comme `threadId` AG-UI afin que chaque partie ait son propre historique de chat, et nous le complétons à 33 caractères pour satisfaire l'exigence de longueur d'id de session d'AgentCore.
</Aside>

<Aside type="note" title="Initialisation du premier message">
Au premier montage, nous vérifions `agent.messages.length === 0` et, si vide, nous envoyons un seul message utilisateur d'ouverture : `I am <player>, playing in the <genre> genre.` Le prompt système indique à l'agent de commencer l'histoire immédiatement à partir de cette indication. Par la suite, chaque interaction (y compris les rafraîchissements de page et les retours ultérieurs) recharge simplement la conversation via la session AG-UI — aucun endpoint d'historique personnalisé requis.
</Aside>
</TabItem>
</Tabs>

Une fois ces fichiers enregistrés, votre serveur de développement local (http://localhost:4200/) devrait maintenant afficher votre jeu prêt à être joué.

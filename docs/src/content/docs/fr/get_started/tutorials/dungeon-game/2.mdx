## Tâche 1 : Implémenter l'API de jeu

Nous allons implémenter les API suivantes dans cette section :

1. `saveGame` - créer ou mettre à jour une partie.
2. `queryGames` - retourner une liste paginée des parties précédemment sauvegardées.
3. `saveAction` - sauvegarder une action pour une partie donnée.
4. `queryActions` - retourner une liste paginée de toutes les actions liées à une partie.
5. `queryInventory` - retourner une liste paginée des objets dans l'inventaire d'un joueur.

:::tip[Configuration d'exécution]
Le nom de la table DynamoDB est enregistré dans la <Link path="guides/runtime-config">Configuration d'exécution</Link> sous l'espace de noms `tables`, stocké dans AWS AppConfig. Les fonctions Lambda et le serveur MCP le récupèrent au moment de l'exécution à l'aide de Lambda Powertools, plutôt que de le recevoir comme variable d'environnement. Un avantage de cette approche est qu'à mesure que le nombre de tables ou d'autres identifiants d'infrastructure augmente, vous n'avez besoin que d'une seule variable d'environnement `RUNTIME_CONFIG_APP_ID` lors de l'exécution en local.

### Déployer votre application
Pour déployer votre application, exécutez la commande suivante :

<NxCommands commands={['deploy infra "dungeon-adventure-infra-sandbox/*"']} />

:::caution
Pour bootstrap votre compte AWS d'abord, exécutez :

<NxCommands commands={['bootstrap infra']} />
:::

Le premier déploiement prendra environ 8 minutes. Les déploiements suivants prendront environ 2 minutes.

:::tip
Si vous itérez sur les modifications de code Lambda, vous pouvez déployer avec le flag `--hotswap` après compilation du codebase pour un temps de déploiement beaucoup plus court (2-3 secondes).

<NxCommands commands={['run @dungeon-adventure/infra:deploy "dungeon-adventure-infra-sandbox/*" --hotswap']} />
:::

Une fois le déploiement terminé, vous verrez des sorties similaires à ceci _(certaines valeurs ont été masquées)_ :

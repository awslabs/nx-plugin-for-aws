## Tâche 1 : Implémenter le Story Agent

Le Story Agent est un agent [Strands](https://strandsagents.com/) exposé via le protocole [AG-UI](https://ag-ui.com/). AgentCore route chaque `runtimeSessionId` vers une microVM dédiée, de sorte qu'un seul `Agent` persistant par conteneur sert chaque tour pour cette session. Lorsque le stockage de session géré par AgentCore est attaché, le `agent.py` produit par le générateur persiste l'historique de conversation dans le `SESSION_STORAGE_PATH` monté via `FileSessionManager`, de sorte qu'une aventure peut survivre à un arrêt/reprise.

Nous allons mettre à jour le prompt système de l'agent pour jouer une aventure de donjon, et supprimer l'outil d'exemple.

### Implémentation de l'agent

Mettez à jour `packages/story/dungeon_adventure_story/agent/agent.py` :

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

Le prompt système indique à l'agent que le premier message sera de la forme `I am <player>, playing in the <genre> genre.` — notre interface utilisateur envoie exactement ceci comme message utilisateur d'ouverture afin que l'agent ait tout ce dont il a besoin pour démarrer l'histoire. Comme la bibliothèque AG-UI conserve automatiquement l'historique de conversation de chaque thread, nous n'avons pas besoin de transmettre le nom du joueur ou le genre lors des tours suivants.

Le générateur de connexion a déjà configuré le client Inventory MCP dans le <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, donc l'agent a accès aux outils d'inventaire.

:::note[Où est `main.py` ?]
Le wrapper FastAPI du protocole AG-UI gère la traduction des requêtes/réponses pour nous. Le `main.py` que le générateur a produit crée déjà le wrapper autour de `get_agent(...)` — nous n'avons pas besoin d'y toucher.
:::

### Créer un utilisateur test Cognito

Avant de tester l'agent déployé (et avant de jouer au jeu via l'interface utilisateur dans le prochain module), créez un utilisateur test Cognito. Nous avons désactivé l'auto-inscription plus tôt, donc cette étape est requise que vous testiez localement ou contre le runtime déployé :

```bash
# Depuis les sorties CDK
export POOL_ID="<UserPoolId depuis les sorties CDK>"
export CLIENT_ID="<UserPoolClientId depuis les sorties CDK>"
export REGION="<votre-région>"

# Désactiver MFA pour simplifier la création d'utilisateur
aws cognito-idp set-user-pool-mfa-config \
  --mfa-configuration OFF \
  --user-pool-id $POOL_ID \
  --region $REGION

# Créer un utilisateur test
aws cognito-idp admin-create-user \
  --user-pool-id $POOL_ID \
  --username "test" \
  --temporary-password "TempPass123-" \
  --region $REGION \
  --message-action SUPPRESS > /dev/null

# Définir un mot de passe permanent (remplacez par quelque chose de plus sécurisé !)
aws cognito-idp admin-set-user-password \
  --user-pool-id $POOL_ID \
  --username "test" \
  --password "PermanentPass123-" \
  --region $REGION \
  --permanent > /dev/null
```

### Tester votre Agent

Vous pouvez tester votre Agent via :
<ul>
<li>Le démarrage d'une instance locale du serveur Agent et discuter avec lui de manière interactive en utilisant la cible `agent-chat`, ou</li>
<li>L'appel de l'API déployée en utilisant curl avec un token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  La cible `agent-chat` démarre l'agent localement (avec ses dépendances, via `agent-serve-local`) et vous place dans une invite de discussion interactive. Exécutez :

    <NxCommands commands={["agent-chat story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    Si vous souhaitez discuter avec l'agent en utilisant votre serveur MCP **déployé** au lieu d'un serveur local, démarrez le serveur directement avec `agent-serve` et passez `RUNTIME_CONFIG_APP_ID` / `AWS_REGION` :

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve story"]} />

    puis dans un autre shell exécutez `agent-chat-cli agui http://localhost:8081/invocations`.
    :::

    Lorsque vous y êtes invité, essayez de démarrer le jeu avec quelque chose comme :

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="Deployed">
    Échangez les identifiants de l'utilisateur test contre un token d'accès Cognito :

    ```bash
    # Authentifier l'utilisateur test et capturer son token d'accès
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoquez l'agent déployé en utilisant l'URL du runtime Bedrock AgentCore. Le protocole AG-UI attend une charge utile `RunAgentInput` — une charge minimale qui envoie le message utilisateur d'ouverture ressemble à ceci :

    ```bash
    # Définir l'ARN du Story Agent depuis les sorties CDK
    export AGENT_ARN="<StoryAgentArn depuis les sorties CDK>"

    # Encoder l'ARN en URL
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construire l'URL d'invocation
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoquer l'agent
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[Encodage URL]
    L'encodage URL est nécessaire pour formater correctement l'ARN pour le point de terminaison de l'API Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Si la commande s'exécute avec succès, vous devriez commencer à voir des événements Server-Sent Events AG-UI en flux continu — les événements `TEXT_MESSAGE_CONTENT` transportent le texte de l'histoire morceau par morceau.

Félicitations. Vous avez construit et déployé votre premier agent Strands sur Bedrock AgentCore Runtime ! 🎉🎉🎉

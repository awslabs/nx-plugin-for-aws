## Module 3 : Implémentation de l'agent narratif

<Aside type="caution">
Assurez-vous que votre compte AWS a activé l'accès au [modèle par défaut Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) dans Bedrock pour votre région cible en suivant les étapes décrites dans [ce guide](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access-modify.html). Au moment de la rédaction, il s'agit d'Anthropic Claude Sonnet 4.
</Aside>

L'agent narratif est un agent [Strands](https://strandsagents.com/) qui, étant donné un `Game` et une liste d'`Action`s comme contexte, fait progresser une histoire. Nous configurerons l'agent pour interagir avec notre serveur MCP d'inventaire afin de gérer les objets disponibles du joueur.

:::note
Pour simplifier ce tutoriel, nous implémentons l'agent de manière "sans état" où chaque invocation est une session indépendante incluant l'historique complet des messages. Pour la production, envisagez d'intégrer [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html). Plus de documentation sur l'utilisation de la mémoire AgentCore avec Strands est [disponible ici](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

  <TabItem label="Déployé">
    Pour tester l'agent déployé, authentifiez-vous avec Cognito et obtenez un token JWT. Configurez d'abord vos variables d'environnement :

    ```bash
    # Définissez votre ID de pool utilisateur Cognito et ID client depuis les sorties CDK
    export POOL_ID="<UserPoolId from CDK outputs>"
    export CLIENT_ID="<UserPoolClientId from CDK outputs>"
    export REGION="<your-region>"
    ```

    Créez un utilisateur de test et obtenez un token :

    ```bash
    # Création utilisateur
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "testuser" \
      --temporary-password "TempPass123!" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Définition mot de passe permanent
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "testuser" \
      --password "PermanentPass123!" \
      --region $REGION \
      --permanent > /dev/null

    # Authentification et récupération token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='testuser',PASSWORD='PermanentPass123!' \
      --region $REGION | jq -r '.AuthenticationResult.IdToken')
    ```

    :::caution
    Utilisez la sortie `UserIdentityUserIdentityUserPoolIdXXX` pour `POOL_ID` et vérifiez l'ID client correspondant dans vos sorties CDK.
    :::

    Invoquez maintenant l'agent déployé via l'URL Bedrock AgentCore :

    ```bash
    # Définissez l'ARN de l'agent narratif depuis les sorties CDK
    export AGENT_ARN="<StoryAgentArn from CDK outputs>"

    # Encodage URL de l'ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construction URL d'invocation
    export MCP_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invocation de l'agent
    curl -N -X POST "$MCP_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}'
    ```

    :::note
    L'encodage URL est nécessaire pour formater correctement l'ARN dans l'endpoint API Bedrock AgentCore.
    :::
  </TabItem>

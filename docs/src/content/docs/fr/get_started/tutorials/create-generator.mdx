## Créer un générateur

Créons un nouveau générateur. Notre objectif sera de générer une nouvelle procédure pour notre API tRPC.

### Vérifier le plugin

Commencez par cloner le plugin :

```bash
git clone git@github.com:awslabs/nx-plugin-for-aws.git
```

Ensuite, installez et construisez :

```bash
cd nx-plugin-for-aws
pnpm i
pnpm nx run-many --target build --all
```

:::note
Notre générateur reçoit un `Tree` en entrée ainsi que les options définies dans notre schéma. Le `Tree` est essentiellement un système de fichiers virtuel permettant de lire et écrire pour créer ou mettre à jour des fichiers projet. Nous évitons de modifier directement le système de fichiers pour ne pas effectuer de changements lors d'un exécution en mode "dry-run".
:::

Connectons maintenant notre générateur en mettant à jour `packages/nx-plugin/generators.json` :

```json
 ...
  "generators": {
    ...
    "ts#trpc-api#procedure": {
      "factory": "./src/trpc/procedure/generator",
      "schema": "./src/trpc/procedure/schema.json",
      "description": "Ajoute une procédure à une API tRPC"
    }
  },
...
```

### Implémenter le générateur

Pour ajouter une procédure à une API tRPC, nous devons :

1. Créer un fichier TypeScript pour la nouvelle procédure
2. Ajouter la procédure au routeur

#### Créer la nouvelle procédure

Pour créer le fichier TypeScript, nous utilisons `generateFiles` avec un template [EJS](https://ejs.co/). Définissons notre template dans `packages/nx-plugin/src/trpc/procedure/files/procedures/__procedureNameKebabCase__.ts.template` :

```ts title="files/procedures/__procedureNameKebabCase__.ts.template"
import { publicProcedure } from '../init.js';
import { z } from 'zod';

export const <%- procedureNameCamelCase %> = publicProcedure
  .input(z.object({
    // TODO: définir l'entrée
  }))
  .output(z.object({
    // TODO: définir la sortie
  }))
  .<%- procedureType %>(async ({ input, ctx }) => {
    // TODO: implémenter !
    return {};
  });
```

:::tip
`generateFiles` remplace les références `__<variable>__` dans les noms de fichiers/répertoires et supprime l'extension `.template`. Le contenu utilise la syntaxe [EJS](https://ejs.co/).
:::

Mettons à jour notre générateur pour passer les variables nécessaires :

```ts title="procedure/generator.ts" {8-19}
import { generateFiles, joinPathFragments, readProjectConfiguration, Tree } from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';

export const trpcProcedureGenerator = async (tree: Tree, options: TrpcProcedureSchema) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(tree, joinPathFragments(__dirname, 'files'), projectConfig.sourceRoot, {
    procedureNameCamelCase,
    procedureNameKebabCase,
    procedureType: options.type,
  });

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
`formatFilesInSubtree` formate les fichiers générés selon les paramètres [prettier](https://prettier.io/) de l'utilisateur.
:::

#### Ajouter la procédure au routeur

Mettons à jour le routeur en manipulant l'AST TypeScript :

```ts title="procedure/generator.ts" {6, 23-33}
import { generateFiles, joinPathFragments, readProjectConfiguration, Tree } from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';
import { destructuredImport, replace } from '../../utils/ast';
import { factory, ObjectLiteralExpression } from 'typescript';

export const trpcProcedureGenerator = async (tree: Tree, options: TrpcProcedureSchema) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(tree, joinPathFragments(__dirname, 'files'), projectConfig.sourceRoot, {
    procedureNameCamelCase,
    procedureNameKebabCase,
    procedureType: options.type,
  });

  const routerPath = joinPathFragments(projectConfig.sourceRoot, 'router.ts');

  destructuredImport(tree, routerPath, [procedureNameCamelCase], `./procedures/${procedureNameKebabCase}.js`);

  replace(
    tree,
    routerPath,
    'CallExpression[expression.name="router"] > ObjectLiteralExpression',
    (node) => factory.createObjectLiteralExpression([
      ...(node as ObjectLiteralExpression).properties,
      factory.createShorthandPropertyAssignment(procedureNameCamelCase),
    ]),
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
La fonction `replace` utilise un sélecteur [tsquery](https://github.com/phenomnomnominal/tsquery) pour trouver l'argument de la fonction `router`.

Utilisez le [tsquery playground](https://tsquery-playground.firebaseapp.com/) pour tester des sélecteurs.
:::

Compilons le générateur :

```bash
pnpm nx run @aws/nx-plugin:compile
```

### Exercices

Suggestions d'améliorations pour le générateur :

#### 1. Opérations imbriquées

Prendre en charge les routeurs imbriqués en :

* Acceptant la notation pointée pour `procedure` (ex: `games.query`)
* Générant des noms inversés (ex: `queryGames`)
* Ajoutant/mettant à jour les routeurs imbriqués

#### 2. Validation

Vérifier que le `project` sélectionné est bien une API tRPC (voir l'exemple `api-connection`).

#### 3. Tests unitaires

Implémenter des tests qui :

1. Créent un espace de travail vide avec `createTreeUsingTsSolutionSetup()`
2. Ajoutent les fichiers nécessaires
3. Exécutent le générateur
4. Vérifient les modifications

#### 4. Tests end-to-end

Mettre à jour les tests existants pour inclure le nouveau générateur.

#### 5. Contribuer !

Proposez une pull request si ce générateur n'existe pas encore dans `@aws/nx-plugin`, ou contribuez un autre générateur.

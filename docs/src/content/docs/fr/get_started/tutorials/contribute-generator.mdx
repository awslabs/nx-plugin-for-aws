Créons un nouveau générateur pour contribuer à `@aws/nx-plugin`. Notre objectif sera de générer une nouvelle procédure pour notre API tRPC.

### Découvrez le Plugin

Commencez par cloner le plugin :

```bash
git clone git@github.com:awslabs/nx-plugin-for-aws.git
```

Ensuite, installez et compilez :

```bash
cd nx-plugin-for-aws
pnpm i
pnpm nx run-many --target build --all
```

### Créez un Générateur Vide

Créons le nouveau générateur dans `packages/nx-plugin/src/trpc/procedure`.

Nous fournissons un générateur de générateur pour échafauder rapidement votre nouveau générateur ! Exécutez-le ainsi :

:::note
Notre générateur reçoit un `Tree` en entrée, ainsi que les options du schéma. Le `Tree` est un système de fichiers virtuel permettant de lire/écrire des fichiers projet. Nous évitons de modifier directement le système de fichiers pour préserver le mode "dry-run".
:::

```json
    }
  },
...
```

### Implémentez le Générateur

Pour ajouter une procédure tRPC, nous devons :

1. Créer un fichier TypeScript pour la procédure
2. Ajouter la procédure au routeur

#### Créez la Nouvelle Procédure

Utilisons `generateFiles` avec un template EJS dans `packages/nx-plugin/src/trpc/procedure/files/procedures/__procedureNameKebabCase__.ts.template` :

```ts title="files/procedures/__procedureNameKebabCase__.ts.template"
import { publicProcedure } from '../init.js';
import { z } from 'zod';

export const <%- procedureNameCamelCase %> = publicProcedure
  .input(z.object({
    // TODO: define input
  }))
  .output(z.object({
    // TODO: define output
  }))
  .<%- procedureType %>(async ({ input, ctx }) => {
    // TODO: implement!
    return {};
  });
```

:::tip
`generateFiles` remplace les variables `__<variable>__` dans les noms de fichiers et utilise le contenu EJS `<% ... %>`. Le suffixe `.template` est supprimé.
:::

Mettons à jour le générateur pour injecter les variables :

```ts title="procedure/generator.ts" {8-19}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
`formatFilesInSubtree` formate les fichiers générés selon les règles Prettier de l'utilisateur.
:::

#### Ajoutez la Procédure au Routeur

Manipulons l'AST TypeScript pour mettre à jour le routeur :

```ts title="procedure/generator.ts" {6, 23-33}
import {
  generateFiles,
  joinPathFragments,
  readProjectConfiguration,
  Tree,
} from '@nx/devkit';
import { TrpcProcedureSchema } from './schema';
import { formatFilesInSubtree } from '../../utils/format';
import camelCase from 'lodash.camelcase';
import kebabCase from 'lodash.kebabcase';
import { destructuredImport, replace } from '../../utils/ast';
import { factory, ObjectLiteralExpression } from 'typescript';

export const trpcProcedureGenerator = async (
  tree: Tree,
  options: TrpcProcedureSchema,
) => {
  const projectConfig = readProjectConfiguration(tree, options.project);

  const procedureNameCamelCase = camelCase(options.procedure);
  const procedureNameKebabCase = kebabCase(options.procedure);

  generateFiles(
    tree,
    joinPathFragments(__dirname, 'files'),
    projectConfig.sourceRoot,
    {
      procedureNameCamelCase,
      procedureNameKebabCase,
      procedureType: options.type,
    },
  );

  const routerPath = joinPathFragments(projectConfig.sourceRoot, 'router.ts');

  destructuredImport(
    tree,
    routerPath,
    [procedureNameCamelCase],
    `./procedures/${procedureNameKebabCase}.js`,
  );

  replace(
    tree,
    routerPath,
    'CallExpression[expression.name="router"] > ObjectLiteralExpression',
    (node) =>
      factory.createObjectLiteralExpression([
        ...(node as ObjectLiteralExpression).properties,
        factory.createShorthandPropertyAssignment(procedureNameCamelCase),
      ]),
  );

  await formatFilesInSubtree(tree);
};

export default trpcProcedureGenerator;
```

:::tip
La fonction `replace` utilise un sélecteur [tsquery](https://github.com/phenomnomnominal/tsquery) pour trouver l'argument du routeur. Testez vos sélecteurs dans le [tsquery playground](https://tsquery-playground.firebaseapp.com/).
:::

Compilons le générateur :

```bash
pnpm nx run @aws/nx-plugin:compile
```

### Exercices

Pour approfondir votre maîtrise des générateurs Nx, voici quelques suggestions :

#### 1. Opérations Imbriquées

Dans dungeon adventure, nous utilisons des routeurs imbriqués. Améliorez le générateur pour :

- Accepter une notation pointée pour `procedure` (ex: `games.query`)
- Générer des noms inversés (ex: `queryGames`)
- Créer/mettre à jour les routeurs imbriqués

#### 2. Validation

Empêchez l'utilisation sur des projets non tRPC. Inspirez-vous du générateur `api-connection`.

#### 3. Tests Unitaires

Implémentez des tests unitaires avec ce flux :

1. Créez un workspace vide avec `createTreeUsingTsSolutionSetup()`
2. Ajoutez les fichiers existants nécessaires
3. Exécutez le générateur
4. Vérifiez les modifications dans le `Tree`

#### 4. Tests End-to-End

Mettez à jour le "smoke test" existant pour inclure le nouveau générateur.

#### 5. Contribuez !

Si ce générateur n'existe pas encore dans `@aws/nx-plugin`, proposez une pull request ! Sinon, imaginez un autre générateur utile.

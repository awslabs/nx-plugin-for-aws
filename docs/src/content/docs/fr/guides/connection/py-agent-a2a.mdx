Le générateur `connection` peut connecter votre <Link path="guides/py-agent">Agent Python</Link> à un agent [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) distant — soit <Link path="guides/ts-agent">TypeScript</Link> soit <Link path="guides/py-agent">Python</Link> — afin que votre agent puisse déléguer à un autre agent en tant qu'outil.

Le générateur configure tout le câblage nécessaire pour que votre agent puisse découvrir et invoquer l'agent A2A distant, à la fois lorsqu'il est déployé sur AWS (via Bedrock AgentCore) et lorsqu'il s'exécute localement.

## Prérequis

Avant d'utiliser ce générateur, assurez-vous d'avoir :

1. Un projet Python avec un composant <Link path="guides/py-agent">Agent Python</Link> (Strands ou LangChain)
2. Un projet avec un composant Agent généré avec `--protocol=A2A` et `--auth=IAM` (soit <Link path="guides/ts-agent">`ts#agent`</Link> soit <Link path="guides/py-agent">`py#agent`</Link>)
3. Les deux composants créés avec `infra: agentcore`

## Sortie du générateur

Le générateur crée un projet Python partagé `agent_connection` dans `packages/common/agent_connection/` (s'il n'existe pas déjà). Les modules clients par connexion sont générés dans ce projet partagé :

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py Ré-exporte les clients par connexion
    - core
      - agentcore\_endpoints.py Résolution ARN/URL indépendante du framework
      - agentcore\_a2a\_client\_config.py Configuration client A2A indépendante du framework (`ClientConfig` signé)
      - agentcore\_a2a\_client\_strands.py Client de transport A2A encapsulant la configuration
      - auth/ `httpx.Auth` SigV4 / transfert de session indépendant du framework
    - app
      - \<target\_agent\_name>\_client\_strands.py Client de transport A2A par connexion pour chaque agent A2A

</FileTree>

Le transport A2A (un `A2AAgent` Strands) ne porte aucun modèle, donc les agents Strands et LangChain le réutilisent — seul le câblage des outils dans `agent.py` diffère selon le framework.

De plus, le générateur :
- Transforme le fichier `agent.py` de votre agent pour enregistrer l'agent A2A distant comme outil en utilisant `@tool`
- Ajoute le projet `agent_connection` comme dépendance d'espace de travail de votre projet d'agent
- Met à jour la cible `serve-local` de l'agent pour dépendre de la cible `serve-local` de l'agent cible

## Utilisation de l'agent A2A connecté

Le générateur transforme le fichier `agent.py` de votre agent pour encapsuler l'agent A2A distant en tant qu'outil. L'agent distant est enregistré avec un délégué décoré par `@tool` — l'import du décorateur et le constructeur de l'agent diffèrent selon le framework :

<Tabs syncKey="framework">
<TabItem label="Strands">

```python title="packages/my-project/my_module/agent/agent.py" {4,9-15,21}
from contextlib import contextmanager
from strands import Agent, tool

from my_scope_agent_connection import RemoteAgentClientStrands

@contextmanager
def get_agent(session_id: str):
    remote_agent = RemoteAgentClientStrands.create(session_id=session_id)

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    yield Agent(
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
<TabItem label="LangChain">
```python title="packages/my-project/my_module/agent/agent.py" {4,8-13,17}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse
from langchain_core.tools import tool

from my_scope_agent_connection import RemoteAgentClientStrands

def get_agent():
    remote_agent = RemoteAgentClientStrands.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
</Tabs>

En interne, `RemoteAgentClientStrands.create()` retourne un `A2AAgent` Strands (utilisé purement comme transport A2A — il ne porte aucun modèle, donc les agents LangChain le réutilisent) configuré avec un `httpx.AsyncClient` qui signe les requêtes avec SigV4 lorsqu'il est déployé sur AWS, et un point de terminaison `http://localhost:<port>/` simple lorsque `SERVE_LOCAL=true`. Pour les agents Strands, le `session_id` est transmis depuis l'appelant, garantissant la cohérence pour [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html).

## Développement local

Le générateur configure la cible `serve-local` de l'agent hôte pour :
1. Démarrer automatiquement le(s) agent(s) A2A connecté(s)
2. Définir `SERVE_LOCAL=true` afin que le client généré se connecte directement à `http://localhost:<port>/` au lieu d'AgentCore

Exécutez l'agent localement avec :

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Cela démarrera à la fois l'agent hôte et tous les agents A2A connectés, l'agent hôte appelant les agents distants via HTTP simple sur leurs ports locaux assignés.

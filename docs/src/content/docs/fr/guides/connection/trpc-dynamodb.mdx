Le générateur `connection` relie une <Link path="guides/trpc">API tRPC</Link> à un projet <Link path="guides/ts-dynamodb">DynamoDB</Link>, en configurant le développement local pour que les deux démarrent automatiquement ensemble.

## Prérequis

Avant d'utiliser ce générateur, assurez-vous d'avoir :

1. Un projet <Link path="guides/trpc">`ts#trpc-api`</Link>
2. Un projet <Link path="guides/ts-dynamodb">`ts#dynamodb`</Link>

## Utilisation

### Exécuter le générateur

<RunGenerator generator="connection" />

Sélectionnez votre projet d'API tRPC comme source et votre projet DynamoDB comme cible.

### Options

<GeneratorParameters generator="connection" />

## Sortie du générateur

Le générateur met à jour le fichier `project.json` de votre API tRPC pour ajouter une dépendance de sa cible `serve-local` vers la cible `serve-local` du projet DynamoDB. Aucun fichier source n'est modifié.

## Utilisation de DynamoDB dans les procédures

Importez les factories d'entités depuis le package DynamoDB et utilisez-les dans vos procédures tRPC :

```ts title="packages/api/src/procedures/example.ts"
import { createExampleEntity } from ':my-scope/my-table';
import { publicProcedure } from '../init.js';

export const listExamples = publicProcedure
  .query(async () => {
    const entity = await createExampleEntity();
    const result = await entity.scan.go();
    return result.data;
  });
```

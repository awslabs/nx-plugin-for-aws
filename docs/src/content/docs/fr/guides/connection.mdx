Ce générateur permet de connecter des projets entre eux, comme des sites web appelant des API. Sélectionnez simplement le projet source (par exemple le projet qui appellera votre API) et le projet cible (par exemple votre projet d'API), et ce générateur se chargera d'intégrer les deux.

Le générateur de connexion utilise la <Link path="guides/runtime-config">Configuration d'exécution</Link> pour transmettre des valeurs au moment du déploiement (telles que les URL d'API, les paramètres Cognito et les ARN d'exécution d'agent) entre les projets et composants générés au moment de l'exécution afin qu'ils puissent se découvrir et se connecter les uns aux autres.
:::

Les projets connectés peuvent être exécutés sur votre machine avec les cibles `serve` et `serve-local`. Consultez le guide <Link path="guides/local-development">Développement local</Link> pour plus de détails.
:::

Considérons un espace de travail avec les connexions suivantes : un site web qui appelle une `api` tRPC, et qui appelle également un `agent` qui à son tour appelle un serveur `mcp`.

### `serve`

La cible `serve` exécute **uniquement le projet ciblé** localement. Tous les autres projets auxquels il se connecte sont censés être **déployés**, et sont accessibles à leur URL AWS réelle.

<NxCommands commands={['serve website']} />

Étant donné que le projet ciblé communique avec des ressources déployées, il doit savoir où elles se trouvent. Pour un site web, cela est fourni par son `runtime-config.json` (voir <Link path="guides/react-website#local-runtime-config">Configuration d'exécution locale</Link>), que vous chargez depuis une application déployée. Pour les projets côté serveur (API et agents) qui lisent la <Link path="guides/runtime-config">Configuration d'exécution</Link>, définissez la variable d'environnement `RUNTIME_CONFIG_APP_ID` pour pointer vers votre application AppConfig déployée.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
agent: agent {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
mcp: mcp {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Utilisez `serve` lorsque vous souhaitez itérer sur un seul projet par rapport aux versions « réelles » déployées de tout ce dont il dépend.

### `serve-local`

La cible `serve-local` exécute le projet ciblé **et tous les projets qui lui sont connectés de manière transitive**, tous sur votre machine. Le générateur de connexion configure cela automatiquement — l'exécution de `serve-local` sur le site web démarre également des serveurs locaux pour l'`api`, l'`agent` et le serveur `mcp` qu'il atteint via l'agent.

<NxCommands commands={['serve-local website']} />

Lorsqu'il est exécuté de cette manière, le `runtime-config.json` du site web est automatiquement remplacé (via le [`MODE` de Vite](https://vite.dev/guide/env-and-mode), défini sur `serve-local`) afin qu'il pointe vers vos serveurs s'exécutant localement au lieu des URL déployées.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#c8e6c9"
}
agent: agent {
  style.fill: "#c8e6c9"
}
mcp: mcp {
  style.fill: "#c8e6c9"
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Utilisez `serve-local` lorsque vous travaillez sur plusieurs projets connectés à la fois et que vous souhaitez itérer rapidement sans déployer votre infrastructure.

:::note[Ressources AWS dans `serve-local`]
Vos serveurs locaux s'exécutent avec les informations d'identification AWS que vous avez configurées localement, ils peuvent donc toujours accéder aux ressources AWS déployées (telles que les tables DynamoDB ou les modèles Bedrock) si nécessaire. Vous pouvez fournir toutes les variables d'environnement dont vos serveurs locaux ont besoin, par exemple :

<NxCommands env={{RUNTIME_CONFIG_APP_ID: 'xxxxx'}} commands={['serve-local website']} />
:::

Pour plus de détails sur le serveur de développement local de chaque type de projet, consultez le guide correspondant — par exemple le guide <Link path="guides/react-website#local-development-server">Site web React</Link>.

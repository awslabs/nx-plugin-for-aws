---
title: "Métriques d'utilisation"
description: "Comment @aws/nx-plugin collecte les métriques"
---



Pour nous aider à améliorer le Nx Plugin pour AWS, nous collectons des métriques sur l'utilisation des générateurs. Ce processus ne nécessite aucune requête réseau supplémentaire - nous utilisons simplement les mesures d'utilisation existantes d'AWS.

Pour garantir la transparence, nous ajoutons un ID, une version et une liste d'IDs de générateurs aux descriptions des piles CloudFormation. Cette opération est réalisée dans le `MetricsAspect` du dossier `common/constructs` pour CDK, ou dans la pile CloudFormation `metrics.tf` pour Terraform.

## Désactivation

Le code de métriques réside entièrement dans votre espace de travail, vous pouvez donc le désactiver en le supprimant :

- **CDK**: supprimez la classe `MetricsAspect` et l'appel `Aspects.of(this).add(new MetricsAspect())` de `packages/common/constructs/src/core/app.ts`.
- **Terraform**: supprimez `packages/common/terraform/src/metrics/metrics.tf`.

Les générateurs ne rajouteront pas le code de métriques une fois supprimé.

Cette page fournit des conseils de dépannage pour les problèmes que vous pourriez rencontrer lors de l'utilisation de Nx.

## Nx Daemon

Le [Nx Daemon](https://nx.dev/docs/concepts/nx-daemon) est un processus qui surveille en continu les modifications de l'espace de travail afin de recalculer son graphe de projets. Cela permet d'améliorer les performances, notamment pour les grands projets, mais peut parfois être à l'origine de problèmes selon votre environnement.

Cela peut se manifester de différentes manières, notamment par l'erreur suivante :

```bash
 NX   Could not find project "@scope/project"

Pass --verbose to see the stacktrace.
```

Pour cette erreur, et toute autre erreur inattendue, il est toujours utile de réinitialiser Nx et de réessayer en cas de problème lié au daemon. Vous pouvez utiliser la commande suivante :

<NxCommands commands={['reset']} />

Si cela ne fonctionne pas, vous pouvez essayer de [désactiver complètement le daemon](https://nx.dev/docs/concepts/nx-daemon#turning-it-off).

:::warning
La commande `nx watch` nécessite le daemon, donc certaines de vos cibles de rechargement à chaud pourraient ne plus fonctionner si vous le désactivez.
:::

---
title: Exigences SSL lors de la connexion sans RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

Pour les connexions directes au cluster Aurora depuis les runtimes Lambda Node.js 20 ou ultérieurs, chargez le bundle CA Amazon RDS en définissant `NODE_EXTRA_CA_CERTS` :

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "api" {
  source = "..."
  ...

  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/var/runtime/ca-cert.pem"
  }
}
```

</Fragment>
</Infrastructure>

Pour plus de détails, consultez les [exigences SSL/TLS pour les connexions Amazon RDS](https://docs.aws.amazon.com/lambda/latest/dg/services-rds.html#services-rds-tls) d'AWS Lambda et la [documentation TLS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-proxy.howitworks.html#rds-proxy-security.tls) d'Amazon RDS Proxy. Lors de l'utilisation de RDS Proxy, vous n'avez pas besoin de configurer le bundle CA RDS dans votre fonction Lambda.

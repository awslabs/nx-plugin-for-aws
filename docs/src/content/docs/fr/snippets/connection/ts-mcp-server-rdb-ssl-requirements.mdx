---
title: Exigences SSL lors de la connexion sans RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

Le générateur de connexion met à jour le Dockerfile pour installer le bundle CA Amazon RDS à l'emplacement `/usr/local/share/ca-certificates/rds-bundle.crt`. Définissez `NODE_EXTRA_CA_CERTS` sur ce chemin afin que Node.js fasse confiance au certificat lors de la connexion sans RDS Proxy :

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
new MyMcpServer(this, 'MyMcpServer', {
  ...
  environmentVariables: {
    NODE_EXTRA_CA_CERTS: '/usr/local/share/ca-certificates/rds-bundle.crt',
  },
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_mcp_server" {
  ...
  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/usr/local/share/ca-certificates/rds-bundle.crt"
  }
}
```

</Fragment>
</Infrastructure>

Pour plus de détails, consultez la [documentation Amazon RDS SSL/TLS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/UsingWithRDS.SSL.html). Lors de l'utilisation de RDS Proxy, vous n'avez pas besoin de configurer `NODE_EXTRA_CA_CERTS`.

---
title: Analyse des images de conteneur
---

L'image Docker construite pour ce projet est analysée pour détecter les vulnérabilités dans le cadre de la construction en utilisant [Trivy](https://trivy.dev/), exécuté à partir de l'[image Trivy hébergée sur ECR](https://gallery.ecr.aws/aquasecurity/trivy).

L'analyse s'exécute après la construction de l'image et **fait échouer la construction** si une vulnérabilité de gravité `HIGH` ou `CRITICAL` corrigeable est trouvée. Les vulnérabilités sans correctif disponible sont ignorées afin que les avis d'image de base non exploitables ne bloquent pas votre construction.

L'analyse utilise le même moteur de conteneur que votre construction d'image (`docker` ou `finch`), donc aucun outillage supplémentaire n'est requis.

:::caution[Images de base saines]
Le `Dockerfile` généré utilise une image de base sans vulnérabilité `HIGH` ou `CRITICAL` connue au moment de la génération, et met à niveau les outils intégrés (tels que `npm`) pour le maintenir ainsi. De nouvelles vulnérabilités sont divulguées au fil du temps, donc une construction qui réussit aujourd'hui peut échouer plus tard lorsque la base de données de vulnérabilités est mise à jour. Lorsque cela se produit, mettez à jour vers une image de base plus récente où la vulnérabilité est corrigée, ou supprimez le résultat comme décrit ci-dessous.
:::

#### Suppression des résultats Trivy

Il peut y avoir des cas où vous souhaitez supprimer une vulnérabilité spécifique, par exemple lorsqu'aucun correctif n'est encore disponible et que vous avez évalué le risque comme acceptable.

Ajoutez l'ID de vulnérabilité (un par ligne) au fichier `.trivyignore` à la racine de votre projet :

```text
// .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

Pour plus de détails sur le filtrage des résultats, consultez la [documentation de filtrage Trivy](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids).

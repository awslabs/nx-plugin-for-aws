[CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html) construit des graphes de vos ressources CDK connectées et fournit deux plugins :

#### Diagram Plugin

Le [CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html) génère des diagrammes d'architecture AWS à partir de votre infrastructure CDK.

Pour une approche déterministe similaire, une alternative viable est [CDK-Dia](https://github.com/pistazie/cdk-dia).

Avec les avancées en IA générative, de nombreux modèles de base sont capables de créer des diagrammes de haute qualité à partir de votre infrastructure CDK. Nous recommandons d'essayer le [AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server). Consultez [cet article de blog](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/) pour un guide détaillé.

#### Threat Composer Plugin

Le [CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html) génère un modèle de menace initial pour [Threat Composer](https://github.com/awslabs/threat-composer/) à partir de votre code CDK.

Ce plugin fonctionnait simplement en filtrant un [modèle de menace de base](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model) contenant des exemples de menaces, et en les filtrant selon les ressources utilisées par votre stack.

Si ces exemples de menaces spécifiques vous intéressent, vous pouvez copier et filtrer le modèle de menace de base, ou l'utiliser comme contexte pour aider un modèle de base à en générer un similaire.

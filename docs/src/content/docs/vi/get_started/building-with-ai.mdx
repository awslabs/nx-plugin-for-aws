## Kiro Power

Nx Plugin cho AWS đi kèm với một [Kiro Power](https://kiro.dev/docs/powers/) tích hợp MCP server cùng với tài liệu và hướng dẫn quy trình làm việc, cung cấp cho Kiro ngữ cảnh phong phú về các generator có sẵn và cách sử dụng chúng. Đây là cách được khuyến nghị để bắt đầu với Kiro.

### Cài đặt Power

<Steps>

1. Mở bảng Kiro Powers từ thanh bên
2. Nhấp vào nút `+` để thêm một power tùy chỉnh
3. Dán URL GitHub sau:

   ```
   https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
   ```

4. Nhấp vào install

</Steps>

Sau khi cài đặt, Kiro tự động có quyền truy cập vào MCP server `aws-nx-mcp` và các công cụ của nó. Không cần cấu hình MCP bổ sung nào.

### Power Cung cấp Gì

Power cung cấp cho Kiro quyền truy cập vào các công cụ MCP sau:

- `create_workspace_command` — Tạo lệnh để tạo một workspace Nx mới được cấu hình sẵn với plugin AWS
- `list_generators` — Khám phá tất cả các generator có sẵn và các tham số của chúng
- `generator_guide` — Nhận tài liệu chi tiết cho một generator cụ thể, bao gồm tham số, cấu trúc file và các phương pháp hay nhất
- `general_guidance` — Nhận hướng dẫn chung về việc sử dụng plugin

Nó cũng bao gồm tài liệu hướng dẫn giúp Kiro hiểu toàn bộ các generator có sẵn, quy trình làm việc phổ biến và các phương pháp hay nhất.

### Ví dụ về Lời nhắc

Sau khi power được cài đặt, bạn có thể bắt đầu xây dựng bằng cách trò chuyện với Kiro:

- *"Create a new Nx workspace called my-app using pnpm"* — Kiro sẽ tạo và chạy lệnh tạo workspace phù hợp
- *"Scaffold a React website with a tRPC API backend and CDK infrastructure"* — Kiro sẽ tạo khung cho từng dự án và kết nối chúng lại với nhau
- *"What generators are available in the Nx Plugin for AWS?"* — Kiro sẽ liệt kê tất cả các generator với mô tả và ví dụ sử dụng
- *"Add a Lambda function to my existing TypeScript project"* — Kiro sẽ tìm generator phù hợp và hướng dẫn qua quy trình tạo khung
- *"Build my project"* - Kiro sẽ thực thi lệnh nx build

## Cấu hình MCP Server

Nx Plugin cho AWS bao gồm một [MCP Server](https://modelcontextprotocol.io/introduction) cho phép các trợ lý AI làm việc với Nx Plugin cho AWS. Bằng cách sử dụng MCP Server, bạn có thể tăng tốc quy trình phát triển của mình với AI, đồng thời hưởng lợi từ việc các dự án của bạn được tạo khung một cách xác định, tiết kiệm thời gian và ít tốn cửa sổ ngữ cảnh hơn cho việc thiết lập các thành phần chính.

Cấu hình MCP server cho trợ lý AI mà bạn lựa chọn:

<Tabs>
  <TabItem label="Kiro">
    Nếu bạn chưa cài đặt [Kiro Power](#kiro-power) ở trên, bạn có thể thêm MCP server theo cách thủ công trong `.kiro/mcp.json`:

    <Snippet name="mcp/config" />
  </TabItem>
  <TabItem label="Claude Code">
    Chạy lệnh sau trong terminal của bạn:

    ```bash
    claude mcp add nx-plugin-for-aws -- npx -y @aws/nx-plugin-mcp
    ```

    Xem [tài liệu Claude Code MCP](https://code.claude.com/docs/en/mcp) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Cursor">
    Thêm nội dung sau vào `~/.cursor/mcp.json`:

    <Snippet name="mcp/config" />

    Xem [tài liệu Cursor MCP](https://cursor.com/docs/mcp) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Codex">
    Thêm nội dung sau vào `~/.codex/config.toml`:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    Xem [tài liệu Codex](https://github.com/openai/codex) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Other">
    Hầu hết các trợ lý tương thích MCP sử dụng file cấu hình JSON. Thêm mục sau:

    <Snippet name="mcp/config" />

    Nếu bạn gặp vấn đề như `ENOENT npx`, hãy thay thế lệnh bằng `/full/path/to/npx` (sử dụng `which npx` để tìm đường dẫn này).

    Đối với tài liệu dành riêng cho từng trợ lý:

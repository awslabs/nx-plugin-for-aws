Cài đặt [Claude Plugin](https://code.claude.com/docs/en/plugins), [Kiro Power](https://kiro.dev/docs/powers/), hoặc [MCP Server](https://modelcontextprotocol.io/introduction) để cho phép các trợ lý AI làm việc hiệu quả với plugin. Bằng cách sử dụng những công cụ này, bạn có thể tăng tốc quy trình phát triển của mình với AI trong khi hưởng lợi từ việc các dự án của bạn được tạo khung một cách nhất quán, tiết kiệm thời gian cho việc thiết lập các thành phần chính và hưởng lợi từ tính bảo mật, khả năng quan sát, type-safety và phát triển cục bộ được tích hợp sẵn trong plugin.

## Cấu hình Trợ lý AI của bạn

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    Nx Plugin cho AWS được cung cấp dưới dạng một [Kiro Power](https://kiro.dev/docs/powers/) tích hợp MCP server với tài liệu hướng dẫn và hướng dẫn quy trình làm việc.

    <Steps>

    1. Mở bảng Powers trong Kiro và nhấp vào **Add power from GitHub**
    2. Nhập URL sau:

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. Nhấp vào **Install**

    </Steps>

    Kiro tự động cấu hình MCP server — không cần thiết lập bổ sung nào.
  </TabItem>
  <TabItem label="Kiro CLI">
    Chạy lệnh sau:

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    Xem [tài liệu Kiro CLI MCP](https://kiro.dev/docs/cli/mcp/) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Claude Code">
    Nx Plugin cho AWS được phân phối dưới dạng một [plugin Claude Code](https://code.claude.com/docs/en/plugins) tích hợp một [Skill](https://code.claude.com/docs/en/skills) với MCP server.

    Chạy lệnh sau trong terminal của bạn:

    ```bash
    npx plugins add awslabs/nx-plugin-for-aws
    ```

    Sau đó khởi động lại Claude Code (hoặc chạy `/reload-plugins`) để kích hoạt plugin.
  </TabItem>
  <TabItem label="Cursor">
    Thêm nội dung sau vào `~/.cursor/mcp.json` (hoặc `.cursor/mcp.json` trong dự án của bạn):

    <Snippet name="mcp/config" />

    Xem [tài liệu Cursor MCP](https://cursor.com/docs/mcp) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Codex">
    Thêm nội dung sau vào `~/.codex/config.toml`:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    Xem [tài liệu cấu hình Codex](https://developers.openai.com/codex/config-reference) để biết thêm chi tiết.
  </TabItem>
  <TabItem label="Other">
    Hầu hết các trợ lý tương thích MCP sử dụng file cấu hình JSON. Thêm mục sau:

    <Snippet name="mcp/config" />

    Nếu bạn gặp vấn đề như `ENOENT npx`, hãy thay thế lệnh bằng đường dẫn đầy đủ đến `npx` (tìm nó bằng `which npx`).

    Đối với tài liệu dành riêng cho từng trợ lý:

    <Snippet name="mcp/assistant-docs" />
  </TabItem>
</Tabs>

## Bắt đầu Vibe Coding

Yêu cầu Trợ lý AI của bạn xây dựng thứ gì đó bằng cách sử dụng `nx-plugin-for-aws`! Thông thường nó sẽ tạo một workspace, tạo khung với các generator phù hợp, sau đó điền vào logic nghiệp vụ.

Thử một lời nhắc như:

> *"Build a multi-agent application using the Nx Plugin for AWS. I want a React frontend that talks to a backend orchestrator agent which delegates to specialised research and writing agents."*

Hoặc để bắt đầu với một điểm nhỏ hơn:

> *"Use the Nx Plugin for AWS to scaffold a React website backed by a tRPC API, with Cognito auth and CDK infrastructure."*

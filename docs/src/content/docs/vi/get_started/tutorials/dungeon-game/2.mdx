## Nhiệm vụ 1: Triển khai Game API

Chúng ta sẽ triển khai các API sau trong phần này:

1. `saveGame` - tạo hoặc cập nhật một trò chơi.
2. `queryGames` - trả về danh sách phân trang các trò chơi đã lưu trước đó.
3. `saveAction` - lưu một hành động cho một trò chơi nhất định.
4. `queryActions` - trả về danh sách phân trang tất cả các hành động liên quan đến một trò chơi.
5. `queryInventory` - trả về danh sách phân trang các vật phẩm trong kho của người chơi.

</Aside>

### Mô hình hóa thực thể

Đây là sơ đồ ER cho ứng dụng của chúng ta.

```d2
direction: right

game: Game {
  shape: sql_table
  playerName: "string  PK"
  genre: string
  lastUpdated: string
}

action: Action {
  shape: sql_table
  playerName: "string  PK"
  timestamp: "string  SK"
  role: string
  content: string
}

item: Item {
  shape: sql_table
  playerName: "string  PK"
  itemName: "string  SK"
  emoji: "string  (optional)"
  lastUpdated: string
  quantity: number
}

game -> action: 1..*
game -> item: 1..*
```

Chúng ta sẽ triển khai cơ sở dữ liệu trong DynamoDB và sẽ sử dụng thư viện client DynamoDB [ElectroDB](https://electrodb.dev/en/core-concepts/introduction/) để đơn giản hóa mọi thứ. Để cài đặt `electrodb` và DynamoDB Client, chạy lệnh này:

## Nhiệm vụ 2: Tạo Inventory MCP server

Hãy tạo một MCP server cho phép agent của chúng ta quản lý các vật phẩm trong kho của người chơi.

Chúng ta sẽ định nghĩa các công cụ sau cho agent của chúng ta:

- `list-inventory-items` để lấy các vật phẩm kho hiện tại của người chơi
- `add-to-inventory` để thêm vật phẩm vào kho của người chơi
- `remove-from-inventory` để xóa vật phẩm khỏi kho của người chơi

### Triển khai ứng dụng của bạn
Để triển khai ứng dụng của bạn, chạy lệnh sau:

<NxCommands commands={['deploy infra "dungeon-adventure-infra-sandbox/*"']} />

:::caution[Yêu cầu bootstrap]
Để bootstrap AWS Account của bạn trước, chạy:

<NxCommands commands={['bootstrap infra']} />
:::

Triển khai đầu tiên của bạn sẽ mất khoảng 8 phút để hoàn thành. Các lần triển khai tiếp theo sẽ mất khoảng 2 phút.

:::tip[Lặp nhanh]
Nếu bạn đang lặp lại các thay đổi code Lambda function, bạn có thể triển khai với cờ `--hotswap` sau khi build codebase để có thời gian triển khai ngắn hơn nhiều (2-3 giây).

<NxCommands commands={['deploy infra "dungeon-adventure-infra-sandbox/*" --hotswap']} />
:::

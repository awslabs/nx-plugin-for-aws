## Nhiệm vụ 1: Cấu hình máy chủ phát triển cục bộ

Để bắt đầu xây dựng giao diện người dùng, chúng ta cần cấu hình máy chủ phát triển cục bộ để trỏ đến sandbox đã triển khai của chúng ta. Để thực hiện điều này, hãy chạy lệnh sau:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Lệnh này sẽ tải xuống file `runtime-config.json` đã được triển khai và lưu trữ nó cục bộ trong thư mục `packages/game-ui/public`.

Để khởi động máy chủ phát triển, hãy chạy lệnh sau:

<NxCommands commands={["serve game-ui"]} />

Mở trang web cục bộ của bạn trong trình duyệt, nơi bạn sẽ được nhắc đăng nhập và làm theo các hướng dẫn để tạo người dùng mới. Sau khi hoàn tất, bạn sẽ thấy trang web cơ bản:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution" title="Giữ server phát triển chạy">
Chúng tôi khuyến nghị giữ máy chủ phát triển chạy trong suốt phần còn lại của module này vì nó sẽ tự động tải lại bất kỳ thay đổi nào chúng ta thực hiện.
</Aside>

:::tip[Sử dụng lệnh Nx]
Bạn có thể chạy các lệnh Nx thay thế để khởi động bất kỳ máy chủ API cục bộ nào được kết nối, cho phép bạn lặp lại cả tRPC API và trang web của bạn cùng nhau. <NxCommands commands={["serve-local game-ui"]} env={{RUNTIME_CONFIG_APP_ID:"xxxx"}} />

Sử dụng giá trị đầu ra deploy CDK của `RuntimeConfigApplicationId` cho biến môi trường `RUNTIME_CONFIG_APP_ID`.
:::

## Nhiệm vụ 2: Tạo route '/game' mới

Hãy giới thiệu các khả năng của `@tanstack/react-router` bằng cách tạo một route an toàn kiểu mới.
Để thực hiện điều này, hãy tạo một file trống tại vị trí sau: `packages/game-ui/src/routes/game/index.tsx`. Bạn sẽ nhận thấy file được cập nhật ngay lập tức.

`@tanstack/react-router` tự động cấu hình route mới của bạn và file bạn vừa tạo đã được điền sẵn với đường dẫn route:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

Nếu bạn điều hướng đến `http://localhost:4200/game`, bạn sẽ thấy trang mới của bạn đã được hiển thị.

<Image src={baselineGamePng} alt="baseline-game.png" width="800" height="600" />

Cập nhật file `index.tsx` để tải route `/game` mới của chúng ta theo mặc định. Khi bạn cập nhật trường `to`, bạn có danh sách các route an toàn kiểu để chọn.

## Nhiệm vụ 4: Tích hợp Story Agent

Nhớ lại rằng trong <Link path="get_started/tutorials/dungeon-game/1#task-7-connect-your-components">Module 1</Link>, chúng ta đã sử dụng generator `connection` để kết nối Game UI của chúng ta với Story Agent. Điều này đã thiết lập một OpenAPI client an toàn kiểu để tương tác với agent, cùng với các hook và provider.

Generator connection đã tạo ra những thứ sau cho chúng ta:
- Một component `StoryAgentProvider` bao bọc ứng dụng của chúng ta trong `main.tsx`
- Một hook `useStoryAgent` để tích hợp TanStack Query
- Một hook `useStoryAgentClient` để truy cập client trực tiếp
- Các TypeScript type được tạo ra từ đặc tả OpenAPI của agent

Chúng ta sẽ sử dụng hook `useStoryAgentClient` trong game component của chúng ta để stream các phản hồi câu chuyện từ agent.

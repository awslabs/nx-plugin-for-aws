## Nhiệm vụ 1: Triển khai Story Agent

Story Agent là một agent [Strands](https://strandsagents.com/) mà khi được cung cấp một `Game` và danh sách các `Action` làm ngữ cảnh, sẽ phát triển câu chuyện. Chúng ta sẽ cấu hình agent để tương tác với Inventory MCP Server của chúng ta nhằm quản lý các vật phẩm khả dụng của người chơi.

:::note[Triển khai Python]
Đối với hướng dẫn này, chúng ta sẽ triển khai Agent theo cách "không trạng thái", trong đó mỗi lần gọi là một phiên độc lập bao gồm toàn bộ lịch sử tin nhắn. Đối với môi trường sản xuất, hãy xem xét tích hợp với [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html). Để biết thêm thông tin, tham khảo trang [sử dụng AgentCore Memory với Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

### Kiểm thử Agent của bạn

Bạn có thể kiểm thử Agent của mình bằng một trong hai cách:
<ul>
<li>Khởi động một instance cục bộ của Agent server và gọi nó bằng `curl`, hoặc</li>
<li>Gọi API đã triển khai bằng curl với JWT token.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  Khởi động Agent server cục bộ của bạn bằng cách chạy lệnh sau:
    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    <Aside type="caution" title="Sử dụng giá trị đầu ra CDK">
    Sử dụng giá trị output `RuntimeConfigApplicationId` từ CDK deploy cho biến môi trường `RUNTIME_CONFIG_APP_ID`, và region AWS đã triển khai của bạn cho `AWS_REGION`.
    </Aside>

    :::note[`agent-serve` vs `agent-serve-local`]
    Target `agent-serve` khởi động agent cục bộ nhưng kết nối với MCP server **đã triển khai** của bạn (thông qua `RUNTIME_CONFIG_APP_ID`). Nếu bạn muốn chạy MCP server cục bộ, hãy sử dụng `agent-serve-local` thay thế, target này sẽ khởi động tất cả các dịch vụ phụ thuộc và đặt `SERVE_LOCAL=true` cho các kết nối HTTP trực tiếp.
    :::

    Khi Agent server đã chạy (bạn sẽ không thấy bất kỳ output nào), gọi nó bằng cách chạy lệnh sau:

    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}' \
      -H "Content-Type: application/json"
    ```
  </TabItem>
  <TabItem label="Deployed">
    Để kiểm thử agent đã triển khai, bạn cần xác thực với Cognito và lấy JWT token. Đầu tiên, thiết lập các biến môi trường của bạn:

    ```bash
    # Đặt Cognito User Pool ID và Client ID từ các output của CDK
    export POOL_ID="<UserPoolId từ CDK outputs>"
    export CLIENT_ID="<UserPoolClientId từ CDK outputs>"
    export REGION="<your-region>"
    ```

    Tạo một test user và lấy authentication token:

    ```bash
    # Vô hiệu hóa MFA để đơn giản hóa việc tạo user
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Tạo User
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Đặt Permanent Password (thay thế bằng một mật khẩu an toàn hơn!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Xác thực User và lấy Access Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Gọi agent đã triển khai bằng URL runtime của Bedrock AgentCore:

    ```bash
    # Đặt Story Agent ARN từ CDK outputs
    export AGENT_ARN="<StoryAgentArn từ CDK outputs>"

    # URL-encode ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Xây dựng invocation URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Gọi agent
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}'
    ```

    :::note[URL Encoding]
    URL encoding là cần thiết để định dạng đúng ARN cho endpoint API của Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

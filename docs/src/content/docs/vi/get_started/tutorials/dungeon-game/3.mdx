## Nhiệm vụ 1: Triển khai Story Agent

Story Agent là một agent [Strands](https://strandsagents.com/) được expose thông qua giao thức [AG-UI](https://ag-ui.com/). Mỗi phiên AG-UI (được xác định bởi `threadId`) ánh xạ tới một instance Agent tồn tại lâu dài trên server, do đó lịch sử hội thoại của agent được giữ trong bộ nhớ trong suốt vòng đời của phiên.

Chúng ta sẽ cập nhật system prompt của agent để chơi trò phiêu lưu hầm ngục, và xóa sample tool.

### Triển khai Agent

Cập nhật `packages/story/dungeon_adventure_story/agent/agent.py`:

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

System prompt cho agent biết rằng tin nhắn đầu tiên sẽ có dạng `I am <player>, playing in the <genre> genre.` — UI của chúng ta gửi chính xác điều này làm tin nhắn người dùng mở đầu để agent có mọi thứ cần thiết để bắt đầu câu chuyện. Vì thư viện AG-UI tự động giữ lại lịch sử hội thoại của mỗi thread, chúng ta không cần truyền tên người chơi hoặc thể loại qua các lượt tiếp theo.

Connection generator đã kết nối Inventory MCP client trong <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, do đó agent có quyền truy cập vào các inventory tools.

:::note[`main.py` ở đâu?]
FastAPI wrapper của giao thức AG-UI xử lý việc chuyển đổi request/response cho chúng ta. File `main.py` mà generator tạo ra đã tạo wrapper xung quanh `get_agent(...)` — chúng ta không cần chạm vào nó.
:::

### Tạo Cognito test user

Trước khi kiểm thử agent đã triển khai (và trước khi chúng ta chơi trò chơi qua UI trong module tiếp theo), hãy tạo một Cognito test user. Chúng ta đã vô hiệu hóa tính năng tự đăng ký trước đó, do đó bước này là bắt buộc bất kể bạn kiểm thử cục bộ hay đối với runtime đã triển khai:

```bash
# Từ các CDK outputs
export POOL_ID="<UserPoolId from CDK outputs>"
export CLIENT_ID="<UserPoolClientId from CDK outputs>"
export REGION="<your-region>"

# Vô hiệu hóa MFA để đơn giản hóa việc tạo user
aws cognito-idp set-user-pool-mfa-config \
  --mfa-configuration OFF \
  --user-pool-id $POOL_ID \
  --region $REGION

# Tạo một test user
aws cognito-idp admin-create-user \
  --user-pool-id $POOL_ID \
  --username "test" \
  --temporary-password "TempPass123-" \
  --region $REGION \
  --message-action SUPPRESS > /dev/null

# Đặt permanent password (thay thế bằng một mật khẩu an toàn hơn!)
aws cognito-idp admin-set-user-password \
  --user-pool-id $POOL_ID \
  --username "test" \
  --password "PermanentPass123-" \
  --region $REGION \
  --permanent > /dev/null
```

### Kiểm thử Agent của bạn

Bạn có thể kiểm thử Agent của mình bằng một trong hai cách:
<ul>
<li>Khởi động một instance cục bộ của Agent server và trò chuyện với nó một cách tương tác bằng target `agent-chat`, hoặc</li>
<li>Gọi API đã triển khai bằng curl với JWT token.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  Target `agent-chat` khởi động agent cục bộ (cùng với các dependencies của nó, thông qua `agent-serve-local`) và đưa bạn vào một prompt chat tương tác. Chạy:

    <NxCommands commands={["agent-chat story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    Nếu bạn muốn trò chuyện với agent đối với MCP server **đã triển khai** của bạn thay vì server cục bộ, hãy khởi động server trực tiếp với `agent-serve` và truyền `RUNTIME_CONFIG_APP_ID` / `AWS_REGION`:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve story"]} />

    sau đó trong một shell khác chạy `agent-chat-cli agui http://localhost:8081/invocations`.
    :::

    Khi được nhắc, hãy thử bắt đầu trò chơi với một câu như:

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="Deployed">
    Đổi thông tin đăng nhập của test user để lấy Cognito access token:

    ```bash
    # Xác thực test user và lấy access token của họ
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Gọi agent đã triển khai bằng URL runtime của Bedrock AgentCore. Giao thức AG-UI mong đợi một payload `RunAgentInput` — một payload tối thiểu gửi tin nhắn người dùng mở đầu trông như sau:

    ```bash
    # Đặt Story Agent ARN từ CDK outputs
    export AGENT_ARN="<StoryAgentArn từ CDK outputs>"

    # URL-encode ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Xây dựng invocation URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Gọi agent
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[URL Encoding]
    URL encoding là cần thiết để định dạng đúng ARN cho endpoint API của Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Nếu lệnh chạy thành công, bạn sẽ bắt đầu thấy các AG-UI Server-Sent Events được streaming trở lại — các event `TEXT_MESSAGE_CONTENT` mang văn bản câu chuyện từng phần một.

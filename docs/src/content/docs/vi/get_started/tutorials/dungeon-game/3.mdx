## Nhiệm vụ 1: Triển khai Story Agent

Story Agent là một agent [Strands](https://strandsagents.com/) được tạo với `--protocol=AG-UI` trong <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, để UI có thể stream từ nó qua [Agent-User Interaction protocol](https://docs.copilotkit.ai/aws-strands/protocol) thông qua CopilotKit. Nó sử dụng Inventory MCP Server để quản lý các vật phẩm của người chơi, và [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) tích hợp sẵn của Strands để lưu trữ lịch sử hội thoại vào sessions bucket mà chúng ta đã cung cấp trong Module 2.

Các thay đổi là:

- `main.py` thêm một `session_manager_provider` tạo ra một `S3SessionManager` cho mỗi `thread_id` khi được triển khai, và quay về sử dụng `FileSessionManager` trên đĩa tại `/tmp/strands-sessions` khi chạy dưới `agent-serve-local` (`SERVE_LOCAL=true`). Khi triển khai, S3 bucket là cùng một bucket mà `queryActions` của Game API đọc từ đó, do đó trình duyệt có thể xây dựng lại bản ghi khi truy cập lại; ở cục bộ, agent lưu trữ các session vào đĩa và giao tiếp với local MCP server, mà không cần triển khai.
- `agent.py` loại bỏ sample tool `subtract` và thay thế system prompt bằng một prompt dungeon-master mời tin nhắn đầu tiên của người dùng nêu tên người chơi và thể loại, đồng thời sử dụng các tool của Inventory MCP Server.

### Trò chuyện với Agent của bạn

Target `agent-chat` được tạo ra sẽ mở một REPL tương tác với agent của bạn. Nó chạy độc lập và kết nối với agent đang chạy cục bộ của bạn, vì vậy trước tiên hãy khởi động local server của agent trong một terminal:

<NxCommands commands={["agent-dev story"]} />

Sau đó, trong terminal thứ hai, khởi động chat:

<NxCommands commands={["agent-chat story"]} />

Tin nhắn đầu tiên của bạn nên cho agent biết tên anh hùng của bạn và thể loại (ví dụ: `My name is Alice. Start my zombie adventure.`) và câu chuyện sẽ được stream trả về.

<Aside type="caution" title="Yêu cầu thông tin xác thực AWS">
Mặc dù agent chạy cục bộ, nó gọi [Amazon Bedrock](https://aws.amazon.com/bedrock/) cho LLM. Bạn sẽ cần thông tin xác thực AWS trong môi trường của mình với quyền truy cập vào [Strands default model](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/). Xem [tài liệu model provider của Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) để biết chi tiết.
</Aside>

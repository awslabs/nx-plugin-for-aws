## Nhiệm vụ 1: Triển khai Story Agent

Story Agent là một agent [Strands](https://strandsagents.com/) được tạo với `--protocol=AG-UI` trong <Link path="get_started/tutorials/dungeon-game/1">Module 1</Link>, để UI có thể stream từ nó qua [Agent-User Interaction protocol](https://docs.copilotkit.ai/aws-strands/protocol) thông qua CopilotKit. Nó sử dụng Inventory MCP Server để quản lý các vật phẩm của người chơi, và [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) tích hợp sẵn của Strands để lưu trữ lịch sử hội thoại vào sessions bucket mà chúng ta đã cung cấp trong Module 2.

:::note[Lưu trữ lịch sử hội thoại]
Mỗi request AG-UI mang theo một `thread_id`. UI của chúng ta xây dựng id đó một cách xác định theo dạng ``{player}-{genre}`` được đệm đến độ dài tối thiểu của AgentCore là 33 ký tự, do đó mỗi cặp `(player, genre)` tiếp tục câu chuyện riêng của nó. Ở phía agent, `StrandsAgentConfig.session_manager_provider` cung cấp cho mỗi thread một `S3SessionManager` trỏ đến bucket `StorySessions` — do đó các lượt tiếp theo sẽ phát lại lịch sử tin nhắn đã lưu mà không cần UI phải gửi lại. Bởi vì Game API đọc tin nhắn trực tiếp từ cùng một bucket, trình duyệt cũng có thể hiển thị toàn bộ bản ghi khi tải lại trang. Đối với thiết lập sản xuất, hãy thay thế bằng [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html) thông qua [AgentCore Memory for Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

Các thay đổi là:

- `main.py` thêm một `session_manager_provider` tạo ra một `S3SessionManager` cho mỗi `thread_id` khi được triển khai, và quay về sử dụng `FileSessionManager` trên đĩa tại `/tmp/strands-sessions` khi chạy dưới `agent-serve-local` (`SERVE_LOCAL=true`). Khi triển khai, S3 bucket là cùng một bucket mà `queryActions` của Game API đọc từ đó, do đó trình duyệt có thể xây dựng lại bản ghi khi truy cập lại; ở cục bộ, agent chạy hoàn toàn offline với local MCP server mà không có bất kỳ lời gọi AWS nào.
- `agent.py` loại bỏ sample tool `subtract` và thay thế system prompt bằng một prompt dungeon-master mời tin nhắn đầu tiên của người dùng nêu tên người chơi và thể loại, đồng thời sử dụng các tool của Inventory MCP Server.

### Kiểm thử Agent của bạn

Bạn có thể kiểm thử Agent của mình bằng một trong hai cách:
<ul>
<li>Khởi động một instance cục bộ của Agent server và trò chuyện với nó thông qua target `agent-chat` được tạo ra, hoặc</li>
<li>Gọi API đã triển khai bằng curl với JWT token.</li>
</ul>

<Tabs>
  <TabItem label="Local">
    Mở một REPL tương tác với bản sao được phục vụ cục bộ của AG-UI agent bằng cách sử dụng target `agent-chat` được tạo ra:

    <NxCommands commands={["agent-chat story"]} />

    `agent-chat` có `dependsOn: ['agent-serve-local']`, do đó không cần khởi động agent server riêng biệt — Nx sẽ spawn `agent-serve-local` (lần lượt khởi động Inventory MCP server cục bộ), sau đó khởi chạy [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) AG-UI REPL với `http://localhost:8081/invocations`. Tin nhắn đầu tiên của bạn nên cho agent biết tên anh hùng của bạn và thể loại (ví dụ: `My name is Alice. Start my zombie adventure.`) và câu chuyện sẽ được stream trả về.

    <Aside type="tip" title="Raw curl">
    Nếu bạn muốn một HTTP call thô với local server, cấu trúc `RunAgentInput` của AG-UI là:
    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -H "Content-Type: application/json" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```
    Sử dụng lại cùng một `threadId` trong các lần gọi tiếp theo sẽ tiếp tục cùng một câu chuyện — `S3SessionManager` phát lại các tin nhắn trước đó.
    </Aside>
  </TabItem>
  <TabItem label="Deployed">
    Để kiểm thử agent đã triển khai, bạn cần xác thực với Cognito và lấy JWT token. Đầu tiên, thiết lập các biến môi trường của bạn:

    ```bash
    # Đặt Cognito User Pool ID và Client ID từ các output của CDK
    export POOL_ID="<UserPoolId từ CDK outputs>"
    export CLIENT_ID="<UserPoolClientId từ CDK outputs>"
    export REGION="<your-region>"
    ```

    Tạo một test user và lấy authentication token:

    ```bash
    # Vô hiệu hóa MFA để đơn giản hóa việc tạo user
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Tạo User
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Đặt Permanent Password (thay thế bằng một mật khẩu an toàn hơn!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Xác thực User và lấy Access Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Gọi agent đã triển khai bằng URL runtime của Bedrock AgentCore:

    ```bash
    # Đặt Story Agent ARN từ CDK outputs
    export AGENT_ARN="<StoryAgentArn từ CDK outputs>"

    # URL-encode ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Xây dựng invocation URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Gọi agent với payload AG-UI RunAgentInput
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: Alice-zombie00000000000000000000000" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```

    :::note[URL Encoding]
    URL encoding là cần thiết để định dạng đúng ARN cho endpoint API của Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Nếu lệnh chạy thành công, bạn sẽ bắt đầu thấy câu chuyện được stream trả về dưới dạng một [AG-UI event stream](https://docs.copilotkit.ai/aws-strands/protocol/events) (Server-Sent Events) — các chunk `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` bao bọc các token câu chuyện thực tế:

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

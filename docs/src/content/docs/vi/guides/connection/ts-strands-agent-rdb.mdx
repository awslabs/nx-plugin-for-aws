Generator `connection` kết nối một <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> với một dự án <Link path="guides/ts-rdb">Cơ sở dữ liệu Quan hệ</Link>, làm cho Prisma client có sẵn trong suốt vòng đời yêu cầu của agent.

## Cách hoạt động

### Giao thức HTTP

Đối với các HTTP agent, cơ sở dữ liệu chảy qua tRPC request context — `createContext` tạo một Prisma client cho mỗi yêu cầu và nó trở nên có sẵn cho mọi procedure thông qua `opts.ctx`. Router sau đó truyền nó một cách rõ ràng vào `getAgent`.

#### Context Interface

Kiểu Prisma client được thêm vào tRPC `Context` trong `init.ts`:

```ts title="packages/my-service/src/my-agent/init.ts" {1,5}
import { getPrisma as getMyDb } from ':my-scope/my-db';

export interface Context {
  sessionId: string;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
}
```

#### Request Context

`createContext` được làm async và Prisma client được khởi tạo và trả về:

```ts title="packages/my-service/src/my-agent/index.ts" {1,6,10}
import { getPrisma as getMyDb } from ':my-scope/my-db';

const createContext = async (
  opts: CreateHTTPContextOptions | CreateWSSContextFnOptions,
): Promise<Context> => {
  const myDb = await getMyDb();
  const sessionId = /* ... */;
  return {
    sessionId,
    myDb,
  };
};
```

#### Router và Agent

Cơ sở dữ liệu được truyền từ context vào `getAgent` thông qua subscription handler:

```ts title="packages/my-service/src/my-agent/router.ts" {2,3}
.subscription(async function* (opts) {
  const { myDb } = opts.ctx;
  const agent = await getAgent(opts.ctx.sessionId, { myDb });
  // ...
})
```

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

### Giao thức A2A

Đối với các A2A agent, không có tRPC context. Cơ sở dữ liệu được tạo một lần khi khởi động và được truyền trực tiếp vào `getAgent` trước khi `A2AExpressServer` khởi động.

#### Server Startup

Prisma client được tạo trước `A2AExpressServer` và được truyền vào `getAgent`:

```ts title="packages/my-service/src/my-agent/index.ts" {1,4,6}
import { getPrisma as getMyDb } from ':my-scope/my-db';

void (async () => {
  const myDb = await getMyDb();
  const server = new A2AExpressServer({
    agent: await getAgent('default', { myDb }),
    // ...
  });
})();
```

#### Agent Definition

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

## Nhiều Cơ sở dữ liệu

Chạy generator lần nữa với một target khác sẽ thêm cơ sở dữ liệu thứ hai cùng với cơ sở dữ liệu đầu tiên. Đối với các HTTP agent, cả hai đều xuất hiện trong interface `Context` và được destructure cùng nhau trong router:

```ts title="packages/my-service/src/my-agent/router.ts"
const { postgresDb, mysqlDb } = opts.ctx;
const agent = await getAgent(opts.ctx.sessionId, { postgresDb, mysqlDb });
```

## Infrastructure

Agent construct được tạo ra triển khai `IGrantable` và `IConnectable`, vì vậy bạn có thể cấp quyền truy cập mạng và IAM cho cơ sở dữ liệu trực tiếp trên construct.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myAgent = new MyAgent(this, 'MyAgent', { vpc, ... });

db.allowDefaultPortFrom(myAgent);
db.grantConnect(myAgent);
```

`allowDefaultPortFrom` mở quy tắc security group để agent runtime có thể truy cập cổng cơ sở dữ liệu. `grantConnect` cấp quyền IAM `rds-db:connect` cho execution role của agent.

</Fragment>
<Fragment slot="terraform">

Truyền các output của database module vào agent module của bạn để nó có thể truy cập cơ sở dữ liệu và đọc cấu hình runtime của nó:

```hcl title="packages/infra/src/main.tf"
module "my_database" {
  source              = "../../common/terraform/src/app/dbs/my-database"
  vpc_id              = module.vpc.vpc_id
  database_subnet_ids = module.vpc.private_isolated_subnet_ids
}

module "my_agent" {
  source = "../../common/terraform/src/app/agents/my-agent"

  appconfig_application_id     = module.my_database.appconfig_application_id
  database_cluster_resource_id = module.my_database.cluster_resource_id
  database_runtime_user        = module.my_database.database_runtime_user
  database_security_group_id   = module.my_database.security_group_id
  database_port                = module.my_database.cluster_port
}
```

Đảm bảo execution role của agent có quyền `rds-db:connect` và security group của nó có thể truy cập security group của cơ sở dữ liệu trên cổng cơ sở dữ liệu.

</Fragment>
</Infrastructure>

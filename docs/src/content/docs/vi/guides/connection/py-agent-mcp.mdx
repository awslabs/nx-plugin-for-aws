Trình tạo `connection` có thể kết nối <Link path="guides/py-agent">Python Agent</Link> của bạn với một MCP server (có thể là <Link path="guides/ts-mcp-server">TypeScript</Link> hoặc <Link path="guides/py-mcp-server">Python</Link>).

Trình tạo thiết lập tất cả các kết nối cần thiết để agent của bạn có thể khám phá và gọi các công cụ của MCP server, cả khi triển khai lên AWS (thông qua Bedrock AgentCore) và khi chạy cục bộ.

## Điều kiện tiên quyết

Trước khi sử dụng trình tạo này, hãy đảm bảo bạn có:

1. Một dự án Python với component <Link path="guides/py-agent">Python Agent</Link> (Strands hoặc LangChain)
2. Một dự án với component MCP server (có thể là <Link path="guides/ts-mcp-server">`ts#mcp-server`</Link> hoặc <Link path="guides/py-mcp-server">`py#mcp-server`</Link>)
3. Cả hai component được tạo với `infra: agentcore`

## Kết quả đầu ra của trình tạo

Trình tạo tạo một dự án Python `agent_connection` được chia sẻ tại `packages/common/agent_connection/` (nếu chưa tồn tại). Các module client theo từng kết nối được tạo vào dự án được chia sẻ này:

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py Xuất lại các client theo từng kết nối
    - core
      - agentcore\_endpoints.py Giải quyết ARN/URL không phụ thuộc framework
      - agentcore\_mcp\_transport.py Truyền tải MCP không phụ thuộc framework
      - agentcore\_mcp\_client\_\<framework>.py Client MCP bao bọc truyền tải cho framework của agent
      - auth/ `httpx.Auth` SigV4 / chuyển tiếp session không phụ thuộc framework
    - app
      - \<mcp\_server\_name>\_client\_\<framework>.py Client theo từng kết nối cho mỗi MCP server

</FileTree>

Hậu tố client khớp với framework của agent (`_strands` hoặc `_langchain`).

Ngoài ra, trình tạo còn:
- Chuyển đổi `agent.py` của agent để import và sử dụng các công cụ của MCP server thông qua một client dựa trên class
- Thêm dự án `agent_connection` làm dependency workspace của dự án agent
- Cập nhật target `serve-local` của agent để phụ thuộc vào target serve của MCP server

## Sử dụng MCP Server đã kết nối

Trình tạo chuyển đổi `agent.py` của agent để sử dụng các công cụ của MCP server:

<Tabs syncKey="agent-framework">
<TabItem label="Strands" _filter={{ framework: 'strands' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,9-13}
from contextlib import contextmanager
from strands import Agent

from my_scope_agent_connection import MyMcpServerClientStrands

@contextmanager
def get_agent():
    my_mcp_server = MyMcpServerClientStrands.create()
    with (
        my_mcp_server,
    ):
        yield Agent(
            system_prompt="...",
            tools=[*my_mcp_server.list_tools_sync()],
        )
```

Client Strands là một context manager, được nhập vào trong một khối `with` xung quanh agent.
</TabItem>
<TabItem label="LangChain" _filter={{ framework: 'langchain' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,8,13}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse

from my_scope_agent_connection import MyMcpServerClientLangChain

def get_agent():
    my_mcp_server = MyMcpServerClientLangChain.create()
    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[*my_mcp_server],
    )
```

Client LangChain trả về một danh sách các công cụ được tải thông qua [`langchain-mcp-adapters`](https://docs.langchain.com/oss/python/langchain/mcp). Mỗi công cụ mở một phiên MCP mới cho mỗi lần gọi, vì vậy các công cụ vẫn có thể sử dụng được trong suốt vòng đời của agent — không cần khối `with`.
</TabItem>
</Tabs>

Session ID của AgentCore được tự động truyền đến MCP server thông qua header `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id` cho cả hai framework, đảm bảo tính nhất quán cho [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html).

## Hạ tầng

<Infrastructure>
<Fragment slot="cdk">
Sau khi chạy trình tạo kết nối, bạn cần cấp quyền cho agent để gọi MCP server:

```ts title="packages/infra/src/stacks/application-stack.ts" {5}
const mcpServer = new MyMcpServer(this, 'MyMcpServer');
const myAgent = new MyAgent(this, 'MyAgent');

// Grant the agent permissions to invoke the MCP server
mcpServer.grantInvokeAccess(myAgent);
```

ARN runtime AgentCore của MCP server được tự động đăng ký trong namespace `agentcore` của <Link path="guides/runtime-config">Runtime Configuration</Link> bởi CDK construct được tạo, để agent có thể khám phá nó tại runtime.

```hcl title="packages/infra/src/main.tf" {12-24}
module "inventory_mcp_server" {
  source = "../../common/terraform/src/app/mcp-servers/inventory-mcp"

  source = "../../common/terraform/src/app/agents/story-agent"

# Grant the agent permissions to invoke the MCP server
resource "aws_iam_policy" "agent_invoke_mcp" {
  name = "AgentInvokeMcpPolicy"

    Version = "2012-10-17"
    Statement = [{
      Effect   = "Allow"
      Action   = "bedrock-agentcore:InvokeAgent"
      Resource = module.inventory_mcp_server.agent_core_runtime_arn
    }]
  })
}

resource "aws_iam_role_policy_attachment" "agent_invoke_mcp" {
  role       = module.story_agent.agent_core_runtime_role_arn
  policy_arn = aws_iam_policy.agent_invoke_mcp.arn
```

## Phát triển cục bộ

Trình tạo cấu hình target `serve-local` của agent để:
1. Tự động khởi động (các) MCP server đã kết nối
2. Đặt `SERVE_LOCAL=true` để client được tạo sử dụng truyền tải HTTP trực tiếp thay vì AgentCore

Chạy agent cục bộ với:

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Điều này sẽ khởi động cả agent và tất cả các MCP server đã kết nối, với agent kết nối trực tiếp đến các MCP server thông qua HTTP trên các cổng cục bộ được chỉ định của chúng.

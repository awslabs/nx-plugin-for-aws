Nx Plugin for AWS cung cấp một trình tạo để nhanh chóng tích hợp <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> của bạn với một trang web React. Nó thiết lập tất cả cấu hình cần thiết để kết nối với agent của bạn thông qua tRPC qua WebSocket, bao gồm hỗ trợ xác thực AWS IAM và Cognito. Tích hợp này cung cấp tính an toàn kiểu dữ liệu từ đầu đến cuối hoàn chỉnh giữa frontend của bạn và tRPC router của agent.

## Điều kiện tiên quyết

Trước khi sử dụng trình tạo này, hãy đảm bảo bạn có:

1. Một trang web React (được tạo bằng <Link path="guides/react-website">trình tạo `ts#react-website`</Link>)
2. Một TypeScript Strands Agent (được tạo bằng <Link path="guides/ts-strands-agent">trình tạo `ts#strands-agent`</Link>)
3. Cognito Auth được thêm thông qua <Link path="/guides/react-website-auth">trình tạo `ts#react-website-auth`</Link> nếu kết nối một agent sử dụng xác thực Cognito hoặc IAM

## Cách hoạt động

### Kết nối WebSocket

Client được tạo kết nối với Strands Agent của bạn thông qua tRPC qua WebSocket. Agent hiển thị một tRPC router (bao gồm subscription `invoke` để phát trực tiếp phản hồi của agent) qua một điểm cuối WebSocket.

- **Đã triển khai**: Agent runtime ARN được tải từ <Link path="guides/runtime-config">Runtime Configuration</Link>. ARN được chuyển đổi thành URL WebSocket theo giao thức Bedrock AgentCore Runtime WebSocket: `wss://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/ws`
- **Phát triển cục bộ**: Khi chạy với `serve-local`, ghi đè cấu hình runtime đặt giá trị thành URL `ws://` cục bộ (ví dụ: `ws://localhost:8081/ws`), và client kết nối trực tiếp

### Xác thực

Mã được tạo xử lý xác thực tùy thuộc vào cấu hình agent của bạn:

- **IAM** (mặc định): Sử dụng AWS SigV4 presigned URLs để xác thực kết nối WebSocket. Thông tin xác thực được lấy từ Cognito Identity Pool được cấu hình với xác thực của trang web của bạn
- **Cognito**: Nhúng JWT access token vào header `Sec-WebSocket-Protocol` dưới dạng bearer token được mã hóa base64url, theo [giao thức xác thực AgentCore WebSocket](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-get-started-websocket.html)
- **Không có xác thực**: Kết nối trực tiếp mà không cần xác thực (phù hợp cho phát triển cục bộ)

## Sử dụng mã được tạo

### Phát trực tiếp phản hồi của Agent với useSubscription

Trường hợp sử dụng phổ biến nhất là phát trực tiếp phản hồi của agent bằng subscription `invoke`:

```tsx {1-2,7-22}
import { useSubscription } from '@trpc/tanstack-react-query';
import { useMyAgent } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const trpc = useMyAgent();

  const subscription = useSubscription(
    trpc.invoke.subscriptionOptions(
      { message: 'What can you help me with?' },
      {
        enabled: true,
        onStarted: () => {
          console.log('Agent started responding');
        },
        onData: (token) => {
          console.log('Received token:', token);
        },
        onError: (error) => {
          console.error('Agent error:', error);
        },
      },
    ),
  );

  return (
    <div>
      <p>Status: {subscription.status}</p>
      {subscription.data && <p>Latest token: {subscription.data}</p>}
      {subscription.error && <p>Error: {subscription.error.message}</p>}
    </div>
  );
}
```

### Sử dụng Vanilla tRPC Client

Hook `use<AgentName>AgentClient` cung cấp quyền truy cập vào vanilla tRPC client để kiểm soát nhiều hơn vòng đời subscription:

```tsx {2,5,9-22,25}
import { useState, useEffect } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const client = useMyAgentClient();
  const [messages, setMessages] = useState<string[]>([]);

  const sendMessage = (message: string) => {
    const subscription = client.invoke.subscribe(
      { message },
      {
        onData: (token) => {
          setMessages((prev) => [...prev, token]);
        },
        onComplete: () => {
          console.log('Agent finished');
        },
        onError: (error) => {
          console.error('Error:', error);
        },
      },
    );

    // Clean up when done
    return () => subscription.unsubscribe();
  };

  return (
    <div>
      <button onClick={() => sendMessage('Hello!')}>Send</button>
      <div>
        {messages.map((msg, i) => (
          <span key={i}>{msg}</span>
        ))}
      </div>
    </div>
  );
}
```

### Xây dựng giao diện Chat

Đây là một ví dụ về xây dựng giao diện chat đơn giản với client được tạo:

```tsx
import { useState, useCallback } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function Chat() {
  const client = useMyAgentClient();
  const [input, setInput] = useState('');
  const [response, setResponse] = useState('');
  const [isStreaming, setIsStreaming] = useState(false);

  const handleSend = useCallback(() => {
    if (!input.trim() || isStreaming) return;

    setIsStreaming(true);
    setResponse('');

    client.invoke.subscribe(
      { message: input },
      {
        onData: (token) => {
          setResponse((prev) => prev + token);
        },
        onComplete: () => {
          setIsStreaming(false);
        },
        onError: (error) => {
          console.error(error);
          setIsStreaming(false);
        },
      },
    );

    setInput('');
  }, [client, input, isStreaming]);

  return (
    <div>
      <div style={{ whiteSpace: 'pre-wrap' }}>{response}</div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        onKeyDown={(e) => e.key === 'Enter' && handleSend()}
        disabled={isStreaming}
        placeholder="Ask the agent..."
      />
      <button onClick={handleSend} disabled={isStreaming}>
        {isStreaming ? 'Streaming...' : 'Send'}
      </button>
    </div>
  );
}
```

## Phát triển cục bộ

Trình tạo kết nối tự động cấu hình tích hợp `serve-local`:

1. Chạy `nx run <website>:serve-local` cũng sẽ khởi động máy chủ cục bộ của agent
2. Cấu hình runtime được ghi đè để trỏ đến URL WebSocket cục bộ (ví dụ: `ws://localhost:8081/ws`)
3. Client phát hiện URL cục bộ và kết nối trực tiếp mà không có chi phí xác thực

Điều này có nghĩa là bạn có thể phát triển và kiểm tra tích hợp agent của mình cục bộ bằng một lệnh duy nhất.

## Tính an toàn kiểu dữ liệu

Tích hợp cung cấp tính an toàn kiểu dữ liệu từ đầu đến cuối hoàn chỉnh. IDE của bạn sẽ cung cấp tự động hoàn thành đầy đủ và kiểm tra kiểu cho tất cả các lời gọi thủ tục agent. Các kiểu được suy ra tự động từ định nghĩa tRPC router của agent của bạn, đảm bảo rằng bất kỳ thay đổi nào đối với API của agent của bạn đều được phản ánh ngay lập tức trong mã frontend của bạn.

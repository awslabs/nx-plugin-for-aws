Generator này tạo một dự án cơ sở dữ liệu quan hệ mới được hỗ trợ bởi [Amazon Aurora](https://aws.amazon.com/rds/aurora/) PostgreSQL. Nó tạo ra mã ứng dụng và cơ sở hạ tầng cần thiết để cung cấp và quản lý cơ sở dữ liệu bằng AWS CDK hoặc Terraform.

Cơ sở hạ tầng được tạo ra bao gồm:

- Construct hoặc module Aurora có thể tái sử dụng
- Construct hoặc module cơ sở dữ liệu dành riêng cho ứng dụng
- Handler tạo user cơ sở dữ liệu ứng dụng được xác thực bằng IAM
- Đăng ký cấu hình runtime trong AWS AppConfig
- Migration handler để triển khai Prisma migration tự động

## Triển khai Cơ sở dữ liệu của bạn

### Mô hình hóa Dữ liệu

Dự án được tạo ra sử dụng [Prisma ORM](https://www.prisma.io/docs/orm) để định nghĩa schema cơ sở dữ liệu của bạn và tạo client an toàn về kiểu. Mô hình hóa các bảng, quan hệ và kiểu trường của bạn trong `prisma/schema.prisma`.

Ví dụ model `User`:

```ts title="packages/postgres/prisma/schema.prisma"
model User {
  id          Int      @id @default(autoincrement()) 
  firstName   String
  lastName    String
}
```

Để biết thêm chi tiết, xem hướng dẫn chính thức về [mô hình hóa dữ liệu với Prisma](https://www.prisma.io/docs/orm/core-concepts/data-modeling#data-modeling-with-prisma-orm).

### Tạo Database Client

Generator tự động cấu hình target `generate` để tạo Prisma client TypeScript an toàn về kiểu bất cứ khi nào bạn build dự án. Client được ghi vào `generated/prisma` (đã thêm vào `.gitignore`).

Bạn cũng có thể tạo client thủ công bất cứ lúc nào:

  </TabItem>
  <TabItem label="yarn" icon="seti:yarn">

Runtime wrapper trong `src/prisma.ts` export:

- `DB_PACKAGE_NAME` - key được sử dụng trong namespace runtime config `database` trong AWS AppConfig
- `getPrisma()` - tải cài đặt kết nối cơ sở dữ liệu từ AWS AppConfig và tạo Prisma client sử dụng xác thực IAM

Client tự động:
- Lấy cấu hình cơ sở dữ liệu từ AWS AppConfig sử dụng biến môi trường `RUNTIME_CONFIG_APP_ID`
- Tạo authentication token tạm thời qua AWS RDS Signer cho xác thực IAM
- Quản lý kết nối SSL/TLS với xác thực certificate
- Xử lý connection pooling thông qua các database connection pool bền vững

Ví dụ sử dụng trong một tRPC API:

```ts title="packages/api/src/procedures/list-user.ts"
import { getPrisma } from ':aws-nx-poc/postgres';
import { publicProcedure } from '../init.js';
import { ListUserOutputSchema } from '../schema/index.js';

export const listUser = publicProcedure
  .output(ListUserOutputSchema)
  .query(async () => {
    const prisma = await getPrisma();
    const users = await prisma.user.findMany({
      orderBy: { id: 'asc' },
    });

    return { users };
  });
```

### Tạo Migration

:::note[Chạy Prisma từ Thư mục Package]
Khi sử dụng Prisma CLI, chạy các lệnh từ thư mục package cơ sở dữ liệu được tạo của bạn (ví dụ: `packages/postgres`) để Prisma có thể tìm thấy `prisma/schema.prisma` và `prisma.config.ts`.
:::

Prisma Migrate yêu cầu một [shadow database](https://www.prisma.io/docs/orm/prisma-migrate/understanding-prisma-migrate/shadow-database) để so sánh schema của bạn với trạng thái cơ sở dữ liệu hiện tại và tạo các thay đổi SQL.

Để tạo migration cục bộ, trỏ Prisma đến một cơ sở dữ liệu phát triển bằng cách export `DATABASE_URL`:

```bash
export DATABASE_URL="postgresql://username:password@hostname:5432/dbname"
```

## Triển khai lên AWS

Generator cơ sở dữ liệu quan hệ tạo cơ sở hạ tầng CDK hoặc Terraform dựa trên `iacProvider` bạn đã chọn.

<Infrastructure>
<Fragment slot="cdk">
Construct CDK được tạo trong `common/constructs`. Ví dụ sử dụng:

```ts title="packages/infra/src/stacks/application-stack.ts"
import { Postgres } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const vpc = new Vpc(this, 'Vpc', {
      subnetConfiguration: [
        {
          name: 'public',
          subnetType: SubnetType.PUBLIC,
        },
        {
          name: 'private_with_egress',
          subnetType: SubnetType.PRIVATE_WITH_EGRESS,
        },
        {
          name: 'private_isolated',
          subnetType: SubnetType.PRIVATE_ISOLATED,
        },
      ],
    });

    const db = new Postgres(this, 'Db', {
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_ISOLATED,
      }
    });
  }
}
```

Điều này cung cấp một Aurora cluster với RDS Proxy, admin credentials, application database user, đăng ký runtime config và migration handler. Triển khai vào private isolated subnet để ngăn truy cập trực tiếp từ internet.

Cơ sở hạ tầng được tạo ra tạo hai database user:
- **Admin user** - Được tạo trong quá trình cung cấp cluster với credentials được lưu trong AWS Secrets Manager
- **Application user** - Được tạo thông qua Lambda custom resource với xác thực IAM được bật và có đầy đủ quyền trên application database
</Fragment>
<Fragment slot="terraform">

Điều này cung cấp một Aurora cluster với RDS Proxy, admin credentials, create-db-user Lambda, đăng ký runtime config, migration Lambda và các tài nguyên container registry.

Cơ sở hạ tầng được tạo ra tạo hai database user:

### Kết nối Lambda Function

Để sử dụng Prisma client từ Lambda function, đảm bảo các yêu cầu sau được đáp ứng:

1. Triển khai function trong cùng VPC với cơ sở dữ liệu
2. Cho phép truy cập mạng đến cổng cơ sở dữ liệu
3. Cấp quyền IAM `rds-db:connect` cho runtime user được tạo ra
4. Đặt các biến môi trường bắt buộc:
   - `NODE_EXTRA_CA_CERTS=/var/runtime/ca-cert.pem` (bắt buộc cho xác thực TLS)
   - `RUNTIME_CONFIG_APP_ID` (được đặt tự động bởi cơ sở hạ tầng được tạo ra)
   - `AWS_REGION` (được đặt tự động trong môi trường thực thi Lambda)

:::note[Prisma CLI vs Runtime Client]
**Prisma CLI** commands (`prisma migrate dev`, `prisma migrate deploy`) đọc `DATABASE_URL` từ biến môi trường.

**Runtime client** (`src/prisma.ts`) tải cài đặt kết nối từ AWS AppConfig và sử dụng xác thực IAM.
:::

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { RuntimeConfig } from ':my-scope/common-constructs';

const runtimeConfig = RuntimeConfig.ensure(this);

const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      vpc,
      vpcSubnets: {
        subnetType: SubnetType.PRIVATE_WITH_EGRESS,
      },
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});

Object.entries(api.integrations).forEach(([operation, integration]) => {
  db.allowDefaultPortFrom(
    integration.handler,
    `Allow ${operation} to connect to the database`,
  );
  db.grantConnect(integration.handler);
});
```

Điều này cấu hình mỗi API Lambda function để chạy trong cùng VPC, đọc runtime config từ AppConfig và kết nối với cơ sở dữ liệu sử dụng xác thực IAM.
</Fragment>
<Fragment slot="terraform">
Khi kết nối Lambda function với cơ sở dữ liệu, sử dụng các output của module này:

- `appconfig_application_id` - expose cho môi trường compute
- `cluster_port` - cho phép truy cập mạng
- `database_runtime_user` - cấp quyền `rds-db:connect`

Đảm bảo compute chạy trong cùng VPC và private subnet, và cấp quyền đọc AppConfig cho compute role.
</Fragment>
</Infrastructure>

### RDS Proxy

Cơ sở hạ tầng được tạo ra bao gồm RDS Proxy theo mặc định, nằm giữa ứng dụng của bạn và Aurora cluster. RDS Proxy cung cấp nhiều lợi ích:

- **Connection pooling** - Duy trì một pool các kết nối cơ sở dữ liệu có thể được chia sẻ giữa các instance ứng dụng, giảm overhead của việc thiết lập kết nối mới
- **Connection resilience** - Tự động xử lý failover và kết nối lại trong quá trình thay thế Aurora instance hoặc bảo trì
- **IAM authentication** - Hỗ trợ xác thực cơ sở dữ liệu dựa trên IAM, loại bỏ nhu cầu quản lý credentials cơ sở dữ liệu trong mã ứng dụng của bạn
- **Improved security** - Thực thi mã hóa TLS cho tất cả các kết nối

<Infrastructure>
<Fragment slot="cdk">

Theo mặc định, RDS Proxy được bật. Runtime client được tạo ra (`getPrisma()`) tự động kết nối thông qua proxy endpoint. Bạn có thể tắt nó nếu cần:

```ts title="packages/infra/src/stacks/application-stack.ts"
const db = new Postgres(this, 'Db', {
  ...
  enableRdsProxy: false,
});
```

Khi RDS Proxy bị tắt, ứng dụng của bạn kết nối trực tiếp với Aurora cluster endpoint.
</Fragment>
<Fragment slot="terraform">

Theo mặc định, RDS Proxy được bật. Runtime client được tạo ra (`getPrisma()`) tự động kết nối thông qua proxy endpoint. Bạn có thể tắt nó nếu cần:

```hcl title="packages/infra/src/main.tf"
module "postgres" {
  source = "../../common/terraform/src/app/dbs/postgres"
  ...
  enable_rds_proxy = false
}
```

Khi RDS Proxy bị tắt, ứng dụng của bạn kết nối trực tiếp với Aurora cluster endpoint. Proxy được cấu hình để yêu cầu mã hóa TLS và sử dụng xác thực IAM.
</Fragment>
</Infrastructure>

## Tùy chỉnh Kiến trúc Cơ sở dữ liệu

Cơ sở hạ tầng được tạo ra có thể được tùy chỉnh để phù hợp với yêu cầu workload của bạn. Các ví dụ sau đây minh họa một số tùy chọn tùy chỉnh phổ biến có sẵn.

### Cluster Instance

Cấu hình writer và reader instance cho Aurora cluster của bạn.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"

  ...
  writer: ClusterInstance.serverlessV2('writer'),
  readers: [ClusterInstance.serverlessV2('reader')],
```

</Fragment>
<Fragment slot="terraform">


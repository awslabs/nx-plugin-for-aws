Các workspace mới được tạo bằng `@aws/nx-plugin` bao gồm [git-secrets](https://github.com/awslabs/git-secrets) pre-commit hooks được cấu hình mặc định. Điều này quét các tệp đã staged để tìm các mẫu khớp với thông tin xác thực AWS trước mỗi commit, ngăn chặn việc rò rỉ thông tin xác thực một cách vô tình.

## Những gì được bao gồm

Khi bạn tạo một workspace mới, những thứ sau đây được thiết lập tự động:

<FileTree>
  - .git-secrets/
    - git-secrets Script bash git-secrets (được nhúng từ awslabs/git-secrets)
    - setup.sh Đăng ký các mẫu thông tin xác thực AWS trong cấu hình git cục bộ
  - .husky/
    - pre-commit Chạy quét git-secrets trước mỗi commit
  - package.json Bao gồm husky và script prepare để thiết lập tự động
</FileTree>

## Cách hoạt động

1. Script `prepare` chạy sau `npm install` / `pnpm install`, khởi tạo các git hooks của husky và đăng ký các mẫu thông tin xác thực AWS trong cấu hình git cục bộ của bạn.
2. Ở mỗi lần `git commit`, husky pre-commit hook gọi `git-secrets` để quét các tệp đã staged.
3. Nếu phát hiện một mẫu khớp với thông tin xác thực AWS, commit sẽ bị chặn với một thông báo lỗi.

## Các mẫu được phát hiện

Các mẫu sau được quét:

- **AWS Access Key IDs** — `AKIA...`, `ASIA...`, và các tiền tố khóa IAM khác theo sau bởi 16 ký tự chữ và số viết hoa
- **AWS Secret Access Keys** — Chuỗi base64 40 ký tự đứng trước bởi nhãn `SECRET_ACCESS_KEY`
- **AWS Account IDs** — Số 12 chữ số đứng trước bởi nhãn `ACCOUNT_ID`
- **Bedrock API Keys** — Khóa Bedrock tồn tại lâu dài (`ABSK...`) và ngắn hạn
- **Giá trị tệp Credentials** — Giá trị khóa theo nghĩa đen được tìm thấy trong `~/.aws/credentials`

Các khóa ví dụ nổi tiếng (ví dụ: `AKIAIOSFODNN7EXAMPLE`) được tự động đưa vào danh sách cho phép.

## Cho phép False Positives

Nếu git-secrets chặn một commit có chứa false positive:

```sh
# Cho phép một mẫu cụ thể
git config --add secrets.allowed 'my-pattern-here'
```

Hoặc thêm các mẫu vào tệp `.gitallowed` ở thư mục gốc của repository (một regex mỗi dòng).

Để ghi đè một lần:

```sh
git commit --no-verify
```

## Từ chối sử dụng

Để tạo một workspace không có git-secrets:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

Hoặc truyền `--gitSecrets=false` cho preset generator.

## Yêu cầu

- **Git** phải được khởi tạo trong workspace (`git init`)
- **Bash** phải có sẵn trên PATH (đi kèm với Git for Windows trên Windows)
- Các mẫu được đăng ký khi `prepare` chạy — nếu bạn clone một bản sao mới, hãy chạy lệnh install của package manager để đăng ký lại

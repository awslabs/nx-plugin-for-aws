## Phát triển cục bộ

### Khởi động DynamoDB cục bộ

Generator cấu hình một target `serve-local` để khởi động một phiên bản [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html) và tạo bảng:

<NxCommands commands={['run <project-name>:serve-local']} />

Điều này tự động:
1. Tải image DynamoDB Local (target `pull-image`)
2. Khởi động một container
3. Tạo một bảng cục bộ với các chỉ mục được định nghĩa trong `dynamodb.config.json`

### Mô hình hóa dữ liệu

Dự án được tạo sử dụng [ElectroDB](https://electrodb.dev/) để mô hình hóa thực thể an toàn kiểu trên một bảng DynamoDB đơn. Thêm hoặc cập nhật các tệp thực thể trong `src/entities/`, sử dụng thực thể mẫu được tạo làm điểm khởi đầu.

Ví dụ định nghĩa thực thể:

```ts title="packages/my-table/src/entities/example.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createExampleEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'example',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: {
          type: 'string',
          required: true,
        },
        createdAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: {
            field: 'pk',
            composite: ['id'],
          },
          sk: {
            field: 'sk',
            composite: [],
          },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

Để biết thêm chi tiết, xem [tài liệu thực thể ElectroDB](https://electrodb.dev/en/modeling/entities/).

### Sử dụng DynamoDB Client

Tệp `src/client.ts` được tạo xuất hai tiện ích chính:

- `getDynamoDBClient()` — trả về một singleton `DynamoDBClient` được lưu trong bộ nhớ cache. Khi `SERVE_LOCAL=true`, kết nối đến phiên bản DynamoDB Local cục bộ; nếu không, tạo một client AWS sử dụng chuỗi thông tin xác thực mặc định.
- `resolveTableName()` — trả về tên bảng DynamoDB. Khi `SERVE_LOCAL=true`, trả về hằng số tên bảng cục bộ; nếu không, lấy tên từ AWS AppConfig sử dụng biến môi trường `RUNTIME_CONFIG_APP_ID` và lưu vào bộ nhớ cache cho các lần gọi tiếp theo.

### Dừng DynamoDB cục bộ

Dừng `serve-local` (ví dụ: với `Ctrl+C`) sẽ tự động xóa container DynamoDB Local, nhưng giữ lại volume có tên để dữ liệu của bạn được duy trì qua các lần khởi động lại.

:::caution[Windows]
Do hạn chế với xử lý tín hiệu trên Windows, container không được tự động xóa khi `serve-local` bị dừng. Bạn sẽ cần xóa nó thủ công:

```bash
<engine> rm -f <scope>-dynamodb
```

Thay thế `<engine>` bằng công cụ container của bạn (`docker` hoặc `finch`) và `<scope>` bằng scope workspace Nx của bạn (ví dụ: `proj`).
:::

## Thêm/Xóa Global Secondary Indexes

GSI được định nghĩa trong `dynamodb.config.json` tại thư mục gốc của dự án. Thêm một mục vào `globalSecondaryIndexes` cho mỗi GSI, tuân theo quy ước đặt tên [thiết kế bảng đơn](https://electrodb.dev/en/core-concepts/single-table-relationships/) cho các khóa GSI:

```json title="packages/my-table/dynamodb.config.json"
{
  "globalSecondaryIndexes": [
    {
      "indexName": "gsi1pk-gsi1sk-index",
      "partitionKey": "gsi1pk",
      "sortKey": "gsi1sk"
    }
  ]
}
```

Trường `sortKey` là tùy chọn cho các GSI chỉ có khóa băm.

Tệp cấu hình này được đọc trực tiếp từ thư mục gốc của dự án bởi tất cả các consumer:
- **Phát triển cục bộ** — `serve-local` đọc `dynamodb.config.json` và tạo hoặc cập nhật bảng cục bộ để khớp với danh sách GSI
- **CDK** — construct đọc `dynamodb.config.json` tại thời điểm synth, vì vậy các thay đổi GSI được phản ánh trong lần `cdk deploy` tiếp theo
- **Terraform** — module đọc `dynamodb.config.json` tại thời điểm plan/apply

### Một GSI mỗi lần triển khai

:::caution
DynamoDB [không cho phép tạo hoặc xóa nhiều hơn một GSI trong một lần cập nhật bảng](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/GSI.OnlineOps.html).

Mỗi lần triển khai có thể thêm hoặc xóa **tối đa một GSI**. Nếu bạn cần thêm hoặc xóa nhiều GSI, hãy thực hiện từng cái một — cập nhật `dynamodb.config.json`, triển khai stack, sau đó lặp lại cho thay đổi tiếp theo.
:::

## Kết nối đến bảng

Trong bất kỳ dự án TypeScript nào, import các factory thực thể từ package DynamoDB của bạn và sử dụng chúng trực tiếp:

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

:::note[Runtime config]
Khi chạy trong AWS, `resolveTableName()` lấy tên bảng từ AWS AppConfig sử dụng biến môi trường `RUNTIME_CONFIG_APP_ID`. Các dự án được xây dựng với plugin này (tRPC APIs, Smithy APIs, agents, MCP servers) đã có biến này được cấu hình tự động. Đối với các dự án TypeScript khác, đảm bảo `RUNTIME_CONFIG_APP_ID` được đặt trong môi trường runtime với ID ứng dụng AppConfig được cung cấp bởi cơ sở hạ tầng của bạn. Để biết thêm thông tin, xem <Link path="guides/runtime-config">hướng dẫn Runtime Configuration</Link>.
:::

- `pk` (partition key) và `sk` (sort key), cả hai đều là kiểu `String`

- Thanh toán theo yêu cầu (`PAY_PER_REQUEST`)

#### Tắt bảo vệ xóa

Tắt nó cho các môi trường mà việc xóa bảng được mong đợi, chẳng hạn như các stack phát triển hoặc xem trước ngắn hạn.

### Chế độ thanh toán

Bảng mặc định sử dụng thanh toán theo yêu cầu (`PAY_PER_REQUEST`). Chuyển sang dung lượng được cung cấp cho các khối lượng công việc có thông lượng cao, có thể dự đoán.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { BillingMode } from 'aws-cdk-lib/aws-dynamodb';
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  billingMode: BillingMode.PROVISIONED,
  readCapacity: 5,
  writeCapacity: 5,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source       = "../../common/terraform/src/app/dynamodb/my-table"
```

</Fragment>

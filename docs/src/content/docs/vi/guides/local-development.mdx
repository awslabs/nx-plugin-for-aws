Các dự án được kết nối cung cấp hai target để chạy chúng trên máy của bạn: `serve` và `serve-local`. Sự khác biệt là về **phạm vi** — bao nhiêu phần của ứng dụng chạy cục bộ so với việc trỏ đến cơ sở hạ tầng AWS đã triển khai.

Xem xét một workspace với các kết nối sau: một website gọi một `api` tRPC, và cũng gọi một `agent` mà lần lượt gọi một máy chủ `mcp`.

## `serve`

Target `serve` chỉ chạy **dự án được chỉ định** cục bộ. Mọi dự án khác mà nó kết nối đến được mong đợi là **đã triển khai**, và được truy cập tại URL AWS thực của chúng.

<NxCommands commands={['serve website']} />

Bởi vì dự án được chỉ định giao tiếp với các tài nguyên đã triển khai, nó cần biết chúng ở đâu. Đối với một website, điều này được cung cấp bởi `runtime-config.json` của nó (xem <Link path="guides/react-website#local-runtime-config">Local Runtime Config</Link>), mà bạn tải từ một ứng dụng đã triển khai. Đối với các dự án phía máy chủ (API và agent) đọc <Link path="guides/runtime-config">Runtime Configuration</Link>, đặt biến môi trường `RUNTIME_CONFIG_APP_ID` để trỏ đến ứng dụng AppConfig đã triển khai của bạn.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
}
deployed: Deployed {
  style.stroke-dash: 3
  api: api
  agent: agent
  mcp: mcp
}

local.website -> deployed.api
local.website -> deployed.agent
deployed.agent -> deployed.mcp
```

Sử dụng `serve` khi bạn muốn lặp lại trên một dự án duy nhất so với các phiên bản "thực", đã triển khai của mọi thứ mà nó phụ thuộc vào.

## `serve-local`

Target `serve-local` chạy **dự án được chỉ định** và **mọi dự án được kết nối với nó một cách bắc cầu**, tất cả trên máy của bạn. Trình tạo kết nối tự động thiết lập điều này — chạy `serve-local` trên website cũng khởi động các máy chủ cục bộ cho `api`, `agent`, và máy chủ `mcp` mà nó truy cập thông qua agent.

<NxCommands commands={['serve-local website']} />

Khi chạy theo cách này, `runtime-config.json` của website được tự động ghi đè (thông qua [Vite's `MODE`](https://vite.dev/guide/env-and-mode), được đặt thành `serve-local`) để nó trỏ đến các máy chủ đang chạy cục bộ của bạn thay vì các URL đã triển khai.

```d2
direction: right

local: Local {
  style.stroke-dash: 3
  website: website
  api: api
  agent: agent
  mcp: mcp

  website -> api
  website -> agent
  agent -> mcp
}
```

Mọi dự án đều chạy cục bộ, vì vậy không có phụ thuộc đã triển khai.

Sử dụng `serve-local` khi bạn đang làm việc trên nhiều dự án được kết nối cùng một lúc và muốn lặp lại nhanh chóng mà không cần triển khai cơ sở hạ tầng của bạn.

:::tip[`dev` shortcut]
Một script `dev` gốc được thêm vào như một lối tắt cho website đầu tiên mà bạn tạo trong workspace của mình, vì vậy bạn có thể khởi động website và tất cả các thành phần được kết nối với:

<PackageManagerShortCommand commands={["dev"]} />
:::

Để biết thêm chi tiết về máy chủ phát triển cục bộ của từng loại dự án, xem hướng dẫn liên quan — ví dụ như hướng dẫn <Link path="guides/react-website#local-development-server">React Website</Link>.

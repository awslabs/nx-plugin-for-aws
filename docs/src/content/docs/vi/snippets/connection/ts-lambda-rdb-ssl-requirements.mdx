---
title: Yêu cầu SSL khi Kết nối không sử dụng RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

Đối với các kết nối trực tiếp đến Aurora cluster từ Node.js 20 hoặc các Lambda runtime mới hơn, hãy tải gói Amazon RDS CA bằng cách thiết lập `NODE_EXTRA_CA_CERTS`:

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "api" {
  source = "..."
  ...

  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/var/runtime/ca-cert.pem"
  }
}
```

</Fragment>
</Infrastructure>

Để biết thêm chi tiết, xem tài liệu AWS Lambda về [yêu cầu SSL/TLS cho kết nối Amazon RDS](https://docs.aws.amazon.com/lambda/latest/dg/services-rds.html#services-rds-tls) và tài liệu [TLS của Amazon RDS Proxy](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-proxy.howitworks.html#rds-proxy-security.tls). Khi sử dụng RDS Proxy, bạn không cần cấu hình gói RDS CA trong Lambda function của mình.

---
title: Cảnh báo Serve Local trên Windows
---

Dừng `serve-local` (ví dụ: bằng `Ctrl+C`) sẽ tự động xóa container DynamoDB Local, nhưng vẫn giữ lại named volume để dữ liệu của bạn được bảo toàn qua các lần khởi động lại.

:::caution[Windows]
Do hạn chế trong việc xử lý tín hiệu trên Windows, container sẽ không tự động bị xóa khi `serve-local` dừng lại. Bạn sẽ cần phải xóa nó thủ công:

```bash
<engine> rm -f <scope>-dynamodb
```

Thay thế `<engine>` bằng container engine của bạn (`docker` hoặc `finch`) và `<scope>` bằng scope của Nx workspace (ví dụ: `proj`).
:::

`@aws/nx-plugin`は[Nx](https://nx.dev/)プラグインで、AWS上でフルスタックアプリケーションを構築・デプロイするためのツールキットを提供します。本番環境対応のアプリケーションコード_と_それをデプロイするためのインフラストラクチャをスキャフォールディングする[Generators](#generators)のコレクションを提供し、タイプセーフでローカル実行可能、すぐにデプロイ可能な状態を実現します。

白紙の状態から始めるのではなく、あなた（またはあなたのAIエージェント）が必要なコンポーネント（API、ウェブサイト、認証、AIエージェント、インフラストラクチャなど）を選択し、いくつかの設定オプションを提供すると、プラグインがベストプラクティスに基づいたスターターコードを生成します。さらに、プロジェクト間を自動的に接続し（既存ファイルのAST変換による更新を含む）、フロントエンドとバックエンド間でタイプセーフなクライアントを生成します。

## Nxとそのジェネレータを基盤として構築

[Nx](https://nx.dev)はモノレポを管理するためのスマートビルドフレームワークです。言語に依存せず、TypeScript、Python、インフラストラクチャなどを単一のワークスペース内で共存させ、一緒にビルドできます。ビルドシステムはキャッシュと依存関係グラフ分析を使用して、変更された部分のみを再ビルドし、タスクを並列実行することで、ワークスペースが成長してもテスト、リント、ビルドを高速に保ちます。

`@aws/nx-plugin`のすべての機能は、Nxの[Generator](https://nx.dev/features/generate-code)として提供されます。これは、事前定義されたテンプレートから一貫したコード構造を生成する自動スキャフォールディングです。ジェネレータは[Nx CLI](https://nx.dev/features/generate-code#invoke-generators)、[Nx Console IDEプラグイン](https://nx.dev/getting-started/editor-setup)、または<Link path="/get_started/building-with-ai">Nx Plugin for AWS MCP Server</Link>を通じてAIアシスタントから呼び出すことができます。

すべてのコンポーネントがジェネレータであるため、必要なものを必要なときにのみ追加できます。APIから始めて、後でウェブサイトを追加し、準備ができたら接続する、各ステップは単一のコマンドです。

## 実績のあるフレームワークで構築

`@aws/nx-plugin`は、すでに知っているツールの上にカスタム抽象化を構築することを意図的に避けています。独自のフレームワーク、ランタイム、ラッパーを発明する代わりに、確立された広く採用されているオープンソースプロジェクトを使用して慣用的なコードをスキャフォールディングします。これは、既存のスキルが直接活用でき、各フレームワークの公式ドキュメントに依存でき、このプラグインだけが理解するレイヤーにロックインされることがないことを意味します。

## 自由な修正が可能

生成されたコードはすべて_あなたの_コードです。`@aws/nx-plugin`はビルド時のツールであり、ランタイムの依存関係ではありません。ジェネレータが実行された後は、プラグインは関与しません。生成されたすべてのファイルを読み、コミットし、好きなように変更できます。

これは以下を意味します：

- **エスケープハッチは不要。** ジェネレータが想定していないことを行う必要がある場合に、学習したり格闘したりする独自の設定サーフェスはありません。自分で書いたコードを編集するのと同じように、生成されたコードを直接編集します。
- **新しい抽象化レイヤーはなし。** ジェネレータは、基盤となるフレームワークのための平易で慣用的なコードを出力します。React、tRPC、FastAPI、CDK、その他のツールとあなたの間に位置するラッパーAPIやマジックな間接層はありません。見たものがそのまま実行されます。
- **プラグインは依存関係ではない。** コードを生成して退きます。アプリケーションは実行時に`@aws/nx-plugin`をインポートしたり依存したりしないため、プラグインによってブロックされることもロックインされることもありません。

生成されたコードに不満がある場合は、自由に修正できます。ジェネレータは出発点を提供するものであり、檻ではありません。

## 最小限の依存関係

`@aws/nx-plugin`はグローバル依存関係の数を最小限に抑えるよう努めています。開始するために必要なものは、呼び出すジェネレータによって決まります。

例として、TypeScriptベースのジェネレータでは[Node](https://nodejs.org/en/download)のインストールのみが必要です。Pythonベースのプロジェクトでは[UV](https://docs.astral.sh/uv/)が唯一の要件となります。

## タイプセーフティ

`@aws/nx-plugin`はタイプセーフティを採用し、IDE補完による開発者体験の向上と、非タイプセーフ実装でのみ発生するランタイムエラーの排除を実現します。そのため、提供されるすべてのコンポーネントはデフォルトでタイプセーフです。

タイプセーフティはプロジェクトの境界を越えて流れます。ウェブサイトをAPIに接続すると、生成されたクライアントはバックエンドと型を共有するため、APIコントラクトの変更は本番環境でのバグではなく、フロントエンドでのコンパイル時エラーとして表面化します。自信を持ってリファクタリングできます。

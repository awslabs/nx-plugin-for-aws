## タスク1: ゲームAPIの実装

このセクションでは以下のAPIを実装します:

1. `saveGame` - ゲームの作成または更新
2. `queryGames` - 保存済みゲームのページネーション付きリストを返す
3. `queryInventory` - プレイヤーのインベントリ内アイテムのページネーション付きリストを返す
4. `queryActions` - 指定されたゲームの会話履歴を返す

:::note[各状態データの保存場所]
- `games`と`inventory`は**DynamoDB**（ElectroDB経由）に保存 — Game API lambdaとInventory MCPサーバーがこれらを読み書きします。
- `actions`（チャット記録）は**S3**に保存 — Story Agentの`S3SessionManager`がターンごとに1つのJSONブロブを書き込み、Game APIの`queryActions`がそれを読み取ることで、UIがページリロード時に履歴を再生できます。`saveAction`プロシージャは存在せず、エージェントのみが書き込みを行います。
:::

これがアプリケーションのER図です。


`DungeonDb`コンストラクトは、デプロイされたテーブル名を<Link path="guides/runtime-config">ランタイム設定</Link>の`dynamodb`名前空間に登録し、AWS AppConfigに保存します。Game APIとInventory MCPサーバーは、環境変数として渡す必要なく、Lambda Powertoolsを使用して実行時にこれを取得します。このアプローチの利点は、テーブルや他のインフラストラクチャ識別子の数が増えても、デプロイされたインフラストラクチャに対して実行する際に必要な環境変数は`RUNTIME_CONFIG_APP_ID`の1つだけで済むことです。
:::

## タスク4: Game APIをローカルでテスト

AWSにデプロイせずにAPIを試すことができます。`serve-local`ターゲットは[DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html)に対してGame APIを実行します。モジュール1でGame APIを`DungeonDb`プロジェクトに接続したため、このターゲットはDynamoDB Localも自動的に起動します。

まず、リンターの問題を修正します:

次にコードベースをビルドします:

<PackageManagerShortCommand commands={["build"]} />

### ローカルサーバーの起動

`dev`ターゲットでGame APIをローカルで起動します。これによりDynamoDB Localも起動されます:

<NxCommands commands={["dev game-api"]} />

<Aside type="tip" title="初回実行">
初回実行時はDynamoDB Localコンテナイメージをプルするため、少し時間がかかる場合があります。このターミナルでサーバーを実行したまま、以下の`curl`コマンド用に2つ目のターミナルを開いてください。
</Aside>

### APIのテスト

サーバーが起動したら、（空の）ゲームリストをクエリします:

```bash
curl -X GET 'http://localhost:2022/games.query?input=%7B%7D'
```

:::note[URLエンコーディング]
APIテストで渡す`%7B%7D`はURIエンコードされた空のJSONオブジェクト（`{}`）です。
:::

空のリストが表示されます:

```json
{"result":{"data":{"items":[],"cursor":null}}}
```

次にゲームを保存します:

```bash
curl -X POST 'http://localhost:2022/games.save' \
  -H 'Content-Type: application/json' \
  -d '{"playerName":"Alice","genre":"zombie"}'
```

保存は永続化されたゲームを返します（エンティティが設定する`lastUpdated`タイムスタンプ付き）:

```json
{"result":{"data":{"playerName":"Alice","genre":"zombie","lastUpdated":"..."}}}
```

再度クエリしてDynamoDB Localに永続化されていることを確認します:

```bash
curl -X GET 'http://localhost:2022/games.query?input=%7B%7D'
```

このレスポンスには保存されたゲームが含まれます:

```json
{"result":{"data":{"items":[{"playerName":"Alice","genre":"zombie","lastUpdated":"..."}],"cursor":null}}}
```

完了したらローカルサーバーを停止できます（`Ctrl+C`）。

## タスク5: Inventory MCPサーバーをローカルでテスト

生成された`mcp-server-inspect`ターゲットを使用して、[MCP Inspector](https://github.com/modelcontextprotocol/inspector)でMCPサーバーのツールを試すことができます:

<NxCommands commands={["mcp-server-inspect inventory"]} />

これによりMCPサーバーがローカルで提供され（DynamoDB Localも起動されます）、MCP Inspectorが`http://localhost:6274`で起動し、接続するように事前設定されます。**Connect**をクリックし、**Tools**タブに切り替え、**List Tools**をクリックし、`add-to-inventory`（例: `playerName: Alice`、`itemName: Rusty Sword`、`emoji: ⚔️`）を試した後、`list-inventory-items`でDynamoDB Localに永続化されていることを確認します。完了したらサーバーを停止します（`Ctrl+C`）。

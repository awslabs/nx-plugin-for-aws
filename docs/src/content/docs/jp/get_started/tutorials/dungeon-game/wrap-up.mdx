## まとめ

おめでとうございます！`@aws/nx-plugin` に含まれる様々なジェネレータを活用したエージェンティックAIダンジョンアドベンチャーゲームを作成できました。🎉🎉🎉

## 次は何をしますか？

以下の機能拡張に挑戦して、コードベースをさらに発展させてみましょう：

1. <Link path="guides/license">ライセンスジェネレータ</Link>をプロジェクトに追加し、ワークスペース内のLICENSEファイルとソースコードヘッダーを自動管理
2. <Link path="get_started/tutorials/contribute-generator">独自のジェネレータを開発</Link>して、新しいtRPC APIの追加を簡素化
3. [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html)をStory Agentと統合し、長時間実行される冒険がサーバー再起動後も継続できるようにします。`ag_ui_strands`アダプターは、まさにこの目的のために`session_manager_provider`フックを公開しています。詳細は[using AgentCore Memory with Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands)のドキュメントを参照してください。
4. 左側のナビゲーションを使用して、様々なコンポーネントガイドを探索

## リソースの削除

1. 作成したAWSリソースを削除するには、次のコマンドを実行します：

<NxCommands commands={['destroy infra "dungeon-adventure-infra-sandbox/*"']} />

    このコマンドは削除するスタックのリスト（`dungeon-adventure-infra-sandbox/Application/GameUI/waf` と `dungeon-adventure-infra-sandbox/Application`）を確認するプロンプトを表示します。

2. `Y`を入力して続行します。CloudFormationがスタックの削除を開始します。

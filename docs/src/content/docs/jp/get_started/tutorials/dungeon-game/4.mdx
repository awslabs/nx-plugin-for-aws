## モジュール4: UIの実装

UIの構築を開始するには、ローカル開発サーバーをデプロイされたサンドボックス環境に向ける設定が必要です。以下のコマンドを実行してください:

<NxCommands commands={["run @dungeon-adventure/game-ui:load:runtime-config"]} />

このコマンドはデプロイ済みの`runtime-config.json`を取得し、`packages/game-ui/public`フォルダ内にローカル保存します。

次に開発サーバーを起動します:

<NxCommands commands={["run @dungeon-adventure/game-ui:serve"]} />

ブラウザで`http://localhost:4200`を開くと、ログインが促され、新規ユーザー作成のプロンプトが表示されます。完了するとベースラインのウェブサイトが表示されます:

<Image src={baselineWebsitePng} alt="ベースラインウェブサイト" width="800" height="600" />

<Aside type="caution">
_本モジュールの残りの作業中は開発サーバーを起動したままにしてください。変更は自動的にホットリロードされます_
</Aside>

:::tip
<NxCommands commands={["run @dungeon-adventure/game-ui:serve-local"]} />を代わりに実行すると、接続されたAPIローカルサーバーも同時に起動し、tRPC APIとウェブサイトの同時開発が可能になります。
:::

### 新しい'/game'ルートの作成

`@tanstack/react-router`の機能を活用するため、型安全な新規ルートを作成します。`packages/game-ui/src/routes/game/index.tsx`に空ファイルを作成するだけで、自動的にファイルが更新されます。

`@tanstack/react-router`が自動的に新規ルートを設定し、作成したファイルには既にルートパスが設定されています:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

`http://localhost:4200/game`にアクセスすると、新しいページが表示されます!

<Image src={baselineGamePng} alt="ベースラインゲーム画面" width="800" height="600" />

### ストーリーエージェントの統合

ストーリーエージェントとの連携用クライアントを初期化するフックを作成します。

<Tabs>
<TabItem label="hooks/useStoryAgent.tsx">
<E2ECode path="dungeon-adventure/4/hooks/useStoryAgent.tsx.template" lang="tsx" />
</TabItem>
</Tabs>

この処理では以下を実行します:

- `runtime-config.json`からAgent ARNを取得
- ARNからAgentCore Runtimeの呼び出しURLを構築
- ログインユーザーのJWTトークンとランダムセッションIDでAgentを呼び出し
- ストリーミングメッセージ用の非同期イテレータを返却

:::tip
`api-connection`ジェネレータを使用したため、Game APIとの統合は既に設定済みです。同様のエージェント体験を希望される場合は[GitHubイシュー](https://github.com/awslabs/nx-plugin-for-aws/issues/326)へ+1をお願いします。
:::

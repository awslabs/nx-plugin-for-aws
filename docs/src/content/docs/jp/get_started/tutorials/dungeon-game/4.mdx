## タスク1: ローカル開発サーバーの設定

UIの構築を開始するには、ローカル開発サーバーをデプロイされたサンドボックス環境に向ける設定が必要です。以下のコマンドを実行してください:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

このコマンドはデプロイ済みの`runtime-config.json`を取得し、`packages/game-ui/public`フォルダ内にローカル保存します。

開発サーバーを起動するには、以下のコマンドを実行します:

<NxCommands commands={["serve game-ui"]} />

ブラウザでローカルウェブサイトを開きます。先ほど作成したCognitoユーザーでログインするよう促されます。ログインすると、ベースラインのウェブサイトが表示されます。

<Aside type="caution" title="開発サーバーを起動したまま">
本モジュールの残りの作業中は開発サーバーを起動したままにすることをお勧めします。変更は自動的にホットリロードされます。
</Aside>

## タスク4: ゲームページの作成

<Link path="get_started/tutorials/dungeon-game/1">モジュール1</Link>で、`GameUI`をStory Agentに接続しました。エージェントが`--protocol=AG-UI`で生成されたため、`connection`ジェネレータは以下を配線してくれました:

- アプリをラップする`<AguiProvider>`（[CopilotKit](https://copilotkit.ai/)の`CopilotKitProvider`）。すでに`main.tsx`に注入されています。
- デプロイされたAgentCoreランタイムを指すCognito認証済みAG-UI `HttpAgent`を構築する`useAguiStoryAgent`フック。
- `src/components/copilot`配下のshadcnテーマのチャットプリミティブセット - ウェブサイトの他の部分と一致する事前スタイル済みの`<CopilotChat>`。

これらを使用する2つのルートファイルを作成するだけです:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="パスパラメータ">
`$playerName`構文は`@tanstack/react-router`に`playerName`を[パスパラメータ](https://tanstack.com/router/v1/docs/framework/react/guide/path-params)として扱うよう指示します。プレイヤー名をAG-UIの`threadId`として使用することで、各ゲームが独自のチャット履歴を持ち、AgentCoreのセッションID長要件を満たすために33文字にパディングします。
</Aside>

<Aside type="note" title="最初のメッセージのシード">
最初のマウント時に`agent.messages.length === 0`をチェックし、空の場合は単一の開始ユーザーメッセージ`I am <player>, playing in the <genre> genre.`をプッシュします。システムプロンプトはエージェントにそのキューから即座にストーリーを開始するよう指示します。それ以降、すべてのインタラクション（ページの更新や後で戻ってくることを含む）は、AG-UIセッション経由で会話をリロードするだけです。カスタム履歴エンドポイントは不要です。
</Aside>
</TabItem>
</Tabs>

これらのファイルを保存すると、ローカル開発サーバー(http://localhost:4200/)でゲームをプレイできるようになります。

## モジュール4: UIの実装

UIの構築を開始するには、ローカル開発サーバーをデプロイされたサンドボックス環境に接続するよう設定します。次のコマンドを実行してください:

<NxCommands commands={["run @dungeon-adventure/game-ui:load:runtime-config"]} />

このコマンドはデプロイ済みの`runtime-config.json`を取得し、`packages/game-ui/public`フォルダ内にローカル保存します。

次に開発サーバーを起動します:

<NxCommands commands={["run @dungeon-adventure/game-ui:serve"]} />

ブラウザで`http://localhost:4200`を開くと、ログインが求められ、新規ユーザー作成のプロンプトが表示されます。完了するとベースラインのウェブサイトが表示されます:

<Image src={baselineWebsitePng} alt="ベースラインウェブサイトの画像" width="800" height="600" />

<Aside type="caution">
_本モジュールの残りの作業中は開発サーバーを起動したままにしてください。変更が自動的にホットリロードされます_
</Aside>

:::tip
<NxCommands commands={["run @dungeon-adventure/game-ui:serve-local"]} />を代わりに実行すると、接続されたAPIローカルサーバーも同時に起動します。これによりtRPC APIとウェブサイトの同時開発が可能になります
:::

### 新しい'/game'ルートの作成

`@tanstack/react-router`の機能を活用して型安全な新規ルートを作成します。次の場所に空ファイルを作成してください: `packages/game-ui/src/routes/game/index.tsx`。ファイルが自動更新されるのを確認できます。

`@tanstack/react-router`が自動的に新規ルートを設定し、作成したファイルにルートパスが既に記述されています:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

`http://localhost:4200/game`にアクセスすると新しいページが表示されます!

<Image src={baselineGamePng} alt="ベースラインゲーム画面の画像" width="800" height="600" />

### ストーリーエージェントの統合

ストーリーエージェントとの連携用クライアントを初期化するフックを作成します。

<Tabs>
<TabItem label="hooks/useStoryAgent.tsx">
<E2ECode path="dungeon-adventure/4/hooks/useStoryAgent.tsx.template" lang="tsx" />
</TabItem>
</Tabs>

このコードは以下を実行します:

- `runtime-config.json`からAgent ARNを取得
- ARNからAgentCore Runtimeの呼び出しURLを構築
- ログインユーザーのJWTトークンとランダムセッションIDでAgentを呼び出し
- ストリーミングメッセージの非同期イテレータを返却

:::tip
`api-connection`ジェネレータを使用したため、Game APIとの統合は既に設定済みです。エージェント向けにも同様の機能を希望する場合は[GitHubイシュー](https://github.com/awslabs/nx-plugin-for-aws/issues/326)で+1してください
:::

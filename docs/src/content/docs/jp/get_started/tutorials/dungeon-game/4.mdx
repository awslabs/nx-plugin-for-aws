## モジュール4: UI実装

UIの構築を開始するには、ローカル開発サーバーをデプロイされたサンドボックス環境に接続するよう設定します。次のコマンドを実行してください:

<NxCommands commands={["run @dungeon-adventure/game-ui:load:runtime-config"]} />

このコマンドはデプロイ済みの`runtime-config.json`を取得し、`packages/game-ui/public`フォルダ内にローカル保存します。

次に開発サーバーを起動します:

<NxCommands commands={["run @dungeon-adventure/game-ui:serve"]} />

ブラウザで`http://localhost:4200`を開くと、ログインが促され、新規ユーザー作成のプロンプトが表示されます。完了するとベースラインWebサイトが表示されます:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution">
_このモジュールの残りの部分では、開発サーバーを実行したままにします。変更を加えると自動的にホットリロードされます。_
</Aside>

:::tip
<NxCommands commands={["run @dungeon-adventure/game-ui:serve-local"]} />を代わりに実行すると、接続されたAPIローカルサーバーも起動し、tRPC APIとWebサイトの同時開発が可能になります。
:::

### 新しい'/game'ルートの作成

`@tanstack/react-router`の機能を活用するため、型安全な新規ルートを作成します。`packages/game-ui/src/routes/game/index.tsx`に空ファイルを作成してください。ファイルが自動更新されます。

`@tanstack/react-router`が自動的に新規ルートを設定し、作成したファイルにルートパスが追加されます:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

`http://localhost:4200/game`にアクセスすると新しいページが表示されます!

<Image src={baselineGamePng} alt="baseline-game.png" width="800" height="600" />

`index.tsx`ファイルを更新してデフォルトで`/game`ルートを読み込むように変更します。`to`フィールドを更新する際、型安全なルートのリストから選択できます。

### Story Agent連携

Story Agentと連携するクライアントを初期化するフックを作成します:

<Tabs>
<TabItem label="hooks/useStoryAgent.tsx">
<E2ECode path="dungeon-adventure/4/hooks/useStoryAgent.tsx.template" lang="tsx" />
</TabItem>
</Tabs>

このコードは以下を実行します:

- `runtime-config.json`からAgent ARNを取得
- ARNからAgentCore Runtime呼び出しURLを構築
- ログインユーザーのJWTトークンとランダムセッションIDでAgentを呼び出し
- ストリーミングメッセージ用の非同期イテレータを返却

:::tip
`api-connection`ジェネレータを使用したため、Game APIとの連携は自動設定されています。エージェントでも同様の体験を希望する場合は[GitHub Issue](https://github.com/awslabs/nx-plugin-for-aws/issues/326)で+1してください。
:::

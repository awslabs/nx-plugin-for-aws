## タスク1: ローカル開発サーバーの設定

デプロイされた`runtime-config.json`をローカルに取得して、開発サーバーが実際のバックエンドと通信できるようにします:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

次に開発サーバーを起動します:

<NxCommands commands={["serve game-ui"]} />

ブラウザでローカルウェブサイトを開き、ログインして、新規ユーザー作成のプロンプトに従います。ベースラインのShadcnウェブサイトが表示されます:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution" title="開発サーバーを起動したまま">
このモジュールの残りの作業中は開発サーバーを起動したままにしてください。以下のすべての変更がホットリロードされます。
</Aside>

## タスク2: CopilotKitが既に配線されている場所

<Link path="get_started/tutorials/dungeon-game/1">モジュール1</Link>で`game-ui → story`の`connection`ジェネレータを実行したとき、ShadcnウェブサイトのAG-UI統合が自動生成されました。簡単に確認してみましょう:

<FileTree>
- packages/game-ui/src/
  - components/
    - AguiProvider.tsx すべてのAG-UIエージェントに登録された単一の`CopilotKitProvider`。
    - copilot/
      - index.tsx テーマ化された`CopilotChat` / `CopilotSidebar` / `CopilotPopup`を再エクスポート。
      - ShadcnAssistantMessage.tsx, ShadcnUserMessage.tsx, ShadcnChatInput.tsx, ShadcnCursor.tsx, copilot.css
  - hooks/
    - useAguiStoryAgent.tsx デプロイされたStory Agentを指す`@ag-ui/client`の`HttpAgent`をインスタンス化し、`threadId`をAgentCoreの最小セッションID長である33文字にパディング。
  - main.tsx `<App />`を`<AguiProvider>`でラップ
</FileTree>

必要なのは、ルートに`<CopilotChat agentId="agent" threadId={...} />`を配置するだけです。統合の詳細については、<Link path="guides/connection/react-agui">React → AG-UI接続ガイド</Link>を参照してください。

## タスク3: ダンジョン用のスタイル変更

`packages/game-ui/src/styles.css`を置き換えます。これはスタイリングのために変更する*唯一*のファイルです。共有のShadcnグローバルをインポートし、パレットを松明で照らされたダンジョンテーマに上書きし、CopilotKitがそれらの色を継承するようにします（CopilotKitはチャット画面内で`--background`などをリセットします）:

## タスク4: ゲームルートの作成

2つのルートが必要です。1つはヒーローを選ぶため、もう1つはプレイするためです。どちらもshadcnコンポーネントとCopilotKitチャットを使用します。手作りのチャットUIはありません。

<Tabs>
<TabItem label="routes/index.tsx">
<E2EDiff before="dungeon-adventure/4/routes/index.tsx.old.template" after="dungeon-adventure/4/routes/index.tsx.template" lang="tsx" />

これはゲームピッカーです: 新規ゲームフォーム（shadcnの`Input` + `Button` + `Card`）と、`useGameApi().games.query`によって供給される「続きから」リスト。ゲームを開始すると`(playerName, genre)`ペアを`saveGame`し（次回表示されるように）、プレイルートに移動します。
</TabItem>
<TabItem label="routes/game/$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

これはプレイルートです。決定論的な`threadId`（``{player}-{genre}``を33文字にパディング — AG-UIフックはこれをそのままAgentCoreセッションIDとして送信）を構築し、`<CopilotChat agentId="agent" threadId={threadId} />`をレンダリングし、`useGameApi().inventory.query`からのインベントリを上にオーバーレイします。新しいスレッド（まだメッセージが保存されていない）では、ルートは`useAgent`を使用してプレイヤーの名前とジャンルを含む1つのプライミングメッセージを送信します。エージェントのシステムプロンプトは、それを使用してシーンを設定するように指示します。その後の訪問では、エージェントの`FileSessionManager`が履歴を再生するため、プライミング`useEffect`は何もしません。

<Aside type="tip" title="決定論的なthreadIdにより/game/$playerName?genre=...が共有可能に">
同じURLを再訪問すると — リロード後、再デプロイ後、または別のデバイスでも — 同じ`threadId`に解決されるため、エージェントは同じストーリーを語り続けます。
</Aside>

<Aside type="tip" title="パスパラメータ">
`$playerName`構文は`@tanstack/react-router`に`playerName`を[パスパラメータ](https://tanstack.com/router/v1/docs/framework/react/guide/path-params)として扱うよう指示します。`validateSearch`は`genre`を列挙型に対して厳密に型付けします。
</Aside>
</TabItem>
</Tabs>

スキャフォールドされた`packages/game-ui/src/routes/welcome`ディレクトリは削除できます。ゲームピッカーとして`/`を使用しています。

保存すると、`http://localhost:4200/`の開発サーバーで冒険を開始し、Story Agentとチャットできるようになります。

## タスク1: ストーリーエージェントの実装

ストーリーエージェントは、<Link path="get_started/tutorials/dungeon-game/1">モジュール1</Link>で`--protocol=AG-UI`を指定して生成された[Strands](https://strandsagents.com/)エージェントです。これにより、UIはCopilotKitを介して[Agent-User Interactionプロトコル](https://docs.copilotkit.ai/aws-strands/protocol)でストリーミングできます。インベントリMCPサーバーを使用してプレイヤーのアイテムを管理し、`FileSessionManager`を使用して会話履歴をディスクに永続化します。

:::note[会話履歴の永続化]
各AG-UIリクエストには`thread_id`が含まれます。UIは、このIDを``{player}-{genre}``という形式で決定論的に構築し、AgentCoreの最小長である33文字にパディングします。これにより、各`(player, genre)`ペアが独自のストーリーを再開できます。エージェント側では、`StrandsAgentConfig.session_manager_provider`が各スレッドにディスクディレクトリを指す`FileSessionManager`を提供します。これにより、UIが送信しなくても、後続のターンで保存されたメッセージ履歴が再生されます。本番環境では、[AgentCore Memory for Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands)を介して[Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html)に置き換えてください。
:::

変更内容は以下の通りです：

- `main.py`は、`thread_id`ごとに`FileSessionManager`を作成する`session_manager_provider`を追加します。これにより、同じ`(player, genre)`に対して会話履歴がターンをまたいで保持されます。
- `agent.py`は、サンプルの`subtract`ツールを削除し、システムプロンプトをダンジョンマスター用のものに置き換えます。このプロンプトは、最初のユーザーメッセージでプレイヤーの名前とジャンルを述べるよう促し、インベントリMCPサーバーのツールを使用します。

### エージェントのテスト

エージェントのテスト方法は2通りあります：
<ul>
<li>エージェントサーバーをローカルで起動し、生成された`agent-chat`ターゲットを介してチャットする、または</li>
<li>デプロイ済みAPIをJWTトークン付きのcurlで呼び出す。</li>
</ul>

<Tabs>
  <TabItem label="ローカル">
  ローカルエージェントサーバーを起動します：
    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    <Aside type="caution" title="CDK出力値の使用">
    `RUNTIME_CONFIG_APP_ID`環境変数にはCDKデプロイ出力の`RuntimeConfigApplicationId`の値を使用し、`AWS_REGION`にはデプロイ先のAWSリージョンを指定してください。
    </Aside>

    :::note[`agent-serve` vs `agent-serve-local`]
    `agent-serve`ターゲットはエージェントをローカルで起動しますが、**デプロイ済み**のMCPサーバーに接続します（`RUNTIME_CONFIG_APP_ID`経由）。MCPサーバーもローカルで実行したい場合は、代わりに`agent-serve-local`を使用してください。これにより、すべての依存サービスが起動され、直接HTTP接続用に`SERVE_LOCAL=true`が設定されます。
    :::

    次に、2つ目のターミナルで、生成された`agent-chat`ターゲットを介してローカルAG-UIエージェントに対する対話型REPLを開きます：

    <NxCommands commands={["run dungeon_adventure.story:agent-chat"]} />

    これは[`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli)を使用して`http://localhost:8081/invocations`に対してAG-UIプロトコルで通信します。最初のメッセージでは、エージェントにヒーローの名前とジャンルを伝えてください（例：`My name is Alice. Start my zombie adventure.`）。ストーリーがストリーミングで返されます。

    <Aside type="tip" title="生のcurl">
    生のHTTP呼び出しを希望する場合、AG-UIの`RunAgentInput`の形式は以下の通りです：
    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -H "Content-Type: application/json" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```
    後続の呼び出しで同じ`threadId`を再利用すると、同じストーリーが続きます。`FileSessionManager`が以前のメッセージを再生します。
    </Aside>
  </TabItem>
  <TabItem label="デプロイ済み">
    デプロイ済みエージェントのテストには、Cognito認証によるJWTトークン取得が必要です。まず環境変数を設定します：

    ```bash
    # CDK出力からCognito User Pool IDとClient IDを設定
    export POOL_ID="<CDK出力のUserPoolId>"
    export CLIENT_ID="<CDK出力のUserPoolClientId>"
    export REGION="<リージョン>"
    ```

    テストユーザーを作成し、認証トークンを取得します：

    ```bash
    # ユーザー作成を簡素化するためMFAを無効化
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # ユーザー作成
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # パスワード設定（本番環境ではより安全なパスワードを使用してください）
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 認証とアクセストークン取得
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Bedrock AgentCoreランタイムURLを使用してデプロイ済みエージェントを呼び出します：

    ```bash
    # CDK出力からStory Agent ARNを設定
    export AGENT_ARN="<CDK出力のStoryAgentArn>"

    # ARNをURLエンコード
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 呼び出しURLを構築
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # AG-UI RunAgentInputペイロードでエージェントを呼び出し
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: Alice-zombie00000000000000000000000" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```

    :::note[URLエンコード]
    ARNのURLエンコードは、Bedrock AgentCore APIエンドポイントの正しいフォーマットに必要です。
    :::
  </TabItem>
</Tabs>

コマンドが正常に実行されると、[AG-UIイベントストリーム](https://docs.copilotkit.ai/aws-strands/protocol/events)（Server-Sent Events）としてストーリーがストリーミングされ始めます。`RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT`チャンクが実際のストーリートークンをラップします：

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

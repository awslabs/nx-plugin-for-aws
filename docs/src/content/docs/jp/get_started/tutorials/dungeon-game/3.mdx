## タスク1: ストーリーエージェントの実装

ストーリーエージェントは[AG-UI](https://ag-ui.com/)プロトコル経由で公開される[Strands](https://strandsagents.com/)エージェントです。各AG-UIセッション（`threadId`で識別）はサーバー上の長期間存続するエージェントインスタンスにマッピングされるため、エージェントの会話履歴はセッションの存続期間中メモリに保持されます。

エージェントのシステムプロンプトをダンジョンアドベンチャー用に更新し、サンプルツールを削除します。

### エージェント実装

`packages/story/dungeon_adventure_story/agent/agent.py`を更新します：

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

システムプロンプトは、最初のメッセージが`I am <player>, playing in the <genre> genre.`の形式であることをエージェントに伝えます。UIはまさにこの形式でオープニングユーザーメッセージを送信するため、エージェントはストーリーを開始するために必要なすべての情報を得られます。AG-UIライブラリが各スレッドの会話履歴を自動的に保持するため、後続のターンでプレイヤー名やジャンルを渡す必要はありません。

接続ジェネレーターは<Link path="get_started/tutorials/dungeon-game/1">モジュール1</Link>ですでにインベントリMCPクライアントを設定しているため、エージェントはインベントリツールにアクセスできます。

:::note[`main.py`はどこに？]
AG-UIプロトコルのFastAPIラッパーがリクエスト/レスポンスの変換を処理してくれます。ジェネレーターが生成した`main.py`はすでに`get_agent(...)`のラッパーを作成しているため、変更する必要はありません。
:::

### エージェントのテスト

エージェントのテスト方法は2通りあります：
<ul>
<li>エージェントサーバーをローカルで起動し、`agent-chat`ターゲットを使用して対話的にチャットする、または</li>
<li>デプロイ済みAPIをJWTトークン付きのcurlで呼び出す。</li>
</ul>

<Tabs>
  <TabItem label="ローカル">
  `agent-chat`ターゲットは、エージェントをローカルで起動し（依存関係も`agent-serve-local`経由で起動）、対話型チャットプロンプトを表示します。以下を実行してください：

    <NxCommands commands={["agent-chat dungeon_adventure.story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    ローカルではなく**デプロイ済み**のMCPサーバーに対してエージェントとチャットしたい場合は、`agent-serve`で直接サーバーを起動し、`RUNTIME_CONFIG_APP_ID` / `AWS_REGION`を渡します：

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    その後、別のシェルで`agent-chat-cli agui http://localhost:8081/invocations`を実行します。
    :::

    プロンプトが表示されたら、次のようにゲームを開始してみてください：

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="デプロイ済み">
    デプロイ済みエージェントのテストには、Cognito認証によるJWTトークン取得が必要です。まず環境変数を設定します：

    ```bash
    # CDK出力からCognito User Pool IDとClient IDを設定
    export POOL_ID="<CDK出力のUserPoolId>"
    export CLIENT_ID="<CDK出力のUserPoolClientId>"
    export REGION="<リージョン>"
    ```

    テストユーザーを作成し、認証トークンを取得します：

    ```bash
    # ユーザー作成を簡素化するためMFAを無効化
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # ユーザー作成
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # パスワード設定（本番環境ではより安全なパスワードを使用してください）
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # 認証とアクセストークン取得
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Bedrock AgentCoreランタイムURLを使用してデプロイ済みエージェントを呼び出します。AG-UIプロトコルは`RunAgentInput`ペイロードを期待します。オープニングユーザーメッセージを送信する最小限のペイロードは次のようになります：

    ```bash
    # CDK出力からStory Agent ARNを設定
    export AGENT_ARN="<CDK出力のStoryAgentArn>"

    # ARNをURLエンコード
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # 呼び出しURLを構築
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # エージェント呼び出し
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[URLエンコード]
    ARNのURLエンコードは、Bedrock AgentCore APIエンドポイントの正しいフォーマットに必要です。
    :::
  </TabItem>
</Tabs>

コマンドが正常に実行されると、AG-UIサーバー送信イベントがストリーミングされ始めます。`TEXT_MESSAGE_CONTENT`イベントがストーリーテキストをチャンクごとに運びます。

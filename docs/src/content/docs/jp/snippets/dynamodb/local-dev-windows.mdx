---
title: Local Development Windows の注意事項
---

`dev` を停止する（例：`Ctrl+C` で）と、DynamoDB Local コンテナは自動的に削除されますが、名前付きボリュームは保持されるため、再起動後もデータは永続化されます。

:::caution[Windows]
Windows でのシグナル処理の制限により、`dev` を停止してもコンテナは自動的に削除されません。手動で削除する必要があります：

```bash
<engine> rm -f <scope>-dynamodb
```

`<engine>` をコンテナエンジン（`docker` または `finch`）に、`<scope>` を Nx ワークスペースのスコープ（例：`proj`）に置き換えてください。
:::

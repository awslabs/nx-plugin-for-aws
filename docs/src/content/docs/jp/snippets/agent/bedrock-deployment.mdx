### Infrastructure as Code

`computeType`に`BedrockAgentCoreRuntime`を選択した場合、関連するCDKまたはTerraformインフラストラクチャが生成され、Strands Agentを[Amazon Bedrock AgentCore Runtime](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/agents-tools-runtime.html)にデプロイするために使用できます。

<Infrastructure>
<Fragment slot="cdk">
エージェント用のCDKコンストラクトが生成されます。名前はジェネレーター実行時に選択した`name`に基づくか、デフォルトでは`<ProjectName>Agent`になります。

このCDKコンストラクトをCDKアプリケーションで使用できます:

```ts
import { MyProjectAgent } from ':my-scope/common-constructs';

export class ExampleStack extends Stack {
  constructor(scope: Construct, id: string) {
    new MyProjectAgent(this, 'MyProjectAgent');
  }
}
```

:::note[Alpha CDKコンストラクト]
このコンストラクトは[`@aws-cdk/aws-bedrock-agentcore-alpha`モジュール](https://docs.aws.amazon.com/cdk/api/v2/docs/aws-bedrock-agentcore-alpha-readme.html)を使用します。
:::
</Fragment>
<Fragment slot="terraform">
Terraformモジュールが生成されます。名前はジェネレーター実行時に選択した`name`に基づくか、デフォルトでは`<ProjectName>-agent`になります。

このTerraformモジュールをTerraformプロジェクトで使用できます:

```terraform
# Agent
module "my_project_agent" {
  # Relative path to the generated module in the common/terraform project
  source = "../../common/terraform/src/app/agents/my-project-agent"
}
```
</Fragment>
</Infrastructure>

:::note[Bedrockモデルの権限]
生成されたエージェントコンストラクトは、デフォルトですべてのBedrockファウンデーションモデルおよび推論プロファイルを呼び出す権限を付与します(`arn:aws:bedrock:*:*:foundation-model/*`および`arn:aws:bedrock:*:*:inference-profile/*`に対する`bedrock:InvokeModel`と`bedrock:InvokeModelWithResponseStream`)。

エージェントが使用する特定のモデルにスコープを絞り込むことをお勧めします。Strandsエージェントは[デフォルトモデルとしてClaude 4 Sonnetを使用します](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/#default-model)。別のモデルを設定する場合は、[Strandsモデルプロバイダーガイド](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/)を参照してください。

<Infrastructure>
<Fragment slot="cdk">
スコープを絞り込むには、生成されたエージェントCDKコンストラクトを編集し、デフォルトの`PolicyStatement`リソースを使用する特定のファウンデーションモデルまたは推論プロファイルのARNに置き換えます。
</Fragment>
<Fragment slot="terraform">
スコープを絞り込むには、生成されたエージェントTerraformモジュールを編集し、デフォルトのbedrockモデル`Resource` ARNを使用する特定のファウンデーションモデルまたは推論プロファイルのARNに置き換えます。
</Fragment>
</Infrastructure>
:::

:::caution[Dockerが必要]
Strands Agentをビルドおよびデプロイするには[Docker](https://www.docker.com/)が必要です。次のようなエラーを回避するために、Dockerがインストールされ起動していることを確認してください:

```
ERROR: Cannot connect to the Docker daemon at unix://path/to/docker.sock.
```
:::

### 認証

ジェネレーターは、Strands Agentの認証を設定するための`auth`オプションを提供します。エージェントを生成する際に、`IAM`(デフォルト)または`Cognito`認証を選択できます。

#### IAM

デフォルトでは、Strands AgentはIAM認証を使用して保護されます。引数なしで単純にデプロイしてください:

<Infrastructure>
<Fragment slot="cdk">
```ts {5}
import { MyProjectAgent } from ':my-scope/common-constructs';

export class ExampleStack extends Stack {
  constructor(scope: Construct, id: string) {
    new MyProjectAgent(this, 'MyProjectAgent');
  }
}
```

`grantInvokeAccess`メソッドを使用して、Bedrock AgentCore Runtime上でエージェントを呼び出すアクセス権を付与できます。例:

```ts {8}
import { MyProjectAgent } from ':my-scope/common-constructs';

export class ExampleStack extends Stack {
  constructor(scope: Construct, id: string) {
    const agent = new MyProjectAgent(this, 'MyProjectAgent');
    const lambdaFunction = new Function(this, ...);

    agent.grantInvokeAccess(lambdaFunction);
  }
}
```
</Fragment>
<Fragment slot="terraform">
```terraform
# Agent
module "my_project_agent" {
  # Relative path to the generated module in the common/terraform project
  source = "../../common/terraform/src/app/agents/my-project-agent"
}
```

エージェントを呼び出すアクセス権を付与するには、`module.my_project_agent.agent_core_runtime_arn`出力を参照して、次のようなポリシーを追加する必要があります:

```terraform
{
  Effect = "Allow"
  Action = [
    "bedrock-agentcore:InvokeAgentRuntime"
  ]
  Resource = [
    module.my_project_agent.agent_core_runtime_arn,
    "${module.my_project_agent.agent_core_runtime_arn}/*"
  ]
}
```
</Fragment>
</Infrastructure>

#### Cognito認証

`Cognito`認証を選択すると、ジェネレーターはCognito認証を使用するようにエージェントを設定します。

<Infrastructure>
<Fragment slot="cdk">
生成されたコンストラクトは、Cognito認証を設定する`identity`プロパティを受け入れます:

```ts {8}
import { MyProjectAgent, UserIdentity } from ':my-scope/common-constructs';

export class ExampleStack extends Stack {
  constructor(scope: Construct, id: string) {
    const identity = new UserIdentity(this, 'Identity');

    new MyProjectAgent(this, 'MyProjectAgent', {
      identity,
    });
  }
}
```

`UserIdentity`コンストラクトは、<Link path="/guides/react-website-auth">`ts#react-website#auth`ジェネレーター</Link>を使用して生成するか、独自のCDK `UserPool`と`UserPoolClient`を作成できます。
</Fragment>
<Fragment slot="terraform">
生成されたモジュールは、Cognito認証用の`user_pool_id`と`user_pool_client_ids`変数を受け入れます:

```terraform
module "user_identity" {
  source = "../../common/terraform/src/core/user-identity"
}

module "my_project_agent" {
  source = "../../common/terraform/src/app/agents/my-project-agent"

  user_pool_id         = module.user_identity.user_pool_id
  user_pool_client_ids = [module.user_identity.user_pool_client_id]
}
```
</Fragment>
</Infrastructure>

:::note[カスタムOIDCプロバイダー]
Cognito以外のOIDCプロバイダーを使用したカスタムJWT認証が必要な場合は、エージェント用に生成されたCDKコンストラクトまたはTerraformモジュールを直接変更できます。なお、接続ジェネレーターは`IAM`または`Cognito`認証のみをサポートします。
:::

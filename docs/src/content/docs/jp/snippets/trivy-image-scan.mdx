このプロジェクト用にビルドされたDockerイメージは、[Trivy](https://trivy.dev/)を使用して脆弱性をスキャンできます。Trivyは[ECRホスト版Trivyイメージ](https://gallery.ecr.aws/aquasecurity/trivy)から実行されます。

`trivy`ターゲットがプロジェクトに追加され、ビルドされたイメージをスキャンし、`HIGH`または`CRITICAL`の深刻度の脆弱性が見つかった場合、**非ゼロで終了します**。生成された`Dockerfile`は、生成時点でこれらの深刻度の既知の修正可能な脆弱性がないベースイメージを使用し、バンドルされたツール（`npm`など）をアップグレードしてその状態を維持します。

スキャンはイメージビルドと同じコンテナエンジン（`docker`または`finch`）を使用するため、追加のツールは必要ありません。スキャンはイメージが変更されたときにのみ再実行されるため、変更されていないイメージは再スキャンされません。提供されている`trivy`ルートスクリプトはワークスペース内のすべてのイメージをスキャンします：

<PackageManagerShortCommand commands={['trivy']} />

:::tip[CIでTrivyを実行してください]
スキャンは意図的に`build`に組み込まれていません。これは、Trivyの脆弱性データベースが新しいCVEで継続的に更新されるため、開発中の反復時に不要な摩擦を引き起こす可能性があるためです。

代わりに、本番ステージへのデプロイ前にCIパイプラインの専用ステップとして上記のコマンドを実行することをお勧めします。
:::

:::caution[修正不可能な脆弱性は無視されます]
スキャンは`--ignore-unfixed`で実行されるため、修正が利用できない脆弱性はビルドを失敗させません。これは、`Dockerfile`内のツールをアップグレードしても対処できないためです。これらはビルドのスキャン出力には報告されません。新しい脆弱性が公開されるにつれてスキャン結果は時間の経過とともに変化する可能性があるため、フラグなしで`trivy image <your-image>`を実行してイメージを定期的に確認し、修正が公開されたら新しいベースイメージに対して再ビルド（または利用可能なパッチを適用するコンテナステップを追加）してください。
:::

#### Trivy検出結果の抑制

特定の脆弱性を抑制したい場合があります。例えば、まだ修正が利用できず、リスクを許容可能と評価した場合などです。

プロジェクトのルート（つまり`project.json`の隣）にある`.trivyignore`ファイルに脆弱性ID（1行に1つ）を追加してください：

```gitignore
# .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

検出結果のフィルタリングの詳細については、[Trivyフィルタリングドキュメント](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids)を参照してください。

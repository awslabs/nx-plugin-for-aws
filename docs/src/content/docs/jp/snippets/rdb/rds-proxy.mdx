生成されるインフラストラクチャには、デフォルトでRDS Proxyが含まれており、アプリケーションとAuroraクラスター間に配置されます。RDS Proxyは以下のような利点を提供します：

- **コネクションプーリング** - アプリケーションインスタンス間で共有できるデータベース接続のプールを維持し、新しい接続を確立するオーバーヘッドを削減します
- **接続の回復性** - Auroraインスタンスの交換やメンテナンス中のフェイルオーバーと再接続を自動的に処理します
- **IAM認証** - IAMベースのデータベース認証をサポートし、アプリケーションコードでデータベース認証情報を管理する必要がなくなります
- **セキュリティの向上** - すべての接続に対してTLS暗号化を強制します

:::note[追加コスト]
RDS Proxyはデフォルトで有効になっていますが、Auroraクラスターのコストに加えて追加料金が発生します。詳細については、[AWS RDS Proxyの料金](https://aws.amazon.com/rds/proxy/pricing/)を参照してください。プロキシを無効にしたい場合は、以下のセクションを参照してください。
:::

#### RDS Proxyを無効にする

RDS Proxyは次のように無効にできます：

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', {
  ...
  enableRdsProxy: false,
});
```

RDS Proxyが無効になっている場合、アプリケーションはAuroraクラスターエンドポイントに直接接続します。
</Fragment>
<Fragment slot="terraform">

デフォルトでは、RDS Proxyが有効になっています。必要に応じて無効にできます：

```hcl title="packages/infra/src/main.tf"
module "my_database" {
  source = "../../common/terraform/src/app/dbs/my-database"
  ...
  enable_rds_proxy = false
}
```

RDS Proxyが無効になっている場合、アプリケーションはAuroraクラスターエンドポイントに直接接続します。
</Fragment>
</Infrastructure>

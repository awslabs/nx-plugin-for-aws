[CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html)は接続されたCDKリソースのグラフを構築し、2つのプラグインを提供します：

#### Diagram Plugin

[CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html)はCDKインフラストラクチャからAWSアーキテクチャ図を生成します。

同様の決定論的アプローチとして有効な代替手段は[CDK-Dia](https://github.com/pistazie/cdk-dia)があります。

生成AIの進化に伴い、多くの基盤モデルがCDKインフラストラクチャから高品質な図を作成可能です。[AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server)の使用をお試しください。チュートリアルについては[こちらのブログ記事](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/)を参照してください。

#### Threat Composer Plugin

[CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html)はCDKコードから[Threat Composer](https://github.com/awslabs/threat-composer/)のスターター脅威モデルを生成します。

このプラグインは、サンプルの脅威を含む[ベース脅威モデル](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model)をフィルタリングし、スタックで使用するリソースに基づいて絞り込むことで機能します。

これらの具体的なサンプル脅威に興味がある場合は、ベース脅威モデルをコピーしてフィルタリングするか、基盤モデルが類似モデルを生成する際のコンテキストとして使用できます。

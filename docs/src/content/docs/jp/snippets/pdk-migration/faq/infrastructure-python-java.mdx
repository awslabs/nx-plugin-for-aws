PDKがサポートするPythonおよびJavaで記述されたCDKインフラストラクチャ。現時点ではNx Plugin for AWSではこれをサポートしていません。

推奨される解決策としては、CDKインフラストラクチャをTypeScriptに移行するか、当社のジェネレータを使用して共通コンストラクトパッケージを目的の言語に移行する方法があります。[Amazon Q CLI](https://aws.amazon.com/developer/learning/q-developer-cli/)などの生成AIを活用して、この種の移行を加速できます。AIエージェントにCloudFormationテンプレートの合成結果が完全に一致するまで移作業を反復させることが可能です。

Type Safe APIが生成するPythonやJavaのインフラストラクチャについても同様です。共通コンストラクトパッケージの汎用的な`rest-api.ts`コンストラクトを翻訳し、対象言語向けのシンプルなメタデータジェネレータを独自に実装できます（[OpenAPIでモデル化されたAPI](#apis-modelled-with-openapi)セクションを参照）。

CDKコード用のベースPythonプロジェクトには<Link path="/guides/python-project">`py#project`ジェネレータ</Link>を利用可能です（`cdk.json`ファイルを移行し、関連するターゲットを追加）。JavaプロジェクトにはNxの[`@nx/gradle`](https://nx.dev/technologies/java/introduction)プラグインを、Mavenには[`@jnxplus/nx-maven`](https://github.com/gridatek/jnxplus/tree/main/packages/nx-maven)を使用できます。

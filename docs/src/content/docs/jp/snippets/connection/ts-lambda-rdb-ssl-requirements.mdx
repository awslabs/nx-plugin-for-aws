---
title: RDS Proxyを使用しない接続時のSSL要件
---
import Infrastructure from '@components/infrastructure.astro';

Node.js 20以降のLambdaランタイムからAuroraクラスターに直接接続する場合、`NODE_EXTRA_CA_CERTS`を設定してAmazon RDS CAバンドルを読み込みます：

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "api" {
  source = "..."
  ...

  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/var/runtime/ca-cert.pem"
  }
}
```

</Fragment>
</Infrastructure>

詳細については、AWS Lambdaの[Amazon RDS接続のSSL/TLS要件](https://docs.aws.amazon.com/lambda/latest/dg/services-rds.html#services-rds-tls)およびAmazon RDS Proxyの[TLSドキュメント](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-proxy.howitworks.html#rds-proxy-security.tls)を参照してください。RDS Proxyを使用する場合、Lambda関数でRDS CAバンドルを設定する必要はありません。

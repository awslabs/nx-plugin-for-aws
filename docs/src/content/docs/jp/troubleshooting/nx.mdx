このページでは、Nxの使用中に発生する可能性のある問題のトラブルシューティングガイダンスを提供します。

## Nx Daemon

[Nx Daemon](https://nx.dev/docs/concepts/nx-daemon)は、プロジェクトグラフの再計算のためにワークスペースの変更を継続的に監視するプロセスです。これは特に大規模なプロジェクトでパフォーマンス向上をもたらしますが、環境によっては問題の原因となる場合があります。

これは次のようなエラーとして現れる可能性があります：

```bash
 NX   Could not find project "@scope/project"

Pass --verbose to see the stacktrace.
```

このエラーやその他の予期せぬ問題が発生した場合、デーモン関連の問題の可能性を考慮してNxのリセットを試す価値があります。次のコマンドを使用できます：

<NxCommands commands={['reset']} />

これで解決しない場合は、[デーモンを完全に無効化](https://nx.dev/docs/concepts/nx-daemon#turning-it-off)することも検討してください。

:::warning
`nx watch`コマンドはデーモンを必要とするため、デーモンを無効にするとホットリロード機能が正常に動作しなくなる可能性があります。
:::

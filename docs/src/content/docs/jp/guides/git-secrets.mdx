`@aws/nx-plugin` で作成された新しいワークスペースには、デフォルトで [git-secrets](https://github.com/awslabs/git-secrets) pre-commit フックが設定されています。これにより、各コミット前にステージングされたファイルが AWS 認証情報に一致するパターンをスキャンし、誤って認証情報が漏洩するのを防ぎます。

## 含まれる内容

新しいワークスペースを作成すると、以下が自動的にセットアップされます：

<FileTree>
  - .git-secrets/
    - git-secrets The git-secrets bash script (vendored from awslabs/git-secrets)
    - setup.sh Registers AWS credential patterns in local git config
  - .husky/
    - pre-commit Runs git-secrets scan before each commit
  - package.json Includes husky and prepare script for automatic setup
</FileTree>

## 動作の仕組み

1. `prepare` スクリプトは `npm install` / `pnpm install` の後に実行され、husky git フックを初期化し、ローカル git 設定に AWS 認証情報パターンを登録します。
2. 各 `git commit` で、husky pre-commit フックが `git-secrets` を呼び出してステージングされたファイルをスキャンします。
3. AWS 認証情報に一致するパターンが検出された場合、コミットはエラーメッセージとともにブロックされます。

## 検出されるパターン

以下のパターンがスキャンされます：

- **AWS Access Key IDs** — `AKIA...`、`ASIA...`、およびその他の IAM キープレフィックスに続く 16 文字の大文字英数字
- **AWS Secret Access Keys** — `SECRET_ACCESS_KEY` ラベルの前に付く 40 文字の base64 文字列
- **AWS Account IDs** — `ACCOUNT_ID` ラベルの前に付く 12 桁の数字
- **Bedrock API Keys** — 長期間有効な（`ABSK...`）および短期間有効な Bedrock キー
- **Credentials file values** — `~/.aws/credentials` に含まれるリテラルキー値

よく知られたサンプルキー（例：`AKIAIOSFODNN7EXAMPLE`）は自動的に許可リストに登録されます。

## 誤検知の許可

git-secrets が誤検知を含むコミットをブロックした場合：

```sh
# 特定のパターンを許可
git config --add secrets.allowed 'my-pattern-here'
```

または、リポジトリルートに `.gitallowed` ファイルを追加してパターンを記述します（1 行に 1 つの正規表現）。

1 回限りのオーバーライドの場合：

```sh
git commit --no-verify
```

## オプトアウト

git-secrets なしでワークスペースを作成するには：

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

または、プリセットジェネレーターに `--gitSecrets=false` を渡します。

## 要件

- ワークスペースで **Git** が初期化されている必要があります（`git init`）
- **Bash** が PATH で利用可能である必要があります（Windows では Git for Windows に同梱）
- パターンは `prepare` 実行時に登録されます — 新しいコピーをクローンした場合は、パッケージマネージャーのインストールコマンドを実行して再登録してください

このジェネレーターは、[Amazon DynamoDB](https://aws.amazon.com/dynamodb/)をバックエンドとする新しいTypeScriptプロジェクトを作成します。型安全なエンティティモデリングには[ElectroDB](https://electrodb.dev/)を使用します。AWS CDKまたはTerraformを使用してDynamoDBテーブルをプロビジョニングおよび管理するために必要なアプリケーションコードとインフラストラクチャを生成し、シングルテーブル設計のサポートとDynamoDB Localによる組み込みのローカル開発環境を提供します。

## ローカル開発

### ローカルDynamoDBの起動

ジェネレーターは、[DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html)インスタンスを起動し、テーブルを作成する`serve-local`ターゲットを設定します：

<NxCommands commands={['run <project-name>:serve-local']} />

これにより自動的に以下が実行されます：
1. DynamoDB Localイメージをプルする（`pull-image`ターゲット）
2. コンテナを起動する
3. 事前定義されたインデックスを持つローカルテーブルを作成する

### データモデリング

生成されたプロジェクトは、単一のDynamoDBテーブル上で型安全なエンティティモデリングを行うために[ElectroDB](https://electrodb.dev/)を使用します。生成されたサンプルエンティティを出発点として、`src/entities/`配下にエンティティファイルを追加または更新してください。

エンティティ定義の例：

```ts title="packages/my-table/src/entities/example.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createExampleEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'example',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: {
          type: 'string',
          required: true,
        },
        createdAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: {
            field: 'pk',
            composite: ['id'],
          },
          sk: {
            field: 'sk',
            composite: [],
          },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

詳細については、[ElectroDBエンティティドキュメント](https://electrodb.dev/en/modeling/entities/)を参照してください。

### DynamoDBクライアントの使用

生成された`src/client.ts`は、2つの主要なユーティリティをエクスポートします：

- `getDynamoDBClient()` — キャッシュされたシングルトンの`DynamoDBClient`を返します。`SERVE_LOCAL=true`の場合、ローカルのDynamoDB Localインスタンスに接続します。それ以外の場合は、デフォルトの認証情報チェーンを使用してAWSクライアントを作成します。
- `resolveTableName()` — DynamoDBテーブル名を返します。`SERVE_LOCAL=true`の場合、ローカルテーブル名定数を返します。それ以外の場合は、`RUNTIME_CONFIG_APP_ID`環境変数を使用してAWS AppConfigから名前を取得し、後続の呼び出しのためにキャッシュします。

### ローカルDynamoDBの停止

`serve-local`を停止する（例：`Ctrl+C`）と、DynamoDB Localコンテナは自動的に削除されますが、名前付きボリュームは保持されるため、再起動してもデータは保持されます。

:::caution[Windows]
Windowsでのシグナル処理の制限により、`serve-local`を停止してもコンテナは自動的に削除されません。手動で削除する必要があります：

```bash
<engine> rm -f <scope>-dynamodb
```

`<engine>`をコンテナエンジン（`docker`または`finch`）に、`<scope>`をNxワークスペースのスコープ（例：`proj`）に置き換えてください。
:::

## グローバルセカンダリインデックスの追加/削除

まず、`src/gsi.ts`の`GLOBAL_SECONDARY_INDEXES`を更新します。これがGSI定義の信頼できる情報源であり、次回の`serve-local`実行時にスクリプトが自動的にローカルテーブルのインデックスを追加または削除して新しいリストを反映します。CDKを使用している場合、これらの変更は次回のデプロイ時にAWSテーブルにも自動的に反映されます。

<OptionFilter when={{ iac: 'terraform' }}>
  CDKとは異なり、`src/gsi.ts`への変更はTerraformモジュールに自動的に反映されません。デプロイ前に、対応するGSI定義を追加または削除するためにTerraformモジュールも手動で更新する必要があります。
</OptionFilter>

## テーブルへの接続

任意のTypeScriptプロジェクトで、DynamoDBパッケージからエンティティファクトリをインポートして直接使用できます：

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

:::note[ランタイム設定]
AWSで実行する場合、`resolveTableName()`は`RUNTIME_CONFIG_APP_ID`環境変数を使用してAWS AppConfigからテーブル名を取得します。このプラグインで構築されたプロジェクト（tRPC API、Smithy API、エージェント、MCPサーバー）では、この変数は自動的に設定されます。他のTypeScriptプロジェクトの場合は、インフラストラクチャによってプロビジョニングされたAppConfigアプリケーションIDを使用して、ランタイム環境で`RUNTIME_CONFIG_APP_ID`が設定されていることを確認してください。詳細については、<Link path="guides/runtime-config">ランタイム設定ガイド</Link>を参照してください。
:::

## テーブルのデプロイ

DynamoDBジェネレーターは、選択した`iac`に基づいてCDKまたはTerraformインフラストラクチャを作成します。

<Infrastructure>
<Fragment slot="cdk">
CDKコンストラクトは`common/constructs`に作成されます。使用例：

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const table = new MyTable(this, 'Table');
  }
}
```

これにより、以下の設定でDynamoDBテーブルがプロビジョニングされます：
- `pk`（パーティションキー）と`sk`（ソートキー）、両方とも`String`型
- デフォルトで2つのグローバルセカンダリインデックス（`src/gsi.ts`で定義）
- オンデマンド（`PAY_PER_REQUEST`）課金
- 自動キーローテーション付きのカスタマー管理KMS暗号化
- ポイントインタイムリカバリが有効
- 削除保護が有効
- AWS AppConfigの`dynamodb`名前空間のランタイム設定にテーブル名が登録される
</Fragment>
<Fragment slot="terraform">
Terraformモジュールは`common/terraform`に作成されます。使用例：

```hcl title="packages/infra/src/main.tf"
module "my_table" {
```

### 削除保護

削除保護はデフォルトで有効になっており、誤ってテーブルを削除することを防ぎます。

#### 削除保護を無効にする

短期間の開発環境やプレビュースタックなど、テーブルの削除が予想される環境では無効にしてください。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  deletionProtection: false,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source                      = "../../common/terraform/src/app/dynamodb/my-table"
  deletion_protection_enabled = false
}
```
</Fragment>
</Infrastructure>

### 課金モード

テーブルはデフォルトでオンデマンド（`PAY_PER_REQUEST`）課金になります。予測可能な高スループットワークロードの場合は、プロビジョニング済みキャパシティに切り替えてください。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { BillingMode } from 'aws-cdk-lib/aws-dynamodb';
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  billingMode: BillingMode.PROVISIONED,
  readCapacity: 5,
  writeCapacity: 5,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source       = "../../common/terraform/src/app/dynamodb/my-table"
  billing_mode = "PROVISIONED"
}
```
</Fragment>
</Infrastructure>

### ポイントインタイムリカバリ

[ポイントインタイムリカバリ](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Point-in-time-recovery.html)はデフォルトで有効になっており、過去35日間の任意の時点にテーブルを復元できます。

#### ポイントインタイムリカバリを無効にする

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  pointInTimeRecoverySpecification: { pointInTimeRecoveryEnabled: false },
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source                         = "../../common/terraform/src/app/dynamodb/my-table"
  point_in_time_recovery_enabled = false
}
```
</Fragment>
</Infrastructure>

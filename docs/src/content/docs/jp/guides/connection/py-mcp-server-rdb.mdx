`connection` ジェネレーターは、<Link path="guides/py-mcp-server">Python MCP Server</Link> を <Link path="guides/py-rdb">Python リレーショナルデータベース</Link>プロジェクトに接続し、サーバーに登録されたすべてのツールでデータベースセッションを利用できるようにします。

## 前提条件

このジェネレーターを使用する前に、以下を用意してください：

1. <Link path="guides/py-mcp-server">`py#mcp-server`</Link> プロジェクト
2. <Link path="guides/py-rdb">`py#rdb`</Link> プロジェクト

## ジェネレーターの出力

ジェネレーターは、データベースパッケージをワークスペースの依存関係として追加し、MCP サーバーの `<mcp-server-name>-dev` ターゲットがデータベースの `dev` ターゲットに依存するように更新して、ローカル開発時に両方が一緒に起動するようにします。また、生成された MCP サーバーの Dockerfile を、Aurora への直接接続に使用される RDS CA バンドルで更新します。

## 仕組み

データベースパッケージは `session_context` をエクスポートします。これは、生成されたデータベースクライアントを使用して `AsyncSession` を開き、`async with` ブロックが終了したときにそれを閉じる非同期コンテキストマネージャーです。

ローカル開発では、`session_context()` はローカルのデータベースコンテナに接続します。デプロイされたランタイムでは、AWS AppConfig からデータベース設定を読み取り、IAM 認証トークンを生成し、データベースの TLS 証明書を検証します。

## MCP ツールでのデータベースの使用

データベースパッケージから `session_context` をインポートし、MCP サーバーツール内で使用します：

```python title="packages/my_service/my_service/my_mcp_server/server.py"
from sqlmodel import select
from my_scope.my_db import session_context
from my_scope.my_db.models.example import ExampleModel

@mcp.tool()
async def list_examples() -> str:
    """List all example records."""
    async with session_context() as session:
        items = (await session.exec(select(ExampleModel))).all()
        return str([item.model_dump() for item in items])
```

## 複数のデータベース

異なるターゲットでジェネレーターを再度実行すると、最初のデータベースと並行して2番目のデータベースが追加されます。両方のセッションコンテキストがすべてのツールで利用可能になります：

```python title="packages/my_service/my_service/my_mcp_server/server.py"
from my_scope.my_db import session_context as my_db_session_context
from my_scope.other_db import session_context as other_db_session_context
```

## インフラストラクチャ

生成された MCP サーバーコンストラクトは `IGrantable` と `IConnectable` を実装しているため、コンストラクト上で直接データベースへのネットワークおよび IAM アクセスを許可できます。

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myMcpServer = new MyMcpServer(this, 'MyMcpServer', { vpc, ... });

db.allowDefaultPortFrom(myMcpServer);
db.grantConnect(myMcpServer);
```

`allowDefaultPortFrom` はセキュリティグループルールを開き、MCP サーバーランタイムがデータベースポートに到達できるようにします。`grantConnect` は、サーバーの実行ロールに IAM `rds-db:connect` 権限を付与します。

データベースモジュールの出力を MCP サーバーモジュールに渡して、データベースに到達し、そのランタイム設定を読み取れるようにします：

### RDS Proxy を使用せずに接続する場合の SSL 要件

MCP サーバーが（RDS Proxy を使用せずに）Aurora クラスターに直接接続する場合、接続ジェネレーターは生成された MCP サーバーの Dockerfile を更新して、Amazon RDS CA バンドルをシステムトラストストアにインストールします：

```dockerfile
ADD https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem /usr/local/share/ca-certificates/rds-global-bundle.crt
RUN update-ca-certificates
```

RDS Proxy を使用する場合、MCP サーバーランタイムで RDS CA バンドルを設定する必要はありません。

## ローカル開発

<NxCommands commands={["<mcp-server-name>-dev <project-name>"]} />

これにより、MCP サーバーと接続されたすべてのデータベースが起動します。`LOCAL_DEV=true` 環境変数により、データベースクライアントは Aurora ではなくローカルの Docker データベースに接続します。

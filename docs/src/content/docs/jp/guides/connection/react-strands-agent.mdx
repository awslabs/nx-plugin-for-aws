Nx Plugin for AWSは、<Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link>をReactウェブサイトと迅速に統合するためのジェネレーターを提供します。これは、AWS IAMおよびCognito認証のサポートを含む、tRPC over WebSocketを介してエージェントに接続するために必要なすべての設定をセットアップします。この統合により、フロントエンドとエージェントのtRPCルーター間で完全なエンドツーエンドの型安全性が提供されます。

## 前提条件

このジェネレーターを使用する前に、以下を確認してください:

1. Reactウェブサイト（<Link path="guides/react-website">`ts#react-website`ジェネレーター</Link>を使用して生成）
2. TypeScript Strands Agent（<Link path="guides/ts-strands-agent">`ts#strands-agent`ジェネレーター</Link>を使用して生成）
3. CognitoまたはIAM認証を使用するエージェントに接続する場合は、<Link path="/guides/react-website-auth">`ts#react-website-auth`ジェネレーター</Link>を介して追加されたCognito Auth

## 使用方法

### ジェネレーターの実行

<RunGenerator generator="connection" />

## ジェネレーターの出力

ジェネレーターは、Reactアプリケーション内に以下の構造を作成します:

<FileTree>

- src
  - components
    - \<AgentName>AgentClientProvider.tsx tRPC WebSocketクライアントとエージェントのtRPCルーターへのバインディングをセットアップ
    - QueryClientProvider.tsx TanStack React Queryクライアントプロバイダー
  - hooks
    - useSigV4.tsx SigV4でリクエストに署名するためのフック（IAMのみ）
    - use\<AgentName>Agent.tsx TanStack Query統合用のtRPCオプションプロキシを返すフック
    - use\<AgentName>AgentClient.tsx 直接エージェント呼び出し用のバニラtRPCクライアントを返すフック

</FileTree>

さらに、必要な依存関係をインストールします:

  - `@trpc/client`
  - `@trpc/tanstack-react-query`
  - `@tanstack/react-query`
  - `aws4fetch`（IAM認証を使用する場合）

## 仕組み

### WebSocket接続

生成されたクライアントは、tRPC over WebSocketを介してStrands Agentに接続します。エージェントは、WebSocketエンドポイント上でtRPCルーター（ストリーミングエージェントレスポンス用の`invoke`サブスクリプションを含む）を公開します。

- **デプロイ済み**: エージェントランタイムARNは<Link path="guides/runtime-config">ランタイム設定</Link>から読み込まれます。ARNは、Bedrock AgentCore Runtime WebSocketプロトコルに従ってWebSocket URLに変換されます: `wss://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/ws`
- **ローカル開発**: `serve-local`で実行する場合、ランタイム設定オーバーライドは値をローカルの`ws://` URL（例: `ws://localhost:8081/ws`）に設定し、クライアントは直接接続します

### 認証

生成されたコードは、エージェントの設定に応じて認証を処理します:

- **IAM**（デフォルト）: AWS SigV4事前署名URLを使用してWebSocket接続を認証します。認証情報は、ウェブサイトの認証で設定されたCognito Identity Poolから取得されます
- **Cognito**: JWTアクセストークンを`Sec-WebSocket-Protocol`ヘッダーにbase64urlエンコードされたベアラートークンとして埋め込み、[AgentCore WebSocket認証プロトコル](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-get-started-websocket.html)に従います
- **認証なし**: 認証なしで直接接続します（ローカル開発に適しています）

## 生成されたコードの使用

### useSubscriptionを使用したエージェントレスポンスのストリーミング

最も一般的な使用例は、`invoke`サブスクリプションを使用してエージェントのレスポンスをストリーミングすることです:

```tsx {1-2,7-22}
import { useSubscription } from '@trpc/tanstack-react-query';
import { useMyAgent } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const trpc = useMyAgent();

  const subscription = useSubscription(
    trpc.invoke.subscriptionOptions(
      { message: 'What can you help me with?' },
      {
        enabled: true,
        onStarted: () => {
          console.log('Agent started responding');
        },
        onData: (token) => {
          console.log('Received token:', token);
        },
        onError: (error) => {
          console.error('Agent error:', error);
        },
      },
    ),
  );

  return (
    <div>
      <p>Status: {subscription.status}</p>
      {subscription.data && <p>Latest token: {subscription.data}</p>}
      {subscription.error && <p>Error: {subscription.error.message}</p>}
    </div>
  );
}
```

### バニラtRPCクライアントの使用

`use<AgentName>AgentClient`フックは、サブスクリプションライフサイクルをより細かく制御するためのバニラtRPCクライアントへのアクセスを提供します:

```tsx {2,5,9-22,25}
import { useState, useEffect } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const client = useMyAgentClient();
  const [messages, setMessages] = useState<string[]>([]);

  const sendMessage = (message: string) => {
    const subscription = client.invoke.subscribe(
      { message },
      {
        onData: (token) => {
          setMessages((prev) => [...prev, token]);
        },
        onComplete: () => {
          console.log('Agent finished');
        },
        onError: (error) => {
          console.error('Error:', error);
        },
      },
    );

    // Clean up when done
    return () => subscription.unsubscribe();
  };

  return (
    <div>
      <button onClick={() => sendMessage('Hello!')}>Send</button>
      <div>
        {messages.map((msg, i) => (
          <span key={i}>{msg}</span>
        ))}
      </div>
    </div>
  );
}
```

### チャットインターフェースの構築

生成されたクライアントを使用してシンプルなチャットインターフェースを構築する例を以下に示します:

```tsx
import { useState, useCallback } from 'react';
import { useMyAgentClient } from './hooks/useMyAgentAgent';

function Chat() {
  const client = useMyAgentClient();
  const [input, setInput] = useState('');
  const [response, setResponse] = useState('');
  const [isStreaming, setIsStreaming] = useState(false);

  const handleSend = useCallback(() => {
    if (!input.trim() || isStreaming) return;

    setIsStreaming(true);
    setResponse('');

    client.invoke.subscribe(
      { message: input },
      {
        onData: (token) => {
          setResponse((prev) => prev + token);
        },
        onComplete: () => {
          setIsStreaming(false);
        },
        onError: (error) => {
          console.error(error);
          setIsStreaming(false);
        },
      },
    );

    setInput('');
  }, [client, input, isStreaming]);

  return (
    <div>
      <div style={{ whiteSpace: 'pre-wrap' }}>{response}</div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        onKeyDown={(e) => e.key === 'Enter' && handleSend()}
        disabled={isStreaming}
        placeholder="Ask the agent..."
      />
      <button onClick={handleSend} disabled={isStreaming}>
        {isStreaming ? 'Streaming...' : 'Send'}
      </button>
    </div>
  );
}
```

## ローカル開発

接続ジェネレーターは`serve-local`統合を自動的に設定します:

1. `nx run <website>:serve-local`を実行すると、エージェントのローカルサーバーも起動します
2. ランタイム設定がオーバーライドされ、ローカルWebSocket URL（例: `ws://localhost:8081/ws`）を指すようになります
3. クライアントはローカルURLを検出し、認証のオーバーヘッドなしで直接接続します

つまり、単一のコマンドでエージェント統合をローカルで開発およびテストできます。

## 型安全性

この統合は、完全なエンドツーエンドの型安全性を提供します。IDEは、すべてのエージェントプロシージャ呼び出しに対して完全な自動補完と型チェックを提供します。型は、エージェントのtRPCルーター定義から自動的に推論されるため、エージェントのAPIへの変更がフロントエンドコードに即座に反映されます。

`connection`ジェネレーターは、<Link path="guides/ts-agent">TypeScript Agent</Link>を<Link path="guides/agentcore-gateway">AgentCore Gateway</Link>に接続できます。

このジェネレーターは、デプロイ時にIAM SigV4でGatewayに認証し、ローカル実行時には接続されたMCPサーバーに直接多重化するようにエージェントを構成します。

## 前提条件

このジェネレーターを使用する前に、以下を確認してください：

1. <Link path="guides/ts-agent">Agent</Link>コンポーネント（`infra: agentcore`）を持つTypeScriptプロジェクト
2. <Link path="guides/agentcore-gateway">`cedar#agentcore-gateway`</Link>プロジェクト

## ジェネレーターの出力

ジェネレーターは、共有コアクライアントファイルを`agent-connection`パッケージに出力し、Gateway毎のラッパーを追加し、エージェントを変更します：

<FileTree>

- packages/common/agent-connection
  - src
    - core/
      - agentcore-gateway-mcp-client.ts デプロイされたGateway用のSigV4 MCPクライアント
      - agentcore-gateway-mcp-local-client.ts 接続されたMCPサーバーに展開するローカル多重化クライアント
    - app/
      - \<gateway-kebab>-client.ts `ATTACHED_MCP_SERVERS`リストを持つGateway毎のラッパー
    - index.ts Gatewayクライアントを再エクスポート

</FileTree>

さらに、ジェネレーターは：

- エージェントの`agent.ts`を変更して、Gatewayクライアントクラスをインポートし、`<Gateway>Client.create()`を呼び出し、返されたクライアントを`tools`配列に登録します
- エージェントの`<agent>-serve-local`ターゲットをGatewayの`<gateway>-serve-local`アグリゲーターに依存するように構成します
- 必要なSigV4 / MCP依存関係をインストールします

## 接続されたGatewayの使用

ジェネレーターは、エージェントの`agent.ts`をGatewayクライアントを使用するように変換します：

```ts title="packages/example/src/my-agent/agent.ts" {2,5,8}
import { Agent } from '@strands-agents/sdk';
import { MyGatewayClient } from ':my-scope/agent-connection';

export const getAgent = async () => {
  const myGateway = await MyGatewayClient.create();
  return new Agent({
    systemPrompt: '...',
    tools: [myGateway],
  });
};
```

デプロイ時（`SERVE_LOCAL`が未設定）、`create()`はGatewayのMCPエンドポイントを指し、SigV4で認証された単一の`McpClient`を返します。`SERVE_LOCAL=true`の場合、クライアントの`ATTACHED_MCP_SERVERS`リスト内のMCPサーバー毎に1つの`McpClient`の配列を返します。Strandsの`Agent.tools`はどちらの形式も受け入れる（ネストされたツール配列を透過的にフラット化する）ため、同じ`agent.ts`が両方のモードで統一的に動作します。

セッションIDは下流のMCPサーバーに自動的に伝播されます：基盤となるトランスポートは[Strands TSセッションコンテキスト](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html)を介してリクエストスコープのセッションを読み取り、`X-Amzn-Bedrock-AgentCore-Runtime-Session-Id`ヘッダーとしてGatewayに転送します。

## インフラストラクチャ

ジェネレーターの実行後、エージェントにGatewayを呼び出す権限を付与する必要があります。

## ローカル開発

ジェネレーターは、エージェントの`serve-local`ターゲットを次のように構成します：

1. 接続されたGatewayに接続されているすべてのMCPサーバーを起動
2. `SERVE_LOCAL=true`を設定して、生成されたクライアントがデプロイされたGatewayを呼び出す代わりに、それらのローカルMCPサーバーに展開するようにします

エージェントをローカルで実行するには：

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

エージェントをローカルで**デプロイされたGatewayに対して**実行するには（例えば、Cedarポリシーを実行するため）、エージェントの`serve`ターゲットを使用します。`SERVE_LOCAL`が設定されていない場合、クライアントはランタイム設定からデプロイされたGateway URLを解決し、ローカルのAWS認証情報でリクエストにSigV4署名します：

<NxCommands commands={["<agent-name>-serve <project-name>"]} />

### ローカルの忠実性

ローカル多重化モードはGateway自体をバイパスするため：

- **Cedarポリシー評価なし。** ポリシーに関係なく、すべてのツールがエージェントに表示されます。デプロイされたGatewayに対してポリシーを実行するには、`serve`ターゲットを使用してください。
- **ツール名のプレフィックスは保持されます。** 各ローカルMCPサーバーのツールは、`<target-name>___<tool-name>`の形式の名前を公開するようにラップされ、デプロイされたGatewayが出力するものと一致します。これにより、エージェントのシステムプロンプトと参照するCedarアクション名がローカル実行とデプロイ実行で一貫性を保ちます。

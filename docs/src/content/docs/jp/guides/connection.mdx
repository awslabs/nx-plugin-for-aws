## ローカル開発

接続されたプロジェクトは、マシン上で実行するための2つのターゲットを公開します：`serve` と `serve-local` です。違いは**スコープ**、つまりアプリケーションのどれだけの部分がローカルで実行されるか、デプロイされたAWSインフラストラクチャを指すかです。

tRPC `api` を呼び出し、さらに `mcp` サーバーを呼び出す `agent` も呼び出すウェブサイトを持つワークスペースを考えてみましょう。

### `serve`

`serve` ターゲットは**対象プロジェクトのみ**をローカルで実行します。接続する他のすべてのプロジェクトは**デプロイ済み**であることが期待され、実際のAWS URLでアクセスされます。

<NxCommands commands={['serve website']} />

対象プロジェクトはデプロイされたリソースと通信するため、それらがどこにあるかを知る必要があります。ウェブサイトの場合、これは `runtime-config.json` によって提供されます（<Link path="guides/react-website#local-runtime-config">Local Runtime Config</Link> を参照）。これはデプロイされたアプリケーションから読み込みます。<Link path="guides/runtime-config">ランタイム設定</Link>を読み取るサーバーサイドプロジェクト（APIやエージェント）の場合は、`RUNTIME_CONFIG_APP_ID` 環境変数を設定して、デプロイされたAppConfigアプリケーションを指すようにします。

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
agent: agent {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
mcp: mcp {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

`serve` は、依存するすべてのものの「実際の」デプロイされたバージョンに対して単一のプロジェクトを反復処理したい場合に使用します。

### `serve-local`

`serve-local` ターゲットは、対象プロジェクト**および推移的に接続されているすべてのプロジェクト**をマシン上で実行します。接続ジェネレータはこれを自動的に配線します。ウェブサイトで `serve-local` を実行すると、`api`、`agent`、およびエージェントを通じて到達する `mcp` サーバーのローカルサーバーも起動します。

<NxCommands commands={['serve-local website']} />

この方法で実行すると、ウェブサイトの `runtime-config.json` は自動的にオーバーライドされ（[Viteの `MODE`](https://vite.dev/guide/env-and-mode) を介して `serve-local` に設定）、デプロイされたURLの代わりにローカルで実行されているサーバーを指すようになります。

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#c8e6c9"
}
agent: agent {
  style.fill: "#c8e6c9"
}
mcp: mcp {
  style.fill: "#c8e6c9"
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

`serve-local` は、複数の接続されたプロジェクトを同時に作業し、インフラストラクチャをデプロイせずに迅速に反復処理したい場合に使用します。

:::note[`serve-local` でのAWSリソース]
ローカルサーバーはローカルに設定されたAWS認証情報で実行されるため、必要に応じてデプロイされたAWSリソース（DynamoDBテーブルやBedrockモデルなど）に引き続きアクセスできます。ローカルサーバーが必要とする環境変数を指定できます。例えば：

<NxCommands env={{RUNTIME_CONFIG_APP_ID: 'xxxxx'}} commands={['serve-local website']} />
:::

各プロジェクトタイプのローカル開発サーバーの詳細については、関連するガイド（例：<Link path="guides/react-website#local-development-server">React Website</Link> ガイド）を参照してください。

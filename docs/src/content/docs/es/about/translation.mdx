# Traducción de Documentación

Este proyecto utiliza el modelo Haiku 3.5 de AWS Bedrock para traducir automáticamente documentación del inglés a múltiples idiomas. El sistema de traducción está diseñado para ser eficiente, preciso y fácil de usar.

## Idiomas Soportados

Actualmente, se admiten los siguientes idiomas:

- Japonés (jp)
- Francés (fr)
- Español (es)
- Alemán (de)
- Chino (zh)
- Coreano (ko)

## Cómo Funciona

El sistema de traducción funciona:

1. **Dividiendo documentos por encabezados h2** - Permite un procesamiento más eficiente y mejor contexto para el modelo de traducción.
2. **Preservando formato markdown** - Toda la sintaxis markdown, bloques de código y etiquetas HTML se conservan durante la traducción.
3. **Manejo especial de frontmatter** - El frontmatter YAML se traduce manteniendo su estructura.
4. **Traducción incremental** - Por defecto solo se traducen archivos modificados, ahorrando tiempo y recursos.

## Ejecutar Traducciones Localmente

Para traducir documentación localmente, usa el script `scripts/translate.ts`:

```bash
# Traducir solo archivos modificados a japonés (predeterminado)
./scripts/translate.ts

# Traducir todos los archivos
./scripts/translate.ts --all

# Traducir a idiomas específicos
./scripts/translate.ts --languages jp,fr,es

# Simular traducción sin aplicar cambios
./scripts/translate.ts --dry-run

# Mostrar salida detallada
./scripts/translate.ts --verbose
```

## Flujo de Trabajo en GitHub

Un workflow de GitHub traduce automáticamente la documentación cuando se modifican archivos en solicitudes de extracción. El workflow:

1. Detecta cambios en archivos de documentación en inglés
2. Traduce los archivos modificados usando AWS Bedrock
3. Confirma las traducciones en la rama fuente
4. Actualiza el PR con el estado de traducción

### Activación Manual del Workflow

Puedes activar manualmente el workflow desde la pestaña GitHub Actions. Es útil para:

- Ejecutar traducción completa de toda la documentación
- Traducir a idiomas específicos
- Actualizar traducciones después de modificar el script de traducción

## Configuración de AWS

El sistema usa el modelo Haiku 3.5 de AWS Bedrock. Para usarlo necesitas:

1. **Credenciales AWS** - En desarrollo local, configura tus credenciales usando AWS CLI o variables de entorno.
2. **Rol IAM** - Para GitHub Actions, configura un rol IAM con autenticación OIDC y permisos necesarios para AWS Bedrock.

### Permisos Requeridos

El rol/usuario IAM necesita estos permisos:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel"
      ],
      "Resource": [
        "arn:aws:bedrock:*::foundation-model/anthropic.claude-3-haiku-20240307-v1:0"
      ]
    }
  ]
}
```

## Calidad de Traducción

La calidad es generalmente alta, pero considera:

- **Términos técnicos** - Se conservan en inglés por configuración.
- **Bloques de código** - No se traducen para mantener su forma original.
- **Contexto técnico** - El modelo entiende el contexto de documentación técnica.

## Personalización de Traducción

Puedes personalizar el proceso modificando `scripts/translate.ts`. Algunas opciones:

- Añadir soporte para más idiomas
- Cambiar el modelo de traducción
- Ajustar prompts de traducción
- Modificar cómo se dividen y procesan documentos

## Solución de Problemas

Si encuentras errores:

1. **Verifica credenciales AWS** - Asegúrate de tenerlas configuradas correctamente.
2. **Verifica región AWS** - Usa una región donde Bedrock esté disponible.
3. **Ejecuta con modo verbose** - Usa el flag `--verbose` para ver logs detallados.
4. **Verifica límites de tasa** - Bedrock tiene límites que pueden afectar trabajos grandes.

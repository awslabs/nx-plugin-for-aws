---
title: "Métricas de Uso"
description: "Cómo @aws/nx-plugin recopila métricas"
---



Para ayudarnos a mejorar el Nx Plugin para AWS, recopilamos métricas sobre el uso de los generadores. Este proceso no implica solicitudes de red adicionales, simplemente utilizamos las métricas de uso existentes de AWS.

Para mantener la transparencia, añadimos un ID, una versión y una lista de IDs de generadores a las descripciones de las pilas de CloudFormation. Esto se realiza en el `MetricsAspect` en `common/constructs` para CDK o en la pila de CloudFormation `metrics.tf` para Terraform.

## Optar por No Participar

El código de métricas reside completamente en tu espacio de trabajo, por lo que puedes optar por no participar eliminándolo:

- **CDK**: elimina la clase `MetricsAspect` y la llamada `Aspects.of(this).add(new MetricsAspect())` de `packages/common/constructs/src/core/app.ts`.
- **Terraform**: elimina `packages/common/terraform/src/metrics/metrics.tf`.

Los generadores no volverán a agregar el código de métricas una vez eliminado.

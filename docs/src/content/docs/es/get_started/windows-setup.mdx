Varios generadores en el Nx Plugin for AWS producen objetivos de compilación basados en Docker (por ejemplo, infraestructura CDK con imágenes de contenedor Lambda, compilación de modelos de API Smithy y despliegues de servidores MCP / agentes Strands). Estos objetivos construyen imágenes Docker de **Linux**, lo que requiere un daemon Docker de Linux.

En macOS y Linux esto funciona de inmediato. En Windows, Docker utiliza por defecto el **modo de contenedor de Windows**, por lo que se necesita un paso de configuración adicional para proporcionar un entorno de compilación Linux. Hay dos opciones:

## Opción A: Docker Desktop (Recomendado)

[Docker Desktop](https://www.docker.com/products/docker-desktop/) incluye una VM Linux administrada y maneja de forma transparente las compilaciones de contenedores Linux. Esta es la ruta más simple para el desarrollo local.

<Steps>
1. **Instalar Docker Desktop** desde [docker.com](https://www.docker.com/products/docker-desktop/).

2. **Habilitar el backend de WSL 2** (predeterminado en instalaciones modernas):
   - Abrir Docker Desktop **Settings > General**
   - Asegurarse de que **Use the WSL 2 based engine** esté marcado

3. **Verificar** abriendo una terminal y ejecutando:
   ```bash
   docker version
   docker buildx ls
   ```
   Deberías ver `OS/Arch: linux/amd64` en la sección del servidor.
</Steps>

Eso es todo — Docker Desktop administra la VM Linux, Buildx y el soporte multiplataforma por ti.

## Opción B: Docker Engine en WSL 2 (Sin Docker Desktop)

Si prefieres no instalar Docker Desktop (por ejemplo, debido a licencias), puedes instalar Docker Engine directamente dentro de una distribución Linux de WSL 2 y usar un pequeño script contenedor para que las herramientas del lado de Windows (como Nx) puedan invocar `docker` de forma transparente.

### Requisitos previos

- **Windows 10 versión 2004+** o **Windows 11**
- **WSL 2** habilitado (`wsl --install` si aún no lo está)

### Instalar Ubuntu + Docker Engine

<Steps>
1. **Instalar una distribución de Ubuntu en WSL 2:**

   ```powershell
   wsl --install Ubuntu-24.04 --no-launch
   wsl --set-default Ubuntu-24.04
   ```

2. **Instalar Docker Engine dentro de WSL 2:**

   Abrir el shell de Ubuntu (`wsl -d Ubuntu-24.04`) y ejecutar:

   ```bash
   # Add Docker's official GPG key and repository
   sudo apt-get update -qq
   sudo apt-get install -y ca-certificates curl
   sudo install -m 0755 -d /etc/apt/keyrings
   sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
     -o /etc/apt/keyrings/docker.asc
   sudo chmod a+r /etc/apt/keyrings/docker.asc
   echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc] \
     https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
     | sudo tee /etc/apt/sources.list.d/docker.list

   # Install Docker Engine + Buildx
   sudo apt-get update -qq
   sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
     docker-buildx-plugin
   ```

3. **Iniciar Docker y crear un constructor Buildx:**

   ```bash
   sudo dockerd &>/var/log/dockerd.log &

   # Wait for the daemon to be ready
   until docker info &>/dev/null; do sleep 1; done

   # Set up a Buildx builder for multi-platform builds
   docker buildx create --name linux-builder --driver docker-container --use
   docker buildx inspect --bootstrap --builder linux-builder
   ```

4. **Crear un contenedor `docker.cmd`** para que las herramientas del lado de Windows usen el daemon de WSL 2:

   Guardar lo siguiente como `C:\docker-wsl\docker.cmd` (o cualquier directorio que agregues a tu `PATH`):

   ```bat
   @echo off
   for /f "usebackq tokens=*" %%i in (`wsl -d Ubuntu-24.04 wslpath "%CD%"`) do set WSL_CWD=%%i
   if "%1"=="build" (
     echo %* | findstr /i /c:"--output" >nul
     if errorlevel 1 (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %* --load
     ) else (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %*
     )
   ) else (
     wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- docker %*
   )
   ```

   Luego agregar `C:\docker-wsl` al **frente** de tu `PATH` del sistema para que tenga prioridad sobre cualquier `docker.exe` existente.

5. **Verificar:**

   ```powershell
   docker version
   docker buildx ls
   ```
</Steps>

:::note
Con la Opción B, necesitas iniciar el daemon Docker dentro de WSL 2 cada vez que reinicies. Puedes automatizar esto agregando `sudo dockerd &>/dev/null &` a tu perfil de shell de WSL, o creando una entrada en el Programador de tareas de Windows.
:::

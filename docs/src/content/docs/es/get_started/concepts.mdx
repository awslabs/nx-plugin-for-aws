El `@aws/nx-plugin` es un plugin de [Nx](https://nx.dev/) que provee un conjunto de herramientas para simplificar el desarrollo e implementación de aplicaciones full-stack en AWS. Ofrece plantillas preconfiguradas tanto para aplicaciones como para código de Infraestructura como Código (IaC), reduciendo significativamente el tiempo de configuración inicial. El plugin maneja la complejidad de integración con servicios AWS manteniendo flexibilidad para personalizaciones.

Los usuarios simplemente seleccionan componentes de la lista de [Generadores](#generators) disponibles, proveen opciones de configuración y el `@aws/nx-plugin` genera el código inicial requerido. Este toolkit incluye generadores para crear APIs, sitios web, infraestructura, e incluso funcionalidades avanzadas como integración frontend-backend con clientes type-safe (incluyendo actualizaciones de archivos existentes mediante transformaciones AST).

## Construido sobre Nx

[Nx](https://nx.dev) es un framework de construcción inteligente que destaca en la gestión de proyectos políglotas complejos gracias a dos características clave: un sistema de construcción inteligente y generadores de código.

El sistema de construcción utiliza caché avanzado y análisis de grafos de dependencias para optimizar la ejecución de tareas. Solo reconstruye lo necesario basado en cambios de código, reduciendo drásticamente los tiempos de compilación. Las tareas pueden ejecutarse en paralelo, con Nx determinando automáticamente el orden óptimo según dependencias del proyecto. Esto hace operaciones como testing, linting y building altamente eficientes, especialmente en codebases grandes.

Los generadores sirven como herramientas de scaffolding automatizado que crean estructuras de código consistentes. Pueden generar desde componentes individuales hasta aplicaciones completas, siguiendo plantillas y patrones predefinidos. Los equipos pueden personalizar estos generadores para cumplir con sus estándares específicos.

Juntas, estas características hacen a Nx particularmente efectivo para manejar monorepos y asegurar prácticas de desarrollo consistentes entre equipos, manteniendo alta velocidad de desarrollo.

## Generadores

Todos los componentes disponibles en este plugin se implementan como [Generadores](https://nx.dev/features/generate-code) que automatizan tareas repetitivas y permiten scaffolding consistente tanto en proyectos nuevos como existentes.

Los generadores pueden invocarse mediante la [CLI de Nx](https://nx.dev/features/generate-code#invoke-generators) o el [plugin IDE Nx Console](https://nx.dev/getting-started/editor-setup).

## Abierto a modificaciones

Todo el código generado es tu código, lo que significa que puedes editarlo libremente. Esto simplifica drásticamente la experiencia del desarrollador: si algo no te gusta del código generado, puedes modificarlo directamente.

## Dependencias mínimas

El `@aws/nx-plugin` busca mantener las dependencias al mínimo. Los requisitos globales dependen únicamente de los generadores que uses.

Por ejemplo, generadores basados en Typescript solo requieren [Node](https://nodejs.org/en/download). Para proyectos Python, [UV](https://docs.astral.sh/uv/) es el único requisito.

<Aside>
  UV fue elegido como gestor de paquetes Python por varias razones:
  - Herramienta única que reemplaza `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv`, etc.
  - 10-100x más rápido que `pip`
  - Soporte nativo para [workspaces/monorepos](https://docs.astral.sh/uv/concepts/projects/workspaces/)
</Aside>

## Política de versión única

La configuración predeterminada del monorepo usa una [política de versión única](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) para proyectos Node y Python.

Esto significa que todos los proyectos dentro del monorepo usan la misma versión de dependencias por defecto, reduciendo problemas de incompatibilidad entre paquetes.

Para Node: un solo lockfile en la raíz con un único `node_modules`. Las nuevas dependencias se agregan al `package.json` raíz.

Para Python: un solo `.venv` en la raíz del monorepo con todas las dependencias. Cada proyecto Python tiene su propio `pyproject.toml`, pero las versiones se gestionan mediante el workspace de uv y se registran en el archivo `uv.lock` raíz.

## Seguridad de tipos

El `@aws/nx-plugin` emplea type-safety para simplificar la experiencia del desarrollador mediante autocompletado en IDEs, eliminando errores en runtime comunes en implementaciones no type-safe. Todos los componentes incluidos son type-safe por defecto.

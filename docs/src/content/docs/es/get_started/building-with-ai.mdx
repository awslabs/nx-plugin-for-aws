El Plugin Nx para AWS incluye un [Skill](https://code.claude.com/docs/en/skills)/[Power](https://kiro.dev/docs/powers/) y un [Servidor MCP](https://modelcontextprotocol.io/introduction) que juntos permiten a los asistentes de IA trabajar eficazmente con el plugin. Al utilizarlos, puedes acelerar tu flujo de trabajo de desarrollo con IA mientras te beneficias de que tus proyectos se generen de manera determinista — gastando menos tiempo y menos de tu ventana de contexto en configurar los componentes principales.

## Configura tu Asistente de IA

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    El Plugin Nx para AWS se distribuye como un [Kiro Power](https://kiro.dev/docs/powers/) que agrupa el servidor MCP con documentación de orientación y guías de flujo de trabajo.

    <Steps>

    1. Abre el panel de Powers en Kiro y haz clic en **Add power from GitHub**
    2. Ingresa la siguiente URL:

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. Haz clic en **Install**

    </Steps>

    Kiro configura automáticamente el servidor MCP — no se requiere configuración adicional.
  </TabItem>
  <TabItem label="Kiro CLI">
    Ejecuta el siguiente comando:

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    Consulta la [documentación MCP de Kiro CLI](https://kiro.dev/docs/cli/mcp/) para más detalles.
  </TabItem>
  <TabItem label="Claude Code">
    El Plugin Nx para AWS se distribuye como un [plugin de Claude Code](https://code.claude.com/docs/en/plugins) que agrupa un [Skill](https://code.claude.com/docs/en/skills) con el servidor MCP.

    Ejecuta los siguientes comandos dentro de Claude Code, uno a la vez:

    ```
    /plugin marketplace add awslabs/nx-plugin-for-aws
    ```

    ```
    /plugin install nx-plugin-for-aws@nx-plugin-for-aws
    ```

    Luego ejecuta `/reload-plugins` para activar el plugin.
  </TabItem>
  <TabItem label="Cursor">
    Agrega lo siguiente a `~/.cursor/mcp.json` (o `.cursor/mcp.json` en tu proyecto):

    <Snippet name="mcp/config" />

    Consulta la [documentación MCP de Cursor](https://cursor.com/docs/mcp) para más detalles.
  </TabItem>
  <TabItem label="Codex">
    Agrega lo siguiente a `~/.codex/config.toml`:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    Consulta la [documentación de configuración de Codex](https://developers.openai.com/codex/config-reference) para más detalles.
  </TabItem>

## Comienza con Vibe Coding

¡Pídele a tu asistente de IA que construya algo usando el `nx-plugin-for-aws`! Típicamente creará un workspace, generará componentes con los generadores aplicables, y luego completará la lógica de negocio.

Prueba un prompt como:

> *"Construye una aplicación multi-agente usando el Plugin Nx para AWS. Quiero un frontend React que se comunique con un agente orquestador backend que delegue a agentes especializados de investigación y escritura."*

O para un punto de partida más pequeño:

> *"Usa el Plugin Nx para AWS para generar un sitio web React respaldado por una API tRPC, con autenticación Cognito e infraestructura CDK."*

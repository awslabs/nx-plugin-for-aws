## Tarea 1: Configurar el servidor de desarrollo local

Para comenzar a construir la interfaz de usuario, necesitaremos configurar nuestro servidor de desarrollo local para apuntar a nuestro sandbox desplegado. Para esto, ejecuta el siguiente comando:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Este comando descargará el `runtime-config.json` que está desplegado y lo almacenará localmente en la carpeta `packages/game-ui/public`.

Para iniciar el servidor de desarrollo, ejecuta el siguiente comando:

<NxCommands commands={["serve game-ui"]} />

Abre el sitio web local en tu navegador, donde se te pedirá iniciar sesión y seguir las indicaciones para crear un nuevo usuario. Una vez completado, deberías ver el sitio web base:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution" title="Mantener el servidor de desarrollo activo">
Recomendamos mantener el servidor de desarrollo en ejecución durante el resto de este módulo, ya que recargará automáticamente los cambios que realicemos.
</Aside>

:::tip[Uso de comandos Nx]
Puedes ejecutar comandos Nx en su lugar para iniciar cualquier servidor API local conectado, permitiéndote iterar tanto en tu API tRPC como en tu sitio web simultáneamente. <NxCommands commands={["serve-local game-ui"]} env={{RUNTIME_CONFIG_APP_ID:"xxxx"}} />

Utiliza el valor de salida del despliegue de CDK `RuntimeConfigApplicationId` para la variable de entorno `RUNTIME_CONFIG_APP_ID`.
:::

## Tarea 2: Crear una nueva ruta '/game'

Mostraremos las capacidades de `@tanstack/react-router` creando una nueva ruta tipada.
Para esto, crea un archivo vacío en la siguiente ubicación: `packages/game-ui/src/routes/game/index.tsx`. Notarás que el archivo se actualiza inmediatamente.

El `@tanstack/react-router` configura automáticamente tu nueva ruta y el archivo que acabas de crear ya está poblado con la ruta:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/game/')({
  component: RouteComponent,
})

function RouteComponent() {
  return <div>Hello "/game/"!</div>
}
```

Si navegas a `http://localhost:4200/game`, verás que tu nueva página se ha renderizado.

<Image src={baselineGamePng} alt="baseline-game.png" width="800" height="600" />

Actualiza el archivo `index.tsx` para cargar nuestra nueva ruta `/game` por defecto. Cuando actualices el campo `to`, tendrás una lista de rutas tipadas para elegir.

## Tarea 4: Integración del Agente de Historia

Recuerda que en el <Link path="get_started/tutorials/dungeon-game/1#task-7-connect-your-components">Módulo 1</Link>, usamos el generador `connection` para conectar nuestra Game UI al Agente de Historia. Esto configuró un cliente OpenAPI tipado para interactuar con el agente, junto con hooks y providers.

El generador de conexión creó lo siguiente para nosotros:
- Un componente `StoryAgentProvider` que envuelve nuestra aplicación en `main.tsx`
- Un hook `useStoryAgent` para integración con TanStack Query
- Un hook `useStoryAgentClient` para acceso directo al cliente
- Tipos TypeScript generados desde la especificación OpenAPI del agente

Usaremos el hook `useStoryAgentClient` en nuestro componente de juego para transmitir respuestas de historia desde el agente.

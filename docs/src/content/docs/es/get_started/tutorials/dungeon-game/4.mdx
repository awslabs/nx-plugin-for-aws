## Tarea 1: Configurar el servidor de desarrollo local

Para comenzar a construir la interfaz de usuario, necesitaremos configurar nuestro servidor de desarrollo local para apuntar a nuestro sandbox desplegado. Para esto, ejecuta el siguiente comando:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Este comando descargará el `runtime-config.json` que está desplegado y lo almacenará localmente en la carpeta `packages/game-ui/public`.

Para iniciar el servidor de desarrollo, ejecuta el siguiente comando:

<NxCommands commands={["serve game-ui"]} />

Abre el sitio web local en tu navegador. Se te pedirá iniciar sesión con el usuario de Cognito que creaste anteriormente. Una vez que hayas iniciado sesión, deberías ver el sitio web base.

<Aside type="caution" title="Mantener el servidor de desarrollo activo">
Recomendamos mantener el servidor de desarrollo en ejecución durante el resto de este módulo, ya que recargará automáticamente los cambios que realicemos.
</Aside>

## Tarea 4: Crear las páginas del juego

En el <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, conectamos nuestra `GameUI` al Agente de Historia. Debido a que el agente fue generado con `--protocol=AG-UI`, el generador de conexión configuró un `AguiProvider` de CopilotKit y un hook por agente (`useAguiStoryAgent`) en lugar del cliente OpenAPI habitual. Podemos renderizar el chat con un solo componente `<CopilotChat />` y controlarlo mediante `threadId` (que usamos como el id de sesión de AgentCore).

Crea dos nuevos archivos bajo `packages/game-ui/src/routes/game/`:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="Parámetros de ruta">
La sintaxis `$playerName` indica a `@tanstack/react-router` que trate `playerName` como un [parámetro de ruta](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). Usamos el nombre del jugador como el `threadId` de AG-UI para que cada juego tenga su propio historial de chat, y lo rellenamos a 33 caracteres para satisfacer el requisito de longitud del id de sesión de AgentCore.
</Aside>

<Aside type="note" title="Inicializar el primer mensaje">
En el primer montaje verificamos `agent.messages.length === 0` y, si está vacío, enviamos un único mensaje de usuario de apertura: `I am <player>, playing in the <genre> genre.` El prompt del sistema le indica al agente que comience la historia inmediatamente desde esa señal. A partir de entonces, cada interacción (incluyendo actualizaciones de página y regresos posteriores) simplemente recarga la conversación a través de la sesión AG-UI — no se requiere un endpoint de historial personalizado.
</Aside>
</TabItem>
</Tabs>

Una vez que guardes estos archivos, tu servidor de desarrollo local (http://localhost:4200/) debería tener ahora el juego listo para jugar.

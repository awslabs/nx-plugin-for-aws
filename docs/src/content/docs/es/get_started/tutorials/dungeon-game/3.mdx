## Tarea 1: Implementar el Agente de Historia

El Agente de Historia es un agente de [Strands](https://strandsagents.com/) expuesto a través del protocolo [AG-UI](https://ag-ui.com/). Cada sesión de AG-UI (identificada por un `threadId`) se mapea a una instancia de Agente de larga duración en el servidor, por lo que el historial de conversación del agente se mantiene en memoria durante la vida de la sesión.

Actualizaremos el prompt del sistema del agente para jugar una aventura de mazmorra y eliminaremos la herramienta de muestra.

### Implementación del agente

Actualiza `packages/story/dungeon_adventure_story/agent/agent.py`:

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

El prompt del sistema le indica al agente que el primer mensaje será de la forma `I am <player>, playing in the <genre> genre.` — nuestra interfaz de usuario envía exactamente esto como el mensaje de usuario inicial para que el agente tenga todo lo que necesita para comenzar la historia. Debido a que la biblioteca AG-UI retiene automáticamente el historial de conversación de cada hilo, no necesitamos pasar el nombre del jugador o el género en turnos posteriores.

El generador de conexiones ya configuró el cliente MCP de Inventory en el <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, por lo que el agente tiene acceso a las herramientas de inventario.

:::note[¿Dónde está `main.py`?]
El wrapper FastAPI del protocolo AG-UI maneja la traducción de solicitud/respuesta por nosotros. El `main.py` que produjo el generador ya crea el wrapper alrededor de `get_agent(...)` — no necesitamos modificarlo.
:::

### Probar tu Agente

Puedes probar tu Agente de dos formas:
<ul>
<li>Iniciando una instancia local del servidor del Agente y chateando con él de forma interactiva usando el objetivo `agent-chat`, o</li>
<li>Llamando a la API desplegada usando curl con un token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  El objetivo `agent-chat` inicia el agente localmente (junto con sus dependencias, a través de `agent-serve-local`) y te lleva a un prompt de chat interactivo. Ejecuta:

    <NxCommands commands={["agent-chat dungeon_adventure.story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    Si quieres chatear con el agente contra tu servidor MCP **desplegado** en lugar de uno local, inicia el servidor directamente con `agent-serve` y pasa `RUNTIME_CONFIG_APP_ID` / `AWS_REGION`:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    luego en otro shell ejecuta `agent-chat-cli agui http://localhost:8081/invocations`.
    :::

    Cuando se te solicite, intenta comenzar el juego con algo como:

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="Desplegado">
    Para probar el agente desplegado, necesitarás autenticarte con Cognito y obtener un token JWT. Primero, configura tus variables de entorno:

    ```bash
    # Establece tu User Pool ID y Client ID de Cognito desde los outputs de CDK
    export POOL_ID="<UserPoolId from CDK outputs>"
    export CLIENT_ID="<UserPoolClientId from CDK outputs>"
    export REGION="<your-region>"
    ```

    Crea un usuario de prueba y obtén un token de autenticación:

    ```bash
    # Disable MFA to simplify user creation
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Create User
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Set Permanent Password (replace with something more secure!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Authenticate User and capture ID Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoca el agente desplegado usando la URL de Bedrock AgentCore Runtime. El protocolo AG-UI espera un payload `RunAgentInput` — uno mínimo que envía el mensaje de usuario inicial se ve así:

    ```bash
    # Set the Story Agent ARN from CDK outputs
    export AGENT_ARN="<StoryAgentArn from CDK outputs>"

    # URL-encode the ARN
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construct the invocation URL
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoke the agent
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[Codificación URL]
    La codificación URL es necesaria para formatear correctamente el ARN en el endpoint de API de Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Si el comando se ejecuta correctamente, deberías comenzar a ver eventos Server-Sent de AG-UI transmitiéndose — los eventos `TEXT_MESSAGE_CONTENT` llevan el texto de la historia fragmento por fragmento.

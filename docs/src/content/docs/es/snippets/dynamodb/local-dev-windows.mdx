---
title: Precaución de Desarrollo Local en Windows
---

Detener `dev` (por ejemplo, con `Ctrl+C`) elimina automáticamente el contenedor de DynamoDB Local, pero conserva el volumen nombrado para que tus datos persistan entre reinicios.

:::caution[Windows]
Debido a limitaciones con el manejo de señales en Windows, el contenedor no se elimina automáticamente cuando se detiene `dev`. Necesitarás eliminarlo manualmente:

```bash
<engine> rm -f <scope>-dynamodb
```

Reemplaza `<engine>` con tu motor de contenedores (`docker` o `finch`) y `<scope>` con el scope de tu espacio de trabajo Nx (por ejemplo, `proj`).
:::

title: Requisitos SSL al Conectar sin RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

Para conexiones directas al clúster Aurora desde entornos de ejecución Lambda de Node.js 20 o posterior, cargue el paquete de CA de Amazon RDS configurando `NODE_EXTRA_CA_CERTS`:

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
const api = new Api(this, 'Api', {
  integrations: Api.defaultIntegrations(this)
    .withDefaultOptions({
      environment: {
        NODE_EXTRA_CA_CERTS: '/var/runtime/ca-cert.pem',
      },
    })
    .build(),
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "api" {
  source = "..."
  ...

  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/var/runtime/ca-cert.pem"
  }
}
```

</Fragment>
</Infrastructure>

Para más detalles, consulte los [requisitos SSL/TLS para conexiones de Amazon RDS](https://docs.aws.amazon.com/lambda/latest/dg/services-rds.html#services-rds-tls) de AWS Lambda y la [documentación TLS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/rds-proxy.howitworks.html#rds-proxy-security.tls) de Amazon RDS Proxy. Al usar RDS Proxy, no necesita configurar el paquete de CA de RDS en su función Lambda.

---
title: Requisitos SSL al Conectar sin RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

El generador de conexión actualiza el Dockerfile para instalar el paquete de CA de Amazon RDS en `/usr/local/share/ca-certificates/rds-bundle.crt`. Configura `NODE_EXTRA_CA_CERTS` con esa ruta para que Node.js confíe en el certificado al conectarse sin RDS Proxy:

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
new MyMcpServer(this, 'MyMcpServer', {
  ...
  environmentVariables: {
    NODE_EXTRA_CA_CERTS: '/usr/local/share/ca-certificates/rds-bundle.crt',
  },
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_mcp_server" {
  ...
  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/usr/local/share/ca-certificates/rds-bundle.crt"
  }
}
```

</Fragment>
</Infrastructure>

Para más detalles, consulta la [documentación de SSL/TLS de Amazon RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/UsingWithRDS.SSL.html). Cuando uses RDS Proxy, no necesitas configurar `NODE_EXTRA_CA_CERTS`.

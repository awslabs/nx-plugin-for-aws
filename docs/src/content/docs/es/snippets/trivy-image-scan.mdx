---
title: Escaneo de Imágenes de Contenedor
---

La imagen Docker construida para este proyecto se escanea en busca de vulnerabilidades como parte de la construcción usando [Trivy](https://trivy.dev/), ejecutándose desde la [imagen Trivy alojada en ECR](https://gallery.ecr.aws/aquasecurity/trivy).

El escaneo se ejecuta después de que se construye la imagen y **falla la construcción** si se encuentra alguna vulnerabilidad de severidad `HIGH` o `CRITICAL` que sea corregible. Las vulnerabilidades sin una corrección disponible se ignoran para que los avisos de imagen base no accionables no bloqueen tu construcción.

El escaneo utiliza el mismo motor de contenedores que tu construcción de imagen (`docker` o `finch`), por lo que no se requieren herramientas adicionales.

:::caution[Imágenes Base Limpias]
El `Dockerfile` generado utiliza una imagen base sin vulnerabilidades conocidas de tipo `HIGH` o `CRITICAL` al momento de la generación, y actualiza las herramientas incluidas (como `npm`) para mantenerlo así. Las nuevas vulnerabilidades se divulgan con el tiempo, por lo que una construcción que pasa hoy puede fallar más tarde a medida que se actualiza la base de datos de vulnerabilidades. Cuando esto suceda, actualiza a una imagen base más reciente donde la vulnerabilidad esté corregida, o suprime el hallazgo como se describe a continuación.
:::

#### Supresión de Hallazgos de Trivy

Puede haber casos en los que desees suprimir una vulnerabilidad específica, por ejemplo, cuando aún no hay una corrección disponible y has evaluado el riesgo como aceptable.

Agrega el ID de vulnerabilidad (uno por línea) al archivo `.trivyignore` en la raíz de tu proyecto:

```text
// .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

Para más detalles sobre el filtrado de hallazgos, consulta la [documentación de filtrado de Trivy](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids).

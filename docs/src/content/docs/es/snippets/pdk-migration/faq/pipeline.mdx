El PDK proporcionaba un [`PDKPipelineProject`](https://aws.github.io/aws-pdk/developer_guides/pipeline/index.html) que configuraba un proyecto de infraestructura CDK y utilizaba un constructo CDK que encapsulaba algunos recursos de [CDK Pipelines](https://docs.aws.amazon.com/cdk/api/v2/docs/aws-cdk-lib.pipelines-readme.html).

Para migrar desde esto, puedes usar directamente los constructos de CDK Pipelines. Sin embargo, en la práctica suele ser más sencillo usar soluciones como GitHub Actions o GitLab CI/CD, donde defines [CDK Stages](https://docs.aws.amazon.com/cdk/v2/guide/stages.html) y ejecutas el comando de despliegue para la etapa correspondiente directamente.

Fija una versión específica del motor Aurora.

Por defecto, la imagen del contenedor de base de datos local generada coincide con la versión predeterminada del motor Aurora. Si cambias la versión del motor Aurora, se recomienda también usar una versión de imagen de contenedor local que coincida para máxima compatibilidad. Consulta las notas de lanzamiento de AWS para [versiones de Aurora PostgreSQL](https://docs.aws.amazon.com/AmazonRDS/latest/AuroraPostgreSQLReleaseNotes/aurorapostgresql-release-calendar.html) y [versiones de Aurora MySQL](https://docs.aws.amazon.com/AmazonRDS/latest/AuroraMySQLReleaseNotes/AuroraMySQL.Updates.30Updates.html) para identificar la versión correspondiente de la base de datos comunitaria.

La imagen de la base de datos local se configura en el campo `localDev.image` del archivo `config.json` generado en la raíz de tu proyecto de base de datos. Actualiza ese valor cuando cambies las versiones del motor.

El generador `connection` puede conectar tu <Link path="guides/py-agent">Python Agent</Link> a un agente [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) remoto — ya sea <Link path="guides/ts-agent">TypeScript</Link> o <Link path="guides/py-agent">Python</Link> — para que tu agente pueda delegar a otro agente como una herramienta.

El generador configura todo el cableado necesario para que tu agente pueda descubrir e invocar al agente A2A remoto, tanto cuando se despliega en AWS (a través de Bedrock AgentCore) como cuando se ejecuta localmente.

## Requisitos previos

Antes de usar este generador, asegúrate de tener:

1. Un proyecto Python con un componente <Link path="guides/py-agent">Python Agent</Link> (Strands o LangChain)
2. Un proyecto con un componente Agent generado con `--protocol=A2A` y `--auth=IAM` (ya sea <Link path="guides/ts-agent">`ts#agent`</Link> o <Link path="guides/py-agent">`py#agent`</Link>)
3. Ambos componentes creados con `infra: agentcore`

## Salida del generador

El generador crea un proyecto Python compartido `agent_connection` en `packages/common/agent_connection/` (si aún no existe). Los módulos de cliente por conexión se generan en este proyecto compartido:

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py Re-exporta clientes por conexión
    - core
      - agentcore\_endpoints.py Resolución de ARN/URL independiente del framework
      - agentcore\_a2a\_client\_config.py Configuración de cliente A2A independiente del framework (`ClientConfig` firmado)
      - agentcore\_a2a\_client\_\<framework>.py Cliente A2A que envuelve la configuración para el framework de tu agente
      - auth/ `httpx.Auth` independiente del framework para SigV4 / reenvío de sesión
    - app
      - \<target\_agent\_name>\_client\_\<framework>.py Cliente A2A por conexión para cada agente A2A

</FileTree>

El sufijo del cliente coincide con el framework de tu agente (`_strands` o `_langchain`). Ambos envuelven el mismo `ClientConfig` firmado independiente del framework: el cliente Strands envuelve un `A2AAgent` de Strands, mientras que el cliente LangChain utiliza el [a2a SDK](https://pypi.org/project/a2a-sdk/) directamente, por lo que un agente LangChain no incluye ninguna dependencia de Strands.

Además, el generador:
- Transforma el `agent.py` de tu agente para registrar el agente A2A remoto como una herramienta usando `@tool`
- Agrega el proyecto `agent_connection` como una dependencia de espacio de trabajo de tu proyecto de agente
- Actualiza el objetivo `serve-local` del agente para que dependa del objetivo `serve-local` del agente de destino

## Usar el agente A2A conectado

El generador transforma el `agent.py` de tu agente para envolver el agente A2A remoto como una herramienta. El agente remoto se registra con un delegado decorado con `@tool` — la importación del decorador y el constructor del agente difieren según el framework:

<Tabs syncKey="agent-framework">
<TabItem label="Strands" _filter={{ framework: 'strands' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,8-13,15}
from contextlib import contextmanager
from strands import Agent, tool

from my_scope_agent_connection import RemoteAgentClientStrands

@contextmanager
def get_agent():
    remote_agent = RemoteAgentClientStrands.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    yield Agent(
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
<TabItem label="LangChain" _filter={{ framework: 'langchain' }}>
```python title="packages/my-project/my_module/agent/agent.py" {4,7-12,14}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse
from langchain_core.tools import tool

from my_scope_agent_connection import RemoteAgentClientLangChain

def get_agent():
    remote_agent = RemoteAgentClientLangChain.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
</Tabs>

Ambos clientes son directamente invocables, devolviendo la respuesta del agente remoto, y envuelven un `httpx.AsyncClient` que firma solicitudes con SigV4 cuando se despliega en AWS y utiliza un endpoint simple `http://localhost:<port>/` cuando `SERVE_LOCAL=true`. El cliente Strands envuelve un `A2AAgent` de Strands; el cliente LangChain utiliza el a2a SDK directamente.

El ID de sesión de AgentCore se propaga automáticamente al agente remoto a través del encabezado `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id`, independientemente del framework: el servidor del agente vincula la sesión de la solicitud entrante en un contexto asíncrono, y el `httpx.Auth` firmado del cliente de conexión lo estampa en cada llamada saliente — asegurando consistencia para [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html).

## Desarrollo local

El generador configura el objetivo `serve-local` del agente anfitrión para:
1. Iniciar los agentes A2A conectados automáticamente
2. Establecer `SERVE_LOCAL=true` para que el cliente generado se conecte directamente a `http://localhost:<port>/` en lugar de AgentCore

Ejecuta el agente localmente con:

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Esto iniciará tanto el agente anfitrión como todos los agentes A2A conectados, con el agente anfitrión llamando a los agentes remotos a través de HTTP simple en sus puertos locales asignados.

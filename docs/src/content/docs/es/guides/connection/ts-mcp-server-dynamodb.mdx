El generador `connection` conecta un <Link path="guides/ts-mcp-server">servidor MCP de TypeScript</Link> a un proyecto de <Link path="guides/ts-dynamodb">DynamoDB de TypeScript</Link>, configurando el desarrollo local para que ambos se inicien juntos automáticamente.

## Requisitos previos

Antes de usar este generador, asegúrate de tener:

1. Un proyecto <Link path="guides/ts-mcp-server">`ts#mcp-server`</Link>
2. Un proyecto <Link path="guides/ts-dynamodb">`ts#dynamodb`</Link>

## Uso

### Ejecutar el generador

<RunGenerator generator="connection" />

Selecciona tu proyecto de servidor MCP como origen y tu proyecto de DynamoDB como destino. Si el proyecto contiene múltiples componentes de servidor MCP, especifica `sourceComponent` para desambiguar.

### Opciones

<GeneratorParameters generator="connection" />

## Salida del generador

El generador actualiza el target `<mcp-server-name>-dev` del servidor MCP en `project.json` para que dependa del target `dev` del proyecto de DynamoDB. No se modifican archivos de código fuente.

## Usar DynamoDB en herramientas

Importa las factories de entidades del paquete de DynamoDB y úsalas dentro de `createServer`:

```ts title="packages/my-service/src/my-mcp/server.ts"
import { createExampleEntity } from ':my-scope/my-table';

export const createServer = async () => {
  const server = new McpServer({ name: 'my-service', version: '1.0.0' });

  server.tool('list_examples', 'List all example items', {}, async () => {
    const entity = await createExampleEntity();
    const result = await entity.scan.go();
    return {
      content: [{ type: 'text', text: JSON.stringify(result.data) }],
    };
  });

  return server;
};
```

## Infraestructura

Para permitir que el servidor MCP acceda a la tabla de DynamoDB, otorga los permisos necesarios en tu infraestructura.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table');
const myMcpServer = new MyMcpServer(this, 'MyMcpServer');

table.grantReadWriteData(myMcpServer);
```

`grantReadWriteData` otorga tanto los permisos de DynamoDB como de KMS al rol de ejecución del servidor MCP.
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source = "../../common/terraform/src/app/dynamodb/my-table"
}

resource "aws_iam_role_policy" "dynamodb_access" {
  role = module.my_mcp_server.lambda_role_name

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Effect = "Allow"
        Action = [
          "dynamodb:GetItem", "dynamodb:PutItem", "dynamodb:UpdateItem",
          "dynamodb:DeleteItem", "dynamodb:Query", "dynamodb:Scan",
          "dynamodb:BatchGetItem", "dynamodb:BatchWriteItem",
        ]
        Resource = [
          module.my_table.table_arn,
          "${module.my_table.table_arn}/index/*",
        ]
      },
      {
        Effect = "Allow"
        Action = [
          "kms:Encrypt",
          "kms:Decrypt",
          "kms:ReEncrypt*",
          "kms:GenerateDataKey*",
          "kms:DescribeKey",
        ]
        Resource = [module.my_table.kms_key_arn]
      },
    ]
  })
}
```

El generador `connection` conecta una <Link path="guides/ts-smithy-api">API Smithy</Link> a un proyecto de <Link path="guides/ts-dynamodb">DynamoDB</Link>, configurando el desarrollo local para que ambos se inicien juntos automáticamente.

## Requisitos previos

Antes de usar este generador, asegúrate de tener:

1. Un proyecto <Link path="guides/ts-smithy-api">`ts#smithy-api`</Link> (backend TypeScript)
2. Un proyecto <Link path="guides/ts-dynamodb">`ts#dynamodb`</Link>

## Uso

### Ejecutar el generador

<RunGenerator generator="connection" />

## Salida del generador

El generador actualiza el `project.json` de tu API Smithy para agregar una dependencia desde su target `serve-local` al target `serve-local` del proyecto de DynamoDB. No se modifican archivos de código fuente.

## Usar DynamoDB en operaciones

Importa las factories de entidades desde el paquete de DynamoDB y úsalas dentro de las implementaciones de tus operaciones:

```ts title="packages/api/src/operations/list-examples.ts"
import { createExampleEntity } from ':my-scope/my-table';
import {
  ListExamplesOperationInput,
  ListExamplesOperationOutput,
} from '../generated/ssdk/index.js';
import { ServiceContext } from '../context.js';

export const listExamples = async (
  _input: ListExamplesOperationInput,
  _ctx: ServiceContext,
): Promise<ListExamplesOperationOutput> => {
  const entity = await createExampleEntity();
  const result = await entity.scan.go();
  return { items: result.data };
};
```

## Desarrollo local

El generador configura el target `serve-local` de tu API Smithy para que dependa del target `serve-local` del proyecto de DynamoDB. Al ejecutar:

<NxCommands commands={["serve-local <api-project-name>"]} />

se inicia automáticamente la instancia local de DynamoDB junto con tu API. La variable de entorno `SERVE_LOCAL=true` se establece automáticamente, por lo que `getDynamoDBClient()` y `resolveTableName()` se conectan a la instancia local de DynamoDB Local en lugar de AWS.

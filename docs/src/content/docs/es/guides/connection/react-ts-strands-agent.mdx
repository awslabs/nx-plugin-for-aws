Nx Plugin for AWS proporciona un generador para integrar rápidamente tu <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> con un sitio web de React. Configura toda la configuración necesaria para conectarse a tu agente a través de tRPC sobre WebSocket, incluyendo soporte de autenticación de AWS IAM y Cognito. La integración proporciona seguridad de tipos completa de extremo a extremo entre tu frontend y el enrutador tRPC del agente.

## Requisitos previos

Antes de usar este generador, asegúrate de tener:

1. Un sitio web de React (generado usando el <Link path="guides/react-website">generador `ts#react-website`</Link>)
2. Un TypeScript Strands Agent (generado usando el <Link path="guides/ts-strands-agent">generador `ts#strands-agent`</Link>)
3. Cognito Auth agregado a través del <Link path="/guides/react-website-auth">generador `ts#react-website-auth`</Link>

## Uso

### Ejecutar el generador

## Salida del generador

El generador crea la siguiente estructura en tu aplicación React:

<FileTree>

- src
  - components
    - \<AgentName>AgentClientProvider.tsx Configura el cliente tRPC WebSocket y los enlaces al enrutador tRPC de tu agente
    - QueryClientProvider.tsx Proveedor del cliente TanStack React Query
  - hooks
    - useSigV4.tsx Hook para firmar solicitudes con SigV4 (solo IAM)
    - use\<AgentName>Agent.tsx Hooks que devuelven el proxy de opciones tRPC y el cliente tRPC vanilla

</FileTree>

Además, instala las dependencias requeridas:

  - `@trpc/client`
  - `@trpc/tanstack-react-query`
  - `@tanstack/react-query`
  - `aws4fetch` (si se usa autenticación IAM)

## Cómo funciona

### Conexión WebSocket

El cliente generado se conecta a tu Strands Agent a través de tRPC sobre WebSocket. El agente expone un enrutador tRPC (incluyendo la suscripción `invoke` para transmitir respuestas del agente) sobre un endpoint WebSocket.

- **Desplegado**: El ARN del runtime del agente se carga desde <Link path="guides/runtime-config">Runtime Configuration</Link>. Ejecutar este generador de conexión también parchea la construcción CDK/Terraform generada del agente para publicar su ARN en el `runtime-config.json` del sitio web (bajo el namespace `connection`), de modo que solo los agentes que conectes explícitamente se expongan al frontend. El ARN se convierte a una URL WebSocket siguiendo el protocolo WebSocket de Bedrock AgentCore Runtime: `wss://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/ws`
- **Desarrollo local**: Al ejecutar con `serve-local`, la anulación de configuración del runtime establece el valor a una URL `ws://` local (por ejemplo, `ws://localhost:8081/ws`), y el cliente se conecta directamente

### Autenticación

El código generado maneja la autenticación dependiendo de la configuración de tu agente:

- **IAM** (predeterminado): Usa URLs prefirmadas AWS SigV4 para autenticar la conexión WebSocket. Las credenciales se obtienen del Cognito Identity Pool configurado con la autenticación de tu sitio web. En modo `serve-local`, la firma se omite automáticamente cuando <Link path="guides/react-website#runtime-configuration">`runtime-config.json`</Link> no está presente
- **Cognito**: Incrusta el token de acceso JWT en el encabezado `Sec-WebSocket-Protocol` como un token bearer codificado en base64url, siguiendo el [protocolo de autenticación WebSocket de AgentCore](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-get-started-websocket.html)

## Usando el código generado

### Usando TanStack Query

El caso de uso más común es transmitir la respuesta del agente usando la suscripción `invoke` con el hook `use<AgentName>Agent`, que devuelve un [proxy de opciones tRPC](https://trpc.io/docs/client/tanstack-react-query) para usar con TanStack Query:

```tsx {1-2,7-22}
import { useSubscription } from '@trpc/tanstack-react-query';
import { useMyAgentAgent } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const trpc = useMyAgentAgent();

  const subscription = useSubscription(
    trpc.invoke.subscriptionOptions(
      { message: 'What can you help me with?' },
      {
        enabled: true,
        onStarted: () => {
          console.log('Agent started responding');
        },
        onData: (token) => {
          console.log('Received token:', token);
        },
        onError: (error) => {
          console.error('Agent error:', error);
        },
      },
    ),
  );

  return (
    <div>
      <p>Status: {subscription.status}</p>
      {subscription.data && <p>Latest token: {subscription.data}</p>}
      {subscription.error && <p>Error: {subscription.error.message}</p>}
    </div>
  );
}
```

:::note[Proxy de opciones tRPC]
Para más detalles sobre el proxy de opciones tRPC y la integración con TanStack Query, consulta la [documentación de tRPC TanStack React Query](https://trpc.io/docs/client/tanstack-react-query).
:::

### Usando el cliente tRPC Vanilla

El hook `use<AgentName>AgentClient` proporciona acceso al [cliente tRPC vanilla](https://trpc.io/docs/client/vanilla) para tener más control sobre el ciclo de vida de la suscripción:

```tsx {2,5,9-22,25}
import { useState } from 'react';
import { useMyAgentAgentClient } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const client = useMyAgentAgentClient();
  const [messages, setMessages] = useState<string[]>([]);

  const sendMessage = (message: string) => {
    const subscription = client.invoke.subscribe(
      { message },
      {
        onData: (token) => {
          setMessages((prev) => [...prev, token]);
        },
        onComplete: () => {
          console.log('Agent finished');
        },
        onError: (error) => {
          console.error('Error:', error);
        },
      },
    );

    // Clean up when done
    return () => subscription.unsubscribe();
  };

  return (
    <div>
      <button onClick={() => sendMessage('Hello!')}>Send</button>
      <div>
        {messages.map((msg, i) => (
          <span key={i}>{msg}</span>
        ))}
      </div>
    </div>
  );
}
```

:::note[Cliente tRPC vanilla]
Para más detalles sobre el cliente tRPC vanilla, consulta la [documentación del cliente Vanilla de tRPC](https://trpc.io/docs/client/vanilla).
:::

## Desarrollo local

El generador de conexión configura automáticamente la integración con `serve-local` para tu sitio web de React:

1. Ejecutar `nx serve-local <website>` también iniciará el servidor local del agente
2. La configuración del runtime se anula para apuntar a la URL WebSocket local (por ejemplo, `ws://localhost:8081/ws`)
3. Al igual que con las APIs conectadas, la autenticación se omite en modo `serve-local` cuando <Link path="guides/react-website#runtime-configuration">`runtime-config.json`</Link> no está presente

:::tip[Recarga en caliente]
El sitio web y el agente conectado se recargarán automáticamente, permitiéndote iterar rápidamente en ambos juntos sin desplegar a AWS.
:::

## Seguridad de tipos

La integración proporciona seguridad de tipos completa de extremo a extremo. Tu IDE proporcionará autocompletado completo y verificación de tipos para todas las llamadas de procedimientos del agente. Los tipos se infieren automáticamente de la definición del enrutador tRPC de tu agente, asegurando que cualquier cambio en la API de tu agente se refleje inmediatamente en tu código frontend.

Este generador crea un nuevo proyecto TypeScript respaldado por [Amazon DynamoDB](https://aws.amazon.com/dynamodb/), utilizando [ElectroDB](https://electrodb.dev/) para el modelado de entidades con seguridad de tipos. Genera el código de aplicación y la infraestructura necesaria para aprovisionar y gestionar una tabla DynamoDB usando AWS CDK o Terraform, con soporte para diseño de tabla única y desarrollo local integrado mediante DynamoDB Local.

## Desarrollo Local

### Iniciar DynamoDB Local

El generador configura un objetivo `serve-local` que inicia una instancia de [DynamoDB Local](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/DynamoDBLocal.html) y crea la tabla:

<NxCommands commands={['run <project-name>:serve-local']} />

Esto automáticamente:
1. Descarga la imagen de DynamoDB Local (objetivo `pull-image`)
2. Inicia un contenedor
3. Crea una tabla local con índices predefinidos

### Modelado de Datos

El proyecto generado utiliza [ElectroDB](https://electrodb.dev/) para el modelado de entidades con seguridad de tipos en una única tabla DynamoDB. Agrega o actualiza archivos de entidad en `src/entities/`, utilizando la entidad de ejemplo generada como punto de partida.

Ejemplo de definición de entidad:

```ts title="packages/my-table/src/entities/example.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createExampleEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'example',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: {
          type: 'string',
          required: true,
        },
        createdAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: {
            field: 'pk',
            composite: ['id'],
          },
          sk: {
            field: 'sk',
            composite: [],
          },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

Para más detalles, consulta la [documentación de entidades de ElectroDB](https://electrodb.dev/en/modeling/entities/).

### Usar el Cliente DynamoDB

El archivo generado `src/client.ts` exporta dos utilidades clave:

- `getDynamoDBClient()` — devuelve un singleton en caché de `DynamoDBClient`. Cuando `SERVE_LOCAL=true`, se conecta a la instancia local de DynamoDB Local; de lo contrario, crea un cliente AWS usando la cadena de credenciales predeterminada.
- `resolveTableName()` — devuelve el nombre de la tabla DynamoDB. Cuando `SERVE_LOCAL=true`, devuelve la constante del nombre de tabla local; de lo contrario, obtiene el nombre de AWS AppConfig usando la variable de entorno `RUNTIME_CONFIG_APP_ID` y lo almacena en caché para llamadas posteriores.

### Detener DynamoDB Local

Detener `serve-local` (por ejemplo, con `Ctrl+C`) elimina automáticamente el contenedor de DynamoDB Local, pero conserva el volumen nombrado para que tus datos persistan entre reinicios.

:::caution[Windows]
Debido a limitaciones con el manejo de señales en Windows, el contenedor no se elimina automáticamente cuando se detiene `serve-local`. Deberás eliminarlo manualmente:

```bash
<engine> rm -f <scope>-dynamodb
```

Reemplaza `<engine>` con tu motor de contenedores (`docker` o `finch`) y `<scope>` con el scope de tu espacio de trabajo Nx (por ejemplo, `proj`).
:::

## Agregar/Eliminar Índices Secundarios Globales

Comienza actualizando `GLOBAL_SECONDARY_INDEXES` en `src/gsi.ts` — esta es la fuente de verdad para tus definiciones de GSI, y en la próxima ejecución de `serve-local` el script agregará o eliminará automáticamente índices en la tabla local para reflejar la nueva lista. Al usar CDK, estos cambios también se reflejan automáticamente en tu tabla de AWS en el próximo despliegue.

<OptionFilter when={{ iac: 'terraform' }}>
  A diferencia de CDK, los cambios en `src/gsi.ts` no se reflejan automáticamente en tu módulo Terraform. También debes actualizar manualmente tu módulo Terraform para agregar o eliminar las definiciones de GSI correspondientes antes de desplegar.
</OptionFilter>

## Conectarse a la Tabla

En cualquier proyecto TypeScript, importa las fábricas de entidades de tu paquete DynamoDB y úsalas directamente:

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

:::note[Configuración en tiempo de ejecución]
Cuando se ejecuta en AWS, `resolveTableName()` obtiene el nombre de la tabla de AWS AppConfig usando la variable de entorno `RUNTIME_CONFIG_APP_ID`. Los proyectos construidos con este plugin (APIs tRPC, APIs Smithy, agentes, servidores MCP) ya tienen esta variable configurada automáticamente. Para otros proyectos TypeScript, asegúrate de que `RUNTIME_CONFIG_APP_ID` esté configurada en el entorno de ejecución con el ID de aplicación de AppConfig aprovisionado por tu infraestructura. Para más información, consulta la <Link path="guides/runtime-config">guía de Configuración en Tiempo de Ejecución</Link>.
:::

## Desplegar tu Tabla

El generador DynamoDB crea infraestructura CDK o Terraform según tu `iac` seleccionado.

<Infrastructure>
<Fragment slot="cdk">
La construcción CDK se crea en `common/constructs`. Ejemplo de uso:

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

export class ApplicationStack extends Stack {
  constructor(scope: Construct, id: string, props?: StackProps) {
    super(scope, id, props);

    const table = new MyTable(this, 'Table');
  }
}
```

Esto aprovisiona una tabla DynamoDB con:
- `pk` (clave de partición) y `sk` (clave de ordenamiento), ambas de tipo `String`
- 2 Índices Secundarios Globales por defecto, según se define en `src/gsi.ts`
- Facturación bajo demanda (`PAY_PER_REQUEST`)
- Cifrado KMS gestionado por el cliente con rotación automática de claves
- Recuperación point-in-time habilitada
- Protección contra eliminación habilitada
- Nombre de tabla registrado en Runtime Config bajo el espacio de nombres `dynamodb` en AWS AppConfig
</Fragment>

### Protección contra Eliminación

La protección contra eliminación está habilitada por defecto para prevenir la eliminación accidental de la tabla.

#### Deshabilitar la Protección contra Eliminación

Deshabilítala para entornos donde se espera la eliminación de tablas, como stacks de desarrollo o vista previa de corta duración.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  deletionProtection: false,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source                      = "../../common/terraform/src/app/dynamodb/my-table"
  deletion_protection_enabled = false
}
```
</Fragment>
</Infrastructure>

### Modo de Facturación

La tabla usa por defecto facturación bajo demanda (`PAY_PER_REQUEST`). Cambia a capacidad aprovisionada para cargas de trabajo predecibles de alto rendimiento.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { BillingMode } from 'aws-cdk-lib/aws-dynamodb';
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  billingMode: BillingMode.PROVISIONED,
  readCapacity: 5,
  writeCapacity: 5,
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source       = "../../common/terraform/src/app/dynamodb/my-table"
  billing_mode = "PROVISIONED"
}
```
</Fragment>
</Infrastructure>

### Recuperación Point-in-time

La [recuperación point-in-time](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Point-in-time-recovery.html) está habilitada por defecto, permitiéndote restaurar la tabla a cualquier punto en los últimos 35 días.

#### Deshabilitar la Recuperación Point-in-time

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyTable } from ':my-scope/common-constructs';

const table = new MyTable(this, 'Table', {
  pointInTimeRecoverySpecification: { pointInTimeRecoveryEnabled: false },
});
```
</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_table" {
  source                         = "../../common/terraform/src/app/dynamodb/my-table"
  point_in_time_recovery_enabled = false
}
```
</Fragment>
</Infrastructure>

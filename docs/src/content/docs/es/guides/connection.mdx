Este generador se utiliza para conectar proyectos entre sí, como sitios web que llaman a APIs. Simplemente selecciona el proyecto origen (por ejemplo, el proyecto que llamará a tu API) y el proyecto destino (por ejemplo, tu proyecto de API), y este generador se encargará de integrar ambos.

## Desarrollo Local

Los proyectos conectados exponen dos targets para ejecutarlos en tu máquina: `serve` y `serve-local`. La diferencia es una cuestión de **alcance** — cuánto de tu aplicación se ejecuta localmente versus apuntando a infraestructura AWS desplegada.

Considera un workspace con las siguientes conexiones: un sitio web que llama a una `api` tRPC, y también llama a un `agent` que a su vez llama a un servidor `mcp`.

### `serve`

El target `serve` ejecuta **solo el proyecto objetivo** localmente. Cada otro proyecto al que se conecta se espera que esté **desplegado**, y se accede a él en su URL real de AWS.

<NxCommands commands={['serve website']} />

Debido a que el proyecto objetivo habla con recursos desplegados, necesita saber dónde están. Para un sitio web esto se proporciona mediante su `runtime-config.json` (ver <Link path="guides/react-website#local-runtime-config">Configuración en Tiempo de Ejecución Local</Link>), que cargas desde una aplicación desplegada. Para proyectos del lado del servidor (APIs y agentes) que leen <Link path="guides/runtime-config">Configuración en Tiempo de Ejecución</Link>, establece la variable de entorno `RUNTIME_CONFIG_APP_ID` para apuntar a tu aplicación AppConfig desplegada.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
agent: agent {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
mcp: mcp {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Usa `serve` cuando quieras iterar en un solo proyecto contra las versiones "reales" desplegadas de todo de lo que depende.

### `serve-local`

El target `serve-local` ejecuta el proyecto objetivo **y cada proyecto conectado a él de forma transitiva**, todo en tu máquina. El generador de conexión configura esto automáticamente — ejecutar `serve-local` en el sitio web también inicia servidores locales para la `api`, el `agent`, y el servidor `mcp` al que llega a través del agente.

<NxCommands commands={['serve-local website']} />

Cuando se ejecuta de esta manera, el `runtime-config.json` del sitio web se sobrescribe automáticamente (a través del [`MODE` de Vite](https://vite.dev/guide/env-and-mode), establecido en `serve-local`) para que apunte a tus servidores locales en lugar de URLs desplegadas.

```d2
direction: right

website: website {
  style.fill: "#c8e6c9"
}
api: api {
  style.fill: "#c8e6c9"
}
agent: agent {
  style.fill: "#c8e6c9"
}
mcp: mcp {
  style.fill: "#c8e6c9"
}

website -> api
website -> agent
agent -> mcp

local: "Local" {
  style.fill: "#c8e6c9"
}
remote: "Deployed (AWS)" {
  style.fill: "#eceff1"
  style.stroke-dash: 3
}
```

Usa `serve-local` cuando estés trabajando en varios proyectos conectados a la vez y quieras iterar rápidamente sin desplegar tu infraestructura.

:::note[Recursos AWS en `serve-local`]
Tus servidores locales se ejecutan con las credenciales de AWS que has configurado localmente, por lo que aún pueden acceder a recursos AWS desplegados (como tablas DynamoDB o modelos Bedrock) cuando sea necesario. Puedes proporcionar cualquier variable de entorno que tus servidores locales necesiten, por ejemplo:

<NxCommands env={{RUNTIME_CONFIG_APP_ID: 'xxxxx'}} commands={['serve-local website']} />
:::

Para más detalles sobre el servidor de desarrollo local de cada tipo de proyecto, consulta la guía relevante — por ejemplo la guía de <Link path="guides/react-website#local-development-server">Sitio Web React</Link>.

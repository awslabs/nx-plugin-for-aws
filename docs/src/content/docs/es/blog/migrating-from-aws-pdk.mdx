Esta guía te lleva paso a paso a través de un ejemplo de migración de un proyecto [AWS PDK](https://github.com/aws/aws-pdk) al Nx Plugin para AWS, además de proporcionar orientación general sobre este tema.

Migrar al Nx Plugin para AWS ofrece los siguientes beneficios frente a PDK:

- Construcciones más rápidas
- Más fácil de usar (UI y CLI)
- Compatible con vibe-coding (<Link path="/get_started/building-with-ai">¡prueba nuestro servidor MCP!</Link>)
- Tecnologías más modernas
- Desarrollo local de APIs y sitios web
- Más control (modifica archivos provistos para adaptarlos a tu caso de uso)
- ¡Y mucho más!

:::caution
Existen muchos matices y consideraciones al migrar un proyecto. Esta guía es para un ejemplo simple, y pueden faltar pasos aquí que sean necesarios para tu proyecto.

Esta guía opta por un enfoque donde creamos un espacio de trabajo nuevo y copiamos elementos de nuestro proyecto PDK a un nuevo proyecto. Si necesitas migrar de forma diferente, o tienes preguntas sobre cómo pasar de PDK al Nx Plugin para AWS, [¡contacta en GitHub discussions](https://github.com/awslabs/nx-plugin-for-aws/discussions)!

Ten en cuenta también que algunas características de PDK no tienen equivalentes exactos. Consulta la sección de [Preguntas Frecuentes](#frequently-asked-questions) para algunas ideas sobre esto.
:::

:::caution
Esta guía está escrita para un momento específico y probablemente no se actualizará conforme evolucione el Nx Plugin para AWS. Para compensar esto, la guía fija la versión de `@aws/nx-plugin` a `0.50.0`. Puedes intentar con la última versión si sigues los pasos, pero puede haber desviaciones respecto a la guía.
:::

## Ejemplo de Migración: Aplicación de Lista de Compras

En esta guía, usaremos la [Aplicación de Lista de Compras del Tutorial de PDK](https://aws.github.io/aws-pdk/getting_started/shopping_list_app.html) como nuestro proyecto objetivo para migrar. Sigue los pasos de ese tutorial para crear el proyecto objetivo si deseas replicar el proceso.

La aplicación de lista de compras consiste en los siguientes tipos de proyecto PDK:

- `MonorepoTsProject`
- `TypeSafeApiProject`
- `CloudscapeReactTsWebsiteProject`
- `InfrastructureTsProject`

### Crear Espacio de Trabajo

Para comenzar, crearemos un nuevo espacio de trabajo para nuestro proyecto nuevo. Aunque más extremo que una migración in situ, este enfoque nos da el resultado final más limpio. Crear un espacio de trabajo Nx es equivalente a usar el `MonorepoTsProject` de PDK:

<CreateNxWorkspaceCommand workspace="shopping-list" iacProvider="CDK" nxVersion="21.4.1" nxPluginVersion="0.50.0" />

Abre el directorio `shopping-list` que crea este comando en tu IDE favorito.

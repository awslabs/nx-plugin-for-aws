Esta página oferece algumas orientações para solução de problemas que você pode encontrar ao usar o Nx.

## Nx Daemon

O [Nx Daemon](https://nx.dev/docs/concepts/nx-daemon) é um processo que monitora continuamente as alterações no workspace para recalcular seu grafo de projetos. Isso proporciona uma melhoria de desempenho, especialmente em projetos grandes, mas às vezes pode causar problemas dependendo do seu ambiente.

Isso pode se manifestar de diferentes formas, uma delas sendo o seguinte erro:

```bash
 NX   Could not find project "@scope/project"

Pass --verbose to see the stacktrace.
```

Para esse e qualquer outro erro inesperado, sempre vale a pena resetar o Nx e tentar novamente em caso de problemas com o daemon. Você pode usar o seguinte comando:

<NxCommands commands={['reset']} />

Se isso não resolver, você pode tentar [desativar o daemon completamente](https://nx.dev/docs/concepts/nx-daemon#turning-it-off).

:::warning
O comando `nx watch` requer o daemon, portanto alguns destinos de hot-reloading podem não funcionar se você desativá-lo.
:::

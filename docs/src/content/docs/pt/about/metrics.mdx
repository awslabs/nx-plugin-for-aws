---
title: "Métricas de Uso"
description: "Como @aws/nx-plugin coleta métricas"
---



Para nos ajudar a melhorar o Nx Plugin para AWS, coletamos métricas sobre o uso dos geradores. Este processo não envolve solicitações de rede adicionais, simplesmente aproveitamos as métricas de uso existentes da AWS.

Para manter a transparência, adicionamos um ID, uma versão e uma lista de IDs de geradores às descrições das pilhas do CloudFormation. Isso é feito no `MetricsAspect` em `common/constructs` para CDK ou na pilha do CloudFormation `metrics.tf` para Terraform.

## Desativação

O código de métricas reside inteiramente no seu workspace, então você pode desativá-lo removendo-o:

- **CDK**: remova a classe `MetricsAspect` e a chamada `Aspects.of(this).add(new MetricsAspect())` de `packages/common/constructs/src/core/app.ts`.
- **Terraform**: exclua `packages/common/terraform/src/metrics/metrics.tf`.

Os geradores não irão adicionar novamente o código de métricas uma vez removido.

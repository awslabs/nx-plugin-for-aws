# Tradução de Documentação

Este projeto utiliza o modelo Haiku 3.5 da AWS Bedrock para traduzir automaticamente documentações do inglês para múltiplos idiomas. O sistema de tradução foi projetado para ser eficiente, preciso e fácil de usar.

## Idiomas Suportados

Atualmente, os seguintes idiomas são suportados:

- Japonês (jp)
- Francês (fr)
- Espanhol (es)
- Alemão (de)
- Chinês (zh)
- Coreano (ko)

## Como Funciona

O sistema de tradução opera através de:

1. **Divisão de documentos por cabeçalhos h2** - Permite processamento mais eficiente e melhor contexto para o modelo de tradução.
2. **Preservação da formatação markdown** - Toda sintaxe markdown, blocos de código e tags HTML são preservados durante a tradução.
3. **Tratamento especial para frontmatter** - O frontmatter YAML é traduzido mantendo sua estrutura.
4. **Tradução incremental** - Por padrão, apenas arquivos alterados são traduzidos, economizando tempo e recursos.

## Executando Traduções Localmente

Para traduzir documentações localmente, use o script `scripts/translate.ts`:

```bash
# Traduz apenas arquivos alterados para japonês (padrão)
./scripts/translate.ts

# Traduz todos os arquivos
./scripts/translate.ts --all

# Traduz para idiomas específicos
./scripts/translate.ts --languages jp,fr,es

# Simula a tradução sem alterar arquivos
./scripts/translate.ts --dry-run

# Exibe saída detalhada
./scripts/translate.ts --verbose
```

## Workflow do GitHub

Um workflow do GitHub traduz automaticamente a documentação quando há alterações em arquivos de documentação em inglês em pull requests. O workflow:

1. Detecta alterações em arquivos de documentação em inglês
2. Traduz os arquivos alterados usando AWS Bedrock
3. Comita as traduções de volta para o branch de origem
4. Atualiza o PR com o status das traduções

### Gatilho Manual do Workflow

Você também pode acionar manualmente o workflow de tradução na aba GitHub Actions. Isso é útil para:

- Executar uma tradução completa de toda documentação
- Traduzir para idiomas específicos
- Atualizar traduções após alterações no script de tradução

## Configuração AWS

O sistema utiliza o modelo Haiku 3.5 da AWS Bedrock para tradução. Para usar este recurso, você precisa de:

1. **Credenciais AWS** - Para desenvolvimento local, configure suas credenciais usando AWS CLI ou variáveis de ambiente.
2. **Função IAM** - Para GitHub Actions, configure uma função IAM com autenticação OIDC e permissões necessárias para AWS Bedrock.

### Permissões Necessárias

A função ou usuário IAM precisa das seguintes permissões:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel"
      ],
      "Resource": [
        "arn:aws:bedrock:*::foundation-model/anthropic.claude-3-haiku-20240307-v1:0"
      ]
    }
  ]
}
```

## Qualidade da Tradução

A qualidade geral é alta, mas considere estes pontos:

- **Termos técnicos** - O sistema preserva termos técnicos em inglês.
- **Blocos de código** - Não são traduzidos, mantendo sua forma original.
- **Contexto técnico** - O modelo compreende o contexto da documentação, melhorando traduções técnicas.

## Personalizando a Tradução

Modifique o arquivo `scripts/translate.ts` para customizações:

- Adicionar novos idiomas
- Alterar o modelo de tradução
- Ajustar prompts de tradução
- Modificar processamento de documentos

## Solução de Problemas

Caso encontre problemas:

1. **Verifique credenciais AWS** - Confira se estão configuradas corretamente.
2. **Verifique região AWS** - Use uma região onde Bedrock está disponível.
3. **Execute com verbose** - Use `--verbose` para logs detalhados.
4. **Verifique rate limiting** - Bedrock possui limites que podem afetar grandes jobs.

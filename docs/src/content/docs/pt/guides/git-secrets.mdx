Novos workspaces criados com `@aws/nx-plugin` incluem hooks de pré-commit do [git-secrets](https://github.com/awslabs/git-secrets) configurados por padrão. Isso verifica arquivos preparados em busca de padrões que correspondam a credenciais AWS antes de cada commit, prevenindo vazamentos acidentais de credenciais.

## O Que Está Incluído

Quando você cria um novo workspace, o seguinte é configurado automaticamente:

<FileTree>
  - .git-secrets/
    - git-secrets O script bash do git-secrets (vendorizado de awslabs/git-secrets)
    - setup.sh Registra padrões de credenciais AWS na configuração local do git
  - .husky/
    - pre-commit Executa a verificação do git-secrets antes de cada commit
  - package.json Inclui husky e script prepare para configuração automática
</FileTree>

## Como Funciona

1. O script `prepare` é executado após `npm install` / `pnpm install`, que inicializa os hooks git do husky e registra padrões de credenciais AWS na sua configuração local do git.
2. Em cada `git commit`, o hook de pré-commit do husky invoca o `git-secrets` para verificar arquivos preparados.
3. Se um padrão correspondente a uma credencial AWS for detectado, o commit é bloqueado com uma mensagem de erro.

## Padrões Detectados

Os seguintes padrões são verificados:

- **AWS Access Key IDs** — `AKIA...`, `ASIA...`, e outros prefixos de chave IAM seguidos por 16 caracteres alfanuméricos maiúsculos
- **AWS Secret Access Keys** — strings base64 de 40 caracteres precedidas por um rótulo `SECRET_ACCESS_KEY`
- **AWS Account IDs** — números de 12 dígitos precedidos por um rótulo `ACCOUNT_ID`
- **Bedrock API Keys** — chaves Bedrock de longa duração (`ABSK...`) e de curta duração
- **Valores de arquivo de credenciais** — valores de chave literais encontrados em `~/.aws/credentials`

Chaves de exemplo conhecidas (por exemplo, `AKIAIOSFODNN7EXAMPLE`) são automaticamente adicionadas à lista de permissões.

## Permitindo Falsos Positivos

Se o git-secrets bloquear um commit que contém um falso positivo:

```sh
# Allow a specific pattern
git config --add secrets.allowed 'my-pattern-here'
```

Ou adicione padrões a um arquivo `.gitallowed` na raiz do repositório (uma regex por linha).

Para uma substituição única:

```sh
git commit --no-verify
```

## Desativando

Para criar um workspace sem git-secrets:

```sh
pnpm create @aws/nx-workspace my-project --no-gitSecrets
```

Ou passe `--gitSecrets=false` para o gerador de preset.

## Requisitos

- **Git** deve estar inicializado no workspace (`git init`)
- **Bash** deve estar disponível no PATH (incluído com Git for Windows no Windows)
- Padrões são registrados quando `prepare` é executado — se você clonar uma cópia nova, execute o comando de instalação do seu gerenciador de pacotes para re-registrar

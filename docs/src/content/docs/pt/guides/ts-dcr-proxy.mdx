## Consumindo um Servidor MCP com Proxy

O proxy permite que clientes MCP autentiquem contra seu Cognito User Pool sem qualquer configuraĂ§ĂŁo especĂ­fica do cliente alĂ©m da URL do proxy. Quando um cliente se conecta ao endpoint `/mcp`, ele descobre os metadados OAuth (via `/.well-known/oauth-protected-resource` e `/.well-known/oauth-authorization-server`), se registra dinamicamente e conduz o usuĂˇrio atravĂ©s do login do Cognito Hosted UI. O proxy injeta o segredo do App Client durante a troca de token, entĂŁo o cliente nunca precisa dele.

Para conectar um cliente, aponte-o para o `mcpUrl` do proxy (ou seja, `<proxyUrl>/mcp`). Os exemplos abaixo assumem que seu proxy estĂˇ implantado em `https://my-proxy.example.com`.

:::note
Cada cliente usa uma URL de callback durante o fluxo OAuth. Essa URL exata â€” incluindo sua porta â€” deve estar registrada como uma [URL de callback no seu Cognito App Client](#permitindo-uris-de-redirecionamento-de-cliente), caso contrĂˇrio o login falha. Onde um cliente escolheria uma porta aleatĂłria, fixe-a em um valor fixo e registre esse valor.
:::

### Claude Code

Adicione o servidor com proxy com o comando [`claude mcp add`](https://code.claude.com/docs/en/mcp), usando o transporte HTTP. Por padrĂŁo, o Claude Code escuta em uma porta de callback aleatĂłria; passe `--callback-port` para fixĂˇ-la na porta registrada no seu App Client (`41100` nos exemplos acima). O Claude Code sempre usa o caminho `/callback`, entĂŁo o URI de redirecionamento resultante Ă© `http://localhost:41100/callback`:

```bash

```

Quando vocĂŞ invoca uma ferramenta do servidor pela primeira vez, o Claude Code abre o Cognito Hosted UI para autenticar antes que a requisiĂ§ĂŁo seja enviada via proxy upstream.

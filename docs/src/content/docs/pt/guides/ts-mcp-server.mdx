Gere um servidor TypeScript do [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) para fornecer contexto a Large Language Models (LLMs), e opcionalmente implante-o no [Amazon Bedrock AgentCore](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/).

## O que é o MCP?

O [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) é um padrão aberto que permite a assistentes de IA interagirem com ferramentas e recursos externos. Ele fornece uma maneira consistente para LLMs:

- Executar ferramentas (funções) que realizam ações ou recuperam informações
- Acessar recursos que fornecem contexto ou dados

## Trabalhando com Seu Servidor MCP

### Adicionando Ferramentas

Ferramentas são funções que o assistente de IA pode chamar para realizar ações. Você pode adicionar novas ferramentas no arquivo `server.ts`:

```typescript
server.registerTool("toolName", {
  description: "tool description",
  inputSchema: { param1: z.string(), param2: z.number() } // Esquema de entrada usando Zod
},
  async ({ param1, param2 }) => {
    // Implementação da ferramenta
    return {
      content: [{ type: "text", text: "Result" }]
    };
  }
);
```

### Adicionando Recursos

Recursos fornecem contexto ao assistente de IA. Você pode adicionar recursos estáticos de arquivos ou recursos dinâmicos:

```typescript
const exampleContext = 'algum contexto para retornar';

server.registerResource('resource-name', 'example://resource', {}, async (uri) => ({
  contents: [{ uri: uri.href, text: exampleContext }],
}));

// Recurso dinâmico
server.registerResource('dynamic-resource', 'dynamic://resource', {}, async (uri) => {
  const data = await fetchSomeData();
  return {
    contents: [{ uri: uri.href, text: data }],
  };
});
```

## Executando Seu Servidor MCP

### Inspector

O gerador configura um alvo chamado `<your-server-name>-inspect`, que inicia seu servidor MCP localmente (através do alvo `<your-server-name>-dev`, incluindo quaisquer dependências conectadas, como um banco de dados local) e lança o [MCP Inspector](https://github.com/modelcontextprotocol/inspector) pré-configurado para conectar a ele através do transporte HTTP Streamable.

<NxCommands commands={['your-server-name-inspect your-project']} />

Isso iniciará o inspector em `http://localhost:6274`. Comece clicando no botão "Connect".

:::tip
Para executar seu servidor MCP (e quaisquer dependências conectadas) sem o inspector, use o alvo `dev` do projeto. Se você gerou múltiplos componentes no mesmo projeto, execute um único servidor MCP por conta própria com seu alvo `<your-server-name>-dev`.
:::

:::tip
Para inspecionar o servidor usando transporte STDIO, use o alvo `<your-server-name>-inspect-stdio`, que lança o inspector contra uma instância STDIO do seu servidor.
:::

### STDIO

A maneira mais fácil de testar e usar um servidor MCP é usando o inspector ou configurando-o com um assistente de IA (como acima).

Você também pode executar seu servidor com [transporte STDIO](https://modelcontextprotocol.io/docs/concepts/transports#standard-input%2Foutput-stdio) diretamente usando o alvo `<your-server-name>-serve-stdio`.

<NxCommands commands={['your-server-name-serve-stdio your-project']} />

Este comando usa `tsx --watch` para reiniciar automaticamente o servidor quando arquivos mudarem.

### HTTP Streamable

Se você deseja executar seu servidor MCP localmente usando [transporte HTTP Streamable](https://modelcontextprotocol.io/docs/concepts/transports#streamable-http), pode usar o alvo `<your-server-name>-serve`.

<NxCommands commands={['your-server-name-serve your-project']} />

Este comando usa `tsx --watch` para reiniciar automaticamente o servidor quando arquivos mudarem.

### Alvo de Bundle

<Snippet name="ts-bundle" />

O alvo de bundle usa `http.ts` como ponto de entrada para o servidor MCP HTTP Streamable hospedado no Bedrock AgentCore Runtime.

### Alvo Docker

O gerador configura um alvo `<your-server-name>-docker` que copia o `Dockerfile` do diretório de origem do servidor MCP para o diretório de saída do bundle. Isso coloca o `Dockerfile` junto com os artefatos empacotados, permitindo que o CDK construa a imagem Docker diretamente usando `AgentRuntimeArtifact.fromAsset`.

Um alvo `docker` também é gerado para preparar o contexto docker para todos servidores MCP se você tiver múltiplos definidos.

[tRPC](https://trpc.io/) é um framework para construir APIs em TypeScript com segurança de tipos end-to-end. Usando tRPC, atualizações nas entradas e saídas das operações da API são refletidas imediatamente no código do cliente e visíveis na sua IDE sem necessidade de reconstruir o projeto.

O gerador de API tRPC cria uma nova API tRPC com configuração de infraestrutura AWS CDK. O backend gerado utiliza AWS Lambda para deploy serverless e inclui validação de esquema usando [Zod](https://zod.dev/). Configura [AWS Lambda Powertools](https://docs.powertools.aws.dev/lambda/typescript/latest/) para observabilidade, incluindo logging, rastreamento com AWS X-Ray e métricas no CloudWatch.

## Saída do Gerador

O gerador criará a seguinte estrutura de projeto no diretório `<directory>/<api-name>`:

<FileTree>
  - src
    - init.ts Inicialização do backend tRPC
    - router.ts Definição do roteador tRPC (ponto de entrada da API no handler Lambda)
    - schema Definições de esquema usando Zod
      - echo.ts Exemplo de definições para entrada e saída do procedimento "echo"
    - procedures Procedimentos (ou operações) expostos pela sua API
      - echo.ts Procedimento de exemplo
    - middleware
      - error.ts Middleware para tratamento de erros
      - logger.ts middleware para configurar AWS Powertools para logging em Lambda
      - tracer.ts middleware para configurar AWS Powertools para rastreamento em Lambda 
      - metrics.ts middleware para configurar AWS Powertools para métricas em Lambda
    - local-server.ts Ponto de entrada do adaptador standalone tRPC para servidor de desenvolvimento local
    - client
      - index.ts Cliente type-safe para chamadas máquina-a-máquina
  - tsconfig.json Configuração TypeScript
  - project.json Configuração do projeto e targets de build
</FileTree>

O gerador também criará constructs CDK para deploy da API, localizados em `packages/common/constructs`.

## Implementando sua API tRPC

Em alto nível, APIs tRPC consistem em um roteador que delega requisições para procedimentos específicos. Cada procedimento tem uma entrada e saída definidas como esquemas Zod.

### Schema

O diretório `src/schema` contém tipos compartilhados entre client e server. Esses tipos são definidos usando [Zod](https://zod.dev/), uma biblioteca TypeScript-first para declaração e validação de schemas.

Um exemplo de schema:

```ts
import { z } from 'zod';

// Definição do schema
export const UserSchema = z.object({
  name: z.string(),
  height: z.number(),
  dateOfBirth: z.string().datetime(),
});

// Tipo TypeScript correspondente
export type User = z.TypeOf<typeof UserSchema>;
```

O tipo `User` acima é equivalente a:

```ts
interface User {
  name: string;
  height: number;
  dateOfBirth: string;
}
```

Schemas são compartilhados entre client e server, permitindo atualizações centralizadas nas estruturas usadas na API.

A validação automática de schemas é feita pelo tRPC em runtime, eliminando a necessidade de lógica de validação manual.

Zod fornece utilitários poderosos como `.merge`, `.pick`, `.omit` e outros. Mais informações na [documentação do Zod](https://zod.dev/?id=basic-usage).

### Roteador e Procedimentos

O ponto de entrada da API está em `src/router.ts`, que contém o handler Lambda que roteia requisições para os procedimentos. Cada procedimento define entrada, saída e implementação.

O roteador de exemplo possui uma operação `echo`:

```ts
import { echo } from './procedures/echo.js';

export const appRouter = router({
  echo,
});
```

O procedimento `echo` em `src/procedures/echo.ts`:

```ts
export const echo = publicProcedure
  .input(EchoInputSchema)
  .output(EchoOutputSchema)
  .query((opts) => ({ result: opts.input.message }));
```

Explicação:
- `publicProcedure` define método público com middleware configurado em `src/middleware` (incluindo AWS Lambda Powertools)
- `input` aceita schema Zod para validação automática
- `output` define schema de saída, gerando erros de tipo se não conformado
- `query` define implementação para operações não mutativas. Use `mutation` para operações mutativas

Novos procedimentos devem ser registrados no roteador em `src/router.ts`.

## Personalizando sua API tRPC

### Erros

Retorne erros lançando `TRPCError`:

```ts
throw new TRPCError({
  code: 'NOT_FOUND',
  message: 'Recurso não encontrado',
});
```

### Organização de Operações

Agrupe operações com roteadores aninhados:

```ts
const appRouter = router({
   users: router({
      get: getUser,
      list: listUsers,
   }),
})
```

Chamada no cliente:

```ts
client.users.list.query();
```

### Logging

O logger AWS Lambda Powertools está acessível via `opts.ctx.logger`:

```ts {5}
export const echo = publicProcedure
   .query(async (opts) => {
      opts.ctx.logger.info('Chamada com input', opts.input);
   });
```

Consulte a [documentação do Logger](https://docs.powertools.aws.dev/lambda/typescript/latest/core/logger/).

### Métricas

Métricas são acessíveis via `opts.ctx.metrics`:

```ts {5}
export const echo = publicProcedure
   .query(async (opts) => {
      opts.ctx.metrics.addMetric('Invocations', 'Count', 1);
   });
```

Veja [documentação de Métricas](https://docs.powertools.aws.dev/lambda/typescript/latest/core/metrics/).

### Rastreamento com X-Ray

O tracer está acessível via `opts.ctx.tracer`:

```ts {5-7}
export const echo = publicProcedure
   .query(async (opts) => {
      const subSegment = opts.ctx.tracer.getSegment()!.addNewSubsegment('MyAlgorithm');
      // ... lógica
      subSegment.close();
   });
```

Documentação do [Tracer](https://docs.powertools.aws.dev/lambda/typescript/latest/core/tracer/).

### Middleware Customizado

Adicione contexto personalizado implementando middleware. Exemplo em `src/middleware/identity.ts`:

```ts
export interface IIdentityContext {
  identity?: {
    sub: string;
    username: string;
  };
}
```

## Deploy da API tRPC

Use o construct CDK gerado em `common/constructs`:

```ts {6-8}
import { MyApi } from ':my-scope/common-constructs`;

export class ExampleStack extends Stack {
   constructor(scope: Construct, id: string) {
      const api = new MyApi(this, 'MyApi', {
        integrations: MyApi.defaultIntegrations(this).build(),
      });
   }
}
```

Para autenticação Cognito:

```ts {9}
import { MyApi, UserIdentity } from ':my-scope/common-constructs';

export class ExampleStack extends Stack {
  constructor(scope: Construct, id: string) {
    const identity = new UserIdentity(this, 'Identity');

    const api = new MyApi(this, 'MyApi', {
      identity,
    });
  }
}
```

O construct `UserIdentity` pode ser gerado com <Link path="/guides/cloudscape-website-auth">`ts#cloudscape-website-auth`</Link>.

## Invocando a API

Crie client type-safe usando `src/client/index.ts`:

```ts
import { createMyApiClient } from ':my-scope/my-api';

const client = createMyApiClient({ url: 'https://my-api-url.example.com/' });

await client.echo.query({ message: 'Hello world!' });
```

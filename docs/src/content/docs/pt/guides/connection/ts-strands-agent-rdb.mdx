O gerador `connection` conecta um <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> a um projeto de <Link path="guides/ts-rdb">Banco de Dados Relacional</Link>, disponibilizando um cliente Prisma durante todo o ciclo de vida da requisição do seu agente.

A estratégia de injeção depende do protocolo do agente — **HTTP** (baseado em tRPC) ou **A2A** (Agent-to-Agent).

## Pré-requisitos

Antes de usar este gerador, certifique-se de ter:

1. Um projeto <Link path="guides/ts-strands-agent">`ts#strands-agent`</Link>
2. Um projeto <Link path="guides/ts-rdb">`ts#rdb`</Link>

## Saída do Gerador

Os arquivos modificados dependem do protocolo do agente.

**Protocolo HTTP** — modifica quatro arquivos:

<FileTree>

- packages/my-service/src/my-agent
  - init.ts propriedade `db` adicionada à interface `Context`
  - index.ts `createContext` tornado assíncrono; cliente Prisma criado e retornado no contexto
  - router.ts `db` desestruturado de `opts.ctx` e passado para `getAgent`
  - agent.ts `getAgent` atualizado para aceitar `{ db }` como segundo parâmetro

</FileTree>

**Protocolo A2A** — modifica dois arquivos:

<FileTree>

- packages/my-service/src/my-agent
  - index.ts Cliente Prisma criado antes do `A2AExpressServer`; passado para `getAgent`
  - agent.ts `getAgent` atualizado para aceitar `{ db }` como segundo parâmetro

</FileTree>

Além disso, o target `<agent-name>-serve-local` do agente é atualizado para depender do target `serve-local` do banco de dados.

## Como Funciona

### Protocolo HTTP

Para agentes HTTP, o banco de dados flui através do contexto de requisição tRPC — `createContext` cria um cliente Prisma por requisição e ele fica disponível para cada procedimento via `opts.ctx`. O roteador então o passa explicitamente para `getAgent`.

#### Interface de Contexto

O tipo do cliente Prisma é adicionado ao `Context` do tRPC em `init.ts`:

```ts title="packages/my-service/src/my-agent/init.ts" {1,5}
import { getPrisma as getMyDb } from ':my-scope/my-db';

export interface Context {
  sessionId: string;
  myDb: Awaited<ReturnType<typeof getMyDb>>;
}
```

#### Contexto de Requisição

`createContext` é tornado assíncrono e o cliente Prisma é instanciado e retornado:

```ts title="packages/my-service/src/my-agent/index.ts" {1,6,10}
import { getPrisma as getMyDb } from ':my-scope/my-db';

const createContext = async (
  opts: CreateHTTPContextOptions | CreateWSSContextFnOptions,
): Promise<Context> => {
  const myDb = await getMyDb();
  const sessionId = /* ... */;
  return {
    sessionId,
    myDb,
  };
};
```

#### Roteador e Agente

O banco de dados é passado do contexto para `getAgent` através do manipulador de assinatura:

```ts title="packages/my-service/src/my-agent/router.ts" {2,3}
.subscription(async function* (opts) {
  const { myDb } = opts.ctx;
  const agent = await getAgent(opts.ctx.sessionId, { myDb });
  // ...
})
```

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

### Protocolo A2A

Para agentes A2A, não há contexto tRPC. O banco de dados é criado uma vez na inicialização e passado diretamente para `getAgent` antes do `A2AExpressServer` iniciar.

#### Inicialização do Servidor

O cliente Prisma é criado antes do `A2AExpressServer` e passado para `getAgent`:

```ts title="packages/my-service/src/my-agent/index.ts" {1,4,6}
import { getPrisma as getMyDb } from ':my-scope/my-db';

void (async () => {
  const myDb = await getMyDb();
  const server = new A2AExpressServer({
    agent: await getAgent('default', { myDb }),
    // ...
  });
})();
```

#### Definição do Agente

```ts title="packages/my-service/src/my-agent/agent.ts" {3}
export const getAgent = async (
  sessionId: string,
  { myDb }: { myDb: Awaited<ReturnType<typeof getMyDb>> },
) => {
  return new Agent({ /* use myDb in tools */ });
};
```

## Múltiplos Bancos de Dados

Executar o gerador novamente com um destino diferente adiciona o segundo banco de dados ao lado do primeiro. Para agentes HTTP, ambos aparecem na interface `Context` e são desestruturados juntos no roteador:

```ts title="packages/my-service/src/my-agent/router.ts"
const { postgresDb, mysqlDb } = opts.ctx;
const agent = await getAgent(opts.ctx.sessionId, { postgresDb, mysqlDb });
```

## Infraestrutura

O construto de agente gerado implementa `IGrantable` e `IConnectable`, então você pode conceder acesso de rede e IAM ao banco de dados diretamente no construto.

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
import { MyDatabase } from ':my-scope/common-constructs';

const db = new MyDatabase(this, 'Db', { vpc, ... });
const myAgent = new MyAgent(this, 'MyAgent', { vpc, ... });

db.allowDefaultPortFrom(myAgent);
db.grantConnect(myAgent);
```

`allowDefaultPortFrom` abre a regra do grupo de segurança para que o runtime do agente possa alcançar a porta do banco de dados. `grantConnect` concede permissão IAM `rds-db:connect` à função de execução do agente.

</Fragment>
<Fragment slot="terraform">

Passe as saídas do módulo de banco de dados para o módulo do seu agente para que ele possa alcançar o banco de dados e ler sua configuração de runtime:

```hcl title="packages/infra/src/main.tf"
module "my_database" {
  source              = "../../common/terraform/src/app/dbs/my-database"
  vpc_id              = module.vpc.vpc_id
  database_subnet_ids = module.vpc.private_isolated_subnet_ids
}

module "my_agent" {
  source = "../../common/terraform/src/app/agents/my-agent"

  appconfig_application_id     = module.my_database.appconfig_application_id
  database_cluster_resource_id = module.my_database.cluster_resource_id
  database_runtime_user        = module.my_database.database_runtime_user
  database_security_group_id   = module.my_database.security_group_id
  database_port                = module.my_database.cluster_port
}
```

Certifique-se de que a função de execução do agente tenha permissão `rds-db:connect` e que seu grupo de segurança possa alcançar o grupo de segurança do banco de dados na porta do banco de dados.

</Fragment>
</Infrastructure>

## Desenvolvimento Local

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Isso inicia o agente e todos os bancos de dados conectados. A variável de ambiente `SERVE_LOCAL=true` faz com que cada cliente Prisma se conecte ao seu banco de dados Docker local em vez do Aurora.

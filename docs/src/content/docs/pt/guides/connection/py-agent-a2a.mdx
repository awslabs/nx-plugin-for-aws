O gerador `connection` pode conectar seu <Link path="guides/py-agent">Python Agent</Link> a um agente [A2A](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-a2a.html) remoto — seja <Link path="guides/ts-agent">TypeScript</Link> ou <Link path="guides/py-agent">Python</Link> — para que seu agente possa delegar a outro agente como uma ferramenta.

O gerador configura toda a infraestrutura necessária para que seu agente possa descobrir e invocar o agente A2A remoto, tanto quando implantado na AWS (via Bedrock AgentCore) quanto quando executado localmente.

## Pré-requisitos

Antes de usar este gerador, certifique-se de ter:

1. Um projeto Python com um componente <Link path="guides/py-agent">Python Agent</Link> (Strands ou LangChain)
2. Um projeto com um componente Agent gerado com `--protocol=A2A` e `--auth=IAM` (seja <Link path="guides/ts-agent">`ts#agent`</Link> ou <Link path="guides/py-agent">`py#agent`</Link>)
3. Ambos os componentes criados com `infra: agentcore`

## Saída do Gerador

O gerador cria um projeto Python compartilhado `agent_connection` em `packages/common/agent_connection/` (se ainda não existir). Módulos cliente por conexão são gerados neste projeto compartilhado:

<FileTree>

- packages/common/agent\_connection
  - \<scope>\_agent\_connection
    - \_\_init\_\_.py Re-exporta clientes por conexão
    - core
      - agentcore\_endpoints.py Resolução de ARN/URL independente de framework
      - agentcore\_a2a\_client\_config.py Configuração de cliente A2A independente de framework (`ClientConfig` assinado)
      - agentcore\_a2a\_client\_strands.py Cliente de transporte A2A encapsulando a configuração
      - auth/ `httpx.Auth` SigV4 / encaminhamento de sessão independente de framework
    - app
      - \<target\_agent\_name>\_client\_strands.py Cliente de transporte A2A por conexão para cada agente A2A

</FileTree>

O transporte A2A (um `A2AAgent` Strands) não carrega nenhum modelo, então tanto agentes Strands quanto LangChain o reutilizam — apenas a configuração de ferramentas do `agent.py` difere por framework.

Além disso, o gerador:
- Transforma o `agent.py` do seu agente para registrar o agente A2A remoto como uma ferramenta usando `@tool`
- Adiciona o projeto `agent_connection` como uma dependência de workspace do seu projeto de agente
- Atualiza o target `serve-local` do agente para depender do target `serve-local` do agente de destino

## Usando o Agente A2A Conectado

O gerador transforma o `agent.py` do seu agente para encapsular o agente A2A remoto como uma ferramenta. O agente remoto é registrado com um delegado decorado com `@tool` — a importação do decorador e o construtor do agente diferem por framework:

<Tabs syncKey="framework">
<TabItem label="Strands">
```python title="packages/my-project/my_module/agent/agent.py" {4,9-15,21}
from contextlib import contextmanager
from strands import Agent, tool

from my_scope_agent_connection import RemoteAgentClientStrands

@contextmanager
def get_agent(session_id: str):
    remote_agent = RemoteAgentClientStrands.create(session_id=session_id)

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    yield Agent(
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
<TabItem label="LangChain">
```python title="packages/my-project/my_module/agent/agent.py" {4,8-13,17}
from langchain.agents import create_agent
from langchain_aws import ChatBedrockConverse
from langchain_core.tools import tool

from my_scope_agent_connection import RemoteAgentClientStrands

def get_agent():
    remote_agent = RemoteAgentClientStrands.create()

    @tool
    def ask_remote_agent(prompt: str) -> str:
        """Delegate a question to the remote RemoteAgent A2A agent and return its reply."""
        return str(remote_agent(prompt))

    return create_agent(
        model=ChatBedrockConverse(model=MODEL_ID, region_name=REGION),
        system_prompt="...",
        tools=[ask_remote_agent],
    )
```
</TabItem>
</Tabs>

Internamente, `RemoteAgentClientStrands.create()` retorna um `A2AAgent` Strands (usado puramente como um transporte A2A — ele não carrega nenhum modelo, então agentes LangChain o reutilizam) configurado com um `httpx.AsyncClient` que assina requisições com SigV4 quando implantado na AWS, e um endpoint simples `http://localhost:<port>/` quando `SERVE_LOCAL=true`. Para agentes Strands, o `session_id` é propagado a partir do chamador, garantindo consistência para [Bedrock AgentCore Observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability.html).

## Desenvolvimento Local

O gerador configura o target `serve-local` do agente host para:
1. Iniciar o(s) agente(s) A2A conectado(s) automaticamente
2. Definir `SERVE_LOCAL=true` para que o cliente gerado se conecte diretamente a `http://localhost:<port>/` em vez do AgentCore

Execute o agente localmente com:

<NxCommands commands={["<agent-name>-serve-local <project-name>"]} />

Isso iniciará tanto o agente host quanto todos os agentes A2A conectados, com o agente host chamando os agentes remotos via HTTP simples em suas portas locais atribuídas.

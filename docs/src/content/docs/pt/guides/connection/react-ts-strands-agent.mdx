O Nx Plugin for AWS fornece um gerador para integrar rapidamente seu <Link path="guides/ts-strands-agent">TypeScript Strands Agent</Link> com um site React. Ele configura toda a configuração necessária para conectar ao seu agente via tRPC sobre WebSocket, incluindo suporte de autenticação AWS IAM e Cognito. A integração fornece segurança de tipo completa de ponta a ponta entre seu frontend e o roteador tRPC do agente.

## Pré-requisitos

Antes de usar este gerador, certifique-se de ter:

1. Um site React (gerado usando o <Link path="guides/react-website">gerador `ts#react-website`</Link>)
2. Um TypeScript Strands Agent (gerado usando o <Link path="guides/ts-strands-agent">gerador `ts#strands-agent`</Link>)
3. Cognito Auth adicionado via o <Link path="/guides/react-website-auth">gerador `ts#react-website-auth`</Link>

## Saída do Gerador

O gerador cria a seguinte estrutura em sua aplicação React:

<FileTree>

- src
  - components
    - \<AgentName>AgentClientProvider.tsx Configura o cliente tRPC WebSocket e vinculações ao roteador tRPC do seu agente
    - QueryClientProvider.tsx Provedor do cliente TanStack React Query
  - hooks
    - useSigV4.tsx Hook para assinar requisições com SigV4 (apenas IAM)
    - use\<AgentName>Agent.tsx Hooks retornando o proxy de opções tRPC e cliente tRPC vanilla

</FileTree>

Adicionalmente, ele instala as dependências necessárias:

  - `@trpc/client`
  - `@trpc/tanstack-react-query`
  - `@tanstack/react-query`
  - `aws4fetch` (se estiver usando autenticação IAM)

## Como Funciona

### Conexão WebSocket

O cliente gerado se conecta ao seu Strands Agent via tRPC sobre WebSocket. O agente expõe um roteador tRPC (incluindo a assinatura `invoke` para transmitir respostas do agente em streaming) sobre um endpoint WebSocket.

- **Implantado**: O ARN do runtime do agente é carregado da <Link path="guides/runtime-config">Configuração de Runtime</Link>. O ARN é convertido para uma URL WebSocket seguindo o protocolo WebSocket do Bedrock AgentCore Runtime: `wss://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/ws`
- **Desenvolvimento local**: Ao executar com `serve-local`, a substituição da configuração de runtime define o valor para uma URL `ws://` local (por exemplo, `ws://localhost:8081/ws`), e o cliente se conecta diretamente

### Autenticação

O código gerado lida com a autenticação dependendo da configuração do seu agente:

- **IAM** (padrão): Usa URLs pré-assinadas AWS SigV4 para autenticar a conexão WebSocket. As credenciais são obtidas do Cognito Identity Pool configurado com a autenticação do seu site. No modo `serve-local`, a assinatura é automaticamente ignorada quando <Link path="guides/react-website#runtime-configuration">`runtime-config.json`</Link> não está presente
- **Cognito**: Incorpora o token de acesso JWT no cabeçalho `Sec-WebSocket-Protocol` como um token bearer codificado em base64url, seguindo o [protocolo de autenticação WebSocket do AgentCore](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-get-started-websocket.html)


## Infraestrutura

Se seu agente usa autenticação IAM, a função autenticada do Cognito Identity Pool deve ter a permissão `bedrock-agentcore:InvokeAgentRuntimeWithWebSocketStream` para invocar agentes via WebSocket:

```ts
const identity = new UserIdentity(this, 'Identity');
const myAgent = new MyAgent(this, 'MyAgent');

identity.identityPool.authenticatedRole.addToPrincipalPolicy(
  new PolicyStatement({
    actions: ['bedrock-agentcore:InvokeAgentRuntimeWithWebSocketStream'],
    resources: [myAgent.agentCoreRuntime.agentRuntimeArn],
  }),
);
```

:::note
No momento da escrita, o método `grantInvoke` do agent core runtime ainda não inclui esta permissão automaticamente (veja [aws-cdk#37495](https://github.com/aws/aws-cdk/issues/37495)).
:::

Se seu agente usa autenticação Cognito, você não precisa definir nenhuma infraestrutura adicional para conectar seu site ao seu agente.

## Usando o Código Gerado

### Usando TanStack Query

O caso de uso mais comum é transmitir a resposta do agente em streaming usando a assinatura `invoke` com o hook `use<AgentName>Agent`, que retorna um [proxy de opções tRPC](https://trpc.io/docs/client/tanstack-react-query) para uso com TanStack Query:

```tsx {1-2,7-22}
import { useSubscription } from '@trpc/tanstack-react-query';
import { useMyAgentAgent } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const trpc = useMyAgentAgent();

  const subscription = useSubscription(
    trpc.invoke.subscriptionOptions(
      { message: 'What can you help me with?' },
      {
        enabled: true,
        onStarted: () => {
          console.log('Agent started responding');
        },
        onData: (token) => {
          console.log('Received token:', token);
        },
        onError: (error) => {
          console.error('Agent error:', error);
        },
      },
    ),
  );

  return (
    <div>
      <p>Status: {subscription.status}</p>
      {subscription.data && <p>Latest token: {subscription.data}</p>}
      {subscription.error && <p>Error: {subscription.error.message}</p>}
    </div>
  );
}
```

:::note
Para mais detalhes sobre o proxy de opções tRPC e integração com TanStack Query, consulte a [documentação tRPC TanStack React Query](https://trpc.io/docs/client/tanstack-react-query).
:::

### Usando o Cliente tRPC Vanilla

O hook `use<AgentName>AgentClient` fornece acesso ao [cliente tRPC vanilla](https://trpc.io/docs/client/vanilla) para mais controle sobre o ciclo de vida da assinatura:

```tsx {2,5,9-22,25}
import { useState } from 'react';
import { useMyAgentAgentClient } from './hooks/useMyAgentAgent';

function ChatComponent() {
  const client = useMyAgentAgentClient();
  const [messages, setMessages] = useState<string[]>([]);

  const sendMessage = (message: string) => {
    const subscription = client.invoke.subscribe(
      { message },
      {
        onData: (token) => {
          setMessages((prev) => [...prev, token]);
        },
        onComplete: () => {
          console.log('Agent finished');
        },
        onError: (error) => {
          console.error('Error:', error);
        },
      },
    );

    // Clean up when done
    return () => subscription.unsubscribe();
  };

  return (
    <div>
      <button onClick={() => sendMessage('Hello!')}>Send</button>
      <div>
        {messages.map((msg, i) => (
          <span key={i}>{msg}</span>
        ))}
      </div>
    </div>
  );
}
```

:::note
Para mais detalhes sobre o cliente tRPC vanilla, consulte a [documentação do Cliente Vanilla tRPC](https://trpc.io/docs/client/vanilla).
:::

## Desenvolvimento Local

O gerador de conexão configura automaticamente a integração `serve-local` para seu site react:

1. Executar `nx run <website>:serve-local` também iniciará o servidor local do agente
2. A configuração de runtime é substituída para apontar para a URL WebSocket local (por exemplo, `ws://localhost:8081/ws`)
3. Como com APIs conectadas, a autenticação é ignorada no modo `serve-local` quando <Link path="guides/react-website#runtime-configuration">`runtime-config.json`</Link> não está presente

:::tip[Hot Reloading]
O site e o agente conectado farão hot-reload, permitindo que você itere rapidamente em ambos juntos sem implantar na AWS.
:::

## Segurança de Tipo

A integração fornece segurança de tipo completa de ponta a ponta. Seu IDE fornecerá autocompletar completo e verificação de tipo para todas as chamadas de procedimento do agente. Os tipos são automaticamente inferidos da definição do roteador tRPC do seu agente, garantindo que quaisquer mudanças na API do seu agente sejam imediatamente refletidas no código do seu frontend.

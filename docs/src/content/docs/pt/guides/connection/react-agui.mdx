O Nx Plugin for AWS fornece um gerador para conectar um website React a um Strands Agent que expõe o [protocolo AG-UI](https://docs.ag-ui.com/). Ele configura o [CopilotKit](https://docs.copilotkit.ai/aws-strands) com um `HttpAgent` [`@ag-ui/client`](https://docs.ag-ui.com/) no seu website, com suporte para autenticação AWS IAM e Cognito.

:::caution[TypeScript Strands Agents]
AG-UI ainda não é suportado para TypeScript Strands Agents. Veja [strands-agents/sdk-typescript#347](https://github.com/strands-agents/sdk-typescript/issues/347) para o status mais recente. Por enquanto, use o <Link path="guides/py-strands-agent">gerador `py#strands-agent`</Link> com `protocol=AG-UI`.
:::

## Pré-requisitos

Antes de usar este gerador, certifique-se de ter:

1. Um website React (gerado usando o <Link path="guides/react-website">gerador `ts#react-website`</Link>)
2. Um Python Strands Agent com `protocol=AG-UI` (gerado usando o <Link path="guides/py-strands-agent">gerador `py#strands-agent`</Link>)
3. Para agentes implantados, Cognito Auth adicionado via o <Link path="/guides/react-website-auth">gerador `ts#react-website-auth`</Link>

## Saída do Gerador

O gerador cria um componente `AguiProvider` **único compartilhado** e um hook por agente conectado:

<FileTree>

- src
  - components
    - AguiProvider.tsx `CopilotKitProvider` único para cada agente AG-UI. Regenerado em cada conexão para que seu registro `selfManagedAgents` sempre corresponda ao conjunto de hooks abaixo.
  - hooks
    - useAgui\<AgentName>.tsx Registra um agente AG-UI. Um arquivo por execução de `connection`.
    - useSigV4.tsx Assinatura SigV4 (somente IAM)

</FileTree>

Executar `connection` uma segunda vez para um agente diferente **adiciona um novo hook `useAgui<AgentName>.tsx`** e reescreve `AguiProvider.tsx` para registrar ambos os hooks. `main.tsx` mantém seu único wrapper `<AguiProvider>` — você nunca acaba com providers aninhados.

As seguintes dependências são adicionadas ao `package.json` raiz:

- `@copilotkit/react-core` — fornece `CopilotKitProvider` e componentes de chat (`CopilotChat`, `CopilotSidebar`, `CopilotPopup`)
- `@ag-ui/client` — `HttpAgent` usado pelos hooks gerados
- `aws4fetch`, `oidc-client-ts`, `react-oidc-context`, `@aws-sdk/credential-providers` — somente autenticação IAM
- `react-oidc-context` — autenticação Cognito

## Como Funciona

### Conexão AG-UI

Cada hook `useAgui<AgentName>` lê o valor de runtime do seu agente da <Link path="guides/runtime-config">Configuração de Runtime</Link> e instancia um `HttpAgent` `@ag-ui/client`:

- **Implantado**: o valor de runtime é um Bedrock AgentCore Runtime ARN, que é convertido para o endpoint HTTPS do AgentCore: `https://bedrock-agentcore.<region>.amazonaws.com/runtimes/<encoded-arn>/invocations?qualifier=DEFAULT`
- **Desenvolvimento local**: `serve-local` substitui o valor para a URL local do agente (ex: `http://localhost:8081`)

O `AguiProvider` compartilhado chama cada hook gerado e distribui cada um em `selfManagedAgents` em um único `CopilotKitProvider`, que os expõe todos aos componentes CopilotKit.

### Integração CopilotKit

[CopilotKit](https://docs.copilotkit.ai/aws-strands) é o cliente React de referência de primeira parte para o protocolo AG-UI e fornece componentes de chat prontos:

- `<CopilotChat />` — interface de chat completa
- `<CopilotSidebar />` — chat em painel lateral fixo
- `<CopilotPopup />` — popup de chat flutuante

Coloque qualquer um destes em qualquer lugar dentro do wrapper `<AguiProvider>` (já configurado em `main.tsx` para você).

### Autenticação

O código gerado lida com a autenticação dependendo da configuração do seu agente:

- **IAM** (padrão): usa requisições HTTP assinadas com AWS SigV4. As credenciais são obtidas do Cognito Identity Pool configurado com a autenticação do seu website.
- **Cognito**: incorpora o token de acesso JWT no cabeçalho `Authorization` como um token Bearer.
- **None**: sem autenticação (útil para desenvolvimento local).

## Usando o Código Gerado

### Adicionando uma Interface de Chat

Cada agente é registrado sob seu `agentName` (não `"default"`), então passe `agentId` no componente de chat para escolher com qual agente o chat conversa:

```tsx
import { CopilotChat } from '@copilotkit/react-core/v2';

function ChatPage() {
  return (
    <CopilotChat
      agentId="agent" // corresponde ao nome do agente de py#strands-agent
      labels={{
        welcomeMessageText: 'How can I help you today?',
        chatInputPlaceholder: 'Ask me anything...',
      }}
    />
  );
}
```

### Usando uma Sidebar ou Popup

```tsx
import { CopilotSidebar } from '@copilotkit/react-core/v2';

function App() {
  return (
    <div>
      <YourMainContent />
      <CopilotSidebar agentId="agent" />
    </div>
  );
}
```

### Conectando Múltiplos Agentes AG-UI

Execute o gerador `connection` uma vez por agente. O gerador adiciona um novo hook `useAgui<AgentName>` e regenera `AguiProvider.tsx` para registrar cada hook:

```tsx
// AguiProvider.tsx gerado (exemplo com dois agentes)
export const AguiProvider: FC<PropsWithChildren> = ({ children }) => {
  const researchAgentAgents = useAguiResearchAgent();
  const storyAgentAgents = useAguiStoryAgent();

  const selfManagedAgents = useMemo<Record<string, AbstractAgent>>(
    () => ({
      ...researchAgentAgents,
      ...storyAgentAgents,
    }),
    [researchAgentAgents, storyAgentAgents],
  );

  return (
    <CopilotKitProvider selfManagedAgents={selfManagedAgents}>
      {children}
    </CopilotKitProvider>
  );
};
```

Todos os agentes são visíveis de qualquer lugar no aplicativo — roteie cada chat com `agentId`:

```tsx
<CopilotChat agentId="story" />    {/* conversa com StoryAgent */}
<CopilotChat agentId="research" /> {/* conversa com ResearchAgent */}
```

:::caution[AguiProvider é regenerado]
`AguiProvider.tsx` é sobrescrito toda vez que você executa o gerador. Se você precisar adicionar lógica personalizada, envolva `<AguiProvider>` em seu próprio componente ao invés de editar o arquivo gerado.
:::

## Personalizando a Aparência

`<CopilotChat />` (e `<CopilotSidebar />`, `<CopilotPopup />`) usam um **sistema de slots** recursivo — você pode substituir qualquer sub-componente com uma string de classe Tailwind, um objeto de prop ou um componente React personalizado. Veja o [guia de slots do CopilotKit](https://docs.copilotkit.ai/aws-strands/custom-look-and-feel/slots) para a árvore completa de slots.

### Estilização Tailwind via slots

```tsx
<CopilotChat
  agentId="agent"
  // estiliza o input e seus filhos
  input={{
    textArea: 'text-blue-600',
    sendButton: 'bg-blue-600 hover:bg-blue-700',
  }}
  // estiliza slots de mensagem aninhados
  messageView={{
    assistantMessage: 'bg-blue-50 rounded-xl p-2',
    userMessage: 'bg-blue-100 rounded-xl',
  }}
/>
```

### Slots de nível superior

Os slots raiz disponíveis em `<CopilotChat />`, `<CopilotSidebar />` e `<CopilotPopup />`:

- `messageView` — container da lista de mensagens
- `scrollView` — container de rolagem
- `input` — entrada de texto + controles de envio
- `suggestionView` — pílulas de sugestão
- `welcomeScreen` — UI de estado vazio mostrada antes da primeira mensagem
- `labels` — mensagem de boas-vindas, placeholder, etc.

Sidebar e popup adicionalmente expõem `header` e `toggleButton`.

### Substituindo um slot com um componente personalizado

Qualquer slot pode receber um componente React ao invés de um className, então você pode substituir o padrão completamente:

```tsx
import { CopilotChat } from '@copilotkit/react-core/v2';

const MySendButton: React.FC<{ onClick: () => void }> = ({ onClick }) => (
  <button onClick={onClick} className="my-send-btn">
    ✨ Send
  </button>
);

<CopilotChat
  agentId="agent"
  input={{ sendButton: MySendButton }}
/>;
```

Substituições mais profundas seguem a mesma forma — ex: substitua apenas o botão de copiar nas mensagens do assistente:

```tsx
<CopilotChat
  agentId="agent"
  messageView={{
    assistantMessage: {
      copyButton: ({ onClick }) => <button onClick={onClick}>Copy</button>,
    },
  }}
/>
```

### Substituindo toda a visualização de chat

Para uma superfície de chat totalmente personalizada, passe um className (ou componente) para o slot `chatView`:

```tsx
<CopilotChat
  agentId="agent"
  chatView="bg-gray-50 rounded-xl shadow-lg"
/>
```

## Desenvolvimento Local

O gerador de conexão configura automaticamente a integração `serve-local`:

1. Executar `nx serve-local <website>` também iniciará o servidor local do agente
2. A configuração de runtime é substituída para apontar para a URL AG-UI local (ex: `http://localhost:8081`)
3. Tanto o website quanto o agente recarregam automaticamente juntos

<NxCommands commands={['serve-local <WebsiteProject>']} />

:::tip[Hot Reloading]
O website e o agente conectado recarregam automaticamente juntos, permitindo que você itere rapidamente em ambos os lados sem implantar na AWS.
:::

## Infraestrutura

Se seu agente usa autenticação IAM, a role autenticada do Cognito Identity Pool deve receber permissão para invocar o agente. Consulte o construto de infraestrutura do seu agente para o método `grantInvokeAccess` apropriado.

Este gerador cria um novo projeto TypeScript DynamoDB apoiado pelo [Amazon DynamoDB](https://aws.amazon.com/dynamodb/), usando [ElectroDB](https://electrodb.dev/) para modelagem de entidades com segurança de tipos. Ele gera o código da aplicação e a infraestrutura necessária para provisionar e gerenciar uma tabela DynamoDB usando AWS CDK ou Terraform, com suporte a design de tabela única e desenvolvimento local integrado via DynamoDB Local.

### Modelagem de Dados

O projeto gerado usa [ElectroDB](https://electrodb.dev/) para modelagem de entidades com segurança de tipos em uma única tabela DynamoDB, seguindo o [design de tabela única do DynamoDB](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/data-modeling-foundations.html). Adicione ou atualize arquivos de entidade em `src/entities/`, usando a entidade de exemplo gerada como ponto de partida.

Exemplo de definição de entidade:

```ts title="packages/my-table/src/entities/example.ts"
import { Entity } from 'electrodb';
import { getDynamoDBClient, resolveTableName } from '../client.js';

export const createExampleEntity = async () =>
  new Entity(
    {
      model: {
        entity: 'example',
        version: '1',
        service: 'MyTable',
      },
      attributes: {
        id: {
          type: 'string',
          required: true,
        },
        createdAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          readOnly: true,
        },
        updatedAt: {
          type: 'string',
          required: true,
          default: () => new Date().toISOString(),
          watch: '*',
          set: () => new Date().toISOString(),
        },
      },
      indexes: {
        primary: {
          pk: {
            field: 'pk',
            composite: ['id'],
          },
          sk: {
            field: 'sk',
            composite: [],
          },
        },
      },
    },
    { client: getDynamoDBClient(), table: await resolveTableName() },
  );
```

Para mais detalhes, consulte a [documentação de entidades do ElectroDB](https://electrodb.dev/en/modeling/entities/).

### Usando o Cliente DynamoDB

O arquivo `src/client.ts` gerado exporta dois utilitários principais:

- `getDynamoDBClient()` — retorna um singleton `DynamoDBClient` em cache. Quando `SERVE_LOCAL=true`, conecta-se à instância local do DynamoDB Local; caso contrário, cria um cliente AWS usando a cadeia de credenciais padrão.
- `resolveTableName()` — retorna o nome da tabela DynamoDB. Quando `SERVE_LOCAL=true`, retorna a constante do nome da tabela local; caso contrário, busca o nome do AWS AppConfig usando a variável de ambiente `RUNTIME_CONFIG_APP_ID` e o armazena em cache para chamadas subsequentes.

## Conectando à Tabela

Em qualquer projeto TypeScript, importe as fábricas de entidades do seu pacote DynamoDB e use-as diretamente:

```ts
import { createExampleEntity } from ':my-scope/my-table';

const entity = await createExampleEntity();
const result = await entity.query.primary({ id: '123' }).go();
```

:::note[Configuração de runtime]
Ao executar na AWS, `resolveTableName()` busca o nome da tabela do AWS AppConfig usando a variável de ambiente `RUNTIME_CONFIG_APP_ID`. Projetos construídos com este plugin (APIs tRPC, APIs Smithy, agentes, servidores MCP) já têm essa variável configurada automaticamente. Para outros projetos TypeScript, certifique-se de que `RUNTIME_CONFIG_APP_ID` esteja definida no ambiente de runtime com o ID da aplicação AppConfig provisionado pela sua infraestrutura. Para mais informações, consulte o <Link path="guides/runtime-config">guia de Configuração de Runtime</Link>.
:::

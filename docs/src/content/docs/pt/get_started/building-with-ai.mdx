O Plugin Nx para AWS inclui uma [Skill](https://code.claude.com/docs/en/skills)/[Power](https://kiro.dev/docs/powers/) e um [Servidor MCP](https://modelcontextprotocol.io/introduction) que juntos permitem que assistentes de IA trabalhem efetivamente com o plugin. Ao usá-los, você pode acelerar seu fluxo de desenvolvimento com IA enquanto se beneficia de seus projetos sendo gerados de forma determinística — gastando menos tempo e menos da sua janela de contexto na configuração dos componentes principais.

## Configure seu Assistente de IA

<Tabs syncKey="ai-assistant">
  <TabItem label="Kiro">
    O Plugin Nx para AWS é distribuído como um [Kiro Power](https://kiro.dev/docs/powers/) que agrupa o servidor MCP com documentação de direcionamento e guias de fluxo de trabalho.

    <Steps>

    1. Abra o painel Powers no Kiro e clique em **Add power from GitHub**
    2. Digite a seguinte URL:

       ```
       https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
       ```

    3. Clique em **Install**

    </Steps>

    O Kiro configura automaticamente o servidor MCP — nenhuma configuração adicional necessária.
  </TabItem>
  <TabItem label="Kiro CLI">
    Execute o seguinte comando:

    ```bash
    kiro-cli mcp add \
      --name nx-plugin-for-aws \
      --scope global \
      --command npx \
      --args "-y,@aws/nx-plugin-mcp"
    ```

    Consulte a [documentação MCP do Kiro CLI](https://kiro.dev/docs/cli/mcp/) para mais detalhes.
  </TabItem>
  <TabItem label="Claude Code">
    O Plugin Nx para AWS é distribuído como um [plugin do Claude Code](https://code.claude.com/docs/en/plugins) que agrupa uma [Skill](https://code.claude.com/docs/en/skills) com o servidor MCP.

    Execute os seguintes comandos dentro do Claude Code:

    ```
    /plugin marketplace add awslabs/nx-plugin-for-aws
    /plugin install nx-plugin-for-aws@nx-plugin-for-aws
    ```

    Em seguida, execute `/reload-plugins` para ativar o plugin.
  </TabItem>
  <TabItem label="Cursor">
    Adicione o seguinte a `~/.cursor/mcp.json` (ou `.cursor/mcp.json` no seu projeto):

    <Snippet name="mcp/config" />

    Consulte a [documentação MCP do Cursor](https://cursor.com/docs/mcp) para mais detalhes.
  </TabItem>
  <TabItem label="Codex">
    Adicione o seguinte a `~/.codex/config.toml`:

    ```toml
    [mcp_servers.nx-plugin-for-aws]
    command = "npx"
    args = ["-y", "@aws/nx-plugin-mcp"]
    ```

    Consulte a [documentação de configuração do Codex](https://developers.openai.com/codex/config-reference) para mais detalhes.

## Iniciar Vibe Coding

Peça ao seu Assistente de IA para construir algo usando o `nx-plugin-for-aws`! Ele normalmente criará um workspace, gerará componentes com os generators aplicáveis e depois preencherá a lógica de negócios.

Experimente um prompt como:

> *"Construa uma aplicação multi-agente usando o Plugin Nx para AWS. Eu quero um frontend React que se comunique com um agente orquestrador backend que delega para agentes especializados de pesquisa e escrita."*

Ou para um ponto de partida menor:

> *"Use o Plugin Nx para AWS para gerar um site React apoiado por uma API tRPC, com autenticação Cognito e infraestrutura CDK."*

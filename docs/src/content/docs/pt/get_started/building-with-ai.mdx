## Kiro Power

O Plugin Nx para AWS vem com um [Kiro Power](https://kiro.dev/docs/powers/) que agrupa o servidor MCP junto com documentação e guias de fluxo de trabalho, fornecendo ao Kiro um contexto rico sobre os generators disponíveis e como usá-los. Esta é a maneira recomendada de começar com o Kiro.

### Instalando o Power

<Steps>

1. Abra o painel Kiro Powers na barra lateral
2. Clique no botão `+` para adicionar um power personalizado
3. Cole a seguinte URL do GitHub:

   ```
   https://github.com/awslabs/nx-plugin-for-aws/tree/main/powers/nx-plugin-for-aws
   ```

4. Clique em instalar

</Steps>

Uma vez instalado, o Kiro automaticamente tem acesso ao servidor MCP `aws-nx-mcp` e suas ferramentas. Nenhuma configuração adicional de MCP é necessária.

### O que o Power Fornece

O power dá ao Kiro acesso às seguintes ferramentas MCP:

- `create_workspace_command` — Gera o comando para criar um novo workspace Nx pré-configurado com o plugin AWS
- `list_generators` — Descobre todos os generators disponíveis e seus parâmetros
- `generator_guide` — Obtém documentação detalhada para um generator específico, incluindo parâmetros, estrutura de arquivos e melhores práticas
- `general_guidance` — Obtém orientação geral sobre o uso do plugin

Também inclui documentação de direcionamento que ajuda o Kiro a entender o conjunto completo de generators disponíveis, fluxos de trabalho comuns e melhores práticas.

### Exemplos de Prompts

Uma vez que o power esteja instalado, você pode começar a construir conversando com o Kiro:

- *"Crie um novo workspace Nx chamado my-app usando pnpm"* — O Kiro irá gerar e executar o comando apropriado de criação de workspace
- *"Gere um site React com um backend de API tRPC e infraestrutura CDK"* — O Kiro irá gerar cada projeto e conectá-los
- *"Quais generators estão disponíveis no Plugin Nx para AWS?"* — O Kiro irá listar todos os generators com descrições e exemplos de uso
- *"Adicione uma função Lambda ao meu projeto TypeScript existente"* — O Kiro irá encontrar o generator correto e orientar no processo de geração
- *"Construa meu projeto"* - O Kiro irá executar o comando nx build

## Configuração do Servidor MCP

O Plugin Nx para AWS inclui um [Servidor MCP](https://modelcontextprotocol.io/introduction) que permite que assistentes de IA trabalhem com o Plugin Nx para AWS. Ao utilizar o Servidor MCP, você pode acelerar seu fluxo de desenvolvimento com IA, aproveitando projetos gerados de forma determinística, gastando menos tempo e menos do seu contexto na configuração dos componentes principais.

Use o Servidor MCP com seu Assistente de IA preferido, como Kiro, Amazon Q Developer, Cline, Claude Code ou Cursor.

Para outros Assistentes de IA, você pode configurar o servidor MCP manualmente. A maioria dos Assistentes de IA possui um arquivo JSON para configuração do servidor MCP.

O `@aws/nx-plugin` é um plugin do [Nx](https://nx.dev/) que fornece um conjunto de ferramentas para simplificar a construção e implantação de aplicações full-stack na AWS. Ele oferece aos desenvolvedores templates pré-configurados para códigos de aplicação e IaC, reduzindo significativamente o tempo gasto em configuração inicial. O plugin gerencia a complexidade da integração com serviços AWS enquanto mantém flexibilidade para personalizações.

Os usuários simplesmente escolhem quais componentes desejam na lista de [Geradores](#geradores) disponíveis, fornecem opções de configuração e deixam o `@aws/nx-plugin` gerar o código inicial necessário. Vários geradores existem neste toolkit, capazes de criar APIs, sites, infraestrutura e até realizar tarefas mais complexas como integrar frontend a backend (incluindo atualizações de arquivos existentes via transformações AST!) com clientes type-safe.

## Construído sobre o Nx

O [Nx](https://nx.dev) é um framework de build inteligente que gerencia projetos poliglotas complexos através de dois pilares: um sistema de build inteligente e geradores de código.

O sistema de build usa cache avançado e análise de grafo de dependências para otimizar a execução de tarefas. Ele reconstroi apenas o necessário com base em alterações de código, reduzindo drasticamente os tempos de build. Tarefas podem rodar em paralelo, com o Nx determinando automaticamente a ordem ideal de execução baseada em dependências do projeto. Isso torna operações como testes, linting e builds altamente eficientes, especialmente em bases de código grandes.

Geradores funcionam como ferramentas automatizadas de scaffolding que criam estruturas de código consistentes entre projetos. Eles podem produzir desde componentes individuais até aplicações completas, seguindo templates e padrões pré-definidos. Equipes podem personalizar esses geradores para atender requisitos específicos e padrões de codificação.

Juntas, essas características tornam o Nx particularmente eficaz para gerenciar monorepos e garantir práticas de desenvolvimento consistentes entre equipes, mantendo alta velocidade de desenvolvimento.

## Geradores

Todos os componentes disponíveis neste plugin são implementados como [Geradores](https://nx.dev/features/generate-code) que automatizam tarefas repetitivas e permitem scaffolding consistente em projetos novos e existentes.

Geradores podem ser invocados via [CLI do Nx](https://nx.dev/features/generate-code#invoke-generators) ou [plugin IDE Nx Console](https://nx.dev/getting-started/editor-setup).

## Aberto para modificações

Todo código gerado é seu código, podendo ser editado livremente. Isso simplifica drasticamente a experiência do desenvolvedor, pois se algo não agradar no código gerado, você pode modificá-lo diretamente.

## Dependências mínimas

O `@aws/nx-plugin` busca manter o número de dependências no mínimo, portanto os requisitos globais dependem apenas dos geradores utilizados.

Por exemplo, geradores baseados em Typescript só requerem [Node](https://nodejs.org/en/download). Para projetos Python, apenas [UV](https://docs.astral.sh/uv/) é necessário.

<Aside>
  UV foi escolhido como gerenciador de pacotes e projetos Python por vários motivos:
  - Ferramenta única que substitui `pip`, `pip-tools`, `pipx`, `poetry`, 
    `pyenv`, `twine`, `virtualenv` e outros
  - 10-100x mais rápido que `pip`
  - Suporte nativo a [workspaces/monorepos](https://docs.astral.sh/uv/concepts/projects/workspaces/)
</Aside>

## Política de versão única

A configuração padrão de monorepo utiliza uma [política de versão única](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) para projetos Node e Python.

Isso significa que todos os projetos no monorepo usam a mesma versão de dependências por padrão, reduzindo problemas de incompatibilidade entre pacotes.

Para Node, isso implica um único lockfile na raiz com uma única `node_modules` contendo todas as dependências. Novas dependências são adicionadas no `package.json` raiz.

Para Python, há um único `.venv` na raiz do monorepo com todas as dependências. Cada projeto Python tem seu próprio `pyproject.toml`, mas as versões são gerenciadas pelo workspace UV e registradas no arquivo `uv.lock` raiz.

## Segurança de tipos

O `@aws/nx-plugin` emprega type-safety para simplificar a experiência do desenvolvedor via autocompletions de IDE, além de eliminar erros em runtime comuns em implementações não type-safe. Todos os componentes fornecidos são type-safe por padrão.

O `@aws/nx-plugin` é um plugin [Nx](https://nx.dev/) que fornece um conjunto de ferramentas para simplificar a construção e implantação de aplicações full-stack na AWS. Ele oferece templates pré-configurados para códigos de aplicação e Infraestrutura como Código (IaC), reduzindo significativamente o tempo gasto em configurações. O plugin gerencia a complexidade da integração com serviços AWS enquanto mantém flexibilidade para personalizações.

Os usuários podem escolher componentes específicos da lista de [geradores](#generators) disponíveis, fornecer opções de configuração e deixar que o `@aws/nx-plugin` gere o código inicial necessário. Você pode usar geradores específicos deste toolkit para criar APIs, sites, infraestrutura e elementos complexos como integração de frontend com backend (incluindo atualizações de arquivos existentes via transformações de Árvore Sintática Abstrata (AST)) com clientes type-safe.

## Construído sobre o Nx

[Nx](https://nx.dev) é um framework de build inteligente que gerencia projetos poliglotas complexos através de dois pilares: um sistema de build avançado e geradores de código.

O sistema de build usa cache inteligente e análise de grafo de dependências para otimizar execução de tarefas. Ele reconstrói apenas o necessário com base em mudanças no código, reduzindo drasticamente tempos de build. Tarefas rodam em paralelo, com o Nx determinando automaticamente a ordem ideal de execução baseada em dependências. Isso torna operações como testes, linting e builds altamente eficientes, especialmente em codebases grandes.

Geradores funcionam como ferramentas de scaffolding automatizado que criam estruturas de código consistentes. Eles podem gerar desde componentes individuais até aplicações completas, seguindo templates pré-definidos. Equipes podem personalizar esses geradores para atender requisitos específicos e padrões de codificação.

Juntas, essas características tornam o Nx particularmente eficaz para gerenciar monorepos e garantir práticas de desenvolvimento consistentes entre equipes, mantendo alta velocidade de desenvolvimento.

## Geradores

Todos os componentes deste plugin são implementados como [Geradores](https://nx.dev/features/generate-code) que automatizam tarefas repetitivas e permitem scaffolding consistente em projetos novos e existentes.

Você pode invocar geradores via [CLI do Nx](https://nx.dev/features/generate-code#invoke-generators) ou [plugin IDE Nx Console](https://nx.dev/getting-started/editor-setup).

## Aberto para modificações

Todo código gerado é seu código, ou seja, pode ser editado livremente. Isso simplifica drasticamente a experiência do desenvolvedor - se algo no código gerado não lhe agradar, você pode modificá-lo diretamente.

## Dependências mínimas

O `@aws/nx-plugin` busca minimizar o número de dependências, portanto os requisitos globais dependem dos geradores utilizados.

Exemplos:

- Geradores baseados em TypeScript exigem apenas [Node](https://nodejs.org/en/download) instalado
- Projetos Python exigem apenas [UV](https://docs.astral.sh/uv/)

<Aside>
  Escolhemos o UV como gerenciador de pacotes Python pelos seguintes motivos:
  - Substitui múltiplas ferramentas: `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv`
  - É 10-100x mais rápido que `pip`
  - Suporte nativo a [workspaces/monorepos](https://docs.astral.sh/uv/concepts/projects/workspaces/)
</Aside>

## Política de versão única

A configuração padrão de monorepo usa uma [política de versão única](https://nx.dev/concepts/decisions/dependency-management#single-version-policy) para projetos Node e Python.

Isso significa que todos os projetos no monorepo usam a mesma versão de dependências por padrão, reduzindo problemas de incompatibilidade entre pacotes.

Para projetos Node:
- Um único lockfile na raiz com um único `node_modules` contém todas as dependências
- Novas dependências devem ser adicionadas no `package.json` da raiz

Para projetos Python:
- Um único `.venv` na raiz contém todas as dependências instaladas
- Cada projeto Python tem seu próprio `pyproject.toml`, mas as versões são gerenciadas pelo workspace UV e registradas no arquivo `uv.lock` da raiz

## Type safety

O `@aws/nx-plugin` utiliza type-safety para simplificar a experiência do desenvolvedor via autocompletar na IDE, além de eliminar erros em runtime que ocorreriam em implementações não type-safe. Todos os componentes fornecidos são type-safe por padrão.

Vários geradores no Nx Plugin for AWS produzem destinos de build baseados em Docker (por exemplo, infraestrutura CDK com imagens de contêiner Lambda, compilação de modelo de API Smithy e implantações de servidor MCP / agente Strands). Esses destinos constroem imagens Docker **Linux**, o que requer um daemon Docker Linux.

No macOS e Linux isso funciona imediatamente. No Windows, o Docker usa por padrão o **modo de contêiner Windows**, então um passo de configuração extra é necessário para fornecer um ambiente de build Linux. Existem duas opções:

## Opção A: Docker Desktop (Recomendado)

O [Docker Desktop](https://www.docker.com/products/docker-desktop/) inclui uma VM Linux gerenciada e lida transparentemente com builds de contêiner Linux. Este é o caminho mais simples para desenvolvimento local.

<Steps>
1. **Instale o Docker Desktop** em [docker.com](https://www.docker.com/products/docker-desktop/).

2. **Habilite o backend WSL 2** (padrão em instalações modernas):
   - Abra o Docker Desktop **Settings > General**
   - Certifique-se de que **Use the WSL 2 based engine** esteja marcado

3. **Verifique** abrindo um terminal e executando:
   ```bash
   docker version
   docker buildx ls
   ```
   Você deve ver `OS/Arch: linux/amd64` na seção do servidor.
</Steps>

É isso — o Docker Desktop gerencia a VM Linux, Buildx e suporte multiplataforma para você.

## Opção B: Docker Engine no WSL 2 (Sem Docker Desktop)

Se você preferir não instalar o Docker Desktop (por exemplo, devido ao licenciamento), você pode instalar o Docker Engine diretamente dentro de uma distribuição Linux WSL 2 e usar um pequeno script wrapper para que as ferramentas do lado do Windows (como Nx) possam invocar o `docker` de forma transparente.

### Pré-requisitos

- **Windows 10 versão 2004+** ou **Windows 11**
- **WSL 2** habilitado (`wsl --install` se ainda não estiver)

### Instalar Ubuntu + Docker Engine

<Steps>
1. **Instale uma distribuição Ubuntu no WSL 2:**

   ```powershell
   wsl --install Ubuntu-24.04 --no-launch
   wsl --set-default Ubuntu-24.04
   ```

2. **Instale o Docker Engine dentro do WSL 2:**

   Abra o shell Ubuntu (`wsl -d Ubuntu-24.04`) e execute:

   ```bash
   # Add Docker's official GPG key and repository
   sudo apt-get update -qq
   sudo apt-get install -y ca-certificates curl
   sudo install -m 0755 -d /etc/apt/keyrings
   sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
     -o /etc/apt/keyrings/docker.asc
   sudo chmod a+r /etc/apt/keyrings/docker.asc
   echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc] \
     https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
     | sudo tee /etc/apt/sources.list.d/docker.list

   # Install Docker Engine + Buildx
   sudo apt-get update -qq
   sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
     docker-buildx-plugin
   ```

3. **Inicie o Docker e crie um builder Buildx:**

   ```bash
   sudo dockerd &>/var/log/dockerd.log &

   # Wait for the daemon to be ready
   until docker info &>/dev/null; do sleep 1; done

   # Set up a Buildx builder for multi-platform builds
   docker buildx create --name linux-builder --driver docker-container --use
   docker buildx inspect --bootstrap --builder linux-builder
   ```

4. **Crie um wrapper `docker.cmd`** para que as ferramentas do lado do Windows usem o daemon WSL 2:

   Salve o seguinte como `C:\docker-wsl\docker.cmd` (ou qualquer diretório que você adicione ao seu `PATH`):

   ```bat
   @echo off
   for /f "usebackq tokens=*" %%i in (`wsl -d Ubuntu-24.04 wslpath "%CD%"`) do set WSL_CWD=%%i
   if "%1"=="build" (
     echo %* | findstr /i /c:"--output" >nul
     if errorlevel 1 (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %* --load
     ) else (
       wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- env BUILDX_BUILDER=linux-builder docker %*
     )
   ) else (
     wsl -d Ubuntu-24.04 -u root --cd "%WSL_CWD%" -- docker %*
   )
   ```

   Em seguida, adicione `C:\docker-wsl` à **frente** do seu `PATH` do sistema para que tenha prioridade sobre qualquer `docker.exe` existente.

5. **Verifique:**

   ```powershell
   docker version
   docker buildx ls
   ```
</Steps>

:::note
Com a Opção B, você precisa iniciar o daemon Docker dentro do WSL 2 cada vez que reiniciar. Você pode automatizar isso adicionando `sudo dockerd &>/dev/null &` ao seu perfil de shell WSL, ou criando uma entrada no Agendador de Tarefas do Windows.
:::

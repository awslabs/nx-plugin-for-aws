## Tarefa 1: Configurar o servidor de desenvolvimento local

Para começar a construir a UI, precisamos configurar nosso servidor de desenvolvimento local para apontar para o sandbox implantado. Para fazer isso, execute o seguinte comando:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Este comando irá baixar o `runtime-config.json` que está implantado e armazená-lo localmente na pasta `packages/game-ui/public`.

Para iniciar o servidor de desenvolvimento, execute o seguinte comando:

<NxCommands commands={["serve game-ui"]} />

Abra seu site local em um navegador. Você será solicitado a fazer login com o usuário Cognito que você criou anteriormente. Após fazer login, você deverá ver o site base.

<Aside type="caution" title="Manter o servidor de dev ativo">
Recomendamos manter o servidor de desenvolvimento em execução durante o restante deste módulo, pois ele recarregará automaticamente quaisquer alterações que fizermos.
</Aside>

## Tarefa 4: Criar as páginas do jogo

No <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, conectamos nossa `GameUI` ao Story Agent. Como o agente foi gerado com `--protocol=AG-UI`, o gerador de conexão configurou um `AguiProvider` do CopilotKit e um hook por agente (`useAguiStoryAgent`) em vez do cliente OpenAPI usual. Podemos renderizar o chat com um único componente `<CopilotChat />` e controlá-lo por `threadId` (que usamos como o id de sessão do AgentCore).

Crie dois novos arquivos em `packages/game-ui/src/routes/game/`:

<Tabs>
<TabItem label="index.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/index.tsx.template" lang="tsx" />
</TabItem>
<TabItem label="$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

<Aside type="tip" title="Parâmetros de caminho">
A sintaxe `$playerName` instrui o `@tanstack/react-router` a tratar `playerName` como um [parâmetro de caminho](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). Usamos o nome do jogador como o `threadId` do AG-UI para que cada jogo tenha seu próprio histórico de chat, e o preenchemos com 33 caracteres para satisfazer o requisito de comprimento do id de sessão do AgentCore.
</Aside>

<Aside type="note" title="Iniciando a primeira mensagem">
Na primeira montagem, verificamos `agent.messages.length === 0` e, se estiver vazio, enviamos uma única mensagem de usuário de abertura: `I am <player>, playing in the <genre> genre.` O prompt do sistema instrui o agente a começar a história imediatamente a partir dessa deixa. A partir de então, cada interação (incluindo atualizações de página e retornos posteriores) apenas recarrega a conversa através da sessão AG-UI — nenhum endpoint de histórico customizado necessário.
</Aside>
</TabItem>
</Tabs>

Após salvar esses arquivos, seu servidor de desenvolvimento local (http://localhost:4200/) deve ter seu jogo pronto para jogar.

Para compilar seu código, execute o seguinte comando:

### Implantar sua aplicação
Para implantar sua aplicação, execute o seguinte comando:

<NxCommands commands={['deploy infra dungeon-adventure-infra-sandbox/*']} />

Após a implantação, navegue até sua URL do Cloudfront. Para encontrar esta URL, inspecione as saídas do deploy do CDK.

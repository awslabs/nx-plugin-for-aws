## Tarefa 1: Configurar o servidor de desenvolvimento local

Baixe o `runtime-config.json` implantado localmente para que o servidor de desenvolvimento possa se comunicar com o backend real:

<NxCommands commands={["run game-ui:load:runtime-config"]} />

Em seguida, inicie o servidor de desenvolvimento:

<NxCommands commands={["serve game-ui"]} />

Abra seu site local em um navegador, faça login e siga as instruções para criar um novo usuário. Você deverá ver o site base Shadcn:

<Image src={baselineWebsitePng} alt="baseline-website.png" width="800" height="600" />

<Aside type="caution" title="Manter o servidor de dev ativo">
Deixe o servidor de desenvolvimento em execução durante o restante deste módulo — ele recarrega automaticamente cada alteração abaixo.
</Aside>

## Tarefa 2: Onde o CopilotKit já está configurado

Quando você executou o gerador `connection` para `game-ui → story` no <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, a integração AG-UI do site Shadcn foi gerada para você. Vale a pena dar uma olhada rápida:

<FileTree>
- packages/game-ui/src/
  - components/
    - AguiProvider.tsx Single `CopilotKitProvider` registered with every AG-UI agent.
    - copilot/
      - index.tsx Re-exports themed `CopilotChat` / `CopilotSidebar` / `CopilotPopup`.
      - ShadcnAssistantMessage.tsx, ShadcnUserMessage.tsx, ShadcnChatInput.tsx, ShadcnCursor.tsx, copilot.css
  - hooks/
    - useAguiStoryAgent.tsx Instantiates an `@ag-ui/client` `HttpAgent` pointing at the deployed Story Agent and pads `threadId` to AgentCore's 33-character minimum session id.
  - main.tsx Wraps `<App />` in `<AguiProvider>`
</FileTree>

Tudo o que precisamos fazer é inserir um `<CopilotChat agentId="agent" threadId={...} />` em uma rota. Para mais detalhes sobre como a integração é montada, consulte o <Link path="guides/connection/react-agui">guia de conexão React → AG-UI</Link>.

## Tarefa 4: Criar as rotas do jogo

Precisamos de duas rotas — uma para escolher um herói, outra para jogar. Ambas usam componentes shadcn e o chat do CopilotKit; não há UI de chat feita manualmente.

<Tabs>
<TabItem label="routes/index.tsx">
<E2EDiff before="dungeon-adventure/4/routes/index.tsx.old.template" after="dungeon-adventure/4/routes/index.tsx.template" lang="tsx" />

Este é o seletor de jogo: formulário de novo jogo (shadcn `Input` + `Button` + `Card`) mais uma lista "Continuar" alimentada por `useGameApi().games.query`. Iniciar um jogo executa `saveGame` com o par `(playerName, genre)` (para que apareça na próxima vez) e navega para a rota de jogo.
</TabItem>
<TabItem label="routes/game/$playerName.tsx">
<E2ECode path="dungeon-adventure/4/routes/game/$playerName.tsx.template" lang="tsx" />

Esta é a rota de jogo. Ela constrói o `threadId` determinístico (``{player}-{genre}`` preenchido para 33 caracteres — o hook AG-UI o enviará literalmente como o id de sessão do AgentCore), renderiza `<CopilotChat agentId="agent" threadId={threadId} />` e sobrepõe o inventário de `useGameApi().inventory.query` por cima. Em um thread novo (sem mensagens armazenadas ainda), a rota usa `useAgent` para enviar uma mensagem de preparação com o nome do jogador e o gênero; o prompt do sistema do agente diz a ele para usar isso para definir a cena. Visitas subsequentes acessam o `FileSessionManager` do agente, que reproduz o histórico, então o `useEffect` de preparação não faz nada.

<Aside type="tip" title="threadIds determinísticos tornam /game/$playerName?genre=... compartilhável">
Revisitar a mesma URL — seja após um reload, um redeploy ou em outro dispositivo — resolve para o mesmo `threadId`, então o agente continua contando a mesma história.
</Aside>

<Aside type="tip" title="Parâmetros de caminho">
A sintaxe `$playerName` diz ao `@tanstack/react-router` para tratar `playerName` como um [parâmetro de caminho](https://tanstack.com/router/v1/docs/framework/react/guide/path-params). `validateSearch` torna `genre` fortemente tipado contra nosso enum.
</Aside>
</TabItem>
</Tabs>

Você pode excluir o diretório scaffolded `packages/game-ui/src/routes/welcome` — estamos usando `/` como nosso seletor de jogo.

Após salvar, o servidor de desenvolvimento em `http://localhost:4200/` deve agora permitir que você inicie uma aventura e converse com o Story Agent.

## Conclusão

Parabéns! Você criou um jogo de aventura em masmorra com IA Agêntica que usa muitos dos geradores disponíveis no `@aws/nx-plugin`. 🎉🎉🎉

## Próximos passos?

Recomendamos que você tente expandir a base de código com as seguintes capacidades por conta própria:

1. Adicione o <Link path="guides/license">gerador de licença</Link> ao seu projeto para gerenciar automaticamente arquivos LICENSE e cabeçalhos no código-fonte.
2. <Link path="get_started/tutorials/contribute-generator">Contribua com seu próprio gerador</Link> para simplificar a adição de novas APIs tRPC.
3. Integre o [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html) com o Story Agent para que aventuras de longa duração sobrevivam a uma reinicialização do servidor. O adaptador `ag_ui_strands` expõe um hook `session_manager_provider` exatamente para isso - veja a documentação sobre [usando AgentCore Memory com Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
4. Explore os diversos guias de componentes usando a navegação à esquerda.

## Destruindo recursos

1. Para destruir os recursos AWS criados, execute o comando:

<NxCommands commands={['destroy infra "dungeon-adventure-infra-sandbox/*"']} />

    Isso solicitará uma lista de stacks para deletar, que devem incluir `dungeon-adventure-infra-sandbox/Application/GameUI/waf` e `dungeon-adventure-infra-sandbox/Application`.

2. Digite `Y` para continuar. O CloudFormation destruirá suas stacks.

## Tarefa 1: Implementar o Agente de História

O Agente de História é um agente [Strands](https://strandsagents.com/) exposto via protocolo [AG-UI](https://ag-ui.com/). Cada sessão AG-UI (identificada por um `threadId`) mapeia para uma instância de Agente de longa duração no servidor, então o histórico de conversação do agente é mantido em memória durante a vida da sessão.

Atualizaremos o prompt de sistema do agente para jogar uma aventura de dungeon e removeremos a ferramenta de exemplo.

### Implementação do Agente

Atualize `packages/story/dungeon_adventure_story/agent/agent.py`:

<E2EDiff lang="python" before="dungeon-adventure/3/agent.py.old.template" after="dungeon-adventure/3/agent.py.template" />

O prompt de sistema informa ao agente que a primeira mensagem será da forma `I am <player>, playing in the <genre> genre.` — nossa UI envia exatamente isso como a mensagem de usuário de abertura para que o agente tenha tudo o que precisa para iniciar a história. Como a biblioteca AG-UI retém o histórico de conversação de cada thread automaticamente, não precisamos passar o nome do jogador ou gênero nas rodadas subsequentes.

O gerador de conexões já configurou o cliente Inventory MCP no <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, então o agente tem acesso às ferramentas de inventário.

:::note[Onde está o `main.py`?]
O wrapper FastAPI do protocolo AG-UI lida com a tradução de requisição/resposta para nós. O `main.py` que o gerador produziu já cria o wrapper em torno de `get_agent(...)` — não precisamos tocá-lo.
:::

### Criar um usuário de teste no Cognito

Antes de testar o agente implantado (e antes de jogarmos o jogo via UI no próximo módulo), crie um usuário de teste no Cognito. Desabilitamos o auto-registro anteriormente, então este passo é necessário independentemente de você testar localmente ou contra o runtime implantado:

```bash
# Das saídas do CDK
export POOL_ID="<UserPoolId das saídas do CDK>"
export CLIENT_ID="<UserPoolClientId das saídas do CDK>"
export REGION="<sua-regiao>"

# Desabilite MFA para simplificar a criação do usuário
aws cognito-idp set-user-pool-mfa-config \
  --mfa-configuration OFF \
  --user-pool-id $POOL_ID \
  --region $REGION

# Criar um usuário de teste
aws cognito-idp admin-create-user \
  --user-pool-id $POOL_ID \
  --username "test" \
  --temporary-password "TempPass123-" \
  --region $REGION \
  --message-action SUPPRESS > /dev/null

# Definir uma senha permanente (substitua por algo mais seguro!)
aws cognito-idp admin-set-user-password \
  --user-pool-id $POOL_ID \
  --username "test" \
  --password "PermanentPass123-" \
  --region $REGION \
  --permanent > /dev/null
```

### Testar seu Agente

Você pode testar seu Agente de duas formas:
<ul>
<li>Iniciando uma instância local do servidor Agent e conversando com ele interativamente usando o destino `agent-chat`, ou</li>
<li>Chamando a API implantada usando curl com um token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  O destino `agent-chat` inicia o agente localmente (junto com suas dependências, via `agent-serve-local`) e coloca você em um prompt de chat interativo. Execute:

    <NxCommands commands={["agent-chat story"]} />

    :::note[`agent-serve` vs `agent-serve-local`]
    Se você quiser conversar com o agente contra seu servidor MCP **implantado** em vez de um local, inicie o servidor diretamente com `agent-serve` e passe `RUNTIME_CONFIG_APP_ID` / `AWS_REGION`:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve story"]} />

    então em outro shell execute `agent-chat-cli agui http://localhost:8081/invocations`.
    :::

    Quando solicitado, tente iniciar o jogo com algo como:

    ```
    I am Aria, playing in the medieval genre.
    ```
  </TabItem>
  <TabItem label="Implantado">
    Troque as credenciais do usuário de teste por um token de acesso do Cognito:

    ```bash
    # Autentique o usuário de teste e capture seu token de acesso
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoque o agente implantado usando a URL do runtime do Bedrock AgentCore. O protocolo AG-UI espera um payload `RunAgentInput` — um mínimo que envia a mensagem de usuário de abertura se parece com:

    ```bash
    # Defina a ARN do Story Agent das saídas do CDK
    export AGENT_ARN="<StoryAgentArn das saídas do CDK>"

    # Codifique a ARN em URL
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construa a URL de invocação
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoque o agente
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "Accept: text/event-stream" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{
        "threadId": "abcdefghijklmnopqrstuvwxyz-123456789",
        "runId": "run-1",
        "messages": [{"id": "m1", "role": "user", "content": "I am UnnamedHero, playing in the superhero genre."}],
        "tools": [], "context": [], "forwardedProps": {}, "state": {}
      }'
    ```

    :::note[Codificação de URL]
    A codificação URL é necessária para formatar corretamente a ARN para o endpoint da API do Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Se o comando for executado com sucesso, você começará a ver eventos Server-Sent do AG-UI sendo transmitidos de volta — eventos `TEXT_MESSAGE_CONTENT` carregam o texto da história pedaço por pedaço.

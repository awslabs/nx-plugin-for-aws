## Módulo 3: Implementação do Agente de História

<Aside type="caution">
Certifique-se que sua conta AWS habilitou acesso ao [modelo padrão Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) no Bedrock para sua região alvo seguindo os passos do [guia oficial](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access-modify.html). Na data desta escrita, o modelo é Anthropic Claude Sonnet 4.
</Aside>

O Story Agent é um agente [Strands](https://strandsagents.com/) que, dado um `Game` e uma lista de `Action`s como contexto, irá progredir uma história. Configuraremos o agente para interagir com nosso Inventory MCP Server e gerenciar os itens disponíveis do jogador.

:::note
Para simplificar este tutorial, implementamos o Agente de forma "stateless" onde cada invocação é uma sessão independente incluindo todo histórico. Para produção, considere integrar com [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html). Mais documentação sobre uso com Strands está [disponível aqui](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

  <TabItem label="Deployed">
    Para testar a versão implantada, autentique via Cognito para obter token JWT. Primeiro configure variáveis:

    ```bash
    export POOL_ID="<UserPoolId dos outputs CDK>"
    export CLIENT_ID="<UserPoolClientId dos outputs CDK>"
    export REGION="<sua-regiao>"
    ```

    Crie usuário teste e obtenha token:

    ```bash
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "testuser" \
      --temporary-password "TempPass123!" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "testuser" \
      --password "PermanentPass123!" \
      --region $REGION \
      --permanent > /dev/null

    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='testuser',PASSWORD='PermanentPass123!' \
      --region $REGION | jq -r '.AuthenticationResult.IdToken')
    ```

    :::caution
    Use o output `UserIdentityUserIdentityUserPoolIdXXX` para `POOL_ID` e verifique o Client ID nos outputs CDK.
    :::

    Invoke o agente implantado:

    ```bash
    export AGENT_ARN="<StoryAgentArn dos outputs CDK>"
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')
    export MCP_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    curl -N -X POST "$MCP_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}'
    ```

    :::note
    A codificação URL é necessária para formatar corretamente a ARN no endpoint Bedrock AgentCore.
    :::
  </TabItem>

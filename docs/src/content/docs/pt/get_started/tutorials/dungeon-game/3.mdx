## Tarefa 1: Implementar o Agente de História

O Agente de História é um agente [Strands](https://strandsagents.com/) gerado com `--protocol=AG-UI` no <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, para que a UI possa transmitir a partir dele através do [protocolo Agent-User Interaction](https://docs.copilotkit.ai/aws-strands/protocol) via CopilotKit. Ele usa o Inventory MCP Server para gerenciar os itens do jogador, e o [`S3SessionManager`](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/agents/sessions/) integrado do Strands para persistir o histórico de conversação no bucket de sessões que provisionamos no Módulo 2.

As alterações são:

- `main.py` adiciona um `session_manager_provider` que cria um `S3SessionManager` por `thread_id` quando implantado, e retorna para um `FileSessionManager` em disco sob `/tmp/strands-sessions` quando executado sob `agent-dev` (`LOCAL_DEV=true`). Quando implantado, o bucket S3 é o mesmo do qual o `queryActions` da Game API lê, para que o navegador possa reconstruir transcrições ao revisitar; localmente o agente persiste sessões em disco e se comunica com o servidor MCP local, sem uma implantação.
- `agent.py` remove a ferramenta de exemplo `subtract` e substitui o prompt de sistema por um de mestre de dungeon que convida a primeira mensagem do usuário a declarar o nome do jogador e o gênero, e usa as ferramentas do Inventory MCP Server.

### Conversar com seu Agente

O destino `agent-chat` gerado abre um REPL interativo contra seu agente. Ele é executado de forma independente e se conecta ao seu agente em execução local, então primeiro inicie o servidor local do agente em um terminal:

<NxCommands commands={["agent-dev story"]} />

Em seguida, em um segundo terminal, inicie o chat:

<NxCommands commands={["agent-chat story"]} />

Sua primeira mensagem deve informar ao agente o nome do seu herói e o gênero (por exemplo: `My name is Alice. Start my zombie adventure.`) e a história será transmitida de volta.

<Aside type="caution" title="Credenciais AWS necessárias">
Embora o agente seja executado localmente, ele invoca o [Amazon Bedrock](https://aws.amazon.com/bedrock/) para o LLM. Você precisará de credenciais AWS em seu ambiente com acesso ao [modelo padrão do Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/). Consulte a [documentação do provedor de modelo do Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) para obter detalhes.
</Aside>

## Tarefa 1: Implementar o Agente de História

O Agente de História é um agente [Strands](https://strandsagents.com/) gerado com `--protocol=AG-UI` no <Link path="get_started/tutorials/dungeon-game/1">Módulo 1</Link>, para que a UI possa transmitir a partir dele através do [protocolo Agent-User Interaction](https://docs.copilotkit.ai/aws-strands/protocol) via CopilotKit. Ele usa o Inventory MCP Server para gerenciar os itens do jogador, e um `FileSessionManager` para persistir o histórico de conversação em disco.

:::note[Persistindo o histórico de conversação]
Cada requisição AG-UI carrega um `thread_id`. Nossa UI constrói esse id de forma determinística como ``{player}-{genre}`` preenchido até o comprimento mínimo do AgentCore de 33 caracteres, para que cada par `(player, genre)` retome sua própria história. No lado do agente, `StrandsAgentConfig.session_manager_provider` fornece a cada thread um `FileSessionManager` apontado para um diretório em disco — assim, turnos subsequentes reproduzem o histórico de mensagens armazenado sem que a UI precise enviá-lo. Para uma configuração de produção, substitua isso por [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html) via [AgentCore Memory for Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

### Testar seu Agente

Você pode testar seu Agente de duas formas:
<ul>
<li>Iniciando uma instância local do servidor Agent e conversando com ele através do destino `agent-chat` gerado, ou</li>
<li>Chamando a API implantada usando curl com um token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  Inicie seu servidor Agent local:
    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["agent-serve dungeon_adventure.story"]} />

    <Aside type="caution" title="Usar valores de saída do CDK">
    Use o valor de saída `RuntimeConfigApplicationId` do CDK deploy para a variável de ambiente `RUNTIME_CONFIG_APP_ID`, e sua região AWS implantada para `AWS_REGION`.
    </Aside>

    :::note[`agent-serve` vs `agent-serve-local`]
    O destino `agent-serve` inicia o agente localmente mas conecta-se ao seu servidor MCP **implantado** (via `RUNTIME_CONFIG_APP_ID`). Se você preferir executar o servidor MCP localmente também, use `agent-serve-local` em vez disso, que inicia todos os serviços dependentes e define `SERVE_LOCAL=true` para conexões HTTP diretas.
    :::

    Então, em um segundo terminal, abra um REPL interativo contra o agente AG-UI local através do destino `agent-chat` gerado:

    <NxCommands commands={["run dungeon_adventure.story:agent-chat"]} />

    Isso usa [`agent-chat-cli`](https://www.npmjs.com/package/agent-chat-cli) para falar o protocolo AG-UI contra `http://localhost:8081/invocations`. Sua primeira mensagem deve informar ao agente o nome do seu herói e o gênero (por exemplo: `My name is Alice. Start my zombie adventure.`) e a história será transmitida de volta.

    <Aside type="tip" title="curl bruto">
    Se você preferir uma chamada HTTP bruta, a forma `RunAgentInput` do AG-UI é:
    ```bash
    curl -N -X POST http://127.0.0.1:8081/invocations \
      -H "Content-Type: application/json" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```
    Reutilizar o mesmo `threadId` em chamadas subsequentes continuará a mesma história — o `FileSessionManager` reproduz as mensagens anteriores.
    </Aside>
  </TabItem>
  <TabItem label="Implantado">
    Para testar o agente implantado, você precisará autenticar com Cognito e obter um token JWT. Primeiro, configure suas variáveis de ambiente:

    ```bash
    # Defina seu Cognito User Pool ID e Client ID das saídas do CDK
    export POOL_ID="<UserPoolId das saídas do CDK>"
    export CLIENT_ID="<UserPoolClientId das saídas do CDK>"
    export REGION="<sua-regiao>"
    ```

    Crie um usuário de teste e obtenha um token de autenticação:

    ```bash
    # Desabilite MFA para simplificar a criação do usuário
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Criar Usuário
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Definir Senha Permanente (substitua por algo mais seguro!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Autenticar Usuário e capturar Access Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoque o agente implantado usando a URL do runtime do Bedrock AgentCore:

    ```bash
    # Defina a ARN do Story Agent das saídas do CDK
    export AGENT_ARN="<StoryAgentArn das saídas do CDK>"

    # Codifique a ARN em URL
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construa a URL de invocação
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoque o agente com um payload AG-UI RunAgentInput
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: Alice-zombie00000000000000000000000" \
      -d '{
        "threadId": "Alice-zombie00000000000000000000000",
        "runId": "run-1",
        "messages": [{ "id": "m1", "role": "user", "content": "My name is Alice. Start my zombie adventure." }],
        "tools": [], "context": [], "state": {}, "forwardedProps": {}
      }'
    ```

    :::note[Codificação de URL]
    A codificação URL é necessária para formatar corretamente a ARN para o endpoint da API do Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

Se o comando for executado com sucesso, você começará a ver a história sendo transmitida de volta como um [fluxo de eventos AG-UI](https://docs.copilotkit.ai/aws-strands/protocol/events) (Server-Sent Events) — os blocos `RUN_STARTED` / `TEXT_MESSAGE_START` / `TEXT_MESSAGE_CONTENT` envolvem os tokens reais da história:

```
data: {"type":"RUN_STARTED","threadId":"Alice-zombie00000000000000000000000",...}
data: {"type":"TEXT_MESSAGE_START","messageId":"...","role":"assistant"}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"Greetings, Alice. "}
data: {"type":"TEXT_MESSAGE_CONTENT","messageId":"...","delta":"The moans grow louder beyond the barricaded door..."}
```

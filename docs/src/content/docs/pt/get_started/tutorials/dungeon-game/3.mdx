## Tarefa 1: Implementar o Agente de História

O Agente de História é um agente [Strands](https://strandsagents.com/) que, dado um `Game` e uma lista de `Action`s como contexto, irá progredir uma narrativa. Configuraremos o agente para interagir com nosso Inventory MCP Server e gerenciar os itens disponíveis de um jogador.

:::note[Implementação Python]
Para este tutorial, implementaremos o Agente de forma "stateless", onde cada invocação é uma sessão independente incluindo todo o histórico de mensagens. Para produção, considere integrar com [Bedrock AgentCore Memory](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/memory.html). Para mais informações, consulte a página [usando AgentCore Memory com Strands](https://github.com/aws/bedrock-agentcore-sdk-python/tree/main/src/bedrock_agentcore/memory/integrations/strands).
:::

### Testar seu Agente

Você pode testar seu Agente de duas formas:
<ul>
<li>Iniciando uma instância local do servidor Agent e invocando-o usando `curl`, ou</li>
<li>Chamando a API implantada usando curl com um token JWT.</li>
</ul>

<Tabs>
  <TabItem label="Local">
  O gerador de agentes fornece uma CLI de chat interativo de terminal (`story-chat`) que usa o cliente TypeScript type-safe gerado a partir da especificação OpenAPI do agente. Como alteramos a forma de entrada do agente para aceitar `playerName`, `genre` e `actions`, primeiro precisamos atualizar o script de chat gerado para fornecer esses campos.

  Substitua o conteúdo de `packages/story/scripts/story/chat.ts` por:

    ```ts
    // packages/story/scripts/story/chat.ts
    import { chatLoop, type ChatAdapter } from 'agent-chat-cli';
    import { Story } from './generated/client.gen.js';

    const port = process.env.PORT ?? '8081';

    class StoryAdapter implements ChatAdapter {
      private client!: Story;
      private readonly actions: { role: string; content: string }[] = [];

      async connect(url: string) {
        this.client = new Story({ url });
        return { agentName: 'Story' };
      }

      async *sendMessage(text: string): AsyncIterable<string> {
        this.actions.push({ role: 'user', content: text });
        let response = '';
        for await (const chunk of this.client.invoke({
          playerName: 'UnnamedHero',
          genre: 'superhero',
          actions: this.actions,
        })) {
          response += chunk.content;
          yield chunk.content;
        }
        this.actions.push({ role: 'assistant', content: response });
      }
    }

    await chatLoop(new StoryAdapter(), `http://localhost:${port}`);
    ```

    Em seguida, inicie um chat interativo com seu agente Story em execução local:

    <NxCommands highlights={['xxxx', 'region']} env={{RUNTIME_CONFIG_APP_ID:"xxxx", AWS_REGION: '<region>'}} commands={["run @dungeon-adventure/story:story-chat"]} />

    <Aside type="caution" title="Usar valores de saída do CDK">
    Use o valor de saída `RuntimeConfigApplicationId` do CDK deploy para a variável de ambiente `RUNTIME_CONFIG_APP_ID`, e sua região AWS implantada para `AWS_REGION`.
    </Aside>

    O destino `story-chat` inicia automaticamente o `story-serve-local` (que inicia o agente e quaisquer dependências do servidor MCP com `SERVE_LOCAL=true`) e executa a geração da especificação OpenAPI + cliente antes de colocá-lo no chat.

    :::note[`story-serve` vs `story-serve-local`]
    `story-chat` depende de `story-serve-local`, que inicia o agente com `SERVE_LOCAL=true` e também inicia quaisquer dependências do servidor MCP localmente. Se você preferir que o agente se conecte ao seu servidor MCP **implantado** (via `RUNTIME_CONFIG_APP_ID`), execute o destino `story-serve` em um terminal separado e então invoque o agente manualmente com `agent-chat-cli` ou `curl`.
    :::
  </TabItem>
  <TabItem label="Implantado">
    Para testar o agente implantado, você precisará autenticar com Cognito e obter um token JWT. Primeiro, configure suas variáveis de ambiente:

    ```bash
    # Defina seu Cognito User Pool ID e Client ID das saídas do CDK
    export POOL_ID="<UserPoolId das saídas do CDK>"
    export CLIENT_ID="<UserPoolClientId das saídas do CDK>"
    export REGION="<sua-regiao>"
    ```

    Crie um usuário de teste e obtenha um token de autenticação:

    ```bash
    # Desabilite MFA para simplificar a criação do usuário
    aws cognito-idp set-user-pool-mfa-config \
      --mfa-configuration OFF \
      --user-pool-id $POOL_ID

    # Criar Usuário
    aws cognito-idp admin-create-user \
      --user-pool-id $POOL_ID \
      --username "test" \
      --temporary-password "TempPass123-" \
      --region $REGION \
      --message-action SUPPRESS > /dev/null

    # Definir Senha Permanente (substitua por algo mais seguro!)
    aws cognito-idp admin-set-user-password \
      --user-pool-id $POOL_ID \
      --username "test" \
      --password "PermanentPass123-" \
      --region $REGION \
      --permanent > /dev/null

    # Autenticar Usuário e capturar Access Token
    export BEARER_TOKEN=$(aws cognito-idp initiate-auth \
      --client-id "$CLIENT_ID" \
      --auth-flow USER_PASSWORD_AUTH \
      --auth-parameters USERNAME='test',PASSWORD='PermanentPass123-' \
      --region $REGION \
      --query "AuthenticationResult.AccessToken" \
      --output text)
    ```

    Invoque o agente implantado usando a URL do runtime do Bedrock AgentCore:

    ```bash
    # Defina a ARN do Story Agent das saídas do CDK
    export AGENT_ARN="<StoryAgentArn das saídas do CDK>"

    # Codifique a ARN em URL
    export ENCODED_ARN=$(echo $AGENT_ARN | sed 's/:/%3A/g' | sed 's/\//%2F/g')

    # Construa a URL de invocação
    export AGENT_URL="https://bedrock-agentcore.$REGION.amazonaws.com/runtimes/$ENCODED_ARN/invocations?qualifier=DEFAULT"

    # Invoque o agente
    curl -N -X POST "$AGENT_URL" \
      -H "authorization: Bearer $BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -H "X-Amzn-Bedrock-AgentCore-Runtime-Session-Id: abcdefghijklmnopqrstuvwxyz-123456789" \
      -d '{"genre":"superhero", "actions":[], "playerName":"UnnamedHero"}'
    ```

    :::note[Codificação de URL]
    A codificação URL é necessária para formatar corretamente a ARN para o endpoint da API do Bedrock AgentCore.
    :::
  </TabItem>
</Tabs>

## Visão Geral

Neste tutorial você construirá um jogo de aventura em masmorra com IA agentica. Este tutorial não assume conhecimento prévio do `@aws/nx-plugin` ou tecnologias relacionadas. As técnicas que você aprenderá são fundamentais para construir qualquer aplicação baseada no `@aws/nx-plugin`, e compreendê-las totalmente dará a você um conhecimento profundo deste pacote e uma noção básica de como usar [NX](https://nx.dev/).

<Aside>Este tutorial é para quem prefere aprender fazendo e deseja criar algo tangível rapidamente. Se prefere aprender conceitos passo a passo, consulte os guias de componentes individuais</Aside>

Ao completar este tutorial, você terá conhecimento para:

- Usar o `@aws/nx-plugin` para criar novas aplicações
- Usar NX para gerenciar/construir bases de código
- Construir APIs com tRPC
- Desenvolver Agentes usando Strands
- Criar servidores MCP em TypeScript
- Usar Tanstack router para criar páginas
- Utilizar Tanstack query para chamar APIs backend
- Criar e implantar infraestrutura CDK

### Componentes da aplicação

O jogo será construído com a seguinte arquitetura de componentes:

<Image style="margin: auto" src={dungeonAdventureArchitecturePng} alt="dungeon-game-architecture.png" height="600" />

- Website frontend React/Vite utilizando:
  - Amazon Cognito/Identity Pools para chamadas API seguras
  - [Tanstack router](https://tanstack.com/router/latest) com roteamento baseado em arquivos e type-safety
  - SDKs gerados para comunicação com a Game API
- API [tRPC](https://trpc.io/) usando [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) para gerenciar estado do jogo
- Agente [Strands](https://strandsagents.com/latest/) hospedado no [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) para execução do jogo
- Servidor MCP TypeScript no [Amazon Bedrock AgentCore](https://aws.amazon.com/bedrock/agentcore/) para gerenciar inventário do jogador
- Infraestrutura [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) para implantação da aplicação

## Pré-requisitos

São necessários os seguintes requisitos globais:

<Snippet name="required-prerequisites" />
- [Docker](https://www.docker.com/) para construção dos componentes AgentCore
- Garanta que sua conta AWS tenha acesso ao [modelo padrão do Strands](https://strandsagents.com/latest/documentation/docs/user-guide/concepts/model-providers/amazon-bedrock/) no Bedrock para sua região-alvo seguindo [este guia](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access-modify.html). Atualmente este modelo é o Anthropic Claude Sonnet 4.

:::tip
Se usar assistentes de IA como Kiro, Amazon Q Developer, Cursor, Claude Code ou Cline, recomendamos <Link path="/get_started/building-with-ai">instalar o Nx Plugin para servidores MCP AWS.</Link>
:::

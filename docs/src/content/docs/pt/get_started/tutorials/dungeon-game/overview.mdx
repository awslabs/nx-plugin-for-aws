## Visão Geral

Neste tutorial você construirá um jogo de aventura em masmorra com IA. Este tutorial não assume nenhum conhecimento prévio do `@aws/nx-plugin` ou tecnologias relacionadas. As técnicas que você aprenderá são fundamentais para construir qualquer aplicação baseada no `@aws/nx-plugin`, e compreendê-las totalmente lhe dará um conhecimento profundo desta ferramenta e uma compreensão básica de como usar o [NX](https://nx.dev/).

<Aside>Este tutorial é ideal para quem prefere aprender fazendo e deseja criar algo tangível rapidamente. Se você prefere aprender cada conceito passo a passo, consulte os Guias individuais na navegação esquerda.</Aside>

Ao completar este tutorial, você aprenderá como:

- Usar o `@aws/nx-plugin` para criar novas aplicações
- Utilizar o NX para gerenciar/construir sua base de código
- Construir APIs usando tRPC e FastAPI
- Usar o TanStack Router para criar novas páginas
- Utilizar o TanStack Query para chamar APIs backend
- Fazer modificações no código gerado para adequá-lo ao propósito
- Criar e implantar infraestrutura CDK

### Componentes da aplicação

Construiremos o jogo usando a seguinte arquitetura de componentes:

<Image class="white-bg" src={dungeonAdventureArchitecturePng} alt="dungeon-game-architecture.png" width="800" height="600" />

1. A API [tRPC](https://trpc.io/) usa [ElectroDB](https://electrodb.dev/)/[DynamoDB](https://aws.amazon.com/dynamodb/) para gerenciar o estado do jogo
2. Função Lambda Python/[FastAPI](https://fastapi.tiangolo.com/) com respostas em streaming para geração de histórias (alimentado por [Amazon Bedrock](https://aws.amazon.com/bedrock/))
3. Frontend React/Vite com:
  - Amazon Cognito/Identity Pools para chamadas de API seguras
  - [TanStack Router](https://tanstack.com/router/latest) com roteamento baseado em arquivos e type-safety
  - SDKs gerados para consumir as APIs de Jogo e História (usando [TanStack Query](https://tanstack.com/query/latest))

Uma infraestrutura [CDK](https://docs.aws.amazon.com/cdk/v2/guide/home.html) para implantar os componentes da aplicação.

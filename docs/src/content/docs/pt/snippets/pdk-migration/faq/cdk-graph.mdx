O [CDK Graph](https://aws.github.io/aws-pdk/developer_guides/cdk-graph/index.html) constrói grafos de seus recursos CDK conectados e fornece dois plugins:

#### Diagram Plugin

O [CDK Graph Diagram Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-diagram/index.html) gera diagramas de arquitetura AWS a partir de sua infraestrutura CDK.

Para uma abordagem determinística similar, uma alternativa viável é o [CDK-Dia](https://github.com/pistazie/cdk-dia).

Com os avanços em IA Generativa, muitos modelos de base são capazes de criar diagramas de alta qualidade a partir de sua infraestrutura CDK. Recomendamos experimentar o [AWS Diagram MCP Server](https://github.com/awslabs/mcp/tree/main/src/aws-diagram-mcp-server). Confira [este post de blog](https://aws.amazon.com/blogs/machine-learning/build-aws-architecture-diagrams-using-amazon-q-cli-and-mcp/) para um passo a passo.

#### Threat Composer Plugin

O [CDK Graph Threat Composer Plugin](https://aws.github.io/aws-pdk/developer_guides/cdk-graph-plugin-threat-composer/index.html) gera um modelo de ameaça inicial para o [Threat Composer](https://github.com/awslabs/threat-composer/) a partir de seu código CDK.

Este plugin funciona simplesmente filtrando um [modelo de ameaça base](https://github.com/aws/aws-pdk/tree/mainline/packages/cdk-graph-plugin-threat-composer/src/model-generator/base-model) contendo ameaças de exemplo, e filtrando-as com base nos recursos utilizados por sua stack.

Se você está interessado nessas ameaças de exemplo específicas, pode copiar e filtrar o modelo de ameaça base, ou usá-lo como contexto para ajudar um modelo de base a gerar um similar.

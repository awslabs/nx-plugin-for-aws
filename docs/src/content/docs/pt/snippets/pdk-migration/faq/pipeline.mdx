O PDK fornecia um [`PDKPipelineProject`](https://aws.github.io/aws-pdk/developer_guides/pipeline/index.html) que configurava um projeto de infraestrutura CDK e utilizava um construto CDK que encapsulava alguns recursos de [CDK Pipelines](https://docs.aws.amazon.com/cdk/api/v2/docs/aws-cdk-lib.pipelines-readme.html).

Para migrar disso, você pode usar os construtos de CDK Pipelines diretamente. Na prática, porém, é provavelmente mais simples usar algo como GitHub Actions ou GitLab CI/CD, onde você define [CDK Stages](https://docs.aws.amazon.com/cdk/v2/guide/stages.html) e executa o comando de deploy para o estágio apropriado diretamente.

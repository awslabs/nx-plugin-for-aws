O PDK suporta infraestrutura CDK escrita em Python e Java. Atualmente, não oferecemos suporte a isso no Nx Plugin for AWS.

A abordagem recomendada seria migrar sua infraestrutura CDK para TypeScript ou usar nossos geradores e migrar o pacote de constructos comuns para sua linguagem desejada. Você pode usar Inteligência Artificial Generativa para acelerar esse tipo de migração, como por exemplo o [Amazon Q CLI](https://aws.amazon.com/developer/learning/q-developer-cli/). Um agente de IA pode iterar na migração até que os modelos CloudFormation sintetizados sejam idênticos.

O mesmo se aplica à infraestrutura gerada por APIs Type Safe em Python ou Java - você pode traduzir o constructo genérico `rest-api.ts` do pacote de constructos comuns e implementar seu próprio gerador de metadados simples para a linguagem alvo (consulte a seção [APIs Modeladas com OpenAPI](#apis-modelled-with-openapi)).

Você pode usar o gerador <Link path="/guides/python-project">`py#project`</Link> para um projeto Python base e adicionar seu código CDK (movendo seu arquivo `cdk.json` e adicionando targets relevantes). Para projetos Java, utilize o plugin [`@nx/gradle`](https://nx.dev/technologies/java/introduction) do Nx ou [`@jnxplus/nx-maven`](https://github.com/gridatek/jnxplus/tree/main/packages/nx-maven) para Maven.

---
title: Requisitos SSL ao Conectar sem RDS Proxy
---
import Infrastructure from '@components/infrastructure.astro';

O gerador de conexão atualiza o Dockerfile para instalar o pacote de CA do Amazon RDS em `/usr/local/share/ca-certificates/rds-bundle.crt`. Defina `NODE_EXTRA_CA_CERTS` para esse caminho para que o Node.js confie no certificado ao conectar sem RDS Proxy:

<Infrastructure>
<Fragment slot="cdk">

```ts title="packages/infra/src/stacks/application-stack.ts"
new MyAgent(this, 'MyAgent', {
  ...
  environmentVariables: {
    NODE_EXTRA_CA_CERTS: '/usr/local/share/ca-certificates/rds-bundle.crt',
  },
});
```

</Fragment>
<Fragment slot="terraform">

```hcl title="packages/infra/src/main.tf"
module "my_agent" {
  ...
  environment_variables = {
    NODE_EXTRA_CA_CERTS = "/usr/local/share/ca-certificates/rds-bundle.crt"
  }
}
```

</Fragment>
</Infrastructure>

Para mais detalhes, consulte a [documentação SSL/TLS do Amazon RDS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/UsingWithRDS.SSL.html). Ao usar RDS Proxy, você não precisa configurar `NODE_EXTRA_CA_CERTS`.

---
title: Verificação de Imagem de Contêiner
---

A imagem Docker construída para este projeto é verificada em busca de vulnerabilidades como parte da construção usando [Trivy](https://trivy.dev/), executando a partir da [imagem Trivy hospedada no ECR](https://gallery.ecr.aws/aquasecurity/trivy).

A verificação é executada após a construção da imagem e **falha a construção** se qualquer vulnerabilidade de severidade `HIGH` ou `CRITICAL` corrigível for encontrada. Vulnerabilidades sem uma correção disponível são ignoradas para que avisos de imagem base não acionáveis não bloqueiem sua construção.

A verificação usa o mesmo mecanismo de contêiner que a construção da sua imagem (`docker` ou `finch`), portanto, nenhuma ferramenta adicional é necessária.

:::caution[Imagens Base Limpas]
O `Dockerfile` gerado usa uma imagem base sem vulnerabilidades `HIGH` ou `CRITICAL` conhecidas no momento da geração, e atualiza ferramentas incluídas (como `npm`) para mantê-la assim. Novas vulnerabilidades são divulgadas ao longo do tempo, então uma construção que passa hoje pode falhar mais tarde conforme o banco de dados de vulnerabilidades é atualizado. Quando isso acontecer, atualize para uma imagem base mais recente onde a vulnerabilidade está corrigida, ou suprima a descoberta conforme descrito abaixo.
:::

#### Suprimindo Descobertas do Trivy

Pode haver casos em que você deseja suprimir uma vulnerabilidade específica, por exemplo, quando nenhuma correção está disponível ainda e você avaliou o risco como aceitável.

Adicione o ID da vulnerabilidade (um por linha) ao arquivo `.trivyignore` na raiz do seu projeto:

```text
// .trivyignore
# node-tar arbitrary file write - not exploitable in our usage
CVE-2024-XXXXX
```

Para mais detalhes sobre filtragem de descobertas, consulte a [documentação de filtragem do Trivy](https://trivy.dev/latest/docs/configuration/filtering/#by-finding-ids).

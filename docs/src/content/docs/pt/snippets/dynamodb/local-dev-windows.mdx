---
title: Aviso sobre Desenvolvimento Local no Windows
---

Parar o `dev` (por exemplo, com `Ctrl+C`) remove automaticamente o contêiner DynamoDB Local, mas preserva o volume nomeado para que seus dados persistam entre reinicializações.

:::caution[Windows]
Devido a limitações no tratamento de sinais no Windows, o contêiner não é removido automaticamente quando o `dev` é interrompido. Você precisará removê-lo manualmente:

```bash
<engine> rm -f <scope>-dynamodb
```

Substitua `<engine>` pelo seu motor de contêiner (`docker` ou `finch`) e `<scope>` pelo escopo do seu workspace Nx (por exemplo, `proj`).
:::

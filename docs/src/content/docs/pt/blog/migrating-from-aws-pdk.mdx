Este guia orienta você através de uma migração exemplo de um projeto [AWS PDK](https://github.com/aws/aws-pdk) para o Nx Plugin for AWS, além de fornecer orientações gerais sobre o tema.

Migrar para o Nx Plugin for AWS oferece os seguintes benefícios em relação ao PDK:

- Compilações mais rápidas
- Mais fácil de usar (UI e CLI)
- Compatível com vibe-coding (<Link path="/get_started/building-with-ai">experimente nosso servidor MCP!</Link>)
- Tecnologias mais modernas
- Desenvolvimento local de APIs e websites
- Mais controle (modifique arquivos fornecidos para seu caso de uso)
- E muito mais!

:::caution
Existem muitas nuances e considerações ao migrar um projeto. Este guia é para um exemplo simples, e podem faltar etapas necessárias para seu projeto específico.

Este guia opta por uma abordagem onde criamos um workspace novo e copiamos partes do projeto PDK. Se precisar migrar de forma diferente, ou tiver dúvidas sobre a migração do PDK para o Nx Plugin for AWS, [participe das discussões no GitHub](https://github.com/awslabs/nx-plugin-for-aws/discussions)!

Note também que alguns recursos do PDK não possuem equivalentes exatos. Consulte a seção [FAQ](#perguntas-frequentes) para algumas dicas/ideias.
:::

:::caution
Este guia foi escrito para uma versão específica e provavelmente não será atualizado conforme o Nx Plugin for AWS evolui. Para compensar, o guia fixa a versão do `@aws/nx-plugin` em `0.50.0`. Você pode tentar com a versão mais recente, mas podem haver diferenças em relação ao guia.
:::

## Exemplo de Migração: Aplicação Lista de Compras

Neste guia, usaremos a [Aplicação Lista de Compras do Tutorial PDK](https://aws.github.io/aws-pdk/getting_started/shopping_list_app.html) como nosso projeto alvo para migração. Siga as etapas daquele tutorial para criar o projeto se quiser acompanhar.

A aplicação lista de compras consiste nos seguintes tipos de projeto PDK:

- `MonorepoTsProject`
- `TypeSafeApiProject` 
- `CloudscapeReactTsWebsiteProject`
- `InfrastructureTsProject`

### Criar Workspace

Começaremos criando um novo workspace para nosso projeto. Embora mais radical que uma migração in-place, esta abordagem fornece o resultado mais limpo. Criar um workspace Nx é equivalente ao `MonorepoTsProject` do PDK:

<CreateNxWorkspaceCommand workspace="shopping-list" iacProvider="CDK" nxVersion="21.4.1" nxPluginVersion="0.50.0" />

Abra o diretório `shopping-list` criado em seu IDE favorito.
